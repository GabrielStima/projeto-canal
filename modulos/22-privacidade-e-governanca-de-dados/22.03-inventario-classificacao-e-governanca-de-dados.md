# Inventário, Classificação e Governança de Dados

## Objetivo

Ao final desta aula, você deve ser capaz de:

- explicar por que inventário de dados é base para privacidade;
- classificar dados por tipo, finalidade, origem, destino e criticidade;
- relacionar governança com ownership, documentação, acesso e retenção;
- criar um mapa simples de dados para uma aplicação.

## Ideia Central

Não dá para proteger bem o que o time nem sabe que existe. Inventário de dados é o mapa dos dados tratados por uma aplicação: quais campos existem, onde entram, onde ficam armazenados, para onde são enviados, quem acessa, quanto tempo ficam e qual finalidade justificou a coleta.

Classificação transforma esse inventário em decisão. Um campo público, um identificador interno, um dado pessoal, um dado sensível, um segredo e um dado anonimizado pedem controles diferentes.

Governança de dados é manter esse mapa vivo. Ela aparece em revisão de pull request, documentação de endpoints, schema de banco, catálogo interno, políticas de acesso, retenção, auditoria e resposta a incidentes.

## Tópicos da Aula

- Inventário de dados como mapa prático.
- Classificação por tipo e criticidade.
- Origem, destino, finalidade e dono do dado.

## Exemplo Prático

O Mapa do Tesouro (Data Mapping):

O dono da PetCare pergunta: 'Onde estão os CPFs dos tutores?'
O Engenheiro responde: 'Ah, no PostgreSQL. E num arquivo Excel exportado de 2021. E num Log de Erro no Datadog. E acho que o sistema de nota fiscal também tem uma cópia no S3...'
Falta Governança. O *Inventário de Dados (Data Mapping)* é a documentação viva (Data Catalog) que lista exatamente em quais tabelas de quais bancos habitam dados Tóxicos vs Dados Públicos.

## Analogias e Intuições

- Inventário de Dados é a planta do Banco Central dizendo onde fica a caixa forte. Se você não sabe onde o ouro está guardado, você gastará fortuna pondo seguranças nos banheiros, enquanto a porta dos fundos fica aberta.

## Erros Comuns

- Mapear apenas tabelas do banco e ignorar logs, filas, caches, backups e SaaS.
- Criar classificações demais e tornar o processo inutilizável.
- Não definir dono para dados e fluxos.
- Deixar inventário desatualizado depois de novas features.
- Tratar governança como trava, não como ferramenta de decisão.

## Checklist de Entendimento
