# Inventário, Classificação e Governança de Dados

## Objetivo

Ao final desta aula, você deve ser capaz de:

- explicar por que inventário de dados é base para privacidade;
- classificar dados por tipo, finalidade, origem, destino e criticidade;
- relacionar governança com ownership, documentação, acesso e retenção;
- criar um mapa simples de dados para uma aplicação.

## Ideia Central

Não dá para proteger bem o que o time nem sabe que existe. Inventário de dados é o mapa dos dados tratados por uma aplicação: quais campos existem, onde entram, onde ficam armazenados, para onde são enviados, quem acessa, quanto tempo ficam e qual finalidade justificou a coleta.

Classificação transforma esse inventário em decisão. Um campo público, um identificador interno, um dado pessoal, um dado sensível, um segredo e um dado anonimizado pedem controles diferentes.

Governança de dados é manter esse mapa vivo. Ela aparece em revisão de pull request, documentação de endpoints, schema de banco, catálogo interno, políticas de acesso, retenção, auditoria e resposta a incidentes.

## Tópicos da Aula

- Inventário de dados como mapa prático.
- Classificação por tipo e criticidade.
- Origem, destino, finalidade e dono do dado.
- Data flow simples: frontend, API, banco, logs, terceiros e IA.
- Governança como processo leve para times de desenvolvimento.

## Exemplo Prático

Um inventário mínimo para `student_profile` poderia ter:

```text
Campo: e-mail
Tipo: dado pessoal
Origem: cadastro
Finalidade: login, comunicação transacional e recuperação de conta
Armazenamento: tabela users
Exposição: GET /me, painel administrativo restrito, ferramenta de suporte
Logs: não deve aparecer em logs de erro
Retenção: enquanto conta estiver ativa e conforme política de encerramento
Dono: time de identidade/conta
```

O valor do inventário não está em virar burocracia. Está em revelar riscos: e-mail indo para analytics sem necessidade, telefone em log, export sem controle, acesso administrativo amplo ou backup sem prazo.

## Analogias e Intuições

- Inventário é como planta baixa: antes de reformar ou proteger, você precisa saber onde estão as coisas.
- Classificação é como etiqueta de fragilidade: cada tipo de dado pede cuidado proporcional.
- Governança é como rotina de manutenção: precisa continuar depois do primeiro documento.

## Erros Comuns

- Mapear apenas tabelas do banco e ignorar logs, filas, caches, backups e SaaS.
- Criar classificações demais e tornar o processo inutilizável.
- Não definir dono para dados e fluxos.
- Deixar inventário desatualizado depois de novas features.
- Tratar governança como trava, não como ferramenta de decisão.

## Checklist de Entendimento

Antes de seguir, a pessoa deve conseguir responder:

- Que perguntas um inventário de dados deve responder?
- Por que classificação ajuda acesso, retenção e logs?
- Onde dados aparecem além do banco?
- Como manter governança leve em um time pequeno?
- O que deveria ser revisado in pull requests que adicionam novos dados?

## Prática Recomendada

- Escolha uma feature e crie um inventário de 5 campos.
- Desenhe o fluxo de um dado do formulário até banco, logs, e-mail e suporte.
- Defina uma classificação simples: público, interno, pessoal, sensível, segredo.
- Crie um checklist para revisar PRs que adicionam ou expõem dados.
