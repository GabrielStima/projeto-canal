# Dados Pessoais, Dados Sensíveis e Ciclo de Vida do Dado

## Objetivo

Ao final desta aula, você deve ser capaz de:

- diferenciar dados pessoais, dados sensíveis, dados técnicos e dados anonimizados;
- reconhecer dados identificáveis mesmo quando não parecem óbvios;
- explicar o ciclo de vida de um dado dentro de uma aplicação;
- mapear riscos em coleta, armazenamento, uso, compartilhamento e descarte.

## Ideia Central

Dados pessoais são informações relacionadas a uma pessoa natural identificada ou identificável. Nome e CPF são exemplos diretos, mas também existem identificadores indiretos: e-mail, telefone, IP, cookie, device ID, endereço, combinações de eventos e histórico de comportamento.

Dados sensíveis recebem cuidado maior porque podem expor aspectos íntimos ou gerar discriminação, como dados de saúde, biometria, origem racial ou étnica, convicção religiosa, opinião política, vida sexual, dados genéticos e outros previstos em regulação.

Para devs, o ponto prático é entender o ciclo de vida: o dado é coletado, validado, enviado, armazenado, consultado, exibido, logado, exportado, compartilhado, copiado em backup e descartado. Risco de privacidade aparece em qualquer etapa.

## Tópicos da Aula

- Dados sensíveis e proteção reforçada.

- Ciclo de vida do dado em uma aplicação fullstack.

## Exemplo Prático

Em uma tabela `users`, estes campos têm riscos diferentes:

```ts
type User = {
  id: string;
  name: string;
  email: string;
  birthDate?: string;
  phone?: string;
  lastLoginIp?: string;
  marketingOptIn: boolean;
  passwordHash: string;
};
```

`name`, `email`, `birthDate`, `phone` e `lastLoginIp` podem ser dados pessoais. `passwordHash` não é a senha original, mas ainda é altamente sensível do ponto de vista de segurança. `marketingOptIn` registra uma escolha do titular. `id` pode parecer inofensivo, mas identifica uma pessoa dentro do sistema.

A mesma informação também pode aparecer em logs, eventos de analytics, tickets de suporte, backups e exports administrativos.

## Analogias e Intuições

- Dado pessoal é como uma etiqueta: pode identificar diretamente ou ajudar a identificar quando combinada com outras etiquetas.
- Ciclo de vida do dado é como uma trilha: o risco não está só no ponto de entrada.
- Dados sensíveis são como materiais perigosos: não basta armazenar, é preciso controlar manuseio, acesso e descarte.

## Erros Comuns

- Achar que apenas CPF e documento contam como dados pessoais.
- Ignorar IP, cookie ID, device ID e dados de comportamento.
- Tratar hash, token e identificador interno como dados sem risco.
- Esquecer que backups e logs também fazem parte do ciclo de vida.
- Chamar dado de anônimo quando ainda existe chave ou contexto para reidentificar.

## Prática Recomendada

- Escolha uma entidade `User`, `Customer` ou `Student` de um projeto e classifique cada campo.
- Liste todos os lugares onde o e-mail do usuário pode aparecer.
- Separe dados necessários para funcionamento, suporte, segurança, analytics e marketing.
- Identifique um campo que poderia ser removido, atrasado ou tornado opcional.
