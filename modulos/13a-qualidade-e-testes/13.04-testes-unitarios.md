# Testes Unitários

## Objetivo

Ao final desta aula, você deve ser capaz de:

- explicar o que um teste unitário valida;
- escolher regras pequenas para testar;
- escrever cenários de entrada, processamento e saída;
- evitar testes acoplados a detalhes internos.

## Ideia Central

Teste unitário valida uma unidade pequena de comportamento. Em TypeScript, isso costuma ser uma função, método ou regra de domínio que pode ser exercitada com entradas controladas e saídas esperadas.

O melhor teste unitário é fácil de entender, rápido de rodar e claro quando falha. Ele deve proteger comportamento, não congelar a implementação.

## Tópicos da Aula

- Unidade de comportamento.
- Arrange, act, assert.
- Casos de sucesso e erro.
- Funções puras.
- Regras de domínio.
- Testes frágeis.

## Exemplo Prático

```ts
// Domínio do PetCare OS
type Tutor = { id: string, tier: 'PADRAO' | 'PREMIUM' };

function calculateDiscount(total: number, tutor: Tutor) {
  if (total <= 0) throw new Error("Total inválido");
  if (tutor.tier === 'PREMIUM') return total * 0.9;
  return total;
}
```

Esse teste examina cenários (`PADRAO`, `PREMIUM`, totais negativos) sem precisar levantar o Express ou o PostgreSQL. Executa em 1 milissegundo.

## Analogias e Intuições

- Teste unitário é como ligar a máquina de raio-x fora da clínica, direto na tomada, e ver se ela emite o feixe corretamente. Você isola o componente do resto do prédio.
*Limite: A máquina pode funcionar fora, mas queimar a fiação quando instalada na clínica (falha de integração).*

## Erros Comuns

- Testar implementação em vez de comportamento.
- Criar um teste por linha de código.
- Ignorar casos negativos.
- Usar banco real para regra que poderia ser pura.
- Criar nomes de teste que não explicam o cenário.

## Checklist de Entendimento

Antes de seguir, você deve conseguir responder:

- Qual comportamento está sendo testado?
- Quais entradas importam?
- Qual saída ou erro é esperado?
- O teste roda rápido?
- O teste continuaria valido após refatoração interna?

## Prática Recomendada

- [Exercício 02 — Testes Unitários e Doubles](exercicios/02-testes-unitarios-e-doubles.md)
- Escreva os três cenários de teste (Arrange, Act, Assert) para a função `calculateDiscount`.
- Refatore a função mentalmente caso precisasse de um banco de dados, separando a busca (impura) da regra de cálculo (pura).
- Evite o uso de dados genéricos obscuros; use `total = 100` e não `total = 384.21` se isso não for relevante.
