# Modulos

## Objetivo

Ao final desta aula, voce deve ser capaz de:

- explicar por que dividir codigo em arquivos;
- usar `export` e `import`;
- diferenciar export nomeado e export default em nivel introdutorio;
- organizar funcoes e tipos em modulos simples;
- preparar a base para projetos maiores.

## Onde Esta Aula Entra no Curso

- Modulo: Programacao com TypeScript
- Aula anterior: Type aliases e interfaces
- Proxima aula: Classes e POO basica
- Pre-requisitos: funcoes, objetos, type aliases e interfaces

## Ideia Central

Modulos permitem dividir o codigo em partes menores. Em vez de colocar tudo em um arquivo, exportamos o que precisa ser usado fora e importamos apenas o que outro arquivo precisa.

Isso melhora organizacao, leitura e manutencao, principalmente quando o projeto cresce.

## Topicos da Aula

- Por que modularizar
- `export`
- `import`
- Exports nomeados
- Export default em nivel introdutorio
- Organizacao de tipos
- Cuidados com dependencias circulares

## Exemplo Pratico

```typescript
// tarefa.ts
export type Tarefa = {
  titulo: string;
  concluida: boolean;
};

export function criarTarefa(titulo: string): Tarefa {
  return {
    titulo,
    concluida: false,
  };
}
```

```typescript
// app.ts
import { criarTarefa } from "./tarefa";

const tarefa = criarTarefa("Estudar TypeScript");
console.log(tarefa);
```

## Analogias e Intuicoes

- Modulo e uma gaveta organizada do projeto.
- Exportar e deixar algo disponivel para fora.
- Importar e pedir algo de outro arquivo.

## Erros Comuns

- Exportar tudo sem criterio.
- Criar arquivos pequenos demais sem motivo.
- Criar arquivos grandes que fazem muitas coisas.
- Misturar regra, exemplo e execucao no mesmo lugar sempre.
- Criar dependencia circular entre arquivos.

## Checklist de Entendimento

Antes de seguir, voce deve conseguir responder:

- Por que dividir codigo em modulos?
- O que `export` faz?
- O que `import` faz?
- Qual a diferenca entre exportar uma funcao e um tipo?
- Que problema dependencia circular pode causar?

## Pratica Recomendada

- Separe um tipo `Tarefa` em um arquivo.
- Separe uma funcao `criarTarefa` no mesmo modulo.
- Importe a funcao em outro arquivo.
- Experimente renomear um export nomeado e ajustar o import.
