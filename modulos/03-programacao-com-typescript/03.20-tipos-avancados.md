# Tipos Avançados

## Objetivo

Ao final desta aula, você deve ser capaz de:

- reconhecer tipos avançados comuns sem precisar domina-los profundamente;
- usar unions discriminadas em exemplos simples;
- entender intersection types;
- ler tipos condicionais em nível introdutório;
- evitar complexidade de tipos antes da necessidade real.

## Onde Esta Aula Entra na Formação

- Módulo: Programação com TypeScript
- Aula anterior: Utility types
- Próxima aula: Decorators
- Pré-requisitos: narrowing, type aliases, interfaces, generics e utility types

## Ideia Central

Tipos avançados permitem modelar relações mais precisas entre dados. Eles ajudam quando o código precisa representar estados, variações e transformações que tipos simples não expressam bem.

Mas tipos avançados também podem tornar o código difícil de ler. O objetivo desta aula e criar repertório inicial, não transformar cada problema em quebra-cabeça de tipos.

## Tópicos da Aula

- Union types mais expressivos
- Unions discriminadas
- Intersection types
- Literal types
- `keyof` em nível introdutório
- Tipos condicionais como leitura
- Custo de complexidade

## Exemplo Prático

```typescript
type ResultadoBuscaPet =
  | { status: "sucesso"; pet: { id: number; nome: string } }
  | { status: "nao_encontrado" }
  | { status: "erro"; mensagem: string };

function mostrarResultado(resultado: ResultadoBuscaPet) {
  if (resultado.status === "sucesso") {
    console.log(`Pet encontrado: ${resultado.pet.nome}`);
    return;
  }

  if (resultado.status === "nao_encontrado") {
    console.log("Pet não encontrado.");
    return;
  }

  console.log("Erro:", resultado.mensagem);
}
```

## Analogias e Intuições

- Union discriminada é uma lista de formatos possíveis com uma etiqueta clara.
- Intersection combina exigencias de dois tipos.
- Tipo avançado deve explicar melhor o domínio, não impressionar.

## Erros Comuns

- Criar tipos mais complexos que o problema.
- Usar tipos condicionais sem conseguir explicar o resultado.
- Esquecer um caso de union.
- Usar `as` para passar por cima de uma modelagem ruim.
- Transformar tipos em código impossível de manter.

## Checklist de Entendimento

Antes de seguir, você deve conseguir responder:

- O que é uma union discriminada?
- Para que serve uma propriedade discriminadora?
- O que intersection type combina?
- O que `keyof` representa em alto nível?
- Quando tipo avançado pode ser exagero?

## Prática Recomendada

- Modele um resultado de API com sucesso e erro.
- Use narrowing pela propriedade `status`.
- Crie uma intersection entre dados de usuário e metadados.
- Leia um tipo com `keyof` e explique com suas palavras.
- Modele o resultado de uma busca por pet com três estados: sucesso, não encontrado e erro.
- Use narrowing para tratar cada estado de forma segura.
- Crie uma intersection entre dados de um pet e metadados de cadastro.
