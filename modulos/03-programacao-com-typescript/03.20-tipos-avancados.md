# Tipos Avancados

## Objetivo

Ao final desta aula, voce deve ser capaz de:

- reconhecer tipos avancados comuns sem precisar domina-los profundamente;
- usar unions discriminadas em exemplos simples;
- entender intersection types;
- ler tipos condicionais em nivel introdutorio;
- evitar complexidade de tipos antes da necessidade real.

## Onde Esta Aula Entra no Curso

- Modulo: Programacao com TypeScript
- Aula anterior: Utility types
- Proxima aula: Decorators
- Pre-requisitos: narrowing, type aliases, interfaces, generics e utility types

## Ideia Central

Tipos avancados permitem modelar relacoes mais precisas entre dados. Eles ajudam quando o codigo precisa representar estados, variacoes e transformacoes que tipos simples nao expressam bem.

Mas tipos avancados tambem podem tornar o codigo dificil de ler. O objetivo desta aula e criar repertorio inicial, nao transformar cada problema em quebra-cabeca de tipos.

## Topicos da Aula

- Union types mais expressivos
- Unions discriminadas
- Intersection types
- Literal types
- `keyof` em nivel introdutorio
- Tipos condicionais como leitura
- Custo de complexidade

## Exemplo Pratico

```typescript
type Resultado =
  | { status: "sucesso"; dados: string[] }
  | { status: "erro"; mensagem: string };

function mostrarResultado(resultado: Resultado) {
  if (resultado.status === "sucesso") {
    console.log(resultado.dados.length);
    return;
  }

  console.log(resultado.mensagem);
}
```

## Analogias e Intuicoes

- Union discriminada e uma lista de formatos possiveis com uma etiqueta clara.
- Intersection combina exigencias de dois tipos.
- Tipo avancado deve explicar melhor o dominio, nao impressionar.

## Erros Comuns

- Criar tipos mais complexos que o problema.
- Usar tipos condicionais sem conseguir explicar o resultado.
- Esquecer um caso de union.
- Usar `as` para passar por cima de uma modelagem ruim.
- Transformar tipos em codigo impossivel de manter.

## Checklist de Entendimento

Antes de seguir, voce deve conseguir responder:

- O que e uma union discriminada?
- Para que serve uma propriedade discriminadora?
- O que intersection type combina?
- O que `keyof` representa em alto nivel?
- Quando tipo avancado pode ser exagero?

## Pratica Recomendada

- Modele um resultado de API com sucesso e erro.
- Use narrowing pela propriedade `status`.
- Crie uma intersection entre dados de usuario e metadados.
- Leia um tipo com `keyof` e explique com suas palavras.
