# TypeScript: O Que E e Como Pensar em Tipos

## Objetivo

Ao final desta aula, voce deve ser capaz de:

- explicar TypeScript como JavaScript com checagem estatica de tipos;
- diferenciar tempo de desenvolvimento e runtime;
- entender que TypeScript nao muda a natureza do JavaScript executado;
- reconhecer o papel dos tipos como documentacao e protecao;
- preparar a base para inferencia e anotacoes.

## Onde Esta Aula Entra no Curso

- Modulo: Programacao com TypeScript
- Aula anterior: Promises e async/await
- Proxima aula: Inferencia de tipos
- Pre-requisitos: JavaScript moderno, funcoes, objetos, arrays, erros e assincronismo

## Ideia Central

TypeScript adiciona tipos ao JavaScript para encontrar problemas antes da execucao. Ele ajuda o editor, o compilador e a pessoa desenvolvedora a entenderem quais valores sao esperados em cada parte do codigo.

O ponto mais importante: TypeScript checa o codigo durante o desenvolvimento, mas o que roda no fim ainda e JavaScript. Por isso, aprender TypeScript tambem exige entender JavaScript.

## Topicos da Aula

- TypeScript como superset de JavaScript
- Checagem estatica
- Runtime vs tempo de checagem
- Tipos como contrato
- Tipos como documentacao
- O que TypeScript consegue e nao consegue garantir
- Por que isso importa em projetos maiores

## Exemplo Pratico

```typescript
function criarSaudacao(nome: string) {
  return `Ola, ${nome}`;
}

criarSaudacao("Ana");
criarSaudacao(42);
```

Neste exemplo, TypeScript aponta o problema antes da execucao: a funcao espera uma `string`, mas recebeu um `number`.

## Analogias e Intuicoes

- TypeScript e uma revisao antecipada do codigo.
- Tipos sao combinados sobre o formato esperado dos dados.
- O runtime ainda precisa ser entendido, porque TypeScript nao executa a aplicacao por voce.

## Erros Comuns

- Achar que TypeScript evita todo bug.
- Pensar que TypeScript e uma linguagem totalmente separada de JavaScript.
- Anotar tudo sem aproveitar inferencia.
- Confundir erro de tipo com erro de regra de negocio.
- Ignorar que dados externos ainda precisam ser validados em runtime.

## Checklist de Entendimento

Antes de seguir, voce deve conseguir responder:

- O que TypeScript adiciona ao JavaScript?
- O que significa checagem estatica?
- O que e runtime?
- Por que tipos ajudam o editor?
- Que tipo de problema TypeScript nao resolve sozinho?

## Pratica Recomendada

- Escreva uma funcao JavaScript simples e adicione tipos aos parametros.
- Provoque um erro passando argumento errado.
- Explique por que o erro aparece antes da execucao.
- Liste tres bugs que TypeScript ajuda a evitar e dois que ele nao evita sozinho.
