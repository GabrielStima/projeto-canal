# Generics

## Objetivo

Ao final desta aula, você deve ser capaz de:

## Ideia Central

Generics permitem criar código reutilizável sem perder informação de tipo. Em vez de dizer que uma função trabalha com `any`, dizemos que ela trabalha com um tipo que será definido no uso.

Isso preserva segurança e flexibilidade ao mesmo tempo.

## Exemplo Prático

```typescript
function pegarPrimeiro<T>(itens: T[]): T | undefined {
  return itens[0];
}

const primeiroPet = pegarPrimeiro([
  { id: "pet-001", nome: "Thor" },
  { id: "pet-002", nome: "Luna" },
]);

const primeiroValor = pegarPrimeiro([120, 85, 200]);
```

Repositório genérico simples:

```typescript
type ComId = { id: string };

function criarRepositorioEmMemoria<T extends ComId>() {
  const itens: T[] = [];

  return {
    adicionar(item: T) {
      itens.push(item);
    },
    buscarPorId(id: string): T | undefined {
      return itens.find((item) => item.id === id);
    },
    listar(): T[] {
      return [...itens];
    },
  };
}
```

## Analogias e Intuições

- Generic é uma variável para tipos.
- Ele deixa a função se adaptar sem esquecer o formato recebido.
- `any` apaga informação; generic preserva.

## Erros Comuns

- Usar generic quando um tipo concreto bastaria.
- Nomear parâmetros de tipo de forma confusa.
- Usar `T` sem preservar nenhuma relacao real.
- Trocar todo `any` por generic sem pensar.
- Criar generics avançados antes de dominar casos simples.

## Prática Recomendada

- Crie uma função `ultimo<T>` que retorna o ultimo item de uma lista.
- Crie um type `RespostaApi<T>` com `dados` e `erro`.
- Use o mesmo type para usuário e tarefa.
- Compare uma solução com `any` e outra com generic.
- Crie um repositório genérico em memória para entidades com `id`.
- Use o mesmo repositório para `Pet` e `Atendimento`.
- Compare a versão genérica com uma versão que usaria `any`.
