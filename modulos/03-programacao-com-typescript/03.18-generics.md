# Generics

## Objetivo

Ao final desta aula, voce deve ser capaz de:

- explicar por que generics existem;
- criar funcoes genericas simples;
- usar generics em arrays e tipos reutilizaveis;
- preservar relacao entre entrada e saida;
- reconhecer quando generic deixa o codigo melhor ou pior.

## Onde Esta Aula Entra no Curso

- Modulo: Programacao com TypeScript
- Aula anterior: Classes e POO basica
- Proxima aula: Utility types
- Pre-requisitos: funcoes, tipos em funcoes, interfaces, type aliases, modulos e classes em nivel basico

## Ideia Central

Generics permitem criar codigo reutilizavel sem perder informacao de tipo. Em vez de dizer que uma funcao trabalha com `any`, dizemos que ela trabalha com um tipo que sera definido no uso.

Isso preserva seguranca e flexibilidade ao mesmo tempo.

## Topicos da Aula

- Problema do `any`
- Parametro de tipo
- Funcao generica
- Type alias generico
- Interface generica
- Constraints em nivel introdutorio
- Quando evitar generics

## Exemplo Pratico

```typescript
function pegarPrimeiro<T>(itens: T[]): T | undefined {
  return itens[0];
}

const primeiroNome = pegarPrimeiro(["Ana", "Bruno"]);
const primeiroNumero = pegarPrimeiro([10, 20]);
```

## Analogias e Intuicoes

- Generic e uma variavel para tipos.
- Ele deixa a funcao se adaptar sem esquecer o formato recebido.
- `any` apaga informacao; generic preserva.

## Erros Comuns

- Usar generic quando um tipo concreto bastaria.
- Nomear parametros de tipo de forma confusa.
- Usar `T` sem preservar nenhuma relacao real.
- Trocar todo `any` por generic sem pensar.
- Criar generics avancados antes de dominar casos simples.

## Checklist de Entendimento

Antes de seguir, voce deve conseguir responder:

- Por que generics existem?
- Qual a diferenca entre `any` e generic?
- O que `T` representa?
- Como um generic preserva o tipo de retorno?
- Quando generic pode ser exagero?

## Pratica Recomendada

- Crie uma funcao `ultimo<T>` que retorna o ultimo item de uma lista.
- Crie um type `RespostaApi<T>` com `dados` e `erro`.
- Use o mesmo type para usuario e tarefa.
- Compare uma solucao com `any` e outra com generic.
