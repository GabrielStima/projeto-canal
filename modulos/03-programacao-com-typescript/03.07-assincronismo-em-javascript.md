# Assincronismo em JavaScript

## Objetivo

Ao final desta aula, você deve ser capaz de:

- explicar o que significa uma operação assíncrona;
- diferenciar código síncrono e assíncrono;
- reconhecer exemplos comuns de assincronismo na Web;
- entender por que JavaScript precisa esperar algumas respostas;
- preparar a base para event loop, Promises e `async/await`.

## Onde Esta Aula Entra na Formação

- Módulo: Programação com TypeScript
- Aula anterior: Tratamento de erros
- Próxima aula: Event loop
- Pré-requisitos: funções, estruturas de controle e tratamento de erros

## Ideia Central

Nem toda operação termina imediatamente. Buscar dados, aguardar um temporizador, ler um arquivo ou esperar uma resposta de rede leva tempo. Assincronismo permite iniciar uma operação e continuar o fluxo sem travar tudo enquanto a resposta não chega.

No desenvolvimento web, isso aparece o tempo todo: formularios, APIs, carregamentos, animacoes e interações dependem de operações que acontecem em momentos diferentes.

## Tópicos da Aula

- Código síncrono
- Código assíncrono
- Espera sem travar a aplicação

- Requisicoes como exemplo conceitual
- Ponte para Promises

## Exemplo Prático

```javascript
console.log("Portal aberto");

setTimeout(() => {
  console.log("Status do pet atualizado");
}, 1000);

console.log("Tutor pode navegar enquanto aguarda");
```

## Analogias e Intuições

- Código síncrono espera uma tarefa terminar antes de seguir.
- Código assíncrono agenda uma tarefa e continua trabalhando.
- Uma resposta de rede e como um pedido: você não sabe exatamente quando volta.

## Erros Comuns

- Esperar que código assíncrono execute na ordem visual em que foi escrito.
- Tentar retornar imediatamente um valor que ainda não chegou.
- Criar callbacks aninhados demais.
- Ignorar erros em operações assíncronas.
- Achar que assincronismo significa executar tudo ao mesmo tempo.

## Checklist de Entendimento

Antes de seguir, você deve conseguir responder:

- O que é código síncrono?
- O que é código assíncrono?
- Por que uma requisição web costuma ser assíncrona?
- O que `setTimeout` demonstra?
- Por que assincronismo pode confundir a ordem do código?

## Prática Recomendada

- Rode um exemplo com `setTimeout`.
- Preveja a ordem dos `console.log` antes de executar.
- Crie duas tarefas com tempos diferentes.
- Explique com suas palavras por que o código não para esperando.
- Simule duas atualizações assíncronas no portal do tutor: uma para status do pet e outra para orçamento disponível.
- Preveja e depois verifique a ordem dos logs.
