# Tipos Primitivos e Estruturas Basicas

## Objetivo

Ao final desta aula, você deve ser capaz de:

- reconhecer tipos primitivos comuns;
- diferenciar `string`, `number`, `boolean`, `null` e `undefined`;
- criar arrays e objetos simples;
- entender por que tipos ajudam a prever operações validas;
- preparar o terreno para TypeScript.

## Onde Esta Aula Entra no Curso

- Módulo: Programação com TypeScript
- Aula anterior: Sintaxe, valores e variáveis
- Próxima aula: Estruturas de controle
- Pré-requisitos: valores, variáveis e sintaxe básica

## Ideia Central

Valores diferentes permitem operações diferentes. Somar numeros, juntar textos, inverter booleanos e acessar propriedades de objetos são ações que dependem do tipo de dado envolvido.

Antes de TypeScript checar tipos, precisamos perceber que os tipos já existem no comportamento do JavaScript.

## Tópicos da Aula

- `string`
- `number`
- `boolean`
- `null` e `undefined`
- Arrays
- Objetos
- Operacoes comuns por tipo

## Exemplo Prático

```javascript
const titulo = "Modulo TypeScript";
const quantidadeDeAulas = 22;
const publicado = false;
const tags = ["javascript", "typescript", "programacao"];

const modulo = {
  titulo,
  quantidadeDeAulas,
  publicado,
};

console.log(tags[0]);
console.log(modulo.titulo);
```

## Analogias e Intuições

- Tipo é uma pista sobre o que da para fazer com um valor.
- Array organiza vários valores em uma lista.
- Objeto organiza valores por nomes.

## Erros Comuns

- Tratar numero vindo de formulario como numero sem converter.
- Confundir `null` com `undefined`.
- Acessar uma posição de array que não existe.
- Criar objetos com nomes de propriedades inconsistentes.
- Misturar tipos em uma lista sem intenção clara.

## Checklist de Entendimento

Antes de seguir, você deve conseguir responder:

- O que diferencia `string` e `number`?
- Para que serve um booleano?
- Quando um valor pode ser `undefined`?
- Como acessar um item de array?
- Como acessar uma propriedade de objeto?
