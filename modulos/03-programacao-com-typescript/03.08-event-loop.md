# Event Loop

Ao final desta aula, você deve ser capaz de:

## Ideia Central

JavaScript executa um trecho de código por vez. O event loop coordena quando tarefas assíncronas podem voltar para execução depois que a pilha principal fica livre.

Essa ideia explica por que um temporizador de zero milissegundos não roda imediatamente e por que Promises podem ter prioridade diferente de outros callbacks.

## Exemplo Prático

```javascript
console.log("A");

setTimeout(() => {
  console.log("B");
}, 0);

Promise.resolve().then(() => {
  console.log("C");
});

console.log("D");
```

Saida esperada:

```text
A
D
C
B
```

## Analogias e Intuições

- A call stack é a mesa de trabalho atual.
- Filas guardam tarefas que estão prontas para voltar.
- O event loop decide quando a mesa está livre para a próxima tarefa.

## Erros Comuns

- Achar que `setTimeout(..., 0)` executa imediatamente.
- Ignorar que Promises usam uma fila própria de alta prioridade.
- Imaginar que JavaScript executa vários trechos ao mesmo tempo.
- Decorar a ordem sem entender a pilha principal.
- Tentar explicar event loop com detalhes demais cedo demais.
