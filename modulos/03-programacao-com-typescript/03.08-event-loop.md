# Event Loop

## Objetivo

Ao final desta aula, voce deve ser capaz de:

- explicar em alto nivel o que o event loop organiza;
- diferenciar call stack e filas de tarefas;
- entender por que callbacks assincronos rodam depois;
- prever a ordem de execucao de exemplos simples;
- conectar event loop com Promises e `async/await`.

## Ideia Central

JavaScript executa um trecho de codigo por vez. O event loop coordena quando tarefas assincronas podem voltar para execucao depois que a pilha principal fica livre.

Essa ideia explica por que um temporizador de zero milissegundos nao roda imediatamente e por que Promises podem ter prioridade diferente de outros callbacks.

## Exemplo Pratico

```javascript
console.log("A");

setTimeout(() => {
  console.log("B");
}, 0);

Promise.resolve().then(() => {
  console.log("C");
});

console.log("D");
```

Saida esperada:

```text
A
D
C
B
```

## Analogias e Intuicoes

- A call stack e a mesa de trabalho atual.
- Filas guardam tarefas que estao prontas para voltar.
- O event loop decide quando a mesa esta livre para a proxima tarefa.

## Erros Comuns

- Achar que `setTimeout(..., 0)` executa imediatamente.
- Ignorar que Promises usam uma fila propria de alta prioridade.
- Imaginar que JavaScript executa varios trechos ao mesmo tempo.
- Decorar a ordem sem entender a pilha principal.
- Tentar explicar event loop com detalhes demais cedo demais.

## Checklist de Entendimento

Antes de seguir, voce deve conseguir responder:

- O que e call stack?
- Por que codigo sincronico roda primeiro?
- O que uma fila de tarefas guarda?
- Por que `Promise.then` pode vir antes de `setTimeout`?
- Como o event loop ajuda a entender bugs de ordem?

## Pratica Recomendada

- Preveja a ordem de tres exemplos com `console.log`.
- Misture `setTimeout` e `Promise.resolve().then`.
- Explique por escrito por que a ordem aconteceu.
- Crie um exemplo em que uma tarefa pesada atrasa um callback.
