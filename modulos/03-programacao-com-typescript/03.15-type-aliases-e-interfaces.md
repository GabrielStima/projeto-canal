# Type Aliases e Interfaces

## Objetivo

Ao final desta aula, você deve ser capaz de:

- criar nomes para formatos de dados;
- usar `type` para aliases;
- usar `interface` para objetos;
- comparar usos comuns de `type` e `interface`;
- escolher uma opção sem transformar a diferença em regra absoluta.

## Ideia Central

Type aliases e interfaces permitem dar nome a formatos que aparecem no código. Isso evita repetir estruturas, melhora leitura e cria contratos mais claros entre funções, arquivos e partes da aplicação.

Na prática, ambos podem descrever objetos. `type` também nomeia unions, primitivos compostos e helpers. `interface` e muito usada para contratos de objetos e pode ser estendida ou reaberta.

## Exemplo Prático

```typescript
type StatusTarefa = "pendente" | "concluida";

interface Tarefa {
  titulo: string;
  status: StatusTarefa;
  prioridade?: "baixa" | "media" | "alta";
}

function concluirTarefa(tarefa: Tarefa): Tarefa {
  return {
    ...tarefa,
    status: "concluida",
  };
}
```

## Analogias e Intuições

- Alias é um apelido para um tipo.
- Interface é um contrato de formato para objetos.
- Nomear tipos ajuda a transformar dados soltos em conceitos do sistema.

## Erros Comuns

- Criar tipos com nomes genéricos demais.
- Separar `type` e `interface` por dogma em vez de contexto.
- Usar propriedade opcional para fugir de modelagem.
- Repetir o mesmo formato em várias funções.
- Criar tipos enormes cedo demais.
