# Type Aliases e Interfaces

## Objetivo

Ao final desta aula, voce deve ser capaz de:

- criar nomes para formatos de dados;
- usar `type` para aliases;
- usar `interface` para objetos;
- comparar usos comuns de `type` e `interface`;
- escolher uma opcao sem transformar a diferenca em regra absoluta.

## Ideia Central

Type aliases e interfaces permitem dar nome a formatos que aparecem no codigo. Isso evita repetir estruturas, melhora leitura e cria contratos mais claros entre funcoes, arquivos e partes da aplicacao.

Na pratica, ambos podem descrever objetos. `type` tambem nomeia unions, primitivos compostos e helpers. `interface` e muito usada para contratos de objetos e pode ser estendida ou reaberta.

## Exemplo Pratico

```typescript
type StatusTarefa = "pendente" | "concluida";

interface Tarefa {
  titulo: string;
  status: StatusTarefa;
  prioridade?: "baixa" | "media" | "alta";
}

function concluirTarefa(tarefa: Tarefa): Tarefa {
  return {
    ...tarefa,
    status: "concluida",
  };
}
```

## Analogias e Intuicoes

- Alias e um apelido para um tipo.
- Interface e um contrato de formato para objetos.
- Nomear tipos ajuda a transformar dados soltos em conceitos do sistema.

## Erros Comuns

- Criar tipos com nomes genericos demais.
- Separar `type` e `interface` por dogma em vez de contexto.
- Usar propriedade opcional para fugir de modelagem.
- Repetir o mesmo formato em varias funcoes.
- Criar tipos enormes cedo demais.
