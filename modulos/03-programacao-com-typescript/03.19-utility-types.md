# Utility Types

## Objetivo

Ao final desta aula, voce deve ser capaz de:

- explicar o que sao utility types;
- usar `Partial`, `Pick`, `Omit` e `Record` em exemplos simples;
- reaproveitar tipos existentes sem duplicar formatos;
- entender a relacao entre utility types e generics;
- evitar tipos derivados que escondem intencao.

## Ideia Central

Utility types sao tipos prontos do TypeScript para transformar outros tipos. Eles ajudam a criar variacoes de um formato sem repetir tudo manualmente.

Isso e util quando uma entidade aparece em contextos diferentes: criacao, atualizacao, exibicao, filtros e mapas.

## Topicos da Aula

- `Partial<T>`
- `Pick<T, K>`
- `Omit<T, K>`
- `Record<K, T>`
- Tipos derivados
- Reaproveitamento sem duplicacao
- Limites de legibilidade

## Exemplo Pratico

```typescript
type Usuario = {
  id: string;
  nome: string;
  email: string;
  ativo: boolean;
};

type AtualizacaoUsuario = Partial<Usuario>;
type UsuarioPublico = Omit<Usuario, "email">;
type ResumoUsuario = Pick<Usuario, "id" | "nome">;
type UsuariosPorId = Record<string, Usuario>;
```

## Analogias e Intuicoes

- Utility type e uma ferramenta para transformar formatos.
- `Pick` escolhe campos.
- `Omit` remove campos.
- `Partial` torna campos opcionais.

## Erros Comuns

- Usar `Partial` onde alguns campos deveriam continuar obrigatorios.
- Criar tipos derivados dificeis de ler.
- Usar utility type para evitar modelar um caso importante.
- Repetir tipos manualmente quando uma derivacao simples resolveria.
- Esquecer que tipos derivados nao validam dados em runtime.

## Checklist de Entendimento

Antes de seguir, voce deve conseguir responder:

- Crie um `Record` de produtos por codigo.
