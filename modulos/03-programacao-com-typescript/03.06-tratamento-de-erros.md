# Tratamento de Erros

## Objetivo

Ao final desta aula, você deve ser capaz de:

- diferenciar erro esperado e erro inesperado;
- usar `throw`, `try` e `catch` em exemplos simples;
- validar entradas antes de processar;
- entender por que erros fazem parte do fluxo real de sistemas;
- preparar a base para erros em código assíncrono.

## Onde Esta Aula Entra na Formação

- Módulo: Programação com TypeScript
- Aula anterior: Funções
- Próxima aula: Assincronismo em JavaScript
- Pré-requisitos: funções, condicionais, valores e tipos básicos

## Ideia Central

Programas lidam com entradas incompletas, estados inválidos e operações que podem falhar. Tratamento de erros é a prática de reconhecer essas situacoes e responder de forma controlada.

Nem todo erro deve ser escondido. Alguns precisam interromper o fluxo; outros podem ser convertidos em mensagens, tentativas novas ou caminhos alternativos.

## Tópicos da Aula

- Erro de sintaxe
- Erro em runtime
- Validacao simples
- `throw`
- `try/catch`
- Mensagens de erro
- Quando não usar `try/catch`

## Exemplo Prático

```javascript
function dividir(a, b) {
  if (b === 0) {
    throw new Error("Não é possível dividir por zero.");
  }

  return a / b;
}

try {
  console.log(dividir(10, 0));
} catch (erro) {
  console.log(erro.message);
}
```

## Analogias e Intuições

- Tratamento de erro é um plano para quando algo sai do caminho esperado.
- Validacao evita que uma função trabalhe com dados que não fazem sentido.
- `catch` não conserta magicamente o problema; ele permite responder a ele.

## Erros Comuns

- Usar `try/catch` para esconder erro sem entender a causa.
- Lancar strings em vez de objetos de erro.
- Validar tarde demais.
- Misturar erro de regra de negócio com erro técnico sem clareza.
- Tratar todo erro do mesmo jeito.

## Checklist de Entendimento

Antes de seguir, você deve conseguir responder:

- O que é erro em runtime?
- Quando usar `throw`?
- O que `try/catch` faz?
- Por que validar entrada?
- Quando um erro deve interromper o fluxo?

## Prática Recomendada

- Crie uma função que valida idade mínima.
- Lance erro quando o valor for invalido.
- Capture o erro e mostre uma mensagem simples.
- Compare um exemplo com validação antes e depois do processamento.
