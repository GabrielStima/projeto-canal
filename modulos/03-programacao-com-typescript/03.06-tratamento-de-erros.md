# Tratamento de Erros

## Objetivo

Ao final desta aula, voce deve ser capaz de:

- diferenciar erro esperado e erro inesperado;
- usar `throw`, `try` e `catch` em exemplos simples;
- validar entradas antes de processar;
- entender por que erros fazem parte do fluxo real de sistemas;
- preparar a base para erros em codigo assincrono.

## Onde Esta Aula Entra no Curso

- Modulo: Programacao com TypeScript
- Aula anterior: Funcoes
- Proxima aula: Assincronismo em JavaScript
- Pre-requisitos: funcoes, condicionais, valores e tipos basicos

## Ideia Central

Programas lidam com entradas incompletas, estados invalidos e operacoes que podem falhar. Tratamento de erros e a pratica de reconhecer essas situacoes e responder de forma controlada.

Nem todo erro deve ser escondido. Alguns precisam interromper o fluxo; outros podem ser convertidos em mensagens, tentativas novas ou caminhos alternativos.

## Topicos da Aula

- Erro de sintaxe
- Erro em runtime
- Validacao simples
- `throw`
- `try/catch`
- Mensagens de erro
- Quando nao usar `try/catch`

## Exemplo Pratico

```javascript
function dividir(a, b) {
  if (b === 0) {
    throw new Error("Nao e possivel dividir por zero.");
  }

  return a / b;
}

try {
  console.log(dividir(10, 0));
} catch (erro) {
  console.log(erro.message);
}
```

## Analogias e Intuicoes

- Tratamento de erro e um plano para quando algo sai do caminho esperado.
- Validacao evita que uma funcao trabalhe com dados que nao fazem sentido.
- `catch` nao conserta magicamente o problema; ele permite responder a ele.

## Erros Comuns

- Usar `try/catch` para esconder erro sem entender a causa.
- Lancar strings em vez de objetos de erro.
- Validar tarde demais.
- Misturar erro de regra de negocio com erro tecnico sem clareza.
- Tratar todo erro do mesmo jeito.

## Checklist de Entendimento

Antes de seguir, voce deve conseguir responder:

- O que e erro em runtime?
- Quando usar `throw`?
- O que `try/catch` faz?
- Por que validar entrada?
- Quando um erro deve interromper o fluxo?

## Pratica Recomendada

- Crie uma funcao que valida idade minima.
- Lance erro quando o valor for invalido.
- Capture o erro e mostre uma mensagem simples.
- Compare um exemplo com validacao antes e depois do processamento.
