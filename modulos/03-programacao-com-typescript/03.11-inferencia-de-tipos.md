# Inferência de Tipos

## Ideia Central

Inferência é a capacidade do TypeScript de descobrir um tipo a partir do valor ou do contexto. Se você declara `const nome = "Ana"`, o TypeScript entende que `nome` e texto sem você escrever `: string`.

Boas anotações ajudam. Anotações obvias demais poluem. Aprender TypeScript também é aprender quando deixar o compilador trabalhar.

## Exemplo Prático

```typescript
const nome = "Ana";
let pontos = 10;
const tecnologias = ["JavaScript", "TypeScript"];

function dobrar(numero: number) {
  return numero * 2;
}

const resultado = dobrar(pontos);
```

O TypeScript consegue inferir `nome`, `pontos`, `tecnologias` e `resultado` a partir dos valores e retornos.

## Analogias e Intuições

- Inferência é o TypeScript lendo pistas do código.
- Anotar tipo obvio e como repetir algo que já está claro.
- Anotar fronteiras importantes deixa a intenção mais firme.

## Erros Comuns

- Escrever `const nome: string = "Ana"` em todo lugar sem necessidade.
- Esperar inferência perfeita em dados vazios ou ambiguos.
- Criar arrays vazios sem tipo quando o uso futuro importa.
- Confundir inferência com conversão de tipo.
- Achar que inferência dispensa pensar nos dados.

## Checklist de Entendimento

Antes de seguir, você deve conseguir responder:

## Prática Recomendada

- Declare variáveis sem anotação e observe os tipos inferidos.
- Crie uma função com retorno inferido.
- Crie um array vazio e depois pense qual tipo ele deveria ter.
- Remova anotações redundantes de um exemplo simples.
