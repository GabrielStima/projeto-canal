# Compatibilidade de Tipos

## Objetivo

Ao final desta aula, voce deve ser capaz de:

- explicar a ideia de tipagem estrutural;
- entender por que dois objetos podem ser compativeis sem terem o mesmo nome;
- reconhecer propriedades obrigatorias e extras;
- prever atribuicoes validas em exemplos simples;
- preparar a base para interfaces, aliases e generics.

## Onde Esta Aula Entra no Curso

- Modulo: Programacao com TypeScript
- Aula anterior: Anotacoes de tipos e tipos em funcoes
- Proxima aula: Narrowing
- Pre-requisitos: objetos, funcoes, anotacoes de tipos e inferencia

## Ideia Central

TypeScript compara tipos principalmente pela estrutura. Se um valor tem as propriedades necessarias para ser usado em certo lugar, ele pode ser compativel, mesmo que tenha vindo de outro tipo nomeado.

Isso combina com a forma como JavaScript trabalha com objetos: o formato do valor importa muito.

## Topicos da Aula

- Tipagem estrutural
- Propriedades obrigatorias
- Propriedades extras
- Compatibilidade em parametros
- Compatibilidade em atribuicoes
- Excesso de propriedades em objetos literais
- Cuidados com flexibilidade demais

## Exemplo Pratico

```typescript
type UsuarioBasico = {
  nome: string;
};

const usuarioCompleto = {
  nome: "Ana",
  email: "ana@example.com",
};

function mostrarNome(usuario: UsuarioBasico) {
  console.log(usuario.nome);
}

mostrarNome(usuarioCompleto);
```

O objeto tem mais propriedades do que o necessario, mas possui a estrutura minima esperada.

## Analogias e Intuicoes

- TypeScript pergunta: este valor tem o formato necessario?
- O nome do tipo ajuda humanos, mas a estrutura decide muita coisa.
- Flexibilidade e boa, desde que voce saiba qual contrato importa.

## Erros Comuns

- Esperar que tipos sejam comparados apenas pelo nome.
- Achar estranho um objeto com propriedades extras ser aceito.
- Usar tipos amplos demais e perder protecao.
- Ignorar mensagens de excesso de propriedades em literais.
- Confundir compatibilidade estrutural com validacao de dados externos.

## Checklist de Entendimento

Antes de seguir, voce deve conseguir responder:

- O que e tipagem estrutural?
- Por que um objeto com propriedade extra pode ser aceito?
- Quando um objeto nao e compativel?
- O que uma funcao realmente exige de um parametro?
- Compatibilidade de tipos valida dados em runtime?

## Pratica Recomendada

- Crie um tipo com uma propriedade obrigatoria.
- Passe um objeto com propriedades extras para uma funcao.
- Remova uma propriedade obrigatoria e observe o erro.
- Explique por escrito por que o TypeScript aceitou ou rejeitou cada caso.
