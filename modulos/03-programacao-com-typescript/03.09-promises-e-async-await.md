## Ideia Central

Uma Promise representa uma operação que ainda pode terminar com sucesso ou falha. `async/await` permite escrever esse fluxo de forma mais parecida com código sequencial, mas sem deixar de ser assíncrono.

Esse padrão será usado constantemente em APIs, bancos de dados, frontend moderno, Next.js e backend.

## Exemplo Prático

```javascript
function buscarResumoDoPet(id) {
  return new Promise((resolve, reject) => {
    setTimeout(() => {
      if (id === "pet-001") {
        resolve({ nome: "Thor", status: "Em Triagem" });
      } else {
        reject(new Error("Pet não encontrado."));
      }
    }, 500);
  });
}

async function mostrarStatusDoPet() {
  try {
    const pet = await buscarResumoDoPet("pet-001");
    console.log(`${pet.nome}: ${pet.status}`);
  } catch (erro) {
    console.log("Falha ao buscar status:", erro.message);
  }
}

mostrarStatusDoPet();
```

## Analogias e Intuições

- Promise é uma promessa de resposta futura.
- `await` pausa aquela função, não o JavaScript inteiro.
- `async` sinaliza que a função trabalha com resultado futuro.

## Erros Comuns

- Esquecer `await` e tentar usar a Promise como valor final.
- Usar `await` fora de contexto sem entender o ambiente.
- Ignorar `catch` ou `try/catch`.
- Misturar muitos estilos sem necessidade.
- Achar que `async/await` transforma assincronismo em código síncrono.
