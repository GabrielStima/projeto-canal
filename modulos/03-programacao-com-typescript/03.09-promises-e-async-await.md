## Ideia Central

Uma Promise representa uma operacao que ainda pode terminar com sucesso ou falha. `async/await` permite escrever esse fluxo de forma mais parecida com codigo sequencial, mas sem deixar de ser assincrono.

Esse padrao sera usado constantemente em APIs, bancos de dados, frontend moderno, Next.js e backend.

## Exemplo Pratico

```javascript
function buscarUsuario() {
  return new Promise((resolve) => {
    setTimeout(() => {
      resolve({ nome: "Ana", ativo: true });
    }, 500);
  });
}

async function mostrarUsuario() {
  try {
    const usuario = await buscarUsuario();
    console.log(usuario.nome);
  } catch (erro) {
    console.log("Falha ao buscar usuario.");
  }
}

mostrarUsuario();
```

## Analogias e Intuicoes

- Promise e uma promessa de resposta futura.
- `await` pausa aquela funcao, nao o JavaScript inteiro.
- `async` sinaliza que a funcao trabalha com resultado futuro.

## Erros Comuns

- Esquecer `await` e tentar usar a Promise como valor final.
- Usar `await` fora de contexto sem entender o ambiente.
- Ignorar `catch` ou `try/catch`.
- Misturar muitos estilos sem necessidade.
- Achar que `async/await` transforma assincronismo em codigo sincronico.
