# Funcoes

## Objetivo

Ao final desta aula, voce deve ser capaz de:

- explicar por que funcoes existem;
- declarar e chamar funcoes;
- usar parametros e retorno;
- diferenciar efeito colateral e valor retornado;
- preparar a base para tipos em funcoes no TypeScript.

## Onde Esta Aula Entra no Curso

- Modulo: Programacao com TypeScript
- Aula anterior: Estruturas de controle
- Proxima aula: Tratamento de erros
- Pre-requisitos: valores, variaveis, tipos basicos e estruturas de controle

## Ideia Central

Funcoes permitem dar nome a uma operacao. Em vez de repetir o mesmo bloco de codigo em varios lugares, criamos uma funcao que recebe entradas, executa uma logica e pode devolver uma saida.

Uma boa funcao torna o codigo mais facil de ler, testar e reutilizar.

## Topicos da Aula

- Declaracao de funcao
- Chamada de funcao
- Parametros
- Retorno
- Arrow functions
- Escopo local
- Efeitos colaterais

## Exemplo Pratico

```javascript
function calcularTotal(preco, quantidade) {
  return preco * quantidade;
}

const formatarResumo = (nome, total) => {
  return `${nome}: R$ ${total}`;
};

const total = calcularTotal(25, 3);
console.log(formatarResumo("Camiseta", total));
```

## Analogias e Intuicoes

- Funcao e uma receita com nome.
- Parametros sao os ingredientes que variam.
- Retorno e o resultado produzido pela receita.

## Erros Comuns

- Criar funcoes que fazem muitas coisas ao mesmo tempo.
- Confundir `return` com `console.log`.
- Depender de variaveis externas sem necessidade.
- Nomear funcoes com substantivos em vez de acoes.
- Retornar tipos diferentes sem uma razao clara.

## Checklist de Entendimento

Antes de seguir, voce deve conseguir responder:

- Para que serve uma funcao?
- O que e um parametro?
- O que e retorno?
- Qual a diferenca entre imprimir e retornar?
- O que e escopo local?

## Pratica Recomendada

- Crie uma funcao que calcula desconto.
- Crie uma funcao que recebe nome e retorna uma saudacao.
- Reescreva um bloco repetido usando funcao.
- Crie uma funcao pequena sem efeito colateral.
