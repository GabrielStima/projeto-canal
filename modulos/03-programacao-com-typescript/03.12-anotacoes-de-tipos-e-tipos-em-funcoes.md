# Anotacoes de Tipos e Tipos em Funcoes

## Objetivo

Ao final desta aula, voce deve ser capaz de:

- anotar tipos em variaveis quando necessario;
- tipar parametros de funcoes;
- tipar retornos de funcoes;
- usar parametros opcionais com cuidado;
- entender por que funcoes sao uma das fronteiras mais importantes para tipos.

## Onde Esta Aula Entra no Curso

- Modulo: Programacao com TypeScript
- Aula anterior: Inferencia de tipos
- Proxima aula: Compatibilidade de tipos
- Pre-requisitos: inferencia, funcoes, tipos primitivos, arrays e objetos simples

## Ideia Central

Anotacoes de tipo tornam expectativas explicitas. Em funcoes, isso e especialmente importante porque parametros sao entradas e retorno e saida. Se essas fronteiras ficam claras, o restante do codigo fica mais facil de usar e revisar.

O objetivo nao e anotar tudo, mas anotar onde a intencao precisa ficar evidente.

## Topicos da Aula

- Sintaxe de anotacao
- Parametros tipados
- Retorno tipado
- Parametros opcionais
- Valores padrao
- Funcoes sem retorno util
- Tipos de arrays em parametros

## Exemplo Pratico

```typescript
function calcularDesconto(preco: number, percentual: number): number {
  return preco - preco * percentual;
}

function criarMensagem(nome: string, saudacao = "Ola"): string {
  return `${saudacao}, ${nome}`;
}

function registrarLog(mensagem: string): void {
  console.log(mensagem);
}
```

## Analogias e Intuicoes

- Tipar uma funcao e explicar que entrada ela aceita e que saida promete.
- Parametro opcional deve ser uma escolha, nao uma desculpa para dado incerto.
- `void` indica que a funcao produz efeito, nao um valor util.

## Erros Comuns

- Anotar retorno errado e brigar com o compilador.
- Usar muitos parametros opcionais sem regra clara.
- Retornar `undefined` sem querer.
- Usar `any` para escapar de uma duvida simples.
- Esquecer que dados externos ainda podem chegar invalidos.

## Checklist de Entendimento

Antes de seguir, voce deve conseguir responder:

- Onde fica a anotacao de parametro?
- Onde fica a anotacao de retorno?
- Quando usar parametro opcional?
- O que significa `void`?
- Por que funcoes sao fronteiras importantes?

## Pratica Recomendada

- Tipar uma funcao que soma dois numeros.
- Tipar uma funcao que recebe uma lista de nomes.
- Criar uma funcao com parametro opcional.
- Criar uma funcao que apenas imprime uma mensagem e retorna `void`.
