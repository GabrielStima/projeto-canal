# Anotações de Tipos e Tipos em Funções

## Objetivo

Ao final desta aula, você deve ser capaz de:

- anotar tipos em variáveis quando necessário;
- tipar parâmetros de funções;
- tipar retornos de funções;
- usar parâmetros opcionais com cuidado;
- entender por que funções são uma das fronteiras mais importantes para tipos.

## Onde Esta Aula Entra na Formação

- Módulo: Programação com TypeScript
- Aula anterior: Inferência de tipos
- Próxima aula: Compatibilidade de tipos
- Pré-requisitos: inferência, funções, tipos primitivos, arrays e objetos simples

## Ideia Central

Anotações de tipo tornam expectativas explícitas. Em funções, isso é especialmente importante porque parâmetros são entradas e retorno é saída. Se essas fronteiras ficam claras, o restante do código fica mais fácil de usar e revisar.

O objetivo não é anotar tudo, mas anotar onde a intenção precisa ficar evidente.

## Tópicos da Aula

- Sintaxe de anotação
- Parametros tipados
- Retorno tipado
- Parametros opcionais
- Valores padrão

- Tipos de arrays em parâmetros

## Exemplo Prático

```typescript
function calcularDesconto(preco: number, percentual: number): number {
  return preco - preco * percentual;
}

function criarMensagem(nome: string, saudacao = "Olá"): string {
  return `${saudacao}, ${nome}`;
}

function registrarLog(mensagem: string): void {
  console.log(mensagem);
}

function calcularTotalDoItem(preco: number, quantidade: number): number {
  return preco * quantidade;
}

function formatarLinhaDoOrcamento(
  descricao: string,
  total: number,
  observacao?: string,
): string {
  const linha = `${descricao}: R$ ${total.toFixed(2)}`;
  return observacao ? `${linha} (${observacao})` : linha;
}
```

## Analogias e Intuições

- Tipar uma função e explicar que entrada ela aceita e que saída promete.
- Parametro opcional deve ser uma escolha, não uma desculpa para dado incerto.
- `void` indica que a função produz efeito, não um valor útil.

## Erros Comuns

- Anotar retorno errado e brigar com o compilador.
- Usar muitos parâmetros opcionais sem regra clara.
- Retornar `undefined` sem querer.
- Usar `any` para escapar de uma dúvida simples.
- Esquecer que dados externos ainda podem chegar inválidos.

## Checklist de Entendimento

Antes de seguir, você deve conseguir responder:

- Onde fica a anotação de parâmetro?
- Onde fica a anotação de retorno?

- Por que funções são fronteiras importantes?

## Prática Recomendada

- Tipar uma função que soma dois numeros.
- Tipar uma função que recebe uma lista de nomes.
- Criar uma função com parâmetro opcional.
- Criar uma função que apenas imprime uma mensagem e retorna `void`.
- Crie uma função `calcularTotalDoItem` que recebe preço e quantidade e retorna o total.
- Crie uma função `formatarLinhaDoOrcamento` com descrição, total e observação opcional.
- Teste ambas com dados de um orçamento veterinário realista.
