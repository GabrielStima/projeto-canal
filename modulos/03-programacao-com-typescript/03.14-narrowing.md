## Ideia Central

Narrowing é o processo de reduzir um tipo amplo para um tipo mais especifico com base em uma checagem. Se uma variável pode ser `string` ou `number`, uma condicao com `typeof` permite tratar cada caso com segurança.

Isso é essencial porque dados reais frequentemente podem assumir mais de um formato.

## Exemplo Prático

```typescript
function imprimirIdentificador(id: string | number) {
  if (typeof id === "string") {
    console.log(id.toUpperCase());
    return;
  }

  console.log(id.toFixed(0));
}

function descreverPet(pet: { nome: string; historico?: string }) {
  if (pet.historico) {
    return `${pet.nome}: ${pet.historico}`;
  }

  return `${pet.nome}: sem histórico registrado.`;
}
```

## Analogias e Intuições

- Narrowing é perguntar antes de usar.
- Union type representa possibilidades.
- A checagem elimina possibilidades até sobrar um caso seguro.

## Erros Comuns

- Acessar método de `string` sem checar se o valor e `string`.
- Usar `as` para forcar tipo em vez de checar.
- Criar unions grandes demais sem discriminador claro.
- Esquecer de tratar todos os casos.
- Confundir narrowing com conversão de valor.
