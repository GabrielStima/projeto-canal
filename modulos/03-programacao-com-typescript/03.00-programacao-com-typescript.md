## Ideia Central

TypeScript não é uma linguagem diferente de JavaScript — é JavaScript com um sistema de tipos. Antes de entrar nos tipos, é preciso ter uma base sólida de JavaScript moderno: arrow functions, destructuring, spread, módulos, Promises, async/await, e como o event loop funciona. Esses fundamentos aparecem em todo código real.

Com essa base, o sistema de tipos do TypeScript deixa de ser um obstáculo e passa a ser uma ferramenta de raciocínio. Tipos descrevem o formato dos dados: o que uma função recebe, o que ela devolve, quais campos um objeto tem. Isso torna erros detectáveis antes de executar o código.

Este módulo cobre a linguagem de ponta a ponta — de JavaScript moderno a generics e utility types — para que você possa usá-la com confiança em frontend, backend e em qualquer ponto do sistema.

## Analogias e Intuições

- TypeScript é como um formulário com campos obrigatórios: sem os campos certos, o código nem compila — você sabe do erro antes de colocar em produção.
- Inferência de tipos é o compilador fazendo dedução lógica: se você atribui `42` a uma variável, ele sabe que é `number` sem você precisar escrever.
- Generics são como moldes: você define a forma uma vez e preenche o tipo concreto quando usa.

## Erros Comuns

- Usar `any` para fugir de erros de tipo em vez de modelar corretamente.
- Confundir `interface` e `type alias` sem entender quando cada um é mais adequado.
- Ignorar o tratamento de erros em código assíncrono e deixar Promises silenciosamente rejeitadas.
- Não entender que TypeScript é apagado em runtime — tipos existem só em tempo de compilação.
- Criar tipos excessivamente complexos antes de entender os fundamentos de compatibilidade e estrutura.
