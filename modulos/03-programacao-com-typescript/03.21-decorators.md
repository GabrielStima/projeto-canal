## Ideia Central

Decorators sao uma forma de anexar comportamento ou metadados a classes, metodos, propriedades ou parametros, dependendo do modelo usado. Eles aparecem bastante em frameworks e bibliotecas que precisam descrever rotas, entidades, validacoes ou injecao de dependencias.

Nesta fase, o objetivo e reconhecer a ideia e o formato. O uso profundo fica para momentos em que frameworks especificos exigirem esse recurso.

## Exemplo Pratico

```typescript
function Entidade(nome: string) {
  return function (classe: Function) {
    console.log(`Registrando entidade: ${nome}`);
  };
}

@Entidade("usuarios")
class Usuario {
  constructor(public nome: string) {}
}
```

Este exemplo e introdutorio. Em projetos reais, decorators costumam depender de configuracao e do framework usado.

## Analogias e Intuicoes

- Decorator e uma anotacao executavel sobre uma declaracao.
- Ele marca algo para outra parte do sistema interpretar.
- Frameworks usam decorators para reduzir configuracao repetitiva.

## Erros Comuns

- Usar decorators antes de entender classes e funcoes.
- Achar que decorator e obrigatorio para escrever TypeScript.
- Copiar decorators de framework sem entender o que eles registram.
- Ignorar configuracao necessaria.
- Usar decorator para esconder logica que deveria ser explicita.
