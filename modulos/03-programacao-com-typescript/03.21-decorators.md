## Ideia Central

Decorators são uma forma de anexar comportamento ou metadados a classes, métodos, propriedades ou parâmetros, dependendo do modelo usado. Eles aparecem bastante em frameworks e bibliotecas que precisam descrever rotas, entidades, validações ou injeção de dependencias.

Nesta fase, o objetivo e reconhecer a ideia é o formato. O uso profundo fica para momentos em que frameworks especificos exigirem esse recurso.

## Exemplo Prático

```typescript
function Entidade(nome: string) {
  return function (classe: Function) {
    console.log(`Registrando entidade: ${nome}`);
  };
}

@Entidade("usuarios")
class Usuario {
  constructor(public nome: string) {}
}
```

Este exemplo e introdutório. Em projetos reais, decorators costumam depender de configuração e do framework usado.

## Analogias e Intuições

- Decorator é uma anotação executavel sobre uma declaracao.
- Ele marca algo para outra parte do sistema interpretar.
- Frameworks usam decorators para reduzir configuração repetitiva.

## Erros Comuns

- Usar decorators antes de entender classes e funções.
- Achar que decorator e obrigatório para escrever TypeScript.
- Copiar decorators de framework sem entender o que éles registram.
- Ignorar configuração necessária.
- Usar decorator para esconder lógica que deveria ser explícita.
