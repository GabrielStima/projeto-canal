# Estruturas de Controle

## Objetivo

Ao final desta aula, você deve ser capaz de:

- usar condicionais para tomar decisões;
- usar loops para repetir operações;
- entender como o fluxo do programa muda;
- evitar repeticoes desnecessarias;
- preparar exemplos para funções e tratamento de erros.

## Ideia Central

Um programa raramente executa tudo de cima para baixo sem escolhas. Estruturas de controle permitem decidir quando algo deve acontecer e quantas vezes uma operação deve se repetir.

Condicionais lidam com possibilidades. Loops lidam com repetição.

## Tópicos da Aula

- `if`, `else if` e `else`
- Comparacoes
- Operadores logicos
- `switch`
- `for`
- `for...of`
- `while`

## Exemplo Prático

```javascript
const atendimentos = [
  { pet: "Thor", status: "Em Triagem" },
  { pet: "Luna", status: "Aguardando Aprovação" },
  { pet: "Fred", status: "Finalizado" },
];

for (const atendimento of atendimentos) {
  if (atendimento.status === "Em Triagem") {
    console.log(`${atendimento.pet}: priorizar avaliação clínica.`);
  } else if (atendimento.status === "Aguardando Aprovação") {
    console.log(`${atendimento.pet}: aguardando resposta do tutor.`);
  } else {
    console.log(`${atendimento.pet}: atendimento finalizado.`);
  }
}
```

## Analogias e Intuições

- Condicional é uma bifurcacao no caminho.
- Loop é uma rotina repetida enquanto existe trabalho a fazer.
- Operadores logicos combinam critérios de decisão.

## Erros Comuns

- Criar condicionais que nunca são atingidas.
- Confundir `=` com `===`.
- Esquecer de atualizar a condicao de um `while`.
- Usar loop quando um método de array seria mais legível depois.
- Escrever muitas condicoes aninhadas sem necessidade.

## Prática Recomendada

- Crie uma lista de notas e mostre quais foram aprovadas.
- Escreva uma condicao com dois critérios.
- Use `switch` para tratar status de tarefa.
- Crie um loop que percorre uma lista de objetos.
- Crie uma lista de atendimentos com `pet` e `status`.
- Use `switch` ou `if/else` para exibir uma mensagem diferente para cada status do ciclo de atendimento.
