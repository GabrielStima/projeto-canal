# Projeto Pratico: Mini Aplicacao Tipada

## Objetivo

Ao final desta aula, voce deve ser capaz de:

- integrar JavaScript moderno e TypeScript em um pequeno projeto;
- modelar dados com aliases, interfaces e unions;
- organizar codigo em modulos;
- usar funcoes tipadas, tratamento de erros e assincronismo;
- revisar o modulo inteiro por meio de uma pratica concreta.

## Onde Esta Aula Entra no Curso

- Modulo: Programacao com TypeScript
- Aula anterior: Decorators
- Proxima aula: Algoritmos e Estruturas de Dados
- Pre-requisitos: todas as aulas anteriores do modulo

## Ideia Central

Esta aula fecha o modulo juntando as ideias principais em uma mini aplicacao de tarefas. O objetivo nao e criar uma interface bonita nem usar framework. O foco e provar que voce consegue modelar dados, escrever funcoes, lidar com erros, simular assincronismo e organizar codigo tipado.

O projeto tambem prepara a transicao para algoritmos e estruturas de dados, porque voce passa a manipular listas, filtros, buscas e transformacoes com mais consciencia.

## Topicos da Aula

- Planejamento do dominio
- Modelagem de tarefa
- Funcoes de criacao e atualizacao
- Tratamento de entradas invalidas
- Filtros por status e prioridade
- Simulacao assincrona
- Organizacao em modulos
- Revisao dos tipos usados

## Exemplo Pratico

```typescript
type StatusTarefa = "pendente" | "concluida";
type Prioridade = "baixa" | "media" | "alta";

interface Tarefa {
  id: string;
  titulo: string;
  status: StatusTarefa;
  prioridade: Prioridade;
}

function criarTarefa(titulo: string, prioridade: Prioridade): Tarefa {
  if (titulo.trim() === "") {
    throw new Error("Titulo e obrigatorio.");
  }

  return {
    id: crypto.randomUUID(),
    titulo,
    status: "pendente",
    prioridade,
  };
}
```

## Analogias e Intuicoes

- O projeto e um ensaio geral do modulo.
- Cada tipo nomeado representa uma decisao sobre o dominio.
- Codigo tipado bom ainda precisa de regras claras e validacao.

## Erros Comuns

- Comecar pelo codigo antes de modelar a tarefa.
- Usar `any` quando os formatos ja sao conhecidos.
- Criar tipos avancados sem necessidade.
- Esquecer tratamento de erro para titulo vazio.
- Misturar toda a aplicacao em um unico arquivo sem criterio.

## Checklist de Entendimento

Antes de seguir, voce deve conseguir responder:

- Quais tipos representam o dominio da mini aplicacao?
- Quais funcoes manipulam tarefas?
- Onde erros podem acontecer?
- Onde assincronismo foi usado?
- Como os modulos separam responsabilidades?

## Pratica Recomendada

- Crie uma lista de tarefas em memoria.
- Implemente `criarTarefa`, `concluirTarefa`, `listarTarefas` e `filtrarPorStatus`.
- Lance erro para titulo vazio.
- Simule salvamento assincrono com Promise.
- Separe tipos e funcoes em modulos.
- Revise onde inferencia foi suficiente e onde anotacoes ajudaram.
