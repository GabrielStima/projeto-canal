# Projeto Prático: Mini Aplicação Tipada

## Objetivo

Ao final desta aula, você deve ser capaz de:

- integrar JavaScript moderno e TypeScript em um pequeno projeto;
- modelar dados com aliases, interfaces e unions;
- organizar código em módulos;
- usar funções tipadas, tratamento de erros e assincronismo;
- revisar o módulo inteiro por meio de uma prática concreta.

## Onde Esta Aula Entra na Formação

- Módulo: Programação com TypeScript
- Aula anterior: Decorators
- Próxima aula: Algoritmos e Estruturas de Dados
- Pré-requisitos: todas as aulas anteriores do módulo

## Ideia Central

Esta aula fecha o módulo juntando as ideias principais em uma mini aplicação de tarefas. O objetivo não é criar uma interface bonita nem usar framework. O foco é provar que você consegue modelar dados, escrever funções, lidar com erros, simular assincronismo e organizar código tipado.

O projeto também prepara a transição para algoritmos e estruturas de dados, porque você passa a manipular listas, filtros, buscas e transformações com mais consciência.

## Tópicos da Aula

- Planejamento do domínio
- Modelagem de tarefa
- Funções de criação e atualização
- Tratamento de entradas inválidas
- Filtros por status e prioridade
- Simulação assíncrona
- Organização em módulos
- Revisão dos tipos usados

## Exemplo Prático

```typescript
type StatusTarefa = "pendente" | "concluida";
type Prioridade = "baixa" | "media" | "alta";

interface Tarefa {
  id: string;
  titulo: string;
  status: StatusTarefa;
  prioridade: Prioridade;
}

function criarTarefa(titulo: string, prioridade: Prioridade): Tarefa {
  if (titulo.trim() === "") {
    throw new Error("Título é obrigatório.");
  }

  return {
    id: crypto.randomUUID(),
    titulo,
    status: "pendente",
    prioridade,
  };
}
```

## Analogias e Intuições

- O projeto é um ensaio geral do módulo.
- Cada tipo nomeado representa uma decisão sobre o domínio.
- Código tipado bom ainda precisa de regras claras e validação.

## Erros Comuns

- Começar pelo código antes de modelar a tarefa.
- Usar `any` quando os formatos já são conhecidos.
- Criar tipos avançados sem necessidade.
- Esquecer tratamento de erro para título vazio.
- Misturar toda a aplicação em um único arquivo sem critério.

## Checklist de Entendimento

Antes de seguir, você deve conseguir responder:

- Quais tipos representam o domínio da mini aplicação?
- Quais funções manipulam tarefas?
- Onde erros podem acontecer?
- Onde assincronismo foi usado?
- Como os módulos separam responsabilidades?

## Prática Recomendada

- Crie uma lista de tarefas em memória.
- Implemente `criarTarefa`, `concluirTarefa`, `listarTarefas` e `filtrarPorStatus`.
- Lance erro para título vazio.
- Simule salvamento assíncrono com Promise.
- Separe tipos e funções em módulos.
- Revise onde inferência foi suficiente e onde anotações ajudaram.
