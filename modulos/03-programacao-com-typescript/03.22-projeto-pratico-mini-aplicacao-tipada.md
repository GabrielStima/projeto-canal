# Projeto Prático: Núcleo TypeScript do PetCare OS

## Objetivo

Ao final desta aula, você deve ser capaz de:

- integrar JavaScript moderno e TypeScript em um pequeno projeto coeso;
- modelar dados com aliases, interfaces e unions;
- organizar código em módulos;
- usar funções tipadas, tratamento de erros e assincronismo;
- revisar o módulo inteiro por meio de uma prática concreta.

## Onde Esta Aula Entra na Formação

- Módulo: Programação com TypeScript
- Aula anterior: Decorators
- Próxima aula: Algoritmos e Estruturas de Dados
- Pré-requisitos: todas as aulas anteriores do módulo

## Ideia Central

Esta aula fecha o módulo juntando as ideias principais em um núcleo TypeScript do PetCare OS. O objetivo não é criar uma interface bonita nem usar framework. O foco é provar que você consegue modelar dados, escrever funções, lidar com erros, simular assincronismo e organizar código tipado.

Você vai produzir um pequeno conjunto de arquivos TypeScript que representam entidades e regras do PetCare OS: pets, tutores, atendimentos, orçamentos e status. Esse trabalho será retomado no módulo de Algoritmos e Estruturas de Dados, quando você implementará a fila de prioridade da triagem, e nos módulos futuros de frontend e backend.

## Tópicos da Aula

- Planejamento do domínio do PetCare OS.
- Modelagem de `Pet`, `Tutor`, `Atendimento` e `Orcamento`.
- Funções de criação e atualização de atendimento.
- Cálculo de orçamento com funções tipadas.
- Tratamento de entradas inválidas.
- Simulação assíncrona de busca de dados.
- Organização em módulos.
- Revisão dos tipos usados.

## Exemplo Prático

```typescript
// tipos.ts
export type StatusAtendimento =
  | "Recebido"
  | "Em Triagem"
  | "Aguardando Aprovação"
  | "Em Procedimento"
  | "Pronto para Alta"
  | "Finalizado";

export interface Pet {
  id: number;
  nome: string;
  especie: string;
  tutorId: number;
}

export interface Tutor {
  id: number;
  nome: string;
  email: string;
}

export interface Atendimento {
  id: number;
  petId: number;
  status: StatusAtendimento;
  motivo?: string;
}
```

```typescript
// atendimento.ts
import { Atendimento, StatusAtendimento } from "./tipos";

export function atualizarStatus(
  atendimento: Atendimento,
  novoStatus: StatusAtendimento,
): Atendimento {
  return {
    ...atendimento,
    status: novoStatus,
  };
}
```

```typescript
// orcamento.ts
export interface ItemOrcamento {
  descricao: string;
  preco: number;
  quantidade: number;
}

export function calcularTotal(itens: ItemOrcamento[]): number {
  return itens.reduce(
    (total, item) => total + item.preco * item.quantidade,
    0,
  );
}
```

## Analogias e Intuições

- O projeto é um ensaio geral do módulo.
- Cada tipo nomeado representa uma decisão sobre o domínio.
- Código tipado bom ainda precisa de regras claras e validação.

## Erros Comuns

- Começar pelo código antes de modelar as entidades.
- Usar `any` quando os formatos já são conhecidos.
- Criar tipos avançados sem necessidade.
- Esquecer tratamento de erro para dados inválidos.
- Misturar toda a aplicação em um único arquivo sem critério.

## Checklist de Entendimento

Antes de seguir, você deve conseguir responder:

- Quais tipos representam o domínio do núcleo?
- Quais funções manipulam atendimentos e orçamentos?
- Onde erros podem acontecer?
- Onde assincronismo foi usado?
- Como os módulos separam responsabilidades?
