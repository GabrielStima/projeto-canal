# Mocks, Stubs e Fakes

## Objetivo

Ao final desta aula, voce deve ser capaz de:

- diferenciar mocks, stubs e fakes em nivel pratico;
- reconhecer quando substituir uma dependencia em teste;
- evitar esconder problemas reais com doubles demais;
- escolher o double mais simples para cada cenario.

## Onde Esta Aula Entra no Curso

- Modulo: 13. Qualidade, Testes, Observabilidade e Performance
- Aula anterior: Testes Unitarios
- Proxima aula: Testes de Integracao
- Pre-requisitos: Testes unitarios, TypeScript, funcoes, dependencias, APIs, banco de dados e estrategia de testes.

## Ideia Central

Doubles de teste substituem dependencias para tornar um teste mais controlavel. Um stub devolve uma resposta preparada. Um fake implementa uma versao simples, mas funcional, de uma dependencia. Um mock tambem verifica interacoes esperadas.

Eles sao uteis quando uma dependencia e lenta, externa, cara, instavel ou dificil de controlar. Mas tambem podem deixar testes irreais se substituirem justamente aquilo que precisava ser validado.

## Topicos da Aula

- Dependencias em testes.
- Stub.
- Fake.
- Mock.
- Interacao vs resultado.
- Risco de excesso de doubles.

## Exemplo Pratico

```ts
type Mailer = {
  sendWelcomeEmail(email: string): Promise<void>;
};

async function registerUser(email: string, mailer: Mailer) {
  await mailer.sendWelcomeEmail(email);
  return { email, status: "created" };
}
```

No teste, o envio real de email pode ser substituido por um double.

## Analogias e Intuicoes

- Um double e um dublê: ajuda no ensaio, mas nao prova tudo sobre a cena real.
- Quanto mais falso o ambiente, mais cuidado voce precisa ter com a conclusao.

## Erros Comuns

- Usar mock para tudo por padrao.
- Testar se uma funcao interna foi chamada sem validar comportamento.
- Criar fake mais complexo que o codigo real.
- Substituir banco sempre e nunca testar integracao real.
- Deixar o teste passar com contratos irreais.

## Checklist de Entendimento

Antes de seguir, voce deve conseguir responder:

- Qual dependencia esta sendo substituida?
- Por que ela precisa ser substituida neste teste?
- O teste valida resultado ou interacao?
- O double representa um comportamento realista?
- Existe um teste de integracao cobrindo a dependencia real?

## Pratica Recomendada

- Escolha uma funcao que envie email ou chame API externa.
- Crie um stub de sucesso e um stub de falha.
- Liste o que esse teste cobre.
- Liste o que ele nao cobre e precisaria de outro nivel de teste.
