# Logging para Observabilidade

## Objetivo

Ao final desta aula, voce deve ser capaz de:

- explicar logs como sinal de observabilidade;
- diferenciar log util de ruido;
- escolher eventos relevantes para registrar;
- incluir contexto sem vazar dados sensiveis.

## Onde Esta Aula Entra no Curso

- Modulo: 13. Qualidade, Testes, Observabilidade e Performance
- Aula anterior: Testes de Carga
- Proxima aula: Observabilidade
- Pre-requisitos: Logging no backend, erros padronizados, autenticacao, seguranca em APIs, dados sensiveis, failure modes e analise de performance.

## Ideia Central

Logging para observabilidade nao e espalhar mensagens aleatorias no codigo. E registrar eventos que ajudam a entender o comportamento do sistema quando algo acontece longe dos seus olhos.

Bons logs tem nivel, evento, contexto de requisicao, identificadores relevantes, duracao e cuidado com dados sensiveis. Eles ajudam a investigar falhas, auditoria e comportamento operacional.

## Topicos da Aula

- Log como sinal.
- Evento relevante.
- Nivel de log.
- Contexto de requisicao.
- Logs estruturados.
- Dados sensiveis.

## Exemplo Pratico

```json
{
  "level": "warn",
  "event": "progress_update_denied",
  "requestId": "req_123",
  "userId": "user_1",
  "targetUserId": "user_2",
  "reason": "forbidden"
}
```

O log ajuda a investigar uma tentativa negada sem registrar senha, token ou dados desnecessarios.

## Analogias e Intuicoes

- Log util e uma pista com contexto.
- Log demais sem criterio vira neblina operacional.

## Erros Comuns

- Registrar dados sensiveis.
- Logar apenas erros inesperados.
- Esquecer requestId ou correlacao.
- Usar mensagens soltas impossiveis de filtrar.
- Criar logs que nao respondem perguntas de investigacao.

## Checklist de Entendimento

Antes de seguir, voce deve conseguir responder:

- Que evento merece log?
- Qual nivel faz sentido?
- Que contexto ajuda a investigar?
- Que dado nao deve aparecer?
- Como esse log seria filtrado depois?

## Pratica Recomendada

- Escolha tres endpoints.
- Liste eventos de sucesso, aviso e erro.
- Escreva um log estruturado para cada endpoint.
- Remova qualquer dado sensivel ou inutil.
