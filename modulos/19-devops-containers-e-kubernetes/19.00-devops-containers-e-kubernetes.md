# DevOps, Containers e Kubernetes

## Objetivo

Ao final desta aula, você deve ser capaz de:

- explicar o conceito principal da aula;
- reconhecer quando ele aparece em projetos reais;
- aplicar a ideia em um exemplo simples de entrega, container ou Kubernetes;
- saber o que estudar em seguida.

## Onde Esta Aula Entra no Curso

- Módulo: 19. DevOps, Containers e Kubernetes
- Aula anterior: Projeto Prático: Provisionando a Infraestrutura de uma Aplicação
- Próxima aula: O que é DevOps na Prática
- Pré-requisitos: Git, GitHub, terminal, variáveis de ambiente e code review; Backend com Node.js, APIs, bancos de dados, segurança e observabilidade; Cloud Fundamentals e Infrastructure as Code; Terraform, state, plan, apply, ambientes e segredos.

## Ideia Central

Aplicações modernas não terminam quando o código compila. Elas precisam ser empacotadas, entregues, configuradas, observadas e operadas em ambientes que mudam o tempo todo. Este módulo organiza esse caminho: primeiro a entrega, depois containers, depois Kubernetes.

## Tópicos da Aula

- O problema entre código pronto e aplicação rodando
- DevOps como colaboração, automação e responsabilidade compartilhada
- CI/CD como caminho de entrega
- Containers como unidade de empacotamento
- Kubernetes como orquestração de containers
- O que será aprofundado nas próximas aulas

## Exemplo Prático

Mapear uma API Node.js que sai de um repositório Git, passa por testes, gera uma imagem Docker, publica em um registry e roda em Kubernetes.

Durante a aula, use esse exemplo para conectar conceito, decisão técnica e consequência operacional. Sempre que fizer sentido, mostre:

- entrada ou mudança inicial;
- processamento ou etapa executada;
- saída esperada;
- variações possíveis;
- erros comuns de interpretação.

## Analogias e Intuições

- O módulo é uma linha de montagem: cada etapa reduz improviso entre código e produção.
- Container é uma embalagem padronizada; Kubernetes é o sistema que distribui, reinicia e conecta essas embalagens.

## Erros Comuns

- Achar que DevOps é apenas cargo ou ferramenta.
- Pular CI/CD e ir direto para Kubernetes.
- Tratar container como máquina virtual pequena.
- Usar Kubernetes antes de entender deploy, configuração, rede e logs.

## Checklist de Entendimento

Antes de seguir, a pessoa deve conseguir responder:

- O que este conceito resolve?
- Que problema aparece quando ele é ignorado?
- Como ele se conecta com as aulas anteriores?
- Quando ele deve ser usado com cuidado?
- Qual é o próximo conceito natural depois dele?

## Prática Recomendada

- Desenhe o caminho de uma aplicação do commit até a execução em produção.
- Liste quais etapas ainda são manuais em um projeto seu.
- Separe o que pertence a CI/CD, Docker e Kubernetes nesse fluxo.
