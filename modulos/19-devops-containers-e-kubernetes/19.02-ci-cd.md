# CI/CD

## Objetivo

Ao final desta aula, você deve ser capaz de:

- explicar o conceito principal da aula;
- reconhecer quando ele aparece em projetos reais;
- aplicar a ideia em um exemplo simples de entrega, container ou Kubernetes;
- saber o que estudar em seguida.

## Onde Esta Aula Entra na Formação

- Módulo: 19. DevOps, Containers e Kubernetes
- Aula anterior: O que é DevOps na Prática
- Próxima aula: Deploy
- Pré-requisitos: Git, GitHub, terminal, variáveis de ambiente e code review; Backend com Node.js, APIs, bancos de dados, segurança e observabilidade; Cloud Fundamentals e Infrastructure as Code; Terraform, state, plan, apply, ambientes e segredos.

## Ideia Central

CI/CD transforma mudanças pequenas em um fluxo verificável. A cada alteração, o projeto pode instalar dependências, rodar testes, validar qualidade, gerar artefatos e preparar uma entrega. O objetivo não é apertar menos botões; é descobrir problemas cedo e tornar a entrega repetível.

## Tópicos da Aula

- Integração contínua
- Entrega contínua
- Deploy contínuo
- Pipelines e etapas
- Artefatos de build
- Gates de qualidade
- Ambientes e promoção

## Exemplo Prático

Pipeline de uma API TypeScript: instalar dependências, rodar lint, testes, build, gerar imagem e preparar deploy.

Durante a aula, use esse exemplo para conectar conceito, decisão técnica e consequência operacional. Sempre que fizer sentido, mostre:

- entrada ou mudança inicial;
- processamento ou etapa executada;
- saída esperada;
- variações possíveis;
- erros comuns de interpretação.

## Analogias e Intuições

- CI/CD é uma esteira de validação: cada etapa aumenta a confiança antes da próxima.
- Um pipeline bom parece chato quando tudo vai bem e explícito quando algo quebra.

## Erros Comuns

- Misturar CI, CD e deploy como se fossem a mesma coisa.
- Rodar testes lentos demais em toda mudança sem estratégia.
- Gerar artefatos diferentes para cada ambiente.

## Checklist de Entendimento

Antes de seguir, a pessoa deve conseguir responder:

- O que este conceito resolve?
- Que problema aparece quando ele é ignorado?
- Como ele se conecta com as aulas anteriores?
- Quando ele deve ser usado com cuidado?
- Qual é o próximo conceito natural depois dele?

## Prática Recomendada

- Esboce um pipeline mínimo para uma API Node.js.
- Separe etapas de validação, build e entrega.
- Defina quais falhas devem bloquear a próxima etapa.
