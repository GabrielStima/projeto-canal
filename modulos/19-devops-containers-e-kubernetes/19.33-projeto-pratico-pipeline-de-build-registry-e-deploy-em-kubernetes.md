# Projeto Prático: Pipeline de Build, Registry e Deploy em Kubernetes

## Objetivo

Ao final desta aula, você deve ser capaz de:

- explicar o conceito principal da aula;
- reconhecer quando ele aparece em projetos reais;
- aplicar a ideia em um exemplo simples de entrega, container ou Kubernetes;
- saber o que estudar em seguida.

## Onde Esta Aula Entra na Formação

- Módulo: 19. DevOps, Containers e Kubernetes
- Aula anterior: Tópicos Avançados de Kubernetes
- Próxima aula: SRE, Operação e Incidentes
- Pré-requisitos: Git, GitHub, terminal, variáveis de ambiente e code review; Backend com Node.js, APIs, bancos de dados, segurança e observabilidade; Cloud Fundamentals e Infrastructure as Code; CI/CD, Docker, imagens, containers, registries, networking e segurança em containers.

## Ideia Central

O projeto final consolida o módulo: uma alteração passa por validação, gera imagem, publica no registry e atualiza uma aplicação em Kubernetes. O foco é montar um fluxo pequeno, rastreável e seguro, não uma plataforma empresarial completa.

## Tópicos da Aula

- Escopo do projeto
- Pipeline de validação
- Build de imagem
- Publicação em registry
- Manifestos Kubernetes
- Configuração e secrets
- Deploy e verificação
- Rollback conceitual

## Exemplo Prático

marco de Kubernetes - A Orquestração Suprema:

Aqui acaba as edições isoladas. Este é o cume da formação técnica.
Neste projeto prático as pecas se integram:
1. Construa o Workflow do Github Actions.
2. Faça a API passar no build de linting e unit tests em branch isolation.
3. O Runner do Action compila a imagem (Docker) via `docker build`.
4. O Pipeline (com credenciais OIDC nativas de seguridade alta) sobe a imagem no Container Registry.
5. Aplica-se o HCL (Terraform) do cluster.
6. Aplica-se os manifestos YAML (Deployments, HPA e Services) do Kubernetes. O sistema da clínica é provisionado autonomamente e ganha vida, servindo no Load Balancer publico.

## Analogias e Intuições

- O marco de Kubernetes não é sobre fazer um gol bonito (Escrever uma função JS perfeita). É sobre ser o presidente de todo o time de futebol e ser dono do estádio, da manutenção do gramado automático até os refletores programáveis. Você dita a fundação do universo.

## Erros Comuns

- Tentar montar arquitetura grande demais.
- Usar produção como laboratório.
- Não versionar imagem de forma rastreável.
- Ignorar logs e health check depois do deploy.
