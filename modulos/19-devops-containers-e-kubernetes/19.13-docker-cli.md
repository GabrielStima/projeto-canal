## Ideia Central

A Docker CLI é a interface prática para construir, executar, listar, inspecionar, acompanhar logs e remover imagens e containers. O objetivo não é decorar comandos, mas entender quais perguntas cada comando responde sobre o estado do ambiente.

## Tópicos da Aula

- `docker build`
- `docker images`
- `docker run`
- `docker ps`
- `docker logs`
- `docker exec`
- `docker stop` e `docker rm`
- `docker inspect`

## Exemplo Prático

Investigar por que uma API em container não respondeu: listar containers, ver logs, inspecionar portas e executar um comando interno.

Durante a aula, use esse exemplo para conectar conceito, decisão técnica e consequência operacional. Sempre que fizer sentido, mostre:

- entrada ou mudança inicial;
- processamento ou etapa executada;
- saída esperada;
- variações possíveis;
- erros comuns de interpretação.

## Analogias e Intuições

- A CLI é o painel de instrumentos do ambiente local de containers.
- Cada comando deve responder uma pergunta operacional.

## Erros Comuns

- Remover container sem olhar logs.
- Confundir imagem parada com container parado.
- Executar comandos interativos como solução permanente.

## Checklist de Entendimento

Antes de seguir, a pessoa deve conseguir responder:

- O que este conceito resolve?
- Que problema aparece quando ele é ignorado?
- Como ele se conecta com as aulas anteriores?
- Quando ele deve ser usado com cuidado?
- Qual é o próximo conceito natural depois dele?

## Prática Recomendada

- Liste containers em execução e parados.
- Explique a diferença entre `logs` e `exec`.
- Monte uma sequência de diagnóstico para container que não sobe.
