# Building Container Images

## Objetivo

Ao final desta aula, você deve ser capaz de:

- explicar o conceito principal da aula;
- reconhecer quando ele aparece em projetos reais;
- aplicar a ideia em um exemplo simples de entrega, container ou Kubernetes;
- saber o que estudar em seguida.

## Onde Esta Aula Entra na Formação

- Módulo: 19. DevOps, Containers e Kubernetes
- Aula anterior: Básico de Containers
- Próxima aula: Docker CLI
- Pré-requisitos: Git, GitHub, terminal, variáveis de ambiente e code review; Backend com Node.js, APIs, bancos de dados, segurança e observabilidade; Cloud Fundamentals e Infrastructure as Code; CI/CD, deploy e noções de artefatos de build.

## Ideia Central

Construir uma imagem é transformar código, dependências e instruções de execução em um artefato versionável. A imagem precisa ser reproduzível, pequena o bastante, segura o bastante e adequada ao ambiente onde será executada.

## Tópicos da Aula

- Dockerfile como receita
- Base image
- Camadas e cache
- Instalação de dependências
- Build da aplicação
- Comando de execução
- Tags

## Exemplo Prático

Criar uma imagem para uma API TypeScript com instalação de dependências, build e execução do JavaScript compilado.

Durante a aula, use esse exemplo para conectar conceito, decisão técnica e consequência operacional. Sempre que fizer sentido, mostre:

- entrada ou mudança inicial;
- processamento ou etapa executada;
- saída esperada;
- variações possíveis;
- erros comuns de interpretação.

## Analogias e Intuições

- Dockerfile é uma receita de empacotamento: ordem e ingredientes afetam o resultado.
- Cache de camadas é economia quando a receita muda pouco.

## Erros Comuns

- Copiar arquivos demais para a imagem.
- Instalar dependências de desenvolvimento sem necessidade.
- Usar `latest` sem critério.
- Colocar segredos no build.

## Checklist de Entendimento

- Explique quais arquivos devem ficar fora da imagem.
