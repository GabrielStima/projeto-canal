## Ideia Central

Deploy não é apenas copiar arquivos para um servidor. É trocar uma versão em execução por outra com o menor risco possível. Isso envolve artefato, configuração, ambiente, migração, saúde da aplicação, observabilidade e plano de retorno.

## Tópicos da Aula

- Release vs deploy
- Artefatos e versões

## Exemplo Prático

Publicar uma nova versão de uma API com variável de ambiente nova, migration e verificação de health check.

Durante a aula, use esse exemplo para conectar conceito, decisão técnica e consequência operacional. Sempre que fizer sentido, mostre:

- entrada ou mudança inicial;
- processamento ou etapa executada;
- saída esperada;
- variações possíveis;
- erros comuns de interpretação.

## Analogias e Intuições

- Deploy é uma troca de motor com o carro andando: quanto melhor o procedimento, menor o susto.
- Rollback é saída de emergência, não desculpa para entrar sem olhar.

## Erros Comuns

- Fazer build diferente para produção.
- Não versionar o artefato entregue.
- Ignorar migrations e compatibilidade.
- Não verificar se a aplicação ficou saudável.
