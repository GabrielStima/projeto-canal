## Ideia Central

Containers resolvem o problema de executar uma aplicação com dependências e configuração de forma previsível. Docker popularizou um conjunto de ferramentas para construir imagens, executar containers, conectar serviços e compartilhar artefatos em registries.

## Exemplo Prático

O PetCare no Caixote:

A API Node foi pra produção, mas no servidor do hospital rodava Node v14. O app exigia Node 20. O App crashou.
Com o Docker (Container), o desenvolvedor empacota o código dele *junto* com o Node 20 e o Linux exato dentro de uma caixa estanque. Quando mandamos a caixa para produção, ela roda igualzinha à máquina do estagiário. O 'Funciona na minha máquina' acabou.

## Analogias e Intuições

- Antes dos containers, tentar rodar um app era como mudar de casa tentando instalar os móveis para caber na parede exata. Com Container, você leva a sua casa pronta (parede e teto) em cima do caminhão.

## Erros Comuns

- Confundir container com VM.
- Achar que container resolve configuração mal desenhada.
- Colocar segredos dentro da imagem.
