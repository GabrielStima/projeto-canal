## Ideia Central

Containers têm rede própria. Para uma aplicação responder fora do container, portas precisam ser publicadas. Para containers conversarem entre si, eles precisam compartilhar uma rede e usar nomes ou endereços adequados. Entender isso evita erros clássicos com `localhost`.

## Tópicos da Aula

- Rede do host vs rede do container
- Publicação de portas
- DNS interno entre containers
- `localhost` dentro do container
- Redes bridge
- Comunicação com serviços externos

## Exemplo Prático

Uma API em container tenta acessar `localhost:5432`, mas o banco está em outro container e deve ser acessado pelo nome do serviço na rede.

Durante a aula, use esse exemplo para conectar conceito, decisão técnica e consequência operacional. Sempre que fizer sentido, mostre:

- entrada ou mudança inicial;
- processamento ou etapa executada;
- saída esperada;
- variações possíveis;
- erros comuns de interpretação.

## Analogias e Intuições

- Cada container tem seu próprio endereço; `localhost` é sempre eu mesmo.
- Publicar porta é criar encaminhamento entre duas ruas diferentes.

## Erros Comuns

- Usar `localhost` para acessar outro container.
- Publicar portas desnecessárias.
- Misturar rede interna e exposição pública.
