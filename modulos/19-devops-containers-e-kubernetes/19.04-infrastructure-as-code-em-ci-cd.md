# Infrastructure as Code em CI/CD

## Objetivo

Ao final desta aula, você deve ser capaz de:

- explicar o conceito principal da aula;
- reconhecer quando ele aparece em projetos reais;
- aplicar a ideia em um exemplo simples de entrega, container ou Kubernetes;
- saber o que estudar em seguida.

## Onde Esta Aula Entra na Formação

- Módulo: 19. DevOps, Containers e Kubernetes
- Aula anterior: Deploy
- Próxima aula: Geração e Revisão de Planos de Infraestrutura
- Pré-requisitos: Git, GitHub, terminal, variáveis de ambiente e code review; Backend com Node.js, APIs, bancos de dados, segurança e observabilidade; Cloud Fundamentals e Infrastructure as Code; Terraform, state, plan, apply, ambientes e segredos.

## Ideia Central

Infrastructure as Code ganha força quando entra no fluxo de revisão e entrega. O pipeline pode validar formatação, inicializar providers, gerar planos, publicar artefatos de plano e aplicar mudanças mediante regras. A automação precisa proteger o state, os segredos e os ambientes.

## Tópicos da Aula

- Por que IaC entra no pipeline
- Validação e formatação
- Plan como artefato de revisão
- Apply controlado
- State remoto e locking
- Separação por ambiente

## Exemplo Prático

Um pull request altera Terraform, o CI gera um plano para revisão e a aplicação ocorre apenas após aprovação.

Durante a aula, use esse exemplo para conectar conceito, decisão técnica e consequência operacional. Sempre que fizer sentido, mostre:

- entrada ou mudança inicial;
- processamento ou etapa executada;
- saída esperada;
- variações possíveis;
- erros comuns de interpretação.

## Analogias e Intuições

- O pipeline de IaC é uma catraca: deixa passar mudanças compreendidas, não apenas mudanças digitadas.
- Plan é o orçamento da mudança antes da compra.

## Erros Comuns

- Rodar apply em qualquer branch.
- Não proteger state e credenciais.
- Misturar planos de ambientes diferentes.

## Checklist de Entendimento

Antes de seguir, a pessoa deve conseguir responder:

- O que este conceito resolve?
- Que problema aparece quando ele é ignorado?
- Como ele se conecta com as aulas anteriores?
- Quando ele deve ser usado com cuidado?
- Qual é o próximo conceito natural depois dele?

## Prática Recomendada

- Desenhe um pipeline para Terraform com etapas de validação, plan e apply.
- Defina quem pode aprovar mudanças de produção.
- Liste quais dados do plano não deveriam aparecer publicamente.
