# Análise e Performance

## Objetivo

Ao final desta aula, você deve ser capaz de:

- explicar performance como comportamento medido;
- diferenciar sintoma, hipótese e gargalo;
- reconhecer métricas iniciais de performance;
- evitar otimização antes de diagnóstico.

## Onde Esta Aula Entra na Formação

- Módulo: 13. Qualidade, Testes, Observabilidade é Performance
- Aula anterior: Testabilidade de Arquitetura

## Ideia Central

Performance não é uma sensação genérica de rapidez. Ela precisa ser analisada a partir de sintomas observaveis: latência alta, timeout, uso excessivo de CPU, consumo de memória, query lenta, fila crescendo ou experiência ruim para o usuário.

Antes de otimizar, você precisa medir, formular hipóteses e comparar depois da mudança. Sem isso, performance vira palpite.

## Tópicos da Aula

- Sintoma de performance.
- Latência e throughput.
- CPU, memória e I/O em alto nível.
- Banco e N+1 queries.
- Hipótese e medição.
- Comparação antes e depois.

## Exemplo Prático

Sintoma no PetCare OS: A tela de Histórico Clínico (`GET /api/prontuarios/:id`) demora 3500ms para carregar o pet "Rex".

Hipóteses de Diagnóstico:
- A query no banco não usa índice na coluna `pet_id`.
- N+1 Queries: O backend faz um `SELECT` separado para buscar os exames de sangue de cada atendimento antigo.
- O Event Loop do Node está bloqueado porque o servidor está gerando PDFs de receita médica na mesma thread.
- A API externa de plano de saúde do pet está lenta e segurando a resposta.

Próximo passo: Medir antes de reescrever o código de PDF.

## Analogias e Intuições

- Analisar performance é como o veterinário examinando o Rex: você não prescreve antibiótico (cache no Redis) só porque ele está amuado (sintoma). Você pede um exame de sangue (medição) para achar a infecção (gargalo).

## Erros Comuns

- Otimizar o trecho errado.
- Medir apenas com poucos dados locais.
- Confundir latência média com experiência real.
- Ignorar banco de dados.
- Culpar framework antes de olhar evidências.

## Checklist de Entendimento

Antes de seguir, você deve conseguir responder:

- Qual sintoma está sendo observado?
- Que dado prova que existe problema?
- Quais hipóteses explicam o sintoma?
- Como medir cada hipótese?
- Como comparar antes e depois?
