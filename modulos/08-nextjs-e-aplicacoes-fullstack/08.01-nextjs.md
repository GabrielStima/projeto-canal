# Next.js

## Objetivo

Ao final desta aula, você deve ser capaz de:

- explicar o que e Next.js;
- diferenciar React puro de uma aplicação Next.js;
- entender o papel do App Router;
- reconhecer quando Next.js ajuda em um projeto.

## Ideia Central

Next.js e um framework React para construir aplicações web com recursos de frontend e backend-for-frontend. Ele organiza rotas, layouts, renderização, busca de dados, arquivos estaticos, APIs internas, build e deploy com convenções integradas.

O ponto principal e entender que Next.js não substitui React. Ele usa React e adiciona uma estrutura para resolver problemas comuns de aplicações reais.

## Tópicos da Aula

- O que é um framework React.
- O que Next.js adiciona ao React.
- App Router e pasta `app`.
- Aplicações fullstack no frontend.
- Quando usar e quando não usar Next.js.

## Exemplo Prático

Compare mentalmente dois projetos:

```text
React puro:
- você escolhe biblioteca de rotas;
- você decide como buscar dados;
- você decide como gerar build e servir a aplicação.

Next.js:
- rotas seguem a estrutura de arquivos;
- layouts fazem parte do roteamento;
- componentes podem rodar no servidor ou no cliente;
- o framework oferece build e convenções de deploy.
```

## Analogias e Intuições

- React e o motor da interface; Next.js e o carro montado ao redor desse motor.
- Next.js troca liberdade total por convenções que economizam decisões repetitivas.

## Erros Comuns

- Achar que Next.js e uma linguagem nova.
- Usar Next.js sem entender React.
- Colocar tudo no cliente por costume de SPA.
- Achar que "fullstack" significa backend completo em qualquer caso.
