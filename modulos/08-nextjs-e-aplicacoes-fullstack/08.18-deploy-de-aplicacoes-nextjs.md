# Deploy de aplicações Next.js

## Objetivo

Ao final desta aula, você deve ser capaz de:

- explicar o que acontece entre desenvolvimento e produção;
- rodar build de uma aplicação Next.js;
- configurar variáveis de ambiente em alto nível;
- revisar cuidados básicos antes de publicar.

## Onde Esta Aula Entra na Formação

- Módulo: 08. Next.js e Aplicações Fullstack no Frontend
- Aula anterior: Animações em aplicações Next.js
- Próxima aula: Projeto prático: aplicação fullstack com Next.js
- Pré-requisitos: Terminal, scripts, variáveis de ambiente, build, Git, GitHub e fundamentos de servidores.

## Ideia Central

Deploy é o processo de colocar a aplicação para rodar fora da máquina local. Em Next.js, isso envolve build, configuração de ambiente, assets, rotas, possíveis funções de servidor e plataforma de hospedagem.

Publicar não é só clicar em um botão. É revisar se a aplicação tem configuração correta, dados seguros, rotas funcionando e comportamento adequado em produção.

## Tópicos da Aula

- Desenvolvimento vs produção.
- `next build` em alto nível.
- Variáveis de ambiente.
- Plataforma de deploy.
- Logs e erros iniciais.
- Checklist antes de publicar.

## Exemplo Prático

```text
Checklist para publicar o Portal do Tutor:
- scripts rodam localmente;
- build passa;
- variáveis de ambiente estão configuradas;
- páginas principais carregam (/tutor/pets, /tutor/pets/novo, /tutor/orcamentos);
- formulários e APIs foram testados;
- mensagens de erro não vazam detalhes sensíveis.
```

## Analogias e Intuições

- Deploy é tirar o projeto da bancada e colocar em uso real.
- Produção revela dependências que o ambiente local escondia.

## Erros Comuns

- Fazer deploy sem rodar build local.
- Esquecer variáveis de ambiente.
- Vazar segredo no cliente.
- Testar apenas a página inicial.
- Ignorar logs da plataforma.

## Checklist de Entendimento

Antes de seguir, você deve conseguir responder:

- O que muda entre dev e produção?
- Para que serve o build?
- Onde configurar variáveis de ambiente?
- Que rotas devem ser testadas após deploy?
- Quais dados nunca devem ir para o cliente?

## Prática Recomendada

- Rode o build do Portal do Tutor.
- Corrija erros de TypeScript ou lint se aparecerem.
- Prepare uma lista de rotas para testar após publicar.
