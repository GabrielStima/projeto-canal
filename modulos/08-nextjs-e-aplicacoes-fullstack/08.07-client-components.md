# Client Components

## Objetivo

Ao final desta aula, você deve ser capaz de:

- explicar o que é um Client Component;
- usar a diretiva `'use client'`;
- identificar quando uma tela precisa de interatividade no browser;
- evitar transformar a aplicação inteira em cliente sem necessidade.

## Onde Esta Aula Entra na Formação

- Módulo: 08. Next.js e Aplicações Fullstack no Frontend
- Aula anterior: Server Components
- Próxima aula: Composição entre Server e Client Components
- Pré-requisitos: React, hooks, estado, eventos, componentes e Server Components.

## Ideia Central

Client Components são componentes que rodam no navegador e podem usar recursos interativos do React, como estado, efeitos e manipuladores de evento.

Eles continuam sendo importantes. O cuidado é usar Client Components nos pontos que precisam de interatividade, mantendo o restante da árvore como Server Component quando possível.

No Portal do Tutor, a lista de orçamentos pode ser renderizada no servidor, mas o botão de aprovação precisa reagir ao clique, mudar de aparência e talvez abrir uma confirmação. Essa parte interativa vira Client Component.

## Tópicos da Aula

- Diretiva `'use client'`.
- Estado local com `useState`.
- Eventos do browser.
- Hooks de cliente.
- Custo de JavaScript no navegador.
- Limite entre servidor e cliente.

## Exemplo Prático

```tsx
"use client";

import { useState } from "react";

export function AprovacaoOrcamento({ orcamentoId }: { orcamentoId: string }) {
  const [aprovado, setAprovado] = useState(false);

  return (
    <button
      onClick={() => setAprovado(true)}
      disabled={aprovado}
      aria-live="polite"
    >
      {aprovado ? "Orçamento aprovado" : "Aprovar orçamento"}
    </button>
  );
}
```

O componente acima roda no browser porque precisa de estado local para responder ao clique do tutor. O identificador `orcamentoId` chega como prop do Server Component que monta a página.

## Analogias e Intuições

- Client Component é a parte da tela que precisa responder ao toque do usuário, como um botão de confirmação no portal.
- Nem toda informação exibida precisa virar código interativo no browser. A foto e o nome do pet podem vir do servidor sem JavaScript adicional.

## Erros Comuns

- Colocar `'use client'` no layout raiz sem necessidade.
- Usar Client Component apenas para renderizar texto estático.
- Buscar todo dado com `useEffect` por costume.
- Esquecer que tudo abaixo de um Client Component entra no mundo do cliente.

## Checklist de Entendimento

Antes de seguir, você deve conseguir responder:

- Para que serve `'use client'`?
- Quais recursos exigem Client Component?
- Qual o custo de enviar JavaScript ao browser?
- Quando `useState` é necessário?
- Como evitar Client Components grandes demais?

## Prática Recomendada

- Abra a página de orçamentos do Portal do Tutor.
- Mantenha a listagem e os valores no servidor.
- Crie apenas o botão de aprovação como Client Component, usando `useState` para refletir o estado de confirmação.
