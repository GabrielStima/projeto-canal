# Formulários em Next.js

## Objetivo

Ao final desta aula, você deve ser capaz de:

- construir formulários em uma aplicação Next.js;
- entender diferenças entre formulário nativo, Client Component e Server Action;
- representar estado pendente;
- preparar dados para validação.

## Onde Esta Aula Entra na Formação

- Módulo: 08. Next.js e Aplicações Fullstack no Frontend
- Aula anterior: Route Handlers e APIs no Next.js
- Próxima aula: Tipos e validação
- Pré-requisitos: Formulários HTML, React, Client Components, HTTP, `FormData` e Route Handlers.

## Ideia Central

Formulário é uma ponte entre usuário e sistema. Em Next.js, ele pode ser tratado com HTML nativo, com interatividade no cliente, com chamada para Route Handler ou com Server Actions, dependendo do fluxo.

Antes de escolher ferramenta, entenda o caminho dos dados: entrada, validação, processamento, resposta e feedback.

## Tópicos da Aula

- Formulários HTML em React.
- `FormData`.
- Submissão no cliente.
- Server Actions em alto nível.
- Estado pendente.
- Feedback de sucesso e erro.

## Exemplo Prático

```tsx
export function NovoEstudoForm() {
  return (
    <form action="/api/estudos">
      <input name="titulo" required />
      <button type="submit">Salvar</button>
    </form>
  );
}
```

Depois compare com uma abordagem usando Client Component para controlar estado pendente e mensagens.

## Analogias e Intuições

- Formulário é um contrato: a tela promete enviar campos, o sistema promete validar e responder.
- Feedback após envio é tão importante quanto capturar os dados.

## Erros Comuns

- Validar apenas no cliente.
- Não mostrar estado pendente.
- Não associar label e input.
- Misturar captura, validação e persistência sem clareza.
- Esquecer acessibilidade em mensagens de erro.

## Checklist de Entendimento

Antes de seguir, você deve conseguir responder:

- Quais dados o formulário envia?
- Onde a validação acontece?
- Como o usuário sabe que o envio está pendente?
- Qual resposta aparece em caso de erro?
- Quando usar cliente, Route Handler ou Server Action?

## Prática Recomendada

- Crie um formulário de novo estudo.
- Capture `titulo`, `descricao` e `url`.
- Mostre feedback de envio e prepare a validação para a próxima aula.
