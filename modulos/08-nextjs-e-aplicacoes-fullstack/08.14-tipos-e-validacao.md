## Ideia Central

TypeScript ajuda durante o desenvolvimento, mas nao garante que dados vindos do usuario ou de uma API estejam corretos em runtime. Validacao existe para conferir dados quando eles entram no sistema.

Em formularios e APIs, tipos e validacao trabalham juntos: tipos organizam o codigo; validacao protege as fronteiras.

## Topicos da Aula

- Tipo estatico vs dado real.

- Mensagens de erro.
- Narrowing apos validacao.
- Cuidados com dados externos.

## Exemplo Pratico

```ts
type NovoEstudo = {
  titulo: string;
  url: string;
};

function validarNovoEstudo(dados: NovoEstudo) {
  if (dados.titulo.trim().length < 3) {
    return "O titulo precisa ter pelo menos 3 caracteres.";
  }

  if (!dados.url.startsWith("https://")) {
    return "A URL precisa comecar com https://.";
  }

  return null;
}
```

## Analogias e Intuicoes

- TypeScript revisa o mapa; validacao confere o terreno.
- Toda entrada externa deve ser tratada como desconhecida ate ser validada.

## Erros Comuns

- Achar que TypeScript valida formulario sozinho.
- Confiar em dados enviados pelo cliente.
- Retornar mensagens tecnicas demais.
- Validar tarde demais.
- Nao validar dados de APIs externas.
