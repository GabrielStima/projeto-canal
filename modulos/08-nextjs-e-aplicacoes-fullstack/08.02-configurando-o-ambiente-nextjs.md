# Configurando o Ambiente Next.js

## Objetivo

Ao final desta aula, você deve ser capaz de:

- criar um novo projeto Next.js com TypeScript usando o CLI oficial;
- entender a estrutura de pastas gerada pelo `create-next-app`;
- rodar a aplicação em modo de desenvolvimento;
- identificar os arquivos de configuração essenciais;
- preparar o ambiente para as próximas aulas do módulo.

## Onde Esta Aula Entra na Formação

- Módulo: 08. Next.js e Aplicações Fullstack no Frontend
- Aula anterior: Next.js — O Que É e Por Que Usar (08.01)
- Próxima aula: Rotas no App Router (08.03)
- Pré-requisitos: Node.js instalado, npm ou pnpm, terminal básico, TypeScript introdutório.

## Ideia Central

Antes de aprender rotas, layouts e Server Components, você precisa de um projeto funcionando na sua máquina. O Next.js tem um CLI oficial — `create-next-app` — que gera uma estrutura de pastas padronizada com TypeScript, ESLint e Tailwind opcionais já configurados.

A ideia desta aula não é entender cada arquivo gerado em profundidade. É sair com um servidor local rodando para que as aulas seguintes façam sentido na prática, não apenas na teoria, enquanto constrói a base do Portal do Tutor do PetCare OS.

## Tópicos da Aula

- Pré-requisitos de sistema: Node.js e gerenciador de pacotes.
- Criando um projeto com `create-next-app`.
- Estrutura de pastas do App Router.
- Rodando o servidor de desenvolvimento.
- Arquivos de configuração essenciais.
- Scripts disponíveis no `package.json`.

## Criando o Projeto

Com Node.js instalado, execute no terminal:

```bash
npx create-next-app@latest petcare-portal --typescript --eslint --app --no-tailwind --src-dir --import-alias "@/*"
```

O CLI vai perguntar algumas opções. Para as aulas deste módulo, usar as respostas acima como padrão é suficiente. O projeto `petcare-portal` será a base do Portal do Tutor ao longo do módulo.

Depois de criado, entre na pasta e inicie o servidor:

```bash
cd petcare-portal
npm run dev
```

Acesse `http://localhost:3000` no navegador. Se aparecer a página de boas-vindas do Next.js, o ambiente está pronto.

## Estrutura de Pastas Gerada

```text
petcare-portal/
├── src/
│   └── app/
│       ├── layout.tsx       # Layout raiz da aplicação
│       ├── page.tsx         # Rota raiz (/)
│       └── globals.css      # CSS global
├── public/                  # Arquivos estáticos (imagens, ícones)
├── next.config.ts           # Configuração do Next.js
├── tsconfig.json            # Configuração do TypeScript
├── package.json             # Dependências e scripts
└── .eslintrc.json           # Configuração do ESLint
```

A pasta `src/app/` é o núcleo do App Router. Cada arquivo `page.tsx` dentro dela define uma rota. O `layout.tsx` define a estrutura que envolve as páginas.

## Arquivos de Configuração Essenciais

**`next.config.ts`** — configuração do Next.js. Por enquanto, pode ignorar o conteúdo; ele será ajustado quando necessário.

**`tsconfig.json`** — configuração do TypeScript. O alias `@/*` mapeado para `src/*` permite usar `import { algo } from '@/components/algo'` em vez de caminhos relativos longos.

**`.env.local`** — arquivo para variáveis de ambiente locais (não commitado no Git). Você vai criar este arquivo manualmente quando precisar de credenciais ou URLs de API.

## Scripts do `package.json`

```bash
npm run dev      # inicia o servidor de desenvolvimento (hot reload ativo)
npm run build    # gera o build de produção
npm run start    # inicia o servidor com o build de produção
npm run lint     # executa o ESLint
```

Durante o desenvolvimento, você usará quase exclusivamente `npm run dev`.

## Analogias e Intuições

- `create-next-app` é como um andaime de construção: monta a estrutura básica para que você possa começar a trabalhar sem se preocupar com a fundação.
- A pasta `app/` funciona como um mapa da clínica: a estrutura de pastas dentro dela é o mapa de rotas do Portal do Tutor.
- `.env.local` é o cofre de segredos: senhas, tokens e URLs sensíveis ficam aqui, fora do repositório.

## Erros Comuns

- **Node.js desatualizado.** Next.js 14+ exige Node.js 18.17 ou superior. Verifique com `node --version`.
- **Rodar `npx create-next-app` dentro de uma pasta que já é um projeto.** Crie sempre em uma pasta nova ou vazia.
- **Esquecer de iniciar o servidor antes de tentar acessar `localhost:3000`.** Sem `npm run dev` rodando, o navegador não vai encontrar nada.
- **Commitar `.env.local` acidentalmente.** Verifique se o `.gitignore` gerado inclui `.env.local` (geralmente já inclui, mas vale conferir). Nunca armazene credenciais reais neste arquivo durante o aprendizado.
- **Editar arquivos fora da pasta `src/app/` esperando ver mudanças na tela.** A lógica de rotas e páginas fica em `src/app/`.

## Prática Recomendada

- Crie um projeto Next.js seguindo os passos desta aula.
- Abra a pasta no seu editor e explore cada arquivo listado na estrutura.
- Edite o `page.tsx` e observe o hot reload no navegador.
- Crie um arquivo `.env.local` com uma variável `NEXT_PUBLIC_APP_NAME=Portal do Tutor` e tente exibi-la no `page.tsx` com `process.env.NEXT_PUBLIC_APP_NAME`.
- Crie uma pasta `src/app/tutor/page.tsx` com um título simples para simular a primeira rota do portal.
- Rode `npm run build` e observe o output no terminal para entender o que é gerado.
