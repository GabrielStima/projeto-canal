# Route Handlers e APIs no Next.js

## Objetivo

Ao final desta aula, você deve ser capaz de:

- explicar o que é um Route Handler;
- criar um endpoint simples no App Router;
- retornar JSON a partir de uma rota;
- entender o limite entre API interna e backend completo.

## Onde Esta Aula Entra na Formação

## Ideia Central

Route Handlers permitem criar endpoints dentro da estrutura do App Router. Eles são úteis para casos de backend-for-frontend: adaptar dados, esconder detalhes de uma API externa, receber formulários simples ou criar pequenas rotas de suporte para a interface.

No Portal do Tutor, um Route Handler pode servir a lista de serviços da clínica em `/api/servicos`, deixando o frontend mais simples e evitando que o browser precise conhecer a URL exata de uma API externa.

Eles não eliminam a necessidade de aprender backend. O módulo de Node.js vai aprofundar APIs, arquitetura, banco de dados, segurança e operação.

## Exemplo Prático

```tsx
export async function GET() {
  return Response.json([
    { id: "1", nome: "Consulta de rotina", duracaoMinutos: 30 },
    { id: "2", nome: "Vacinação", duracaoMinutos: 15 },
    { id: "3", nome: "Exame de sangue", duracaoMinutos: 20 },
  ]);
}
```

Crie um endpoint `/api/servicos` para alimentar uma página do catálogo no portal ou no painel da clínica.

## Analogias e Intuições

- Route Handler é uma pequena janela de backend dentro do projeto Next.js.
- Backend-for-frontend adapta o servidor para as necessidades da interface, como uma recepcionista que resume informações antes de repassá-las ao tutor.

## Erros Comuns

- Colocar regra de negócio grande demais em Route Handler.
- Ignorar status HTTP.
- Expor segredo ou token para o cliente.
- Confundir API interna com arquitetura backend completa.
