# Error boundaries

## Objetivo

Ao final desta aula, você deve ser capaz de:

- explicar por que erros precisam de UI própria;
- usar `error.tsx` no App Router;
- entender recuperação de erro em uma rota;
- diferenciar erro esperado de falha inesperada.

## Onde Esta Aula Entra na Formação

- Módulo: 08. Next.js e Aplicações Fullstack no Frontend
- Aula anterior: Loading states e streaming
- Próxima aula: Suspense no contexto do Next.js

## Ideia Central

Aplicações reais falham. Uma API pode cair, uma permissão pode faltar, um dado pode vir inválido. Error boundaries permitem mostrar uma interface controlada quando uma parte da aplicação quebra.

No Portal do Tutor, se a página de detalhes do orçamento não conseguir carregar, o tutor precisa ver uma mensagem clara e ter a opção de tentar novamente, em vez de uma tela quebrada.

No App Router, um arquivo `error.tsx` cria uma fronteira de erro para um segmento de rota.

## Tópicos da Aula

- Falhas em UI.
- Arquivo `error.tsx`.
- Error boundary por segmento.
- Botão de tentar novamente.
- Erros esperados vs inesperados.
- Relação com loading e empty state.

## Exemplo Prático

```tsx
"use client";

export default function Error({ reset }: { reset: () => void }) {
  return (
    <div>
      <p>Não foi possível carregar esta área.</p>
      <button onClick={() => reset()}>Tentar novamente</button>
    </div>
  );
}
```

## Analogias e Intuições

- Error boundary é um disjuntor: limita o estrago quando uma parte falha, como isolar a falha no carregamento de um orçamento sem derrubar todo o Portal do Tutor.
- Uma boa mensagem de erro ajuda o usuário a decidir o próximo passo: tentar de novo, voltar ou entrar em contato.

## Erros Comuns

- Deixar erro quebrar a tela inteira.
- Mostrar mensagem técnica demais para o usuário.
- Tratar erro esperado como exceção inesperada.
- Não oferecer caminho de recuperação.

## Checklist de Entendimento

Antes de seguir, você deve conseguir responder:

- O que `error.tsx` captura?
- Por que ele é um Client Component?
- Quando oferecer tentar novamente?
- Qual a diferença entre erro, loading e vazio?

## Prática Recomendada

- Crie uma rota que simula erro ao carregar os detalhes de um orçamento.
- Adicione `error.tsx`.
- Escreva uma mensagem clara e um botão de recuperação.
