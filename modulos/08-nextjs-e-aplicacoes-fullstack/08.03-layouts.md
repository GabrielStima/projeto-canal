## Ideia Central

Layouts definem partes da interface que envolvem páginas. No Next.js, um layout recebe `children` e pode ser compartilhado por uma rota e suas rotas filhas.

Isso permite manter cabeçalho, navegação, sidebar ou estrutura visual sem repetir esse código em cada página.

## Tópicos da Aula

- `layout.tsx`.
- Root layout.
- Propriedade `children`.
- Layouts aninhados.
- Persistência entre navegacoes.
- Separacao entre estrutura e conteúdo.

## Exemplo Prático

```tsx
export default function Layout({ children }: { children: React.ReactNode }) {
  return (
    <>
      <nav>Menu</nav>
      <main>{children}</main>
    </>
  );
}
```

Use um layout para manter uma navegação fixa nas páginas do painel de estudos.

## Analogias e Intuições

- Layout é a moldura persistente; página e o quadro que muda.
- Um layout de rota funciona como um componente pai compartilhado.

## Erros Comuns

- Repetir a mesma navegação em várias páginas.
- Colocar conteúdo específico de página no layout.
- Esquecer que layouts também seguem a árvore de rotas.
- Transformar todo layout em Client Component sem necessidade.
