# Loading states e streaming

## Objetivo

Ao final desta aula, voce deve ser capaz de:

- explicar por que loading state e parte da experiencia;
- usar `loading.tsx` em rotas do App Router;
- entender streaming em alto nivel;
- evitar telas que parecem travadas.

## Onde Esta Aula Entra no Curso

- Modulo: 07. Next.js e Aplicacoes Fullstack no Frontend
- Aula anterior: Composicao entre Server e Client Components
- Proxima aula: Error boundaries
- Pre-requisitos: Rotas, layouts, Server Components, chamadas de API e estados de UI.

## Ideia Central

Quando uma pagina depende de dados ou trabalho assincro, o usuario precisa receber feedback. No App Router, `loading.tsx` permite mostrar uma UI de carregamento enquanto a rota ou parte dela esta sendo preparada.

Streaming permite enviar partes da interface conforme ficam prontas, melhorando a percepcao de velocidade.

## Topicos da Aula

- Estado de carregamento.
- Arquivo `loading.tsx`.
- Loading por segmento de rota.
- Streaming em alto nivel.
- Skeletons e mensagens simples.
- Experiencia percebida.

## Exemplo Pratico

```tsx
export default function Loading() {
  return <p>Carregando estudos...</p>;
}
```

Crie um `loading.tsx` para uma pagina de detalhes que busca dados externos.

## Analogias e Intuicoes

- Loading e uma placa dizendo que o sistema ainda esta trabalhando.
- Streaming e entregar a tela em partes, em vez de esperar tudo ficar pronto.

## Erros Comuns

- Mostrar tela em branco durante carregamento.
- Criar loading visualmente maior que o conteudo real.
- Esquecer estados vazios depois que os dados chegam.
- Usar feedback generico demais para fluxos importantes.

## Checklist de Entendimento

Antes de seguir, voce deve conseguir responder:

- Quando uma pagina precisa de loading?
- O que `loading.tsx` representa?
- Por que loading melhora confianca do usuario?
- O que e streaming em alto nivel?
- Qual a diferenca entre loading, vazio e erro?

## Pratica Recomendada

- Adicione `loading.tsx` em uma rota com busca de dados.
- Simule uma resposta lenta.
- Compare a experiencia com e sem feedback de carregamento.
