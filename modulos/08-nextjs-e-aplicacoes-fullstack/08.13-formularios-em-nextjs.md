# Formularios em Next.js

## Objetivo

Ao final desta aula, voce deve ser capaz de:

- construir formularios em uma aplicacao Next.js;
- entender diferencas entre formulario nativo, Client Component e Server Action;
- representar estado pendente;
- preparar dados para validacao.

## Onde Esta Aula Entra no Curso

- Modulo: 07. Next.js e Aplicacoes Fullstack no Frontend
- Aula anterior: Route Handlers e APIs no Next.js
- Proxima aula: Tipos e validacao
- Pre-requisitos: Formularios HTML, React, Client Components, HTTP, `FormData` e Route Handlers.

## Ideia Central

Formulario e uma ponte entre usuario e sistema. Em Next.js, ele pode ser tratado com HTML nativo, com interatividade no cliente, com chamada para Route Handler ou com Server Actions, dependendo do fluxo.

Antes de escolher ferramenta, entenda o caminho dos dados: entrada, validacao, processamento, resposta e feedback.

## Topicos da Aula

- Formularios HTML em React.
- `FormData`.
- Submissao no cliente.
- Server Actions em alto nivel.
- Estado pendente.
- Feedback de sucesso e erro.

## Exemplo Pratico

```tsx
export function NovoEstudoForm() {
  return (
    <form action="/api/estudos">
      <input name="titulo" required />
      <button type="submit">Salvar</button>
    </form>
  );
}
```

Depois compare com uma abordagem usando Client Component para controlar estado pendente e mensagens.

## Analogias e Intuicoes

- Formulario e um contrato: a tela promete enviar campos, o sistema promete validar e responder.
- Feedback apos envio e tao importante quanto capturar os dados.

## Erros Comuns

- Validar apenas no cliente.
- Nao mostrar estado pendente.
- Nao associar label e input.
- Misturar captura, validacao e persistencia sem clareza.
- Esquecer acessibilidade em mensagens de erro.

## Checklist de Entendimento

Antes de seguir, voce deve conseguir responder:

- Quais dados o formulario envia?
- Onde a validacao acontece?
- Como o usuario sabe que o envio esta pendente?
- Qual resposta aparece em caso de erro?
- Quando usar cliente, Route Handler ou Server Action?

## Pratica Recomendada

- Crie um formulario de novo estudo.
- Capture `titulo`, `descricao` e `url`.
- Mostre feedback de envio e prepare a validacao para a proxima aula.
