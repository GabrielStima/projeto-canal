# Suspense no contexto do Next.js

## Objetivo

Ao final desta aula, você deve ser capaz de:

- explicar Suspense em alto nível;
- entender sua relação com carregamento assíncrono;
- usar Suspense para isolar partes da interface;
- diferenciar Suspense de tratamento de erro.

## Onde Esta Aula Entra na Formação

- Módulo: 08. Next.js e Aplicações Fullstack no Frontend
- Aula anterior: Error boundaries
- Próxima aula: Chamadas de API e busca de dados
- Pré-requisitos: React, componentes, assincronismo, loading states, streaming e error boundaries.

## Ideia Central

Suspense permite declarar uma interface alternativa enquanto uma parte da árvore ainda não está pronta. No contexto do Next.js, ele se conecta com streaming e com a forma como partes assíncronas da tela podem carregar de maneira independente.

No painel da clínica, o cabeçalho com o número de atendimentos do dia pode aparecer imediatamente, enquanto o resumo financeiro do turno, que depende de mais cálculos, fica em Suspense até ficar pronto.

Suspense não substitui tratamento de erro. Ele lida com espera; error boundaries lidam com falha.

## Tópicos da Aula

- Espera em interfaces React.
- `Suspense` e `fallback`.
- Relação com streaming.
- Componentes assíncronos.
- Suspense vs error boundary.
- Granularidade de carregamento.

## Exemplo Prático

```tsx
import { Suspense } from "react";

export default function Page() {
  return (
    <>
      <h1>Painel da clínica</h1>
      <Suspense fallback={<p>Carregando resumo financeiro...</p>}>
        <ResumoFinanceiroAssincrono />
      </Suspense>
    </>
  );
}
```

## Analogias e Intuições

- Suspense é uma reserva de espaço enquanto uma parte da tela chega, como um card vazio no painel que mostra "calculando valores" até o resumo financeiro estar pronto.
- Loading responde "ainda não"; erro responde "não deu certo".

## Erros Comuns

- Usar Suspense como se fosse `try/catch`.
- Criar fallbacks demais e fragmentar a experiência.
- Mostrar fallback que muda o layout de forma brusca.
- Não pensar em qual parte realmente precisa carregar separadamente.

## Checklist de Entendimento

Antes de seguir, você deve conseguir responder:

- O que Suspense espera?
- Para que serve `fallback`?
- Como Suspense se relaciona com streaming?
- Por que Suspense não trata erro?
- Quando isolar uma parte da tela?

## Prática Recomendada

- Crie uma página do painel da clínica com duas áreas assíncronas.
- Coloque Suspense apenas em uma delas.
- Compare a percepção de carregamento da página.
