# Composição entre Server e Client Components

## Objetivo

Ao final desta aula, você deve ser capaz de:

- combinar Server Components e Client Components;
- decidir onde colocar busca de dados e interatividade;
- passar dados do servidor para componentes de cliente;
- reconhecer fronteiras ruins entre servidor e cliente.

## Onde Esta Aula Entra na Formação

## Ideia Central

Uma aplicação Next.js moderna costuma ser uma composição: dados e estrutura ficam no servidor; interações pontuais ficam no cliente.

No painel da recepção, a lista de atendimentos do dia pode ser buscada e renderizada no servidor, enquanto o filtro por status — que precisa reagir instantaneamente à escolha do recepcionista — pode viver em um Client Component.

O desafio não é escolher um lado para tudo. O desafio é desenhar a árvore para que cada parte rode onde faz mais sentido.

## Tópicos da Aula

- Passagem de props.
- Componentes interativos pequenos.
- Dados serializáveis.
- Organização de arquivos.
- Decisões práticas de composição.

## Exemplo Prático

```tsx
export default async function PaginaAtendimentos() {
  const atendimentos = await carregarAtendimentosDoDia();

  return (
    <>
      <h1>Atendimentos do dia</h1>
      <FiltroAtendimentos atendimentos={atendimentos} />
    </>
  );
}
```

`PaginaAtendimentos` busca dados no servidor. `FiltroAtendimentos` pode ser um Client Component se precisar de estado local para filtrar a lista na tela sem recarregar a página.

## Analogias e Intuições

- O servidor prepara os dados, como uma recepcionista que organiza a fila de atendimentos antes do expediente.
- O cliente manipula o que precisa responder imediatamente ao usuário, como o mesmo recepcionista marcando visualmente quais atendimentos já foram triados.
- Uma boa fronteira reduz trabalho duplicado.

## Erros Comuns

- Fazer fetch no cliente por hábito mesmo quando a página pode buscar no servidor.
- Passar funções ou objetos inadequados entre servidor e cliente.
- Criar Client Components grandes demais.
- Misturar regra de negócio, busca de dados e UI interativa no mesmo arquivo.

## Checklist de Entendimento

Antes de seguir, você deve conseguir responder:

- Qual parte da tela precisa de interação?
- Onde os dados devem ser carregados?
- O que pode ser passado por props?
- Como reduzir JavaScript no cliente?
- Quando dividir um componente em dois?

## Prática Recomendada

- Abra a página de atendimentos do painel da recepção.
- Marque quais partes são servidor e quais são cliente.
- Refatore para deixar a interatividade no menor componente possível, como um filtro de status.
