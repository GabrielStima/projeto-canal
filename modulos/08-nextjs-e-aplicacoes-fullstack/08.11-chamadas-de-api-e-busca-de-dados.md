# Chamadas de API e busca de dados

## Objetivo

Ao final desta aula, você deve ser capaz de:

- buscar dados em uma aplicação Next.js;
- decidir se a busca acontece no servidor ou no cliente;
- entender cache e revalidação em nível introdutório;
- representar estados de dados na interface.

## Onde Esta Aula Entra na Formação

- Módulo: 08. Next.js e Aplicações Fullstack no Frontend
- Aula anterior: Suspense no contexto do Next.js
- Próxima aula: Route Handlers e APIs no Next.js
- Pré-requisitos: HTTP, `fetch`, promises, Server Components, Client Components, loading e erro.

## Ideia Central

Em Next.js, buscar dados não significa automaticamente usar `useEffect`. Muitas páginas podem buscar dados no servidor, perto da rota que precisa deles.

Ainda assim, existem casos em que buscar no cliente faz sentido: interação instantânea, dados que dependem de estado local ou atualizações após a página carregar.

## Tópicos da Aula

- `fetch` no servidor.
- `fetch` no cliente.
- Dados públicos e dados por usuário.
- Cache é revalidação em alto nível.
- Loading, erro e vazio.
- Organização de funções de dados.

## Exemplo Prático

```tsx
async function carregarEstudos() {
  const resposta = await fetch("https://exemplo.com/api/estudos");

  if (!resposta.ok) {
    throw new Error("Falha ao carregar estudos");
  }

  return resposta.json();
}
```

Use essa função em uma página de servidor e trate a falha com a aula de error boundaries.

## Analogias e Intuições

- Buscar dados perto da página reduz viagens desnecessarias entre cliente e servidor.
- Cache é um acordo: ganhar velocidade aceitando uma regra clara sobre atualização.

## Erros Comuns

- Usar `useEffect` para toda busca de dados.
- Ignorar `resposta.ok`.
- Não pensar se o dado e público, privado, estático ou dinâmico.
- Misturar chamada de API com código visual sem separacao mínima.

## Checklist de Entendimento

Antes de seguir, você deve conseguir responder:

- Quando buscar dados no servidor?
- Quando buscar dados no cliente?
- Por que checar `resposta.ok`?
- Como loading e erro aparecem nessa decisão?
- Como cache pode ajudar ou atrapalhar?

## Prática Recomendada

- Crie uma função `carregarEstudos`.
- Use-a em uma página Server Component.
- Simule sucesso, erro e lista vazia.
