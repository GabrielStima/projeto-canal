# Chamadas de API e busca de dados

## Objetivo

Ao final desta aula, voce deve ser capaz de:

- buscar dados em uma aplicacao Next.js;
- decidir se a busca acontece no servidor ou no cliente;
- entender cache e revalidacao em nivel introdutorio;
- representar estados de dados na interface.

## Onde Esta Aula Entra no Curso

- Modulo: 07. Next.js e Aplicacoes Fullstack no Frontend
- Aula anterior: Suspense no contexto do Next.js
- Proxima aula: Route Handlers e APIs no Next.js
- Pre-requisitos: HTTP, `fetch`, promises, Server Components, Client Components, loading e erro.

## Ideia Central

Em Next.js, buscar dados nao significa automaticamente usar `useEffect`. Muitas paginas podem buscar dados no servidor, perto da rota que precisa deles.

Ainda assim, existem casos em que buscar no cliente faz sentido: interacao instantanea, dados que dependem de estado local ou atualizacoes apos a pagina carregar.

## Topicos da Aula

- `fetch` no servidor.
- `fetch` no cliente.
- Dados publicos e dados por usuario.
- Cache e revalidacao em alto nivel.
- Loading, erro e vazio.
- Organizacao de funcoes de dados.

## Exemplo Pratico

```tsx
async function carregarEstudos() {
  const resposta = await fetch("https://exemplo.com/api/estudos");

  if (!resposta.ok) {
    throw new Error("Falha ao carregar estudos");
  }

  return resposta.json();
}
```

Use essa funcao em uma pagina de servidor e trate a falha com a aula de error boundaries.

## Analogias e Intuicoes

- Buscar dados perto da pagina reduz viagens desnecessarias entre cliente e servidor.
- Cache e um acordo: ganhar velocidade aceitando uma regra clara sobre atualizacao.

## Erros Comuns

- Usar `useEffect` para toda busca de dados.
- Ignorar `resposta.ok`.
- Nao pensar se o dado e publico, privado, estatico ou dinamico.
- Misturar chamada de API com codigo visual sem separacao minima.

## Checklist de Entendimento

Antes de seguir, voce deve conseguir responder:

- Quando buscar dados no servidor?
- Quando buscar dados no cliente?
- Por que checar `resposta.ok`?
- Como loading e erro aparecem nessa decisao?
- Como cache pode ajudar ou atrapalhar?

## Pratica Recomendada

- Crie uma funcao `carregarEstudos`.
- Use-a em uma pagina Server Component.
- Simule sucesso, erro e lista vazia.
