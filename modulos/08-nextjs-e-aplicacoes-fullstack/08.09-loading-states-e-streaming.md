# Loading states e streaming

## Objetivo

Ao final desta aula, você deve ser capaz de:

- explicar por que loading state é parte da experiência;
- usar `loading.tsx` em rotas do App Router;
- entender streaming em alto nível;
- evitar telas que parecem travadas.

## Onde Esta Aula Entra na Formação

- Aula anterior: Composição entre Server e Client Components
- Próxima aula: Error boundaries

## Ideia Central

Quando uma página depende de dados ou trabalho assíncrono, o usuário precisa receber feedback. No App Router, `loading.tsx` permite mostrar uma UI de carregamento enquanto a rota ou parte dela está sendo preparada.

No Portal do Tutor, a página de status do pet pode demorar alguns instantes enquanto busca o histórico recente de atendimentos. Em vez de deixar o tutor olhando para uma tela em branco, mostramos uma mensagem ou skeleton.

Streaming permite enviar partes da interface conforme ficam prontas, melhorando a percepção de velocidade.

## Tópicos da Aula

## Exemplo Prático

```tsx
export default function Loading() {
  return <p>Carregando status do pet...</p>;
}
```

Crie um `loading.tsx` para a rota `/tutor/pets/[id]/status`, que busca dados externos sobre o atendimento atual.

## Analogias e Intuições

- Loading é uma placa dizendo que o sistema ainda está trabalhando, como a tela de espera do Portal do Tutor.
- Streaming é entregar a tela em partes, em vez de esperar tudo ficar pronto: o cabeçalho com o nome do pet aparece primeiro, os detalhes do atendimento depois.

## Erros Comuns

- Mostrar tela em branco durante carregamento.
- Criar loading visualmente maior que o conteúdo real.
- Esquecer estados vazios depois que os dados chegam.
- Usar feedback genérico demais para fluxos importantes.
