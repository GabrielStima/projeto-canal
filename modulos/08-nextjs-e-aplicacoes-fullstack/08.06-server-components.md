# Server Components

## Objetivo

Ao final desta aula, você deve ser capaz de:

- explicar o que é um Server Component;
- entender por que componentes no servidor reduzem JavaScript no cliente;
- buscar dados em um componente de servidor;
- reconhecer limites de interatividade.

## Onde Esta Aula Entra na Formação

- Módulo: 08. Next.js e Aplicações Fullstack no Frontend
- Aula anterior: Estratégias de renderização
- Próxima aula: Client Components
- Pré-requisitos: React, componentes, promises, `async/await`, HTTP e renderização.

## Ideia Central

No App Router, componentes são Server Components por padrão. Isso significa que eles podem ser renderizados no servidor e enviar ao navegador o resultado necessário, sem transformar todo o componente em JavaScript interativo.

Server Components são bons para buscar dados, montar estrutura e renderizar conteúdo que não depende de eventos do navegador. No Portal do Tutor, uma página que lista os pets de um tutor é um bom candidato a Server Component: os dados podem ser buscados no servidor e o HTML pode chegar pronto para exibição.

## Tópicos da Aula

- Server Component como padrão.
- Busca de dados no servidor.
- Menos JavaScript no cliente.
- Acesso a recursos do servidor em alto nível.
- Limites: eventos, estado local e hooks de cliente.

## Exemplo Prático

```tsx
type Pet = {
  id: string;
  nome: string;
  especie: string;
};

export default async function PetsPage() {
  const resposta = await fetch("https://api-exemplo.local/pets");
  const pets = (await resposta.json()) as Pet[];

  return <ListaPets pets={pets} />;
}
```

Neste exemplo, `PetsPage` busca a lista de pets no servidor e renderiza o resultado. O componente `ListaPets` pode continuar sendo um Server Component se não precisar de interatividade, ou virar Client Component se precisar de filtros ou animações.

## Analogias e Intuições

- Server Component prepara a refeição do pet internado antes de entregar ao tutor: o trabalho pesado fica nos bastidores.
- Ele é excelente para conteúdo e dados, como lista de vacinas ou histórico de atendimentos, mas não para cliques e estado local.

## Erros Comuns

- Tentar usar `useState` em Server Component.
- Colocar `'use client'` em tudo por reflexo.
- Buscar dados no cliente quando poderia buscar no servidor.
- Esquecer que código de servidor tem limites de ambiente e deploy.
