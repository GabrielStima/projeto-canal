# Composicao entre Server e Client Components

## Objetivo

Ao final desta aula, voce deve ser capaz de:

- combinar Server Components e Client Components;
- decidir onde colocar busca de dados e interatividade;
- passar dados do servidor para componentes de cliente;
- reconhecer fronteiras ruins entre servidor e cliente.

## Onde Esta Aula Entra no Curso

- Modulo: 07. Next.js e Aplicacoes Fullstack no Frontend
- Aula anterior: Client Components
- Proxima aula: Loading states e streaming
- Pre-requisitos: Server Components, Client Components, props, estado e chamadas de API.

## Ideia Central

Uma aplicacao Next.js moderna costuma ser uma composicao: dados e estrutura ficam no servidor; interacoes pontuais ficam no cliente.

O desafio nao e escolher um lado para tudo. O desafio e desenhar a arvore para que cada parte rode onde faz mais sentido.

## Topicos da Aula

- Fronteira servidor-cliente.
- Passagem de props.
- Componentes interativos pequenos.
- Dados serializaveis.
- Organizacao de arquivos.
- Decisoes praticas de composicao.

## Exemplo Pratico

```tsx
export default async function Page() {
  const estudos = await carregarEstudos();

  return (
    <>
      <h1>Estudos</h1>
      <FiltroEstudos estudos={estudos} />
    </>
  );
}
```

`Page` busca dados no servidor. `FiltroEstudos` pode ser um Client Component se precisar de estado local para filtrar a lista na tela.

## Analogias e Intuicoes

- O servidor prepara os dados; o cliente manipula o que precisa responder imediatamente ao usuario.
- Uma boa fronteira reduz trabalho duplicado.

## Erros Comuns

- Fazer fetch no cliente por habito mesmo quando a pagina pode buscar no servidor.
- Passar funcoes ou objetos inadequados entre servidor e cliente.
- Criar Client Components grandes demais.
- Misturar regra de negocio, busca de dados e UI interativa no mesmo arquivo.

## Checklist de Entendimento

Antes de seguir, voce deve conseguir responder:

- Qual parte da tela precisa de interacao?
- Onde os dados devem ser carregados?
- O que pode ser passado por props?
- Como reduzir JavaScript no cliente?
- Quando dividir um componente em dois?

## Pratica Recomendada

- Pegue uma pagina existente.
- Marque quais partes sao servidor e quais sao cliente.
- Refatore para deixar a interatividade no menor componente possivel.
