# Projeto prático: aplicação fullstack com Next.js

## Objetivo

Ao final desta aula, você deve ser capaz de:

- integrar os principais assuntos do módulo em uma aplicação pequena;
- criar rotas, layouts, Server Components e Client Components;
- usar loading, erro, formulários, validação e API interna;
- preparar uma versão inicial para deploy.

## Onde Esta Aula Entra no Curso

- Módulo: 08. Next.js e Aplicações Fullstack no Frontend
- Aula anterior: Deploy de aplicações Next.js
- Próxima aula: 09. Bancos de Dados
- Pré-requisitos: Todas as aulas anteriores do módulo 07.

## Ideia Central

O projeto final transforma conceitos isolados em uma aplicação coerente. A ideia não é criar um produto grande nem antecipar banco de dados real. O foco e montar um fluxo completo de Next.js com dados simulados ou API simples.

Sugestão: um painel de estudos com listagem, detalhes, cadastro, validação, UI condicional de usuário logado e deploy.

## Tópicos da Aula

- Escopo do projeto.
- Árvore de rotas.
- Layout principal.
- Busca de dados no servidor.
- Interatividade no cliente.
- Loading, erro e Suspense.
- Route Handler é formulário.
- Validação é deploy.

## Exemplo Prático

```text
Aplicação: Painel de Estudos

Rotas:
- /
- /estudos
- /estudos/[id]
- /estudos/novo
- /dashboard

Funcionalidades:
- listar estudos;
- ver detalhes;
- cadastrar novo estudo;
- validar título e URL;
- mostrar loading e erro;
- simular usuário logado;
- preparar deploy.
```

## Analogias e Intuições

- O projeto e um ensaio geral antes de banco de dados e backend completo.
- Escopo pequeno bem acabado ensina mais do que arquitetura grande incompleta.

## Erros Comuns

- Adicionar banco de dados antes de consolidar o fluxo.
- Transformar tudo em Client Component.
- Ignorar loading e erro.
- Criar autenticação falsa como se fosse segura.
- Fazer deploy sem testar rotas principais.

## Checklist de Entendimento

Antes de seguir, você deve conseguir responder:

- Quais rotas existem?
- Quais componentes rodam no servidor?
- Quais componentes precisam do cliente?
- Onde os dados são buscados?
- Como formulário e validação funcionam?
- O que precisa mudar quando entrar banco de dados?

## Prática Recomendada

- Construa primeiro a navegação e o layout.
- Adicione listagem e detalhes com dados simulados.
- Crie formulário e validação.
- Adicione Route Handler simples.
- Revise loading, erro e deploy.
