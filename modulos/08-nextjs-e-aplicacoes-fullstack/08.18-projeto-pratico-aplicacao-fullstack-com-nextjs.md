# Projeto pratico: aplicacao fullstack com Next.js

## Objetivo

Ao final desta aula, voce deve ser capaz de:

- integrar os principais assuntos do modulo em uma aplicacao pequena;
- criar rotas, layouts, Server Components e Client Components;
- usar loading, erro, formularios, validacao e API interna;
- preparar uma versao inicial para deploy.

## Onde Esta Aula Entra no Curso

- Modulo: 07. Next.js e Aplicacoes Fullstack no Frontend
- Aula anterior: Deploy de aplicacoes Next.js
- Proxima aula: 08. Bancos de Dados
- Pre-requisitos: Todas as aulas anteriores do modulo 07.

## Ideia Central

O projeto final transforma conceitos isolados em uma aplicacao coerente. A ideia nao e criar um produto grande nem antecipar banco de dados real. O foco e montar um fluxo completo de Next.js com dados simulados ou API simples.

Sugestao: um painel de estudos com listagem, detalhes, cadastro, validacao, UI condicional de usuario logado e deploy.

## Topicos da Aula

- Escopo do projeto.
- Arvore de rotas.
- Layout principal.
- Busca de dados no servidor.
- Interatividade no cliente.
- Loading, erro e Suspense.
- Route Handler e formulario.
- Validacao e deploy.

## Exemplo Pratico

```text
Aplicacao: Painel de Estudos

Rotas:
- /
- /estudos
- /estudos/[id]
- /estudos/novo
- /dashboard

Funcionalidades:
- listar estudos;
- ver detalhes;
- cadastrar novo estudo;
- validar titulo e URL;
- mostrar loading e erro;
- simular usuario logado;
- preparar deploy.
```

## Analogias e Intuicoes

- O projeto e um ensaio geral antes de banco de dados e backend completo.
- Escopo pequeno bem acabado ensina mais do que arquitetura grande incompleta.

## Erros Comuns

- Adicionar banco de dados antes de consolidar o fluxo.
- Transformar tudo em Client Component.
- Ignorar loading e erro.
- Criar autenticacao falsa como se fosse segura.
- Fazer deploy sem testar rotas principais.

## Checklist de Entendimento

Antes de seguir, voce deve conseguir responder:

- Quais rotas existem?
- Quais componentes rodam no servidor?
- Quais componentes precisam do cliente?
- Onde os dados sao buscados?
- Como formulario e validacao funcionam?
- O que precisa mudar quando entrar banco de dados?

## Pratica Recomendada

- Construa primeiro a navegacao e o layout.
- Adicione listagem e detalhes com dados simulados.
- Crie formulario e validacao.
- Adicione Route Handler simples.
- Revise loading, erro e deploy.
