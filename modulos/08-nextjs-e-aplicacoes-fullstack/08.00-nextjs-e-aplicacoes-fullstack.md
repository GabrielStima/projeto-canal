## Ideia Central

React resolve o problema de construir interfaces. Next.js resolve o problema de entregar essas interfaces de forma rápida, indexável e conectada ao servidor. Ele adiciona rotas baseadas em arquivos, estratégias de renderização, Server Components e route handlers que permitem buscar dados e executar lógica server-side sem precisar de um backend separado para casos simples.

A grande virada do Next.js moderno está na composição entre Server Components e Client Components: componentes no servidor buscam dados sem expor credenciais, reduzem o JavaScript enviado ao navegador e melhoram a performance. Componentes no cliente mantêm interatividade, estado e eventos do usuário.

Este módulo cobre desde a estrutura do App Router até autenticação simples no frontend, animações e deploy, preparando você para publicar uma versão inicial do Portal do Tutor do PetCare OS.

## Analogias e Intuições

- Server Components são como a cozinha de uma clínica veterinária que prepara a refeição do pet internado antes de chegar à enfermaria: o trabalho acontece nos bastidores, e o tutor vê apenas o resultado pronto.
- Estratégias de renderização são como políticas de estoque de medicamentos: SSG é pré-fabricado (saco sempre no armário), SSR é sob demanda (preparado na hora da consulta), ISR é reabastecimento periódico (atualiza a lista de vacinas de tempos em tempos) e CSR é autoatendimento (o tutor escolhe os filtros no portal).
- O App Router é como a planta de uma clínica veterinária: cada pasta é um corredor, cada `page.tsx` é uma sala de atendimento, e os `layout.tsx` são as paredes e sinalização em comum.

## Erros Comuns

- Usar Client Components onde Server Components resolveriam com menos JavaScript.
- Misturar busca de dados no cliente quando o servidor seria mais seguro e eficiente.
- Ignorar loading states e deixar a tela em branco durante a busca de dados.
- Não usar o App Router corretamente e misturar convenções de versões antigas.
- Colocar segredos de API em variáveis de ambiente sem o prefixo correto, expondo-os ao navegador.

- Como o Next.js lida com loading states sem bloquear a navegação?
- Que problema o App Router resolve comparado a um roteador manual?
- O que este módulo vai aprofundar?

## Prática Recomendada

- Esboce quais páginas do Portal do Tutor seriam Server Components e quais precisariam ser Client Components.
- Liste quais dados seriam buscados no servidor (status do atendimento, histórico de vacinas) e quais seriam carregados após a interação do tutor (filtros de agenda, aprovação de orçamento).
- Identifique onde um `loading.tsx` e um `error.tsx` fariam diferença na experiência do tutor.
- Desenhe a estrutura de pastas do App Router para o Portal do Tutor com pelo menos três rotas, como `/tutor`, `/tutor/pets/[id]` e `/tutor/orcamentos`.
