# Tipos e validação

- retornar mensagens de erro compreensíveis.

## Ideia Central

TypeScript ajuda durante o desenvolvimento, mas não garante que dados vindos do usuário ou de uma API estejam corretos em runtime. Validação existe para conferir dados quando eles entram no sistema.

Em formulários e APIs, tipos e validação trabalham juntos: tipos organizam o código; validação protege as fronteiras.

## Tópicos da Aula

- Tipo estático vs dado real.
- Fronteiras de entrada.
- Validação de formulário.
- Mensagens de erro.
- Narrowing após validação.
- Cuidados com dados externos.

## Exemplo Prático

```ts
type NovoServico = {
  nome: string;
  descricao: string;
  precoEstimado: number;
};

function validarNovoServico(dados: NovoServico) {
  if (dados.nome.trim().length < 3) {
    return "O nome precisa ter pelo menos 3 caracteres.";
  }

  if (dados.precoEstimado <= 0) {
    return "O preço estimado deve ser maior que zero.";
  }

  return null;
}
```

## Analogias e Intuições

- TypeScript revisa o mapa; validação confere o terreno.
- Toda entrada externa deve ser tratada como desconhecida até ser validada.

## Erros Comuns

- Achar que TypeScript valida formulário sozinho.
- Confiar em dados enviados pelo cliente.
- Retornar mensagens técnicas demais.
- Validar tarde demais.
- Não validar dados de APIs externas.
