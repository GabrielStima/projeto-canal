# Server Components

## Objetivo

Ao final desta aula, você deve ser capaz de:

- explicar o que e um Server Component;
- entender por que componentes no servidor reduzem JavaScript no cliente;
- buscar dados em um componente de servidor;
- reconhecer limites de interatividade.

## Onde Esta Aula Entra no Curso

- Módulo: 08. Next.js e Aplicações Fullstack no Frontend
- Aula anterior: Estratégias de renderização
- Próxima aula: Client Components
- Pré-requisitos: React, componentes, promises, `async/await`, HTTP e renderização.

## Ideia Central

No App Router, componentes são Server Components por padrão. Isso significa que eles podem ser renderizados no servidor e enviar ao navegador o resultado necessário, sem transformar todo o componente em JavaScript interativo.

Server Components são bons para buscar dados, montar estrutura e renderizar conteúdo que não depende de eventos do browser.

## Tópicos da Aula

- Server Component como padrão.
- Busca de dados no servidor.
- Menos JavaScript no cliente.
- Acesso a recursos do servidor em alto nível.
- Limites: eventos, estado local e hooks de cliente.

## Exemplo Prático

```tsx
type Estudo = {
  id: string;
  titulo: string;
};

export default async function EstudosPage() {
  const resposta = await fetch("https://exemplo.com/api/estudos");
  const estudos = (await resposta.json()) as Estudo[];

  return <ListaEstudos estudos={estudos} />;
}
```

## Analogias e Intuições

- Server Component prepara a refeição antes de entregar ao navegador.
- Ele é excelente para conteúdo e dados, mas não para cliques e estado local.

## Erros Comuns

- Tentar usar `useState` em Server Component.
- Colocar `'use client'` em tudo por reflexo.
- Buscar dado no cliente quando poderia buscar no servidor.
- Esquecer que código de servidor tem limites de ambiente e deploy.
