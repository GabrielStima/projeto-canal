# Server Components

## Objetivo

Ao final desta aula, voce deve ser capaz de:

- explicar o que e um Server Component;
- entender por que componentes no servidor reduzem JavaScript no cliente;
- buscar dados em um componente de servidor;
- reconhecer limites de interatividade.

## Onde Esta Aula Entra no Curso

- Modulo: 07. Next.js e Aplicacoes Fullstack no Frontend
- Aula anterior: Estrategias de renderizacao
- Proxima aula: Client Components
- Pre-requisitos: React, componentes, promises, `async/await`, HTTP e renderizacao.

## Ideia Central

No App Router, componentes sao Server Components por padrao. Isso significa que eles podem ser renderizados no servidor e enviar ao navegador o resultado necessario, sem transformar todo o componente em JavaScript interativo.

Server Components sao bons para buscar dados, montar estrutura e renderizar conteudo que nao depende de eventos do browser.

## Topicos da Aula

- Server Component como padrao.
- Busca de dados no servidor.
- Menos JavaScript no cliente.
- Acesso a recursos do servidor em alto nivel.
- Limites: eventos, estado local e hooks de cliente.

## Exemplo Pratico

```tsx
type Estudo = {
  id: string;
  titulo: string;
};

export default async function EstudosPage() {
  const resposta = await fetch("https://exemplo.com/api/estudos");
  const estudos = (await resposta.json()) as Estudo[];

  return <ListaEstudos estudos={estudos} />;
}
```

## Analogias e Intuicoes

- Server Component prepara a refeicao antes de entregar ao navegador.
- Ele e excelente para conteudo e dados, mas nao para cliques e estado local.

## Erros Comuns

- Tentar usar `useState` em Server Component.
- Colocar `'use client'` em tudo por reflexo.
- Buscar dado no cliente quando poderia buscar no servidor.
- Esquecer que codigo de servidor tem limites de ambiente e deploy.

## Checklist de Entendimento

Antes de seguir, voce deve conseguir responder:

- Por que Server Components existem?
- O que eles podem fazer bem?
- O que eles nao podem fazer?
- Como eles afetam o JavaScript enviado ao browser?
- Quando um componente precisa virar Client Component?

## Pratica Recomendada

- Remova estados e efeitos desnecessarios.
- Identifique quais partes da tela realmente precisam de interatividade.
