# Monolito

## Objetivo

Ao final desta aula, você deve ser capaz de:

- explicar o que é um monolito;
- diferenciar monolito de código bagunçado;
- reconhecer vantagens de um monolito bem modularizado;
- entender quando separar serviços pode fazer sentido.

## Onde Esta Aula Entra na Formação

- Módulo: 15. Arquitetura de Software
- Aula anterior: MVC
- Próxima aula: Clean Architecture
- Pré-requisitos: Backend, bancos de dados, APIs, componentes, boundaries e testes.

## Ideia Central

Monolito é um sistema implantado como uma unidade principal. Isso não significa que ele precise ser desorganizado. Um monolito pode ter módulos, boundaries, testes, contratos internos e separação clara de responsabilidades.

Para muitos produtos, o monolito modular é a melhor primeira arquitetura: simples de operar, fácil de entender e suficiente até que limites reais exijam separação maior.

## Tópicos da Aula

- Monolito como unidade de deploy.
- Monolito modular.
- Banco compartilhado.
- Vantagens operacionais.
- Riscos de acoplamento.
- Caminhos para evolução.

## Exemplo Prático

```text
Monolito modular:
- módulo de aulas;
- módulo de progresso;
- módulo de assinaturas;
- módulo de notificações.

Todos no mesmo deploy, mas com dependências internas controladas.
```

## Analogias e Intuições

- Monolito bom é um prédio organizado; monolito ruim é um depósito sem corredor.
- Separar serviços não corrige automaticamente limites ruins.

## Erros Comuns

- Usar monolito como sinônimo de sistema ruim.
- Distribuir cedo demais.
- Deixar todos os módulos acessarem tudo.
- Compartilhar modelos internos sem critério.
- Ignorar testes de integração.

## Checklist de Entendimento

- O sistema é uma unidade de deploy?
- Os módulos internos têm limites claros?
- Existe acoplamento escondido pelo banco?
- Que mudança justificaria separar algo?
- O time consegue operar a alternativa distribuída?

## Prática Recomendada

- Desenhe um monolito modular para a plataforma de estudos.
- Defina quatro módulos principais.
- Liste quais dependências são permitidas.
- Escreva quando você consideraria separar um módulo.
