# Twelve-Factor Apps

## Objetivo

Ao final desta aula, voce deve ser capaz de:

- explicar o objetivo dos Twelve-Factor Apps;
- entender fatores importantes para apps implantaveis e operaveis;
- conectar configuracao, logs, processos e dependencias com arquitetura;
- preparar a entrada nos modulos de cloud e infraestrutura.

## Onde Esta Aula Entra no Curso

- Modulo: 15. Arquitetura de Software
- Aula anterior: Serverless
- Proxima aula: Service Mesh
- Pre-requisitos: Backend, variaveis de ambiente, deploy, logging, processos, APIs e observabilidade.

## Ideia Central

Twelve-Factor Apps e um conjunto de principios para construir aplicacoes que rodam bem em ambientes modernos de deploy. Eles tratam de codigo, dependencias, configuracao, servicos externos, build, release, processos, logs e portabilidade.

No contexto deste curso, eles ajudam a conectar arquitetura de software com operacao e cloud sem entrar ainda em Kubernetes ou infraestrutura profunda.

## Topicos da Aula

- Codigo e dependencias.
- Configuracao no ambiente.
- Servicos externos como recursos anexados.
- Build, release e run.
- Processos stateless.
- Logs como fluxo de eventos.

## Exemplo Pratico

```text
API preparada para deploy:
- configuracao por variaveis de ambiente;
- dependencias declaradas;
- logs enviados para stdout;
- processo stateless;
- banco, cache e fila tratados como recursos externos;
- build separado de execucao.
```

## Analogias e Intuicoes

- Twelve-Factor e uma lista de habitos para aplicacoes portaveis.
- Uma app operavel nasce no design, nao apenas no deploy.

## Erros Comuns

- Colocar configuracao sensivel no codigo.
- Depender de estado local no servidor.
- Tratar logs como arquivo manual.
- Misturar build e runtime sem criterio.
- Ignorar diferenca entre ambiente local e producao.

## Checklist de Entendimento

- A configuracao esta fora do codigo?
- A aplicacao e stateless?
- Logs sao consumiveis pela plataforma?
- Servicos externos sao declarados como dependencias?
- Build, release e run estao separados?

## Pratica Recomendada

- Pegue uma API local.
- Liste configuracoes sensiveis.
- Identifique estado local indevido.
- Descreva como logs deveriam sair.
- Escreva um checklist minimo de deploy.
