# Resources e Data Sources

## Objetivo

Ao final desta aula, você deve ser capaz de:

- diferenciar resource e data source;
- entender quando Terraform cria algo e quando apenas consulta algo;
- reconhecer referências entre recursos;
- evitar gerenciar sem querer recursos que deveriam ser apenas lidos.

## Onde Esta Aula Entra na Formação

- Módulo: 18. Infrastructure as Code
- Aula anterior: Providers
- Próxima aula: Dependências e Grafo de Recursos
- Pré-requisitos: HCL, providers, recursos cloud e leitura de configuração.

## Ideia Central

Resource representa algo que Terraform gerencia: criar, alterar ou destruir. Data source representa algo que Terraform consulta para usar informação existente.

Essa diferença é fundamental. Um banco, bucket ou fila criado pela configuração costuma ser resource. Uma imagem existente, uma rede compartilhada ou uma informação de conta pode ser data source.

## Tópicos da Aula

- Resource como objeto gerenciado.
- Data source como leitura de informação.
- Referências entre blocos.
- Nomes locais e nomes reais.
- Quando consultar em vez de criar.

## Exemplo Prático

Recurso vs Dado no PetCare:

- Resource: `resource "aws_instance" "api_petcare" { ... }` => O Terraform VAI CRIAR essa máquina na AWS.
- Data Source: `data "aws_vpc" "rede_default" { default = true }` => A rede já existe. O Terraform VAI LER essa rede da AWS e trazer os IDs dela pro nosso código poder usar.

## Analogias e Intuições

- Resource é a Ordem de Compra de um tomógrafo novo. Data Source é ir na sala do almoxarifado ler o número de série de um tomógrafo que já existe lá há anos.

## Erros Comuns

- Criar recurso duplicado em vez de consultar um existente.
- Consultar recurso que deveria ser controlado pela mesma configuração.
- Confundir nome local com identificador real.
- Referenciar atributo sem entender se ele é conhecido antes ou depois do apply.

## Checklist de Entendimento

- Este bloco cria algo ou só lê informação?
- Quem é dono do recurso?
- O recurso pode ser destruído por esta configuração?
- A referência está clara?
- Existe dependência implícita entre esses blocos?
