# Dependências e Grafo de Recursos

## Objetivo

Ao final desta aula, você deve ser capaz de:

- explicar como dependências aparecem em Terraform;
- entender por que a ordem de criação não deve depender só da ordem dos arquivos;
- reconhecer dependências implícitas e explícitas;
- conectar grafo de recursos com plano de execução.

## Onde Esta Aula Entra no Curso

- Próxima aula: Variables, Local Values e Outputs

## Ideia Central

Terraform monta um grafo de recursos para entender relações entre blocos. Se um serviço usa o identificador de uma rede, existe uma dependência: a rede precisa estar conhecida antes do serviço.

Muitas dependências surgem por referência direta. Outras podem precisar ser declaradas explicitamente quando a relação não aparece nos atributos, mas existe no comportamento real da plataforma.

## Tópicos da Aula

- Dependências implícitas por referência.
- Dependências explícitas.
- Grafo de recursos.
- Paralelismo e ordem de execução.
- Relação entre plano e dependências.

## Exemplo Prático

```hcl
resource "example_network" "main" {
  name = "main"
}

resource "example_service" "api" {
  name       = "api"
  network_id = example_network.main.id
}
```

## Analogias e Intuições

- O grafo é uma lista de "isso depende daquilo".
- Terraform não lê arquivos como uma receita linear; ele calcula relações.

## Erros Comuns

- Achar que a ordem no arquivo define a ordem de criação.
- Usar dependência explícita para esconder desenho confuso.
- Não perceber dependências externas ao Terraform.
- Ignorar que mudanças em um recurso podem forçar troca em outro.

## Checklist de Entendimento

- Que recurso depende de qual?
- A dependência aparece por referência?
- Existe dependência real que a configuração não mostra?
- O plano indica recriação em cadeia?
- A ordem de execução faz sentido?

## Prática Recomendada

- Desenhe quatro recursos de uma aplicação.
- Trace setas de dependência entre eles.
- Identifique quais dependências apareceriam por referência.
- Identifique uma dependência que talvez precisasse ser explícita.
