# State no Terraform

## Objetivo

Ao final desta aula, você deve ser capaz de:

- explicar o que é state no Terraform;
- entender por que Terraform precisa mapear configuração e recursos reais;
- reconhecer riscos de perder ou expor state;
- diferenciar configuração, state e infraestrutura real.

## Ideia Central

State é o registro que Terraform usa para acompanhar os recursos gerenciados e mapear blocos da configuração para objetos reais no provedor. Sem state, a ferramenta perde parte importante do contexto sobre o que ela administra.

Existem três coisas diferentes: os arquivos `.tf`, o state e a infraestrutura real. O plano nasce da comparação entre esses elementos e as informações do provider.

State é sensível operacionalmente. Perder state pode dificultar o gerenciamento. Expor state pode revelar informações internas e, em alguns casos, dados sensíveis.

## Tópicos da Aula

## Exemplo Prático

```text
Configuração:
- resource "bucket" "uploads"

State:
- bucket.uploads corresponde ao bucket real app-uploads-123

Infraestrutura real:
- bucket existe no provedor cloud com atributos atuais
```

## Analogias e Intuições

- State é o inventário que Terraform usa para saber o que está sob sua responsabilidade.
- Arquivo `.tf` diz o que você quer; state registra o que Terraform conhece.

## Erros Comuns

- Commitar state local no Git.
- Apagar state sem backup.
- Editar state manualmente sem entender impacto.
- Achar que state é igual à configuração.
- Ignorar que state pode conter dados sensíveis.

## Checklist de Entendimento

- Onde está o state?
- Quem tem acesso a ele?
- Que recursos estão mapeados nele?
- O state representa fielmente a realidade?
- Existe dado sensível armazenado nele?

## Prática Recomendada

- Desenhe três colunas: configuração, state e infraestrutura real.
- Coloque o mesmo recurso nas três colunas.
- Explique o que acontece se alguém apaga o recurso manualmente.
- Explique o que acontece se o state for perdido.
