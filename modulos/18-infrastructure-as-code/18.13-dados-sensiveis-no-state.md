# Dados Sensíveis no State

## Objetivo

Ao final desta aula, você deve ser capaz de:

- entender por que state pode conter dados sensíveis;
- diferenciar mascaramento em plano de proteção real do dado;
- reconhecer riscos de outputs e variáveis sensíveis;
- definir cuidados mínimos para proteger state.

## Onde Esta Aula Entra no Curso

- Módulo: 18. Infrastructure as Code
- Aula anterior: Remote State e Locking de State
- Próxima aula: Drift e Importação de Recursos Existentes
- Pré-requisitos: State, remote state, secrets, IAM, variáveis e outputs.

## Ideia Central

Terraform pode mascarar valores sensíveis em saídas de terminal e planos, mas isso não significa que o dado desapareceu. Dependendo do recurso e do provider, valores sensíveis podem ficar registrados no state.

Por isso state deve ser tratado como dado sensível. Controle de acesso, criptografia do backend, segregação por ambiente e cuidado com outputs são parte da segurança de IaC.

Marcar uma variável como sensível ajuda a evitar exposição em logs e plano, mas não substitui secret manager, permissões corretas e proteção do state.

## Tópicos da Aula

- Valores sensíveis em Terraform.
- Mascaramento em plano e logs.
- Dados sensíveis persistidos no state.
- Outputs sensíveis.
- Controle de acesso ao backend.
- Relação com secret managers.

## Exemplo Prático

```hcl
variable "database_password" {
  type      = string
  sensitive = true
}
```

```text
Boa prática:
- não versionar segredo;
- evitar output de segredo;
- proteger acesso ao state;
- preferir integração com secret manager quando possível.
```

## Analogias e Intuições

- Mascarar na tela é cobrir o valor durante a conversa; não é apagar do cofre.
- State deve ser tratado como documento confidencial da infraestrutura.

## Erros Comuns

- Achar que `sensitive = true` protege tudo.
- Expor segredo em output.
- Dar acesso amplo ao backend de state.
- Commitar arquivos `.tfvars` com credenciais reais.
- Misturar segredos de todos os ambientes no mesmo lugar.

## Checklist de Entendimento

- Esse valor pode aparecer no state?
- Quem consegue ler o state?
- O backend criptografa dados?
- Algum output expõe informação sensível?
- O segredo deveria vir de um secret manager?

## Prática Recomendada

- Liste valores sensíveis de uma infraestrutura de API.
- Indique quais poderiam aparecer no state.
- Defina quem pode acessar state de produção.
- Reescreva um output inseguro como output sensível ou remova-o.
