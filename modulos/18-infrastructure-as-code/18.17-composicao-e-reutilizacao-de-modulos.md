# Composição e Reutilização de Módulos

## Objetivo

Ao final desta aula, você deve ser capaz de:

- compor módulos para formar uma infraestrutura maior;
- entender dependências entre módulos;
- reconhecer limites de reutilização;
- evitar acoplamento excessivo entre módulos.

## Onde Esta Aula Entra na Formação

- Módulo: 18. Infrastructure as Code
- Aula anterior: Modules
- Próxima aula: Organização por Ambientes
- Pré-requisitos: Modules, outputs, dependências, recursos cloud e arquitetura básica.

## Ideia Central

Composição é usar módulos menores para construir uma infraestrutura maior. Um módulo de rede pode expor identificadores usados por um módulo de aplicação. Um módulo de banco pode expor endpoint usado por configuração de aplicação.

Reutilização não significa criar uma abstração universal. Módulos devem refletir padrões reais do time e do projeto. Quando um módulo tenta atender todos os casos, ele vira uma caixa preta difícil de evoluir.

## Tópicos da Aula

- Módulos pequenos e focados.
- Outputs como contrato.
- Dependências entre módulos.
- Composição no root module.
- Versionamento de módulos.
- Riscos de abstração excessiva.

## Exemplo Prático

```hcl
module "network" {
  source = "./modules/network"
}

module "api" {
  source     = "./modules/api"
  network_id = module.network.network_id
}
```

## Analogias e Intuições

- Compor módulos é montar infraestrutura com peças que têm encaixes claros.
- Output é parte do contrato público do módulo.

## Erros Comuns

- Fazer módulos que dependem de detalhes internos uns dos outros.
- Usar output para expor tudo.
- Criar módulo compartilhado sem versionamento.
- Reutilizar módulo em contexto incompatível.
- Misturar decisões de ambiente dentro de módulo genérico.

## Checklist de Entendimento

- Que contrato este módulo oferece?
- Que módulo depende de qual output?
- A composição está clara no root module?
- O módulo é reutilizável ou apenas movido de lugar?
- Como mudanças no módulo afetariam consumidores?

## Prática Recomendada

- Desenhe três módulos: network, storage e api.
- Defina entradas e saídas de cada um.
- Mostre como o root module compõe os três.
- Liste uma mudança quebradora em um módulo.
