# Organização por Ambientes

## Objetivo

Ao final desta aula, você deve ser capaz de:

- explicar por que ambientes precisam de separação explícita;
- comparar estratégias de organização por diretórios, variáveis e state;
- reconhecer riscos de misturar dev, staging e produção;
- desenhar uma estrutura simples de ambientes.

## Onde Esta Aula Entra no Curso

- Módulo: 18. Infrastructure as Code
- Aula anterior: Composição e Reutilização de Módulos
- Próxima aula: Workspaces e Suas Limitações
- Pré-requisitos: Modules, remote state, locking, variables, IAM, Git e ambientes de desenvolvimento.

## Ideia Central

Ambientes diferentes têm necessidades diferentes: produção exige mais cuidado, permissões mais restritas, state protegido, revisão forte e mudanças controladas. Desenvolvimento pode ser mais barato e descartável.

Organizar ambientes em IaC é decidir onde ficam configurações, variáveis, state e permissões. Não existe uma única estrutura perfeita. O ponto essencial é evitar que uma mudança de dev atinja produção por acidente.

## Tópicos da Aula

- Dev, staging e produção.
- Separação de state por ambiente.
- Diretórios por ambiente.
- Variáveis por ambiente.
- Permissões por ambiente.
- Revisão e promoção de mudanças.

## Exemplo Prático

```text
infra/
  modules/
    network/
    app/
  environments/
    dev/
    staging/
    production/
```

## Analogias e Intuições

- Ambientes são salas diferentes; cada uma precisa de chave e inventário próprios.
- Produção não deve ser só "dev com nome diferente".

## Erros Comuns

- Usar o mesmo state para todos os ambientes.
- Aplicar produção a partir do diretório errado.
- Dar a todos permissão de escrita em produção.
- Copiar configuração sem entender diferenças.
- Esconder diferenças críticas em variáveis soltas.

## Checklist de Entendimento

- Cada ambiente tem state separado?
- Quem pode aplicar em produção?
- Quais variáveis mudam entre ambientes?
- O custo de dev está controlado?
- Como uma mudança chega de dev a produção?

## Prática Recomendada

- Desenhe uma estrutura de pastas para três ambientes.
- Defina um backend de state para cada ambiente.
- Liste permissões necessárias para dev e produção.
- Explique como evitar aplicar no ambiente errado.
