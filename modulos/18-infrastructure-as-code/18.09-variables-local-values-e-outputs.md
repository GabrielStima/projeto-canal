# Variables, Local Values e Outputs

## Objetivo

Ao final desta aula, você deve ser capaz de:

- diferenciar variables, local values e outputs;
- entender como parametrizar configuração sem duplicação desnecessária;
- expor informações úteis com outputs;
- evitar transformar variáveis em bagunça global.

## Ideia Central

Variables recebem valores de entrada. Local values ajudam a nomear expressões e compor valores internos. Outputs expõem informações geradas pela configuração, como endpoints, nomes, identificadores ou valores necessários para outro módulo.

Esses recursos deixam a configuração mais reutilizável e legível, mas também podem criar indireção demais. O objetivo é tornar intenção clara, não esconder tudo atrás de nomes genéricos.

## Exemplo Prático

Parametrizando a Clínica:

Se tivermos `nome = "PetCare-SP"` chumbado no código inteiro, o código só serve para São Paulo.
Com Variáveis: `variable "filial_nome" {}`. Quando o script rodar, ele pergunta pro usuário. Se passarmos `RJ`, todas as tags da infra nascem com `PetCare-RJ`.
Os *Outputs* devolvem coisas que só a AWS sabe, ex: `output "ip_do_banco" { value = aws_db.banco.endpoint }`.

## Analogias e Intuições

- Variáveis (`variable`) são as opções que você preenche na tela de compra do carro (Cor, Motor). Locals (`local`) são contas que o vendedor faz com essas opções (Imposto de 10%). Outputs são as chaves do carro que ele te entrega no final.

## Erros Comuns

- Criar variável para tudo sem necessidade.
- Usar nomes genéricos como `value`, `config` e `settings`.
- Expor dado sensível em output.
- Duplicar expressões em vez de usar local com intenção clara.
- Confundir output com log.

## Checklist de Entendimento

- Esse valor precisa vir de fora?
- Esse cálculo merece um nome?
- Esse output será usado por alguém?
- Existe dado sensível envolvido?
- A parametrização melhora ou piora a leitura?
