## Ideia Central

HCL é a linguagem de configuração usada pelo Terraform. Ela permite declarar blocos como `terraform`, `provider`, `resource`, `data`, `variable`, `locals` e `output`.

HCL não deve ser tratada como uma linguagem de aplicação comum. O foco é descrever infraestrutura, valores, relações e composição. Quando a configuração começa a parecer um programa cheio de truques, normalmente há um problema de desenho.

## Exemplo Prático

Lendo HCL no Hospital:

```hcl
resource "aws_s3_bucket" "raio_x" {
  bucket = "petcare-raiox-imagens"
}
```
A sintaxe HCL (HashiCorp Configuration Language) é como um JSON com esteróides. É feita pra humanos lerem rápido, suporta comentários (coisa que JSON não suporta) e tem blocos literais fáceis de entender.

## Analogias e Intuições

- HCL não é uma linguagem de programação como JavaScript; você não escreve laços complexos ou algoritmos pesados nela. É um Contrato de intenções.

## Erros Comuns

- Tentar escrever lógica complexa demais em HCL.
- Copiar configuração sem entender os blocos.
- Colocar segredo diretamente em arquivo versionado.
- Confundir nome local do recurso com nome real no provedor.

## Checklist de Entendimento

- O que é um bloco?
- O que é um argumento?
- Como um recurso pode referenciar outro?
- Por que HCL não deve virar código de aplicação?
- Onde valores sensíveis podem aparecer por acidente?

## Prática Recomendada

- Escreva um bloco abstrato em HCL criando um recurso `banco_de_dados` e passando as variáveis `nome` e `tamanho`.
