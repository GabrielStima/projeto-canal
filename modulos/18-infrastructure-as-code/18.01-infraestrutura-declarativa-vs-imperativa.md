# Infraestrutura Declarativa vs Imperativa

## Objetivo

Ao final desta aula, você deve ser capaz de:

- diferenciar abordagem declarativa e imperativa;
- entender por que Terraform usa configuração declarativa;
- reconhecer quando comandos imperativos ainda aparecem;
- explicar o impacto disso no planejamento de mudanças.

## Onde Esta Aula Entra na Formação

- Módulo: 18. Infrastructure as Code
- Aula anterior: Infrastructure as Code
- Próxima aula: Idempotência em Infraestrutura
- Pré-requisitos: Cloud fundamentals, linha de comando, Git e noções de automação.

## Ideia Central

Infraestrutura imperativa descreve passos: crie isso, depois altere aquilo, depois conecte este recurso naquele. Infraestrutura declarativa descreve o estado desejado: quero esta rede, este banco, este bucket, estas permissões e estas relações.

Terraform trabalha principalmente de forma declarativa. Você escreve o estado desejado, e a ferramenta compara configuração, state e infraestrutura real para propor um plano de mudança.

Essa diferença muda a forma de pensar: menos foco em scripts de passos soltos, mais foco em declarar recursos e revisar o plano que leva do estado atual ao estado desejado.

## Tópicos da Aula

- Comandos imperativos.

- Por que IaC declarativa combina com revisão.
- Onde operações imperativas ainda aparecem.

## Exemplo Prático

```text
Imperativo:
1. criar bucket;
2. ativar versionamento;
3. configurar regra de acesso;
4. adicionar tags.

Declarativo:
- existe um bucket;
- versionamento está ativo;
- política de acesso é esta;
- tags esperadas são estas.
```

## Analogias e Intuições

- Imperativo é passar uma lista de passos para alguém executar.
- Declarativo é descrever como a sala deve ficar pronta no final.

## Erros Comuns

- Misturar scripts manuais com Terraform sem registrar mudanças.
- Achar que declarativo significa ausência de ordem.
- Ignorar que algumas operações têm efeitos destrutivos.
- Usar IaC como se fosse apenas shell script com outra sintaxe.

## Checklist de Entendimento

- O que é estado desejado?
- O que muda entre dizer "como fazer" e "o que deve existir"?
- Por que o plano é importante em uma ferramenta declarativa?
- Quando um comando imperativo ainda pode ser necessário?
- Que problema aparece se alguém muda recurso manualmente?

## Prática Recomendada

- Escolha três recursos cloud do módulo anterior.
- Descreva a criação deles em passos imperativos.
- Reescreva como estado desejado.
- Compare qual versão é mais fácil de revisar em code review.
