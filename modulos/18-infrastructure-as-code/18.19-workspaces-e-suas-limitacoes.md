# Workspaces e Suas Limitações

## Objetivo

Ao final desta aula, você deve ser capaz de:

- explicar o que são workspaces no Terraform;
- entender quando workspaces podem ajudar;
- reconhecer limitações para separar ambientes;
- evitar usar workspaces como solução mágica.

## Ideia Central

Workspaces permitem manter múltiplas instâncias de state para uma mesma configuração. Eles podem ser úteis em alguns cenários, especialmente para variações parecidas.

Mas workspaces não resolvem sozinhos diferenças reais de ambiente, permissões, backends, revisão e blast radius. Produção frequentemente precisa de separação mais explícita do que apenas trocar workspace.

O objetivo desta aula é aprender workspaces sem transformar workspaces em arquitetura padrão para tudo.

## Exemplo Prático

```bash
terraform workspace list
terraform workspace new dev
terraform workspace select dev
```

```text
Pergunta antes de usar:
- os ambientes são realmente parecidos?
- permissões e state precisam ser mais isolados?
- o time consegue evitar workspace errado?
```

## Analogias e Intuições

- Workspace é uma gaveta diferente para o state, não uma parede de segurança completa.
- Separação operacional envolve mais do que trocar um nome.

## Erros Comuns

- Usar workspace para produção sem avaliar riscos.
- Esquecer em qual workspace está.
- Misturar diferenças complexas de ambiente na mesma configuração.
- Achar que workspace substitui permissões e revisão.
- Não documentar estratégia de ambientes.

## Checklist de Entendimento

- O que muda entre os workspaces?
- Existe risco de aplicar no workspace errado?
- Produção precisa de isolamento maior?
- As permissões são diferentes por ambiente?
- A estratégia é compreensível para o time?

## Prática Recomendada

- Compare duas estratégias: workspaces e diretórios por ambiente.
- Liste vantagens e riscos de cada uma.
- Escolha uma para dev/staging/produção e justifique.
- Escreva um aviso que deveria aparecer antes de aplicar em produção.
