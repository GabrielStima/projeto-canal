# Providers

## Objetivo

Ao final desta aula, você deve ser capaz de:

- explicar o que é um provider no Terraform;
- entender por que providers precisam ser configurados;
- conectar provider com credenciais, região e permissões;
- reconhecer riscos de versões e autenticação.

## Ideia Central

Provider é o plugin que permite ao Terraform conversar com uma plataforma. Pode ser um provedor cloud, um serviço de DNS, uma ferramenta de monitoramento ou outro sistema que exponha uma API.

Configurar provider não é detalhe burocrático. É onde aparecem região, autenticação, permissões e versões. Uma credencial poderosa demais ou um provider sem controle de versão pode transformar uma mudança simples em risco operacional.

## Tópicos da Aula

- Região e escopo.
- Versionamento de providers.
- Relação entre provider e resources.

## Exemplo Prático

Providers no PetCare:

Como o Terraform consegue criar repositórios no Github e subir Banco na AWS ao mesmo tempo? Providers.
Providers são Plugins (binários invisíveis em GO) baixados na máquina do programador. O Provider `hashicorp/aws` sabe traduzir o código HCL para requests HTTP na API da Amazon.

## Analogias e Intuições

- O Terraform puro é um Maestro surdo-mudo. Os Providers são os Tradutores que ficam do lado dele. Se o Maestro quer mandar a Orquestra (AWS) tocar, ele gesticula pro tradutor, que passa a instrução exata adiante.

## Erros Comuns

- Usar credencial pessoal em automação.
- Dar permissões administrativas sem necessidade.
- Não fixar faixa de versão do provider.
- Configurar região errada.
- Esquecer que providers diferentes têm comportamentos diferentes.

## Checklist de Entendimento

- Que plataforma este provider controla?
- Como ele autentica?
- Quais permissões ele tem?
- Em qual região ou escopo ele atua?
- A versão está controlada?

## Prática Recomendada

- Pense e pesquise: É possível o Terraform criar um usuário no banco PostgreSQL (não o banco, o usuário dentro dele)? Qual Provider faria isso?
