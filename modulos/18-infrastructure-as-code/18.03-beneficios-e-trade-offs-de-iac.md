# Benefícios e Trade-offs de IaC

## Objetivo

Ao final desta aula, você deve ser capaz de:

- listar benefícios reais de IaC;
- reconhecer custos e riscos da prática;
- evitar tratar automação como sinônimo de segurança;
- decidir quando uma mudança de infraestrutura precisa de mais cuidado.

## Onde Esta Aula Entra na Formação

- Módulo: 18. Infrastructure as Code
- Aula anterior: Idempotência em Infraestrutura
- Próxima aula: Terraform e Panorama de Ferramentas IaC
- Pré-requisitos: IaC, infraestrutura declarativa, Git, code review, cloud fundamentals e segurança básica.

## Ideia Central

IaC melhora rastreabilidade, revisão, consistência, reprodução de ambientes e recuperação. Também facilita colaboração: em vez de alguém clicar sozinho no painel, o time revisa uma mudança em código.

Mas IaC também aumenta o alcance do erro. Um plano mal revisado pode destruir recurso, abrir permissão demais, vazar dado sensível ou replicar uma arquitetura ruim em vários ambientes.

A pergunta madura não é "IaC é bom ou ruim?". É "como usamos IaC com revisão, separação de ambientes, controle de acesso, state protegido e leitura cuidadosa de planos?".

## Tópicos da Aula

- Versionamento e histórico.
- Revisão e colaboração.
- Reprodutibilidade de ambientes.
- Risco de automação mal controlada.
- Custo de aprendizado e manutenção.

## Exemplo Prático

```text
Benefício:
- pull request mostra mudança em uma regra de firewall.

Trade-off:
- se a revisão for superficial, a regra insegura pode chegar a produção mais rápido.
```

## Analogias e Intuições

- IaC é uma alavanca: aumenta força, mas também aumenta o estrago de um movimento errado.
- Automação boa amplifica processos bons; automação ruim amplifica confusão.

## Erros Comuns

- Automatizar antes de entender a infraestrutura.
- Não revisar plano antes de aplicar.
- Usar a mesma credencial para tudo.
- Não proteger state.
- Criar módulos genéricos demais cedo demais.

## Checklist de Entendimento

- Que benefício esta mudança traz?
- Que risco ela amplia?
- Quem revisa o plano?
- Como voltar atrás se algo der errado?
- Existe dado sensível envolvido?

## Prática Recomendada

- Liste cinco benefícios de IaC para um time.
- Liste cinco riscos de IaC para o mesmo time.
- Escolha uma mudança perigosa e escreva quais proteções deveriam existir.
