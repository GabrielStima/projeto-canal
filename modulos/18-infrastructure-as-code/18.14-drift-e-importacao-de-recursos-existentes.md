# Drift e Importação de Recursos Existentes

## Objetivo

Ao final desta aula, você deve ser capaz de:

- explicar o que é drift;
- entender como mudanças manuais afetam IaC;
- reconhecer quando importar recurso existente;
- diferenciar importar recurso de escrever configuração correta.

## Onde Esta Aula Entra no Curso

- Módulo: 18. Infrastructure as Code
- Aula anterior: Dados Sensíveis no State
- Próxima aula: Ciclo de Vida dos Recursos
- Pré-requisitos: State, remote state, resources, plano, cloud fundamentals e operação.

## Ideia Central

Drift acontece quando a infraestrutura real muda sem que a configuração acompanhe. Pode ser uma alteração manual no painel, uma automação externa, uma mudança emergencial ou um comportamento do provider.

Importação permite trazer um recurso existente para o gerenciamento do Terraform, associando o recurso real ao state. Mas importar não é mágica: ainda é preciso escrever configuração compatível e revisar o plano para evitar alteração ou destruição inesperada.

## Tópicos da Aula

- O que é drift.
- Causas comuns de drift.
- Como plano revela diferenças.
- Importação de recursos existentes.
- Riscos pós-importação.
- Decisão entre recriar e importar.

## Exemplo Prático

```text
Cenário:
- bucket foi criado manualmente no painel;
- agora o time quer gerenciá-lo por Terraform.

Processo conceitual:
- escrever configuração correspondente;
- importar o recurso para o state;
- rodar plan;
- ajustar configuração até o plano não propor mudanças indevidas.
```

## Analogias e Intuições

- Drift é quando a realidade saiu do combinado.
- Importar é registrar no inventário algo que já existia.

## Erros Comuns

- Alterar produção manualmente e esquecer de refletir no código.
- Importar recurso e aplicar sem revisar o plano.
- Achar que importação cria configuração perfeita automaticamente.
- Não decidir quem é dono do recurso depois da importação.

## Checklist de Entendimento

- O que mudou fora do Terraform?
- A mudança deveria virar código?
- O recurso existente deve ser importado ou recriado?
- A configuração bate com a realidade?
- O plano pós-importação está limpo?

## Prática Recomendada

- Descreva três exemplos de drift.
- Escolha um recurso existente e planeje como importá-lo conceitualmente.
- Liste riscos de aplicar logo após importar.
- Defina uma regra de time para mudanças emergenciais manuais.
