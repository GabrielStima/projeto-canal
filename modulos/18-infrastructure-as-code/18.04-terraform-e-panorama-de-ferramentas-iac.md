# Terraform e Panorama de Ferramentas IaC

## Objetivo

Ao final desta aula, você deve ser capaz de:

- explicar o papel do Terraform;
- reconhecer que existem outras ferramentas de IaC;
- entender por que o módulo usa Terraform como base;
- diferenciar ferramenta, provedor e serviço cloud.

## Ideia Central

Terraform é uma ferramenta de Infrastructure as Code usada para definir, planejar e aplicar mudanças de infraestrutura em vários provedores. Ele usa arquivos de configuração, plugins chamados providers, recursos declarados e um state para acompanhar o que foi gerenciado.

Ele não é a única ferramenta possível. Existem opções nativas de provedores, ferramentas baseadas em linguagens de programação e plataformas gerenciadas. Nesta formação, Terraform será o caminho principal porque ajuda a aprender conceitos transferíveis: configuração declarativa, plano, state, módulos, providers e gestão de ambientes.

## Exemplo Prático

O Canivete Suíço Cloud:

- CloudFormation: Só funciona na AWS.
- ARM Templates: Só funciona na Azure.
- Terraform: Funciona com a API do Github, Datadog, AWS, GCP, Vercel e PagerDuty. A clínica PetCare usa Terraform para gerenciar TUDO num único repositório.

## Analogias e Intuições

- Ferramentas nativas (CloudFormation) são carregadores de celular de marcas exclusivas. Terraform é a tomada USB universal: funciona no carro, no computador e na parede.

## Erros Comuns

- Achar que Terraform substitui conhecimento de cloud.
- Confundir Terraform com um provedor cloud.
- Escolher ferramenta pela moda, não pelo contexto.
- Ignorar limites, bugs e comportamentos específicos de providers.

## Checklist de Entendimento

- Qual problema Terraform resolve?
- O que é um provider?
- Por que state é necessário?
- Que tipo de decisão ainda depende de arquitetura cloud?
- Quando uma ferramenta nativa do provedor poderia fazer sentido?

## Prática Recomendada

- Qual seria a limitação comercial severa de uma empresa (Vendor Lock-in) se ela atrelasse todo seu IaC no CloudFormation?
