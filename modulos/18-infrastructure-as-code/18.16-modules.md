# Modules

## Objetivo

Ao final desta aula, você deve ser capaz de:

- explicar o que é um module no Terraform;
- entender module raiz e módulos chamados;
- reconhecer entradas e saídas de módulos;
- evitar criar abstrações cedo demais.

## Onde Esta Aula Entra na Formação

- Módulo: 18. Infrastructure as Code

- Próxima aula: Composição e Reutilização de Módulos
- Pré-requisitos: Variables, outputs, resources, dependências, state e organização de configuração.

## Ideia Central

Module é uma unidade de configuração Terraform. Todo diretório com configuração Terraform é um módulo. O módulo onde você executa Terraform é o root module. Ele pode chamar outros módulos para organizar e reutilizar infraestrutura.

Módulos ajudam a reduzir repetição e criar padrões, mas também podem esconder detalhes importantes. Um bom módulo tem propósito claro, entradas compreensíveis e outputs úteis.

## Tópicos da Aula

- Root module.
- Child modules.
- Inputs e outputs de módulos.
- Organização de arquivos.
- Reuso com responsabilidade.
- Quando não criar módulo.

## Exemplo Prático

```hcl
module "storage" {
  source      = "./modules/storage"
  environment = "dev"
}

output "bucket_name" {
  value = module.storage.bucket_name
}
```

## Analogias e Intuições

- Module é uma peça reutilizável de infraestrutura.
- Um módulo bom reduz repetição sem esconder decisões críticas.

## Erros Comuns

- Criar módulo antes de entender o caso de uso.
- Fazer módulo genérico demais.
- Expor outputs demais.
- Usar nomes de variáveis vagos.
- Esconder risco de destruição dentro de abstração.

## Checklist de Entendimento

- Qual problema este módulo resolve?
- Quais entradas ele precisa?
- Quais saídas são úteis?
- O módulo esconde alguma decisão perigosa?
- Ele será reutilizado de verdade?

## Prática Recomendada

- Pegue uma configuração repetida.
- Identifique o que poderia virar módulo.
- Defina três inputs e dois outputs.
- Explique por que esse módulo vale ou não vale a pena.
