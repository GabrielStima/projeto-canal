# HTTP

## Objetivo

Ao final desta aula, você deve ser capaz de:

- explicar o que é HTTP;
- entender HTTP como protocolo de comunicação da Web;
- reconhecer os elementos básicos de uma request e de uma response;
- interpretar métodos, status codes, headers e body em nível inicial.

## Ideia Central

HTTP é o protocolo mais importante para entender a comunicação da Web.

Ele define como clientes e servidores trocam mensagens. Uma request HTTP pede algo. Uma response HTTP devolve o resultado desse pedido.

No início, não é preciso decorar todos os métodos, headers e status codes. O essencial é entender a estrutura da conversa e conseguir observar essa conversa nas ferramentas do browser.

## Tópicos da Aula

- O que é HTTP
- Request HTTP
- Response HTTP
- Métodos como GET e POST
- Status codes como 200, 404 e 500
- Headers
- Body
- HTTP como protocolo sem estado por padrão

## Exemplo Prático

Uma request simples pode pedir uma página:

```text
GET /sobre HTTP/1.1
Host: exemplo.com
```

Uma response simplificada poderia ser:

```text
HTTP/1.1 200 OK
Content-Type: text/html

<html>...</html>
```

O cliente pediu o recurso `/sobre`. O servidor respondeu com sucesso e informou que o conteúdo é HTML.

## Analogias e Intuições

- HTTP é a regra da conversa entre cliente e servidor na Web.
- Método é a intenção do pedido.
- Status code é um resumo do resultado.
- Header é metadado sobre a mensagem.
- Body é o conteúdo principal quando existe.

## Erros Comuns

- Achar que HTTP é a mesma coisa que HTML.
- Decorar status codes sem entender a categoria geral.
- Confundir método HTTP com rota.
- Ignorar headers porque eles parecem detalhe.
- Não perceber que HTTP, por padrão, não lembra automaticamente requests anteriores.

## Checklist de Entendimento

Antes de seguir, você deve conseguir responder:

- O que é HTTP?
- Qual a diferença entre request e response HTTP?
- Para que servem métodos?
- Para que servem status codes?
- O que são headers e body?

## Prática Recomendada

- Abra um site no browser.
- Use as ferramentas de desenvolvedor para olhar uma request.
- Identifique método, status, headers e, se existir, body.
- Escreva com suas palavras o que aconteceu naquela troca.
