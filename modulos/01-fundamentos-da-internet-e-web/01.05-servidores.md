# Servidores

## Objetivo

Ao final desta aula, você deve ser capaz de:

- explicar o que é um servidor no contexto da Web;
- diferenciar servidor como máquina, programa e papel;
- entender como servidores respondem requests;
- reconhecer exemplos de servidores em aplicações reais.

## Onde Esta Aula Entra no Curso

- Módulo: Fundamentos da Internet e da Web
- Aula anterior: DNS
- Próxima aula: Browsers
- Pré-requisitos: cliente-servidor, protocolos e DNS

## Ideia Central

Servidor pode significar mais de uma coisa.

Às vezes, falamos de servidor como uma máquina. Às vezes, como um programa que escuta requests. Às vezes, como o papel de responder a um cliente.

No desenvolvimento web, o mais importante no começo é entender a responsabilidade: um servidor recebe pedidos, processa esses pedidos e devolve respostas.

## Tópicos da Aula

- Servidor como papel
- Servidor como programa
- Servidor como máquina ou ambiente
- O que significa "hospedar" uma aplicação
- Servidores estáticos e dinâmicos
- Onde backend entra nessa conversa

## Exemplo Prático

Um site simples pode ter um servidor que devolve arquivos prontos:

- HTML
- CSS
- JavaScript
- imagens

Uma aplicação mais dinâmica pode ter um servidor que:

- valida login;
- busca dados em um banco;
- calcula uma regra;
- grava uma compra;
- devolve JSON para o frontend.

## Analogias e Intuições

- Servidor é quem atende pedidos dentro de uma comunicação.
- Uma mesma máquina pode rodar vários programas servidores.
- Um servidor não precisa ser "grande"; até seu computador pode rodar um servidor local para desenvolvimento.

## Erros Comuns

- Achar que servidor é apenas um computador físico.
- Confundir servidor com backend, mesmo quando há sobreposição.
- Pensar que todo servidor precisa gerar páginas completas.
- Esquecer que servidores também podem devolver arquivos estáticos.
- Imaginar que o servidor sempre sabe quem é o usuário sem receber alguma informação.

## Checklist de Entendimento

Antes de seguir, você deve conseguir responder:

- O que um servidor faz?
- Servidor é uma máquina, um programa ou um papel?
- O que significa hospedar uma aplicação?
- Qual a diferença entre conteúdo estático e dinâmico?
- Como um servidor participa do ciclo request/response?

## Prática Recomendada

- Liste 3 exemplos de coisas que um servidor pode devolver.
- Explique a diferença entre servir um arquivo estático e gerar uma resposta dinâmica.
- Desenhe um fluxo simples: browser envia request, servidor processa, servidor devolve response.
