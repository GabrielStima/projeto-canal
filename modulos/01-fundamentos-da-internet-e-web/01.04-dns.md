# DNS

## Objetivo

Ao final desta aula, você deve ser capaz de:

- explicar o papel do DNS;
- entender por que nomes de domínio são úteis;
- descrever, em alto nível, como um nome pode virar um endereço;
- reconhecer erros comuns envolvendo DNS.

## Ideia Central

DNS é o sistema que ajuda a transformar nomes legíveis, como `exemplo.com`, em endereços que computadores conseguem usar para encontrar o destino.

Pessoas preferem nomes. Máquinas precisam de endereços. O DNS existe para aproximar essas duas necessidades.

Quando você digita uma URL no browser, antes de pedir o conteúdo ao servidor, o sistema precisa descobrir para onde esse pedido deve ir. O DNS participa dessa descoberta.

## Exemplo Prático

Imagine que você quer acessar `www.exemplo.com`.

O browser não "adivinha" onde esse site está. Ele precisa descobrir qual endereço está associado a esse nome. Essa consulta pode passar por caches e servidores DNS até encontrar uma resposta.

Depois disso, o browser consegue tentar se comunicar com o servidor correto.

## Analogias e Intuições

- DNS é como uma agenda que traduz nomes para endereços.
- Domínio é um nome fácil para humanos; endereço é uma referência útil para máquinas.
- Cache de DNS é como lembrar uma resposta recente para não perguntar tudo de novo.

## Erros Comuns

- Achar que comprar um domínio cria automaticamente um site completo.
- Confundir domínio com servidor.
- Confundir DNS com HTTP.
- Esperar que mudanças de DNS apareçam instantaneamente em todos os lugares.
- Esquecer que caches podem manter respostas antigas por algum tempo.
