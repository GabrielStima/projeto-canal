# Cliente, Servidor, Request, Response e Estado

## Objetivo

Ao final desta aula, você deve ser capaz de:

- explicar o que são cliente e servidor;
- descrever o ciclo request/response;
- entender o que significa estado em uma aplicação web;
- reconhecer esses conceitos em exemplos reais.

## Ideia Central

Na Web, uma interação comum começa com um cliente pedindo algo e um servidor respondendo.

O cliente é quem inicia a comunicação. Em muitos casos, ele é um browser, mas também pode ser um aplicativo mobile, uma ferramenta de terminal ou outro sistema.

O servidor é quem recebe a solicitação, processa o pedido e devolve uma resposta.

A request é o pedido. A response é a resposta. O estado é a informação que precisa ser lembrada entre interações, como usuário logado, carrinho de compras, preferências ou progresso de uma ação.

## Exemplo Prático

Imagine uma pessoa acessando uma página de login:

1. O browser envia uma request pedindo a página.
2. O servidor devolve uma response com o conteúdo.
3. A pessoa preenche usuário e senha.
4. O browser envia outra request com esses dados.
5. O servidor valida as informações.
6. A response informa se o login funcionou.
7. A aplicação precisa lembrar que aquela pessoa está autenticada.

Esse "lembrar" é parte do problema de estado.

## Analogias e Intuições

- Cliente é quem faz o pedido; servidor é quem atende.
- Request é a pergunta; response é a resposta.
- Estado é a memória necessária para que interações separadas façam sentido juntas.

## Erros Comuns

- Achar que cliente sempre significa "usuário".
- Achar que servidor é sempre uma máquina física específica.
- Confundir request com response.
- Pensar que a Web lembra automaticamente tudo sobre o usuário.
- Ignorar estado até chegar em autenticação, carrinho ou sessões.
