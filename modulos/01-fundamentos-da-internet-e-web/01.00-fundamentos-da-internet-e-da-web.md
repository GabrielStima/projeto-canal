# Fundamentos da Internet e da Web

## Objetivo

Ao final desta aula, você deve ser capaz de:
- explicar o que é a internet e como ela se diferencia da web;
- descrever o papel de protocolos, DNS e servidores na comunicação entre máquinas;
- entender o ciclo request/response e o que acontece entre o browser e um servidor;
- reconhecer os conceitos que sustentam todo o restante da formação.

## Ideia Central

Antes de escrever uma linha de código para o PetCare OS, você precisa entender onde essa aplicação vai existir. Toda aplicação web roda sobre uma infraestrutura de protocolos, servidores, nomes de domínio e regras de comunicação que existem há décadas e continuam sendo a base da web moderna.

Este módulo cobre o caminho completo que uma requisição percorre: do browser do tutor até o servidor da clínica, passando por DNS, TCP/IP, HTTP e cache. Esses conceitos não são teoria isolada — eles aparecem toda vez que você depura um erro de rede, configura um servidor ou entende por que uma página está lenta.

Ao final do módulo, você terá um mapa mental claro de como a web funciona, que será a base sobre a qual frontend, backend, APIs e segurança farão sentido.

## Tópicos da Aula

- Internet vs Web: dois conceitos distintos.
- Cliente, servidor, request, response e estado.
- Protocolos: o que são e por que existem.
- DNS: como nomes viram endereços.
- Servidores: o que são e o que fazem.
- Browsers: como renderizam e se comunicam.
- HTTP: o protocolo da web.
- Cache HTTP: evitar trabalho repetido.
- Frontend e backend como papéis na web.
- Caminho completo de uma requisição.

## Analogias e Intuições

- A internet é a rodovia; a web é um dos serviços que trafega nela, como o correio trafega em estradas.
- DNS é uma agenda telefônica: você sabe o nome, ela devolve o número (IP).
- HTTP é o idioma que cliente e servidor usam para se entender — sem ele, os dados chegam mas ninguém sabe o que fazer com eles.

## Erros Comuns

- Confundir internet com web e tratar os dois como sinônimos.
- Achar que o browser "baixa o site" de uma vez — na prática são dezenas de requisições separadas.
- Ignorar o papel do cache e tentar debugar comportamentos que são, na verdade, respostas cacheadas.
- Não entender que HTTP é sem estado por padrão, o que afeta autenticação e sessões.
- Tratar frontend e backend como tecnologias, não como papéis com responsabilidades distintas.

## Prática Recomendada

- Abra o DevTools do browser (aba Network) e observe as requisições ao carregar qualquer site.
- Identifique: qual é a primeira requisição? Quais recursos são cacheados?
- Faça um diagrama simples do caminho de uma requisição desde o browser até o servidor e de volta.
- Pesquise o que acontece se o servidor de DNS estiver fora do ar.
