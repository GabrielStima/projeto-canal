# Frontend e Backend Como Papéis na Web

## Objetivo

Ao final desta aula, você deve ser capaz de:

- explicar frontend e backend como papéis;
- entender a relação entre browser, servidor e aplicação;
- diferenciar interface, regras de negócio e dados;
- evitar a ideia limitada de que frontend e backend são apenas nomes de tecnologias.

## Onde Esta Aula Entra no Curso

- Módulo: Fundamentos da Internet e da Web
- Aula anterior: Cache HTTP
- Próxima aula: O caminho completo de uma requisição Web
- Pré-requisitos: browsers, servidores, HTTP e estado

## Ideia Central

Frontend e backend são papéis dentro de uma aplicação web.

Frontend é a parte mais próxima da interação com a pessoa usuária. No contexto da Web, normalmente roda no browser e lida com interface, eventos, apresentação, navegação e chamadas para obter ou enviar dados.

Backend é a parte que roda do lado do servidor ou de serviços remotos. Ele lida com regras, autenticação, persistência, integrações, processamento e respostas para clientes.

Esses papéis podem aparecer de formas diferentes conforme a arquitetura, mas a separação ajuda a entender responsabilidades.

## Exemplo Prático

Em uma tela de cadastro:

- o frontend mostra campos, mensagens e botões;
- o frontend valida algumas informações simples;
- o frontend envia uma request;
- o backend valida regras importantes;
- o backend salva dados ou rejeita o cadastro;
- o backend devolve uma response;
- o frontend mostra o resultado para a pessoa.

Frontend e backend trabalham juntos no mesmo fluxo.

## Analogias e Intuições

- Frontend é a camada de interação; backend é a camada de processamento e regras.
- Frontend pergunta e apresenta; backend decide, busca, grava e responde.
- A fronteira entre frontend e backend é uma decisão de responsabilidade, não só de ferramenta.

## Erros Comuns

- Achar que frontend é "só visual".
- Achar que backend é "só banco de dados".
- Confundir frontend/backend com linguagem específica.
- Colocar regra crítica apenas no frontend.
- Pensar que toda aplicação precisa ter a mesma divisão interna.

## Checklist de Entendimento

- Qual é o papel do browser nessa divisão?
- Qual é o papel do servidor nessa divisão?
- Por que algumas validações não devem ficar só no frontend?
