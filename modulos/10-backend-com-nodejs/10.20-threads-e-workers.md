## Ideia Central

Node.js é excelente para muitas tarefas de I/O, mas código pesado de CPU pode bloquear a thread principal. Quando isso acontece, uma API pode atrasar outras requisições mesmo que elas sejam simples.

Workers permitem executar trabalho pesado em outro contexto de execução. Eles não devem ser usados como solução mágica para tudo, mas são uma ferramenta importante quando o backend precisa processar dados, transformar arquivos, calcular informações ou executar tarefas custosas.

## Exemplo Prático

```text
Sem worker:
- requisição inicia cálculo pesado;
- event loop fica ocupado;
- outras requisições atrasam.

Com worker:
- requisição delega cálculo;
- thread principal continua respondendo;
- resultado volta quando estiver pronto.
```

## Analogias e Intuições

- Worker é chamar outra pessoa para uma tarefa demorada enquanto você continua atendendo.
- Paralelismo ajuda, mas também exige coordenacao.

## Erros Comuns

- Usar worker para operação simples de banco.
- Achar que `async/await` resolve CPU pesada.
- Criar workers demais.
- Ignorar custo de passar dados entre contextos.
- Não tratar erro vindo do worker.
