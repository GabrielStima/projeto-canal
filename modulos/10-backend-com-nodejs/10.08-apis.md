# APIs

## Objetivo

Ao final desta aula, você deve ser capaz de:

- explicar o que é uma API;
- diferenciar API, endpoint, rota e contrato;
- entender consumidor e provedor de uma API;
- preparar o estudo de JSON APIs, REST, SOAP e GraphQL.

## Onde Esta Aula Entra na Formação

- Módulo: 10. Backend com Node.js
- Aula anterior: Frameworks Backend
- Próxima aula: JSON APIs
- Pré-requisitos: HTTP, chamadas de API no frontend, servidor HTTP e frameworks backend em alto nível.

## Ideia Central

API é uma forma definida de uma parte do sistema conversar com outra. No contexto web, normalmente pensamos em APIs HTTP: um cliente envia uma requisição para um servidor, e o servidor devolve uma resposta seguindo algum contrato.

O ponto mais importante é contrato. Uma API não é apenas código no backend. Ela define entradas, saídas, erros, formatos, regras e expectativas para quem consome.

## Tópicos da Aula

- O que é API.
- Consumidor e provedor.
- Endpoint, rota e recurso.
- Contrato de entrada e saída.
- API pública, privada e interna.
- Compatibilidade entre cliente e servidor.

## Exemplo Prático

```text
GET /pets/42

Resposta esperada:
- status: 200
- corpo: dados públicos do pet
- erro possível: pet não encontrado
```

Mesmo sem mostrar implementação, já existe uma promessa entre o Portal do Tutor e o backend. Permissão para acessar o pet será aprofundada no módulo de autenticação e autorização.

## Analogias e Intuições

- API é um cardápio: mostra o que pode ser pedido, como pedir e o que esperar.
- Contrato ruim faz o cliente adivinhar comportamento.

## Erros Comuns

- Chamar qualquer rota solta de API bem desenhada.
- Mudar resposta sem pensar em quem consome.
- Não documentar erros.
- Misturar detalhes internos com contrato público.
- Criar endpoints pensando só no banco.

## Checklist de Entendimento

Antes de seguir, você deve conseguir responder:

- O que uma API define?
- Quem consome uma API?
- O que é endpoint?
- Por que contrato importa?
- O que pode quebrar um cliente?

## Prática Recomendada

- Escolha a tela de detalhes de um pet no Portal do Tutor.
- Descreva qual API essa tela precisaria consumir.
- Especifique método, caminho, entrada, saída e erros.
- Escreva o que aconteceria se a resposta mudasse sem aviso.
