# Runtime

## Objetivo

Ao final desta aula, você deve ser capaz de:

- explicar o que é um runtime;
- entender o papel do processo Node.js;
- reconhecer APIs disponíveis no ambiente Node.js;
- separar responsabilidade da linguagem, do runtime e das bibliotecas.

## Onde Esta Aula Entra no Curso

- Módulo: 10. Backend com Node.js
- Aula anterior: Node.js
- Próxima aula: Event Loop no Node.js
- Pré-requisitos: Node.js, JavaScript moderno, terminal e nocao de processo.

## Ideia Central

Um runtime é o ambiente que torna possível executar um programa. No caso do Node.js, ele inclui o motor JavaScript, integração com o sistema operacional, APIs nativas, event loop, gerenciamento do processo e recursos para lidar com entrada e saída.

Quando você roda `node app.js`, não está apenas "abrindo um arquivo". Você está criando um processo que executa código, consome memória, pode receber sinais, acessar variáveis de ambiente, abrir conexões e terminar com sucesso ou erro.

## Tópicos da Aula

- Linguagem, runtime e biblioteca.
- Processo Node.js.
- `process`, argumentos e variáveis de ambiente.
- APIs nativas.
- Entrada, processamento e saída.
- Ciclo de vida de um programa Node.js.

## Exemplo Prático

```javascript
console.log("PID:", process.pid);
console.log("Ambiente:", process.env.NODE_ENV || "sem NODE_ENV");
console.log("Argumentos:", process.argv.slice(2));
```

Esse exemplo mostra que o programa conhece informações do próprio processo e do ambiente em que está rodando.

## Analogias e Intuições

- A linguagem é a receita; o runtime é a cozinha onde a receita pode ser executada.
- O processo Node.js é uma instância viva do seu programa.

## Erros Comuns

- Achar que tudo vem da linguagem JavaScript.
- Guardar configurações sensíveis direto no código.
- Ignorar que um programa pode terminar com código de erro.
- Confundir dependência instalada com API nativa do runtime.
- Não pensar em ambiente local, teste e produção como contextos diferentes.

## Checklist de Entendimento

Antes de seguir, você deve conseguir responder:

- O que o runtime oferece além da linguagem?
- O que acontece ao executar `node arquivo.js`?
- Para que serve `process.env`?
- O que é uma API nativa?
- Por que backend precisa conhecer o ambiente onde roda?

## Prática Recomendada

- Crie um script que leia argumentos do terminal.
- Leia uma variavel de ambiente.
- Imprima uma mensagem diferente se a variavel não existir.
- Termine o processo com código de erro quando faltar configuração obrigatoria.
