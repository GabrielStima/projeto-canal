# Motores de Busca

## Objetivo

Ao final desta aula, você deve ser capaz de:

- explicar por que bancos comuns nem sempre resolvem busca textual;
- entender o papel de motores de busca em alto nível;
- diferenciar consulta transacional de busca;
- reconhecer cuidados de sincronização entre banco e índice.

## Onde Esta Aula Entra na Formação

- Módulo: 10. Backend com Node.js
- Aula anterior: Message Brokers
- Próxima aula: Projeto Prático: API Backend com Node.js
- Pré-requisitos: APIs, banco de dados, indexes, message brokers e modelagem de dados.

## Ideia Central

Bancos de dados conseguem filtrar e consultar dados, mas busca textual rica tem necessidades próprias: relevância, ranking, tolerância a erro, destaque de termos, filtros combinados e indexação de conteúdo.

Motores de busca entram quando a aplicação precisa encontrar informações de forma mais flexível do que uma query simples. O backend normalmente fica responsável por gravar dados no banco, atualizar o índice de busca e expor endpoints que consultam esse índice.

## Tópicos da Aula

- Busca simples vs busca especializada.
- Índice de busca.
- Documento indexado.
- Relevância.
- Sincronização com banco.
- Atualização assíncrona.
- Consistência eventual em alto nível.

## Exemplo Prático

```text
Aplicação de estudos:
PetCare OS:
- banco guarda pets, tutores e atendimentos;
- um motor poderia indexar apenas campos permitidos para busca operacional;
- a API de busca consultaria o índice;
- resultado aponta para os dados oficiais no banco.
```

Para a primeira versão, filtros e índices do banco podem ser suficientes. Um motor especializado só deve entrar quando requisitos como relevância, tolerância a erro ou volume justificarem a sincronização adicional.

## Analogias e Intuições

- Banco é o arquivo oficial.
- Motor de busca é o índice remissivo inteligente.

## Erros Comuns

- Usar motor de busca como fonte única de verdade sem necessidade.
- Esperar consistência imediata em todo fluxo.
- Indexar dados sensíveis.
- Não planejar reindexação.
- Usar busca especializada para filtro simples que o banco resolveria bem.

## Checklist de Entendimento

Antes de seguir, você deve conseguir responder:

- Por que busca textual pode ser diferente de query comum?
- O que é um índice de busca?
- Qual sistema deve ser fonte de verdade?
- Como dados chegam ao motor de busca?
- Que falhas podem ocorrer na sincronização?

## Prática Recomendada

- Desenhe uma busca operacional de pets por nome e identificador do tutor.
- Separe quais dados ficam no banco e quais entram no índice.
- Liste campos que não deveriam ser indexados.
- Descreva como atualizar o índice quando um pet muda.
- Liste failure modes desse fluxo.
- Faça o [Exercício 06 — Decisões de Integração](exercicios/06-decisoes-de-integracao.md).
