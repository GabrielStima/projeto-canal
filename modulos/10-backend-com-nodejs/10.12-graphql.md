## Ideia Central

GraphQL é uma forma de expor uma API por meio de um schema tipado, onde o cliente descreve os dados que deseja buscar ou alterar. Em vez de muitos endpoints organizados por recurso, é comum ter uma entrada principal que recebe queries e mutations.

Ele pode ajudar quando clientes diferentes precisam de formatos diferentes de dados, mas também adiciona complexidade: schema, resolvers, autorizacao por campo, cache, observabilidade e controle de custo das queries.

## Exemplo Prático

```graphql
query {
  lesson(id: "10.01") {
    id
    title
    status
  }
}
```

O cliente pede os campos que precisa, mas o servidor continua responsavel por regras, segurança, performance e acesso aos dados.

## Analogias e Intuições

- REST oferece portas diferentes para recursos diferentes.
- GraphQL oferece um balcao onde o cliente descreve exatamente o pedido.

## Erros Comuns

- Achar que GraphQL substitui REST em todo projeto.
- Ignorar custo de queries grandes.
- Expor dados demais no schema.
- Esquecer autorizacao em campos sensíveis.
- Confundir flexibilidade do cliente com ausencia de contrato.
