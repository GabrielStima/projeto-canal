# Servidor HTTP com Node.js Puro

## Objetivo

Ao final desta aula, você deve ser capaz de:

- criar um servidor HTTP simples com Node.js;
- reconhecer request e response no código;
- devolver status, headers e corpo de resposta;
- entender o que frameworks backend simplificam.

## Onde Esta Aula Entra no Curso

- Módulo: 10. Backend com Node.js
- Aula anterior: CLI com Node.js
- Próxima aula: Frameworks Backend
- Pré-requisitos: HTTP, Node.js, runtime, event loop, módulos e terminal.

## Ideia Central

Antes de usar um framework, vale criar um servidor HTTP pequeno com Node.js puro. Isso mostra que uma API não nasce mágica: ela recebe uma requisição, decide o que fazer e envia uma resposta.

Frameworks ajudam muito, mas eles constroem em cima desses fundamentos. Entender o mínimo do servidor puro evita tratar rotas, headers, status e corpo como detalhes invisíveis.

## Tópicos da Aula

- Módulo HTTP nativo.
- Request e response.
- Status code.
- Headers.
- Corpo da resposta.
- Rotas simples.
- Limites de fazer tudo sem framework.

## Exemplo Prático

```javascript
import http from "node:http";

const server = http.createServer((req, res) => {
  if (req.url === "/health") {
    res.writeHead(200, { "Content-Type": "application/json" });
    res.end(JSON.stringify({ status: "ok" }));
    return;
  }

  res.writeHead(404, { "Content-Type": "application/json" });
  res.end(JSON.stringify({ error: "Not found" }));
});

server.listen(3000, () => {
  console.log("Servidor rodando em http://localhost:3000");
});
```

## Analogias e Intuições

- O servidor é uma portaria: recebe pedidos, confere o caminho e responde.
- Framework é uma portaria com formulários, regras e atalhos prontos.

## Erros Comuns

- Sempre responder `200` mesmo quando houve erro.
- Esquecer `Content-Type`.
- Misturar roteamento, regra e formatação de resposta no mesmo bloco.
- Ignorar métodos HTTP.
- Não encerrar a resposta.

## Checklist de Entendimento

Antes de seguir, você deve conseguir responder:

- O que é `req`?
- O que é `res`?
- Para que serve status code?
- Por que headers importam?
- Que trabalho repetitivo um framework pode reduzir?

## Prática Recomendada

- Crie uma rota `/health`.
- Crie uma rota `/aulas` que retorna uma lista fixa.
- Retorne `404` para caminhos desconhecidos.
- Altere o `Content-Type` corretamente.
- Escreva quais partes ficaram repetitivas.
