# Logging

## Objetivo

Ao final desta aula, você deve ser capaz de:

- explicar por que logs são essenciais em backend;
- diferenciar log de debug temporario;
- registrar eventos relevantes de uma API;
- evitar expor dados sensíveis em logs.

## Onde Esta Aula Entra no Curso

- Módulo: 10. Backend com Node.js
- Aula anterior: OpenAPI Specs
- Próxima aula: Working with DB

## Ideia Central

Backend roda longe dos olhos do usuário. Quando algo falha, você precisa de pistas para entender o que aconteceu: qual requisição chegou, que operação foi executada, quanto tempo levou, qual erro apareceu e em que contexto.

Logging é a prática de registrar eventos importantes do sistema. Não é sair imprimindo tudo. Bons logs ajudam investigacao, operação e auditoria. Logs ruins geram ruído, escondem problemas e podem vazar informações sensíveis.

## Tópicos da Aula

- Niveis de log.

- Diferença entre log, metrica e trace.

## Exemplo Prático

```json
{
  "level": "info",
  "event": "lesson_created",
  "requestId": "req_123",
  "lessonId": "10.01",
  "durationMs": 42
}
```

Esse log é melhor do que uma frase solta porque pode ser filtrado e analisado por campos.

## Analogias e Intuições

- Log é o diário de bordo do backend.
- Log demais pode ser tao inutil quanto log de menos.

## Erros Comuns

- Usar `console.log` aleatorio como estrategia definitiva.
- Registrar senha, token ou dado pessoal sensível.
- Logar apenas quando da erro.
- Não incluir contexto suficiente para investigar.
- Criar mensagens impossiveis de filtrar.

## Checklist de Entendimento

Antes de seguir, você deve conseguir responder:

- Por que backend precisa de logs?
- O que é nível de log?
- Que dados não devem aparecer em logs?
- O que é log estruturado?

- Revise se algum campo exporia dado sensível.
