# CLI com Node.js

## Objetivo

Ao final desta aula, você deve ser capaz de:

- criar um script executado pelo terminal;
- ler argumentos de linha de comando;
- validar entrada simples;
- entender por que CLIs aparecem em projetos backend.

## Onde Esta Aula Entra na Formação

- Módulo: 10. Backend com Node.js
- Aula anterior: Módulos no Node.js
- Próxima aula: Servidor HTTP com Node.js Puro
- Pré-requisitos: Node.js, runtime, terminal, módulos e tratamento de erros.

## Ideia Central

Nem todo código backend começa como servidor. Muitos projetos precisam de scripts para criar dados iniciais, validar configuração, rodar migrations, gerar arquivos, testar conexões ou automatizar tarefas repetitivas.

Criar uma CLI pequena ajuda a entender entrada, processamento, saída, erros e código de término antes de lidar com requisições HTTP.

## Tópicos da Aula

- O que é uma CLI.
- Argumentos com `process.argv`.
- Saída com `console.log` e `console.error`.
- Validação de entrada.
- Código de saída.
- Scripts no `package.json`.

## Exemplo Prático

```javascript
const nome = process.argv[2];

if (!nome) {
  console.error("Informe um nome.");
  process.exit(1);
}

console.log(`Ola, ${nome}`);
```

Execute:

```bash
node saudacao.js Ana
```

## Analogias e Intuições

- Uma CLI é uma conversa curta com o programa pelo terminal.
- Ela recebe dados, toma uma decisão e devolve uma saída objetiva.

## Erros Comuns

- Não validar argumentos.
- Retornar erro apenas com texto, sem código de saída.
- Criar script que depende de configuração escondida.
- Misturar automação simples com regra de negócio complexa.
- Esquecer que scripts também precisam ser mantidos.

## Checklist de Entendimento

Antes de seguir, você deve conseguir responder:

- Para que serve uma CLI em um projeto backend?
- Como ler argumentos no Node.js?
- Quando usar `console.error`?
- Por que código de saída importa?
- Como transformar um comando em script do projeto?

## Prática Recomendada

- Crie uma CLI que receba o nome de uma aula.
- Valide se o nome foi informado.
- Gere uma mensagem de confirmação.
- Retorne erro quando faltar argumento.
- Adicione um script no `package.json` para executar a CLI.
