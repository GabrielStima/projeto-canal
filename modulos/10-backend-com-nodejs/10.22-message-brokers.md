# Message Brokers

## Objetivo

Ao final desta aula, você deve ser capaz de:

- explicar o que é um message broker em alto nível;
- entender filas e publicação de mensagens;
- reconhecer quando sair do fluxo request/response direto;
- preparar o aprofundamento em comunicação assíncrona no módulo 16.

## Onde Esta Aula Entra na Formação

- Módulo: 10. Backend com Node.js
- Aula anterior: Profiling Performance
- Próxima aula: Motores de Busca
- Pré-requisitos: APIs, failure modes, workers, banco de dados e noção de assincronismo.

## Ideia Central

Nem todo trabalho precisa acontecer durante a requisição do usuário. Enviar email, processar arquivo, atualizar índice de busca, gerar relatório ou notificar outro sistema pode ser feito de forma assíncrona.

Message brokers ajudam sistemas a trocar mensagens de forma mais desacoplada. Neste módulo, a ideia é apenas entender o papel: uma parte publica uma mensagem, outra consome e executa o trabalho. Filas, pub/sub, garantias, reprocessamento e arquitetura serão aprofundados depois.

## Tópicos da Aula

- O que é message broker.
- Mensagem.
- Producer e consumer.
- Fila.
- Pub/sub em alto nível.
- Trabalho assíncrono.
- Falhas e reprocessamento.

## Exemplo Prático

```text
Usuário conclui uma aula:
1. API salva progresso;
2. API publica mensagem "lesson_completed";
3. consumer atualiza estatísticas;
4. consumer envia notificação;
5. requisição original não precisa esperar tudo isso.
```

## Analogias e Intuições

- Broker é uma caixa postal organizada entre sistemas.
- Fila ajuda a separar "receber o pedido" de "fazer todo o trabalho agora".

## Erros Comuns

- Usar broker para esconder endpoint lento sem entender causa.
- Achar que assíncrono significa sem falha.
- Não pensar em mensagens duplicadas.
- Não registrar tentativas e erros.
- Colocar trabalho crítico em fila sem estratégia de recuperação.
