# OpenAPI Specs

## Objetivo

Ao final desta aula, você deve ser capaz de:

- explicar o papel de uma especificação OpenAPI;
- entender contrato documentado de API;
- reconhecer caminhos, métodos, parâmetros, responses e schemas;
- preparar o aprofundamento de OpenAPI no módulo 11.

## Onde Esta Aula Entra na Formação

- Módulo: 10. Backend com Node.js
- Aula anterior: GraphQL
- Próxima aula: Logging
- Pré-requisitos: APIs, JSON APIs, REST, contratos e status codes.

## Ideia Central

OpenAPI é uma forma padronizada de descrever APIs HTTP. Uma especificação pode documentar endpoints, parâmetros, corpos de requisição, respostas, erros e schemas de dados.

Neste módulo, OpenAPI entra como introdução: o aluno precisa entender que API também pode ser descrita como contrato legível por pessoas e ferramentas. O uso profissional com versionamento, governança, contract testing e compatibilidade fica para o módulo 11.

## Exemplo Prático

```yaml
paths:
  /lessons/{id}:
    get:
      summary: Busca uma aula por id
      parameters:
        - name: id
          in: path
          required: true
      responses:
        "200":
          description: Aula encontrada
        "404":
          description: Aula não encontrada
```

## Analogias e Intuições

- OpenAPI é a planta baixa da API.
- Uma spec boa reduz conversa escondida entre frontend e backend.

## Erros Comuns

- Escrever documentação que não acompanha o código.
- Documentar apenas sucesso e esquecer erros.
- Tratar OpenAPI como burocracia inútil.
- Colocar detalhe interno demais na spec.
- Achar que spec substitui boas decisões de design.

## Checklist de Entendimento

Antes de seguir, você deve conseguir responder:

- O que uma spec descreve?
- Para que servem schemas?
- Por que documentar erros?
- Quem usa uma especificação OpenAPI?
- O que será aprofundado no módulo 11?

## Prática Recomendada

- Escolha um endpoint REST simples.
- Descreva caminho, método e parâmetro.
- Liste respostas de sucesso e erro.
- Esboce um schema de resposta.
- Compare a spec com o comportamento esperado da API.
