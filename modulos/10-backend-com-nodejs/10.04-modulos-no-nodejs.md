# Módulos no Node.js

## Objetivo

Ao final desta aula, você deve ser capaz de:

- explicar por que módulos existem;
- organizar código backend em arquivos menores;
- diferenciar módulos internos, externos e locais;
- entender o papel de importacao e exportacao em projetos Node.js.

## Onde Esta Aula Entra no Curso

- Módulo: 10. Backend com Node.js
- Aula anterior: Event Loop no Node.js
- Próxima aula: CLI com Node.js
- Pré-requisitos: TypeScript, módulos, Node.js e terminal.

## Ideia Central

Um backend cresce rápido: rotas, configurações, validacoes, acesso a banco, logs e regras de negocio. Módulos existem para separar responsabilidades e permitir que cada parte do código seja entendida, reutilizada e testada com mais facilidade.

No Node.js, você pode usar módulos nativos, dependências instaladas e arquivos do próprio projeto. O importante não é decorar sintaxe isolada, mas entender a fronteira entre uma parte e outra do sistema.

## Tópicos da Aula

- Por que dividir código em módulos.
- Módulos nativos do Node.js.
- Dependencias externas.
- Módulos locais.
- Exportacao e importacao.
- Organizacao inicial de pastas.

## Exemplo Prático

```javascript
// saudacao.js
export function criarSaudacao(nome) {
  return `Ola, ${nome}`;
}
```

```javascript
// app.js
import { criarSaudacao } from "./saudacao.js";

console.log(criarSaudacao("Canal"));
```

## Analogias e Intuições

- Módulos são gavetas identificadas: você sabe onde guardar e onde buscar cada coisa.
- Um arquivo gigante parece rápido no comeco, mas cobra juros quando o projeto cresce.

## Erros Comuns

- Colocar todas as responsabilidades em `app.js`.
- Criar pastas demais antes de existir complexidade real.
- Misturar código de rota, banco e regra de negocio sem necessidade.
- Importar dependências sem entender se são de runtime ou desenvolvimento.
- Criar ciclos de importacao confusos.

## Checklist de Entendimento

Antes de seguir, você deve conseguir responder:

- O que é módulo nativo?

- Como identificar que um arquivo está fazendo coisa demais?

## Prática Recomendada

- Crie tres arquivos: configuração, regra simples e arquivo principal.
- Exporte uma função de regra.
- Importe essa função no arquivo principal.
- Explique qual responsabilidade ficou em cada arquivo.
