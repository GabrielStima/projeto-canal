# ORMs

## Ideia Central

ORM significa Object-Relational Mapping. Em termos práticos, é uma ferramenta que ajuda a mapear tabelas, registros e relacionamentos para estruturas usadas no código da aplicação.

ORMs podem acelerar desenvolvimento, reduzir repetição e padronizar acesso a dados. Mas também podem esconder queries caras, dificultar casos avançados e gerar falsa sensação de que não é preciso entender banco.

## Exemplo Prático

```text
Com SQL direto:
- você escreve a query;
- controla exatamente o que será executado;
- assume mais repetição.

Com ORM:
- você usa uma API de código;
- ganha conveniência;
- precisa entender a query gerada e seus custos.
```

## Analogias e Intuições

- ORM é um tradutor entre o mundo do código e o mundo relacional.
- Tradutor ajuda, mas você ainda precisa entender a conversa.

## Erros Comuns

- Achar que ORM elimina modelagem.
- Ignorar queries geradas.
- Usar ORM para tudo, mesmo quando SQL direto seria mais claro.
- Criar modelos que não respeitam o banco.
- Não entender migrations antes de alterar schema.
