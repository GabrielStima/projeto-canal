# Projeto Prático: API Backend com Node.js

## Objetivo

Ao final desta aula, você deve ser capaz de:

- integrar os principais conceitos do módulo em uma API pequena;
- construir um fluxo backend com Node.js, API, banco e logs;
- reconhecer pontos de falha e performance;
- preparar a transição para API Design Profissional.

## Onde Esta Aula Entra na Formação

- Módulo: 10. Backend com Node.js
- Aula anterior: Motores de Busca
- Próxima aula: 11. API Design Profissional
- Pré-requisitos: Todas as aulas anteriores do módulo 10.

## Ideia Central

O projeto final deve consolidar fundamentos sem virar arquitetura exagerada. Você já criou a base Node.js, os endpoints e a integração com o banco durante os exercícios. Agora vai demonstrar um fluxo vertical do PetCare OS sem recomeçar a implementação.

O foco não é ter o projeto mais completo possível. O foco é provar que uma requisição atravessa HTTP, validação, regra, banco, transaction, resposta e logs de forma verificável. Contratos profissionais, autenticação real e observabilidade profunda continuam nos módulos seguintes.

## Tópicos da Aula

- Escopo do projeto.
- Setup Node.js com TypeScript.
- CLI de validação.
- Servidor e framework.
- Endpoints JSON.
- Contrato REST introdutório.
- OpenAPI parcial.
- Acesso a banco.
- Uso de transação.
- ORM ou acesso direto.
- Logging.
- Failure modes.
- Worker simples.
- Profiling inicial.
- Possível fila ou busca em alto nível.

## Analogias e Intuições

- Este projeto é uma ponte entre fundamento e desenho profissional de APIs.
- O objetivo é construir pequeno, mas pensar como algo que poderia crescer.

## Erros Comuns

- Tentar implementar todos os temas avançados de uma vez.
- Recriar o banco ou abandonar os arquivos do módulo 09.
- Esquecer erros e logs.
- Usar ORM sem entender o banco.
- Otimizar performance sem medir.
- Adicionar broker ou busca sem necessidade atual.
- Colocar autenticação profunda aqui, antes do módulo específico.

## Checklist de Entendimento

Antes de seguir, você deve conseguir responder:

- Qual problema a API resolve?
- Quais recursos ela expõe?
- Como são as respostas JSON?
- Onde o banco entra?
- Que operações precisam de transação?
- Que eventos devem ser logados?
- Quais failure modes existem?
- Que parte merece profiling?
- O que será melhorado no módulo 11?
