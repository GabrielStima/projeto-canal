# Node.js

## Objetivo

Ao final desta aula, você deve ser capaz de:

- explicar o que é Node.js;
- diferenciar JavaScript, TypeScript, runtime e framework;
- reconhecer por que Node.js é usado em backend;
- saber o que estudar antes de criar uma API com framework.

## Ideia Central

Node.js é um runtime que permite executar JavaScript fora do browser. Isso significa que o mesmo ecossistema de linguagem usado no frontend também pode ser usado para scripts, CLIs, servidores HTTP, APIs, workers e ferramentas de automação.

No backend, Node.js é muito usado porque combina bem com entrada e saída assíncronas: receber requisições, consultar banco, chamar serviços externos, ler arquivos e devolver respostas. Isso não significa que Node seja melhor para todo problema, mas explica por que ele aparece tanto em aplicações web.

## Exemplo Prático

```javascript
console.log("Rodando JavaScript fora do browser");
console.log(process.version);
```

Execute o arquivo no terminal:

```bash
node exemplo.js
```

A ideia não é criar uma API ainda. O primeiro passo é perceber que agora o JavaScript está rodando como processo local, com acesso a recursos que o browser normalmente não oferece.

## Analogias e Intuições

- JavaScript é a língua; Node.js é o ambiente onde essa língua está sendo executada.
- O browser é uma casa com regras próprias. Node.js é outra casa, com outras portas disponíveis.

## Erros Comuns

- Achar que Node.js é uma linguagem.
- Começar por framework sem entender runtime.
- Confundir Node.js com npm.
- Achar que tudo que existe no browser existe igual no Node.js.
- Usar Node.js para tarefas pesadas de CPU sem entender limites.

## Checklist de Entendimento

Antes de seguir, você deve conseguir responder:

- O que o Node.js executa?
- Qual é a diferença entre Node.js e browser?
- Por que backend costuma lidar muito com I/O?
- O que é runtime?
- Por que frameworks backend dependem de fundamentos do Node.js?

## Prática Recomendada

- Execute o arquivo pelo terminal.
- Leia argumentos com `process.argv`.
