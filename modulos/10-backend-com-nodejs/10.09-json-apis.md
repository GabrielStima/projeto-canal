# JSON APIs

## Objetivo

Ao final desta aula, você deve ser capaz de:

- explicar o papel do JSON em APIs web;
- modelar respostas simples e previsíveis;
- diferenciar dados, metadados e erros;
- evitar formatos inconsistentes entre endpoints.

## Ideia Central

JSON é um formato leve para representar dados estruturados. Em APIs web modernas, ele costuma ser usado como corpo de requisições e respostas porque conversa bem com JavaScript, navegadores, servidores e muitas outras linguagens.

Uma JSON API boa não devolve apenas "qualquer objeto". Ela tenta ser previsível: nomes consistentes, tipos esperados, erros padronizados e respostas que o cliente consegue consumir sem adivinhar.

## Tópicos da Aula

- O que é JSON.
- `Content-Type: application/json`.
- Corpo de requisição.
- Corpo de resposta.
- Listas, objetos e metadados.
- Formato de erro.
- Consistência entre endpoints.

## Exemplo Prático

```json
{
  "data": {
    "id": 42,
    "nome": "Luna",
    "especie": "canina",
    "statusAtendimento": "aguardando_aprovacao"
  }
}
```

```json
{
  "error": {
    "code": "PET_NOT_FOUND",
    "message": "O pet solicitado não foi encontrado."
  }
}
```

## Analogias e Intuições

- JSON é uma caixa etiquetada: a estrutura precisa ajudar quem vai abrir.
- Consistência de resposta economiza código no frontend.

## Erros Comuns

- Misturar nomes em português, inglês, camelCase e snake_case sem critério.
- Retornar erro como string solta.
- Mudar tipo de campo dependendo do caso.
- Omitir `Content-Type`.
- Enviar dados internos que o cliente não precisa conhecer.
