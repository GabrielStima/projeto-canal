# Normalization no Backend

## Objetivo

Ao final desta aula, você deve ser capaz de:

- revisar normalização no contexto de APIs;
- entender como modelagem afeta endpoints;
- evitar duplicação perigosa de dados;
- reconhecer trade-offs entre consulta simples e modelo consistente.

## Onde Esta Aula Entra no Curso

- Módulo: 10. Backend com Node.js
- Aula anterior: ACID no Backend
- Próxima aula: ORMs
- Pré-requisitos: Modelagem de dados, integridade, SQL, APIs e Working with DB.

## Ideia Central

Normalização organiza dados para reduzir duplicação e proteger consistência. No backend, isso aparece quando a API precisa montar respostas a partir de tabelas relacionadas, validar referências e decidir o que salvar em cada lugar.

Um endpoint não deve copiar dados sem pensar. Se o nome de uma tag muda, quantos registros precisam mudar? Se uma aula pertence a várias tags, onde essa relação vive? Essas perguntas ligam modelagem ao código backend.

## Tópicos da Aula

- Revisão de normalização.
- Entidades e relacionamentos.
- Dados duplicados.
- Respostas compostas.

- Trade-offs práticos.

## Exemplo Prático

```text
Tabelas:
- lessons
- tags
- lessons_tags

Resposta da API:
{
  "id": "10.01",
  "title": "Node.js",
  "tags": ["backend", "node"]
}
```

O banco pode estar normalizado, enquanto a resposta JSON entrega dados convenientes para o cliente.

## Analogias e Intuições

- O banco guarda a informação de forma disciplinada.
- A API traduz essa disciplina para um formato útil para quem consome.

## Erros Comuns

- Confundir formato de resposta com modelo do banco.
- Duplicar dados sem plano de atualização.
- Criar tabelas pensando apenas em uma tela.
- Montar JSON sem considerar custo de consulta.
- Achar que normalização impede respostas amigáveis.

## Checklist de Entendimento

Antes de seguir, você deve conseguir responder:

- Por que normalização reduz inconsistências?
- Como uma API pode devolver dados combinados?
- Qual a diferença entre modelo do banco e contrato da API?
- Quando duplicação pode ser aceitável?
- Que riscos duplicação traz?

## Prática Recomendada

- Modele aulas e tags de forma normalizada.
- Desenhe a resposta JSON que o frontend gostaria de receber.
- Liste queries necessárias para montar essa resposta.
- Identifique onde poderia existir duplicação perigosa.
