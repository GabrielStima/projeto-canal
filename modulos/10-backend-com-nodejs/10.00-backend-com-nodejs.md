## Ideia Central

O backend é a camada que recebe requisições, executa regras de negócio, conversa com o banco de dados e devolve respostas. Node.js traz JavaScript para o servidor com um modelo baseado em eventos e I/O não bloqueante, tornando-o eficiente para APIs que lidam com muitas requisições simultâneas.

Entender o event loop é entender por que o Node.js se comporta como se comporta: uma única thread que não bloqueia enquanto espera disco ou rede. Esse modelo define como escrever código assíncrono corretamente e onde os erros aparecem quando isso é ignorado.

Este módulo cobre desde o runtime até message brokers e motores de busca, passando por frameworks, APIs REST e GraphQL, ORMs, logging, failure modes e performance, construindo a base para criar serviços backend profissionais.

## Analogias e Intuições

- O event loop do Node.js é como um plantonista de triagem: anota o pedido, encaminha para o especialista e já atende o próximo, sem esperar a resposta chegar.
- Um framework backend é como a estrutura de uma clínica: define onde ficam recepção, consultórios e laboratório — você não precisa construir os corredores do zero.
- Logging estruturado é como o prontuário médico: sem ele, reconstruir o que aconteceu num incidente é trabalho de detetive.

## Erros Comuns

- Bloquear o event loop com operações síncronas pesadas.
- Tratar todos os erros com `try/catch` genérico sem distinguir tipos de falha.
- Misturar regra de negócio, acesso ao banco e resposta HTTP no mesmo lugar.
- Não usar logging estruturado e perder rastreabilidade em produção.
- Ignorar failure modes e deixar a API retornar 500 sem mensagem útil.
