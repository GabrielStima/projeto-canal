# Working with DB

## Objetivo

Ao final desta aula, você deve ser capaz de:

- explicar como um backend conversa com banco de dados;
- diferenciar conexão, query e transação em nível prático;
- entender configuração por ambiente;
- reconhecer cuidados básicos antes de expor dados por API.

## Onde Esta Aula Entra na Formação

- Módulo: 10. Backend com Node.js
- Aula anterior: Logging
- Próxima aula: ACID no Backend
- Pré-requisitos: Bancos de dados, SQL/NoSQL, migrations, variáveis de ambiente, APIs e Node.js.

## Ideia Central

Conectar backend a banco de dados transforma uma API em aplicação com estado persistente. Agora, cada endpoint pode ler, criar, alterar ou remover dados reais. Isso exige cuidado com conexão, configuração, validação, erros, tempo de resposta e consistência.

O backend não deve ser apenas uma passagem direta entre HTTP e banco. Ele precisa validar entrada, aplicar regras, tratar erros, limitar o que retorna e proteger detalhes internos.

## Tópicos da Aula

- String de conexão.
- Configuração por ambiente.
- Cliente de banco.
- Queries.
- Pool de conexões em alto nível.
- Erros de banco.
- Separação entre rota e acesso a dados.

## Exemplo Prático

```text
Fluxo de um endpoint:
1. receber requisição;
2. validar entrada;
3. chamar função de acesso a dados;
4. tratar resultado ou erro;
5. devolver resposta JSON.
```

Essa separação evita que todo o backend vire uma mistura de HTTP, SQL e regra de negócio no mesmo lugar.

## Analogias e Intuições

- O backend é o atendente; o banco é o arquivo. O atendente não entrega a chave do arquivo para qualquer pessoa.
- Uma conexão com o banco é um recurso compartilhado, não um detalhe descartável.

## Erros Comuns

- Montar queries diretamente com texto vindo do usuário.
- Retornar dados internos sem filtro.
- Ignorar falhas de conexão.
- Colocar credenciais no código.
- Misturar rota, SQL e regra em uma função enorme.

## Checklist de Entendimento

Antes de seguir, você deve conseguir responder:

- Como o backend sabe onde está o banco?
- Por que usar variáveis de ambiente?
- O que pode dar errado ao consultar o banco?
- Por que validar antes de salvar?
- Por que separar acesso a dados da rota?

## Prática Recomendada

- Desenhe o fluxo de um endpoint que cria uma aula.
- Liste validações antes de salvar.
- Liste erros possíveis do banco.
- Escreva quais dados devem voltar para o cliente.
- Separe mentalmente rota, regra e acesso a dados.
