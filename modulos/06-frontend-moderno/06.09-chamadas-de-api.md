# Chamadas de API

## Objetivo

Ao final desta aula, voce deve ser capaz de:

- buscar dados em uma API a partir do frontend;
- representar carregamento, erro e sucesso na interface;
- entender cuidados basicos com `useEffect` e `fetch`;
- separar dados remotos de estado puramente visual.

## Onde Esta Aula Entra no Curso

- Modulo: 06. Frontend Moderno
- Aula anterior: SPA vs PWA
- Proxima aula: Formularios no frontend moderno
- Pre-requisitos: HTTP, APIs em alto nivel, promises, `async/await`, React, hooks e estado.

## Ideia Central

Aplicacoes frontend modernas raramente vivem sozinhas. Elas precisam buscar dados, enviar formularios, lidar com falhas de rede e mostrar ao usuario o que esta acontecendo.

Uma chamada de API nao deve ser pensada apenas como "pegar JSON". Ela cria estados de interface: carregando, erro, vazio, sucesso e, as vezes, atualizando.

Nesta etapa, o objetivo e entender o fluxo basico. Bibliotecas especializadas para data fetching podem aparecer depois, mas primeiro o aluno precisa dominar o problema.

## Topicos da Aula

- `fetch` e promises.
- Estado de carregamento.
- Estado de erro.
- Estado de sucesso.
- Efeito para buscar dados.
- Cancelamento e cuidados introdutorios.

## Exemplo Pratico

```tsx
function Produtos() {
  const [status, setStatus] = useState("carregando");
  const [produtos, setProdutos] = useState<Produto[]>([]);

  useEffect(() => {
    async function carregar() {
      try {
        const resposta = await fetch("/api/produtos");
        const dados = await resposta.json();
        setProdutos(dados);
        setStatus("sucesso");
      } catch {
        setStatus("erro");
      }
    }

    carregar();
  }, []);

  if (status === "carregando") return <p>Carregando...</p>;
  if (status === "erro") return <p>Erro ao carregar produtos.</p>;
  return <ListaProdutos produtos={produtos} />;
}
```

## Analogias e Intuicoes

- Chamada de API e uma conversa com outro sistema.
- A interface precisa mostrar o estado da conversa, nao apenas o resultado final.

## Erros Comuns

- Nao tratar erro.
- Mostrar tela vazia enquanto carrega.
- Fazer chamada em loop por dependencia errada no efeito.
- Misturar dados remotos, estado visual e regra de negocio sem organizacao.

## Checklist de Entendimento

Antes de seguir, voce deve conseguir responder:

- Quais estados uma chamada de API pode ter?
- Por que `async/await` nao elimina tratamento de erro?
- Por que buscar dados afeta a UI?
- Quando usar `useEffect` para buscar dados?
- O que pode causar chamadas repetidas sem querer?

## Pratica Recomendada

- Busque uma lista de dados de uma API simples.
- Mostre carregamento, erro e sucesso.
- Adicione um estado de lista vazia.
