# Performance no frontend

## Objetivo

Ao final desta aula, voce deve ser capaz de:

- reconhecer gargalos comuns de performance no frontend;
- diferenciar carregamento inicial, renderizacao e interacao;
- entender por que medir vem antes de otimizar;
- aplicar cuidados basicos sem exagerar em micro-otimizacoes.

## Onde Esta Aula Entra no Curso

- Modulo: 06. Frontend Moderno
- Aula anterior: Testes em frontend
- Proxima aula: Web Components
- Pre-requisitos: Browser, cache, React, renderizacao, estado, chamadas de API e bundlers.

## Ideia Central

Performance no frontend nao e apenas "deixar o codigo mais rapido". Ela envolve quanto tempo a aplicacao demora para carregar, quando o usuario consegue interagir, quanto trabalho o browser precisa fazer e como a UI responde a mudancas.

Antes de otimizar, e preciso medir ou pelo menos identificar o gargalo. Otimizar sem entender o problema pode deixar o codigo mais complexo sem melhorar a experiencia.

No contexto React, performance passa por renderizacoes desnecessarias, listas grandes, calculos caros, imagens pesadas, bundles grandes e chamadas de API mal planejadas.

## Topicos da Aula

- Carregamento inicial.
- Bundle e assets.
- Renderizacao e re-renderizacoes.
- Listas grandes.
- Imagens e rede.
- Medicao antes de otimizacao.
- Performance percebida.

## Exemplo Pratico

```text
Problema: uma pagina de produtos parece lenta.

Possiveis causas:
- imagem grande demais;
- chamada de API lenta;
- lista enorme renderizada de uma vez;
- calculo pesado em toda renderizacao;
- bundle inicial muito grande.
```

Cada causa pede uma solucao diferente.

## Analogias e Intuicoes

- Performance e diagnostico: primeiro descubra onde doi, depois trate.
- O usuario sente tempo de espera, travamento e falta de feedback.

## Erros Comuns

- Otimizar tudo antes de medir.
- Culpar React por problemas de rede ou imagens.
- Usar memoizacao sem entender o custo.
- Ignorar estados de carregamento e feedback visual.

## Checklist de Entendimento

Antes de seguir, voce deve conseguir responder:

- O que pode deixar uma pagina lenta?
- Qual a diferenca entre carregamento e interacao?
- Por que medir antes de otimizar?
- Que problemas podem causar renderizacoes desnecessarias?
- Como feedback visual melhora performance percebida?

## Pratica Recomendada

- Analise uma tela e liste possiveis gargalos.
- Reduza uma imagem grande e observe a diferenca.
- Crie uma lista grande e pense como renderiza-la com cuidado.
