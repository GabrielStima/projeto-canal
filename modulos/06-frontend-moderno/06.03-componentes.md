# Componentes

## Objetivo

Ao final desta aula, voce deve ser capaz de:

- explicar o que e um componente;
- dividir uma interface em componentes menores;
- passar dados com props;
- reconhecer quando um componente esta fazendo coisa demais.

## Onde Esta Aula Entra no Curso

- Modulo: 06. Frontend Moderno
- Aula anterior: React
- Proxima aula: Renderizacao
- Pre-requisitos: React introdutorio, funcoes, objetos, arrays, HTML e CSS.

## Ideia Central

Componentes sao blocos reutilizaveis de interface. Eles ajudam a transformar uma tela grande em partes menores, cada uma com uma responsabilidade mais clara.

Um bom componente recebe dados, renderiza uma parte da UI e deixa claro o que espera de fora. Props funcionam como entradas: quem usa o componente fornece valores, e o componente decide como mostrar esses valores.

Nem todo pedaco visual precisa virar componente imediatamente. A divisao deve ajudar leitura, reutilizacao ou organizacao.

## Topicos da Aula

- Componentes como funcoes de UI.
- Props como entrada.
- Composicao de componentes.
- Componentes pequenos, mas nao fragmentados demais.
- Separacao entre estrutura, dados e estilo.

## Exemplo Pratico

Uma lista pode ser separada em componentes:

```tsx
type Produto = {
  nome: string;
  preco: number;
};

function ProdutoCard({ produto }: { produto: Produto }) {
  return <li>{produto.nome} - R$ {produto.preco}</li>;
}

export function ListaProdutos({ produtos }: { produtos: Produto[] }) {
  return (
    <ul>
      {produtos.map((produto) => (
        <ProdutoCard key={produto.nome} produto={produto} />
      ))}
    </ul>
  );
}
```

## Analogias e Intuicoes

- Componentes sao pecas de montagem: cada peca tem formato e funcao.
- Props sao os dados que voce entrega para a peca ser renderizada corretamente.

## Erros Comuns

- Criar um componente enorme chamado `App` com tudo dentro.
- Criar componentes minusculos sem ganho real.
- Alterar props dentro do componente.
- Usar o indice do array como `key` sem pensar nas consequencias.

## Checklist de Entendimento

Antes de seguir, voce deve conseguir responder:

- O que torna algo um bom candidato a componente?
- O que sao props?
- Por que componentes ajudam a organizar uma aplicacao?
- Quando a divisao em componentes atrapalha?
- Por que listas precisam de `key`?

## Pratica Recomendada

- Pegue uma tela simples e desenhe sua arvore de componentes.
- Crie um componente de card que recebe titulo, descricao e status.
- Renderize uma lista usando esse card.
