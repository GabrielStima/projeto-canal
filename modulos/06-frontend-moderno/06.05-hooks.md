## Ideia Central

Hooks sao funcoes especiais que permitem usar recursos do React dentro de componentes. O primeiro contato costuma ser com `useState`, que guarda memoria local da interface, e `useEffect`, que permite sincronizar o componente com algo externo, como uma API, um timer ou uma assinatura de evento.

O ponto importante e nao tratar `useEffect` como lugar para qualquer codigo. Muitas vezes, valores podem ser calculados durante a renderizacao ou derivados de estado existente.

Hooks devem ser chamados no topo do componente ou de outro hook, nao dentro de condicionais ou loops.

## Exemplo Pratico

```tsx
import { useEffect, useState } from "react";

export function Relogio() {
  const [agora, setAgora] = useState(new Date());

  useEffect(() => {
    const id = window.setInterval(() => setAgora(new Date()), 1000);
    return () => window.clearInterval(id);
  }, []);

  return <time>{agora.toLocaleTimeString()}</time>;
}
```

O efeito sincroniza o componente com um timer externo e limpa esse timer quando nao for mais necessario.

## Analogias e Intuicoes

- `useState` e a memoria de curto prazo do componente.
- `useEffect` e uma ponte controlada entre React e algo de fora da renderizacao.

## Erros Comuns

- Chamar hooks dentro de `if`, `for` ou funcoes comuns.
- Usar `useEffect` para calcular algo que poderia ser derivado diretamente.
- Esquecer limpeza de timers, subscriptions ou listeners.
- Criar loop infinito atualizando estado dentro de efeito sem cuidado.
