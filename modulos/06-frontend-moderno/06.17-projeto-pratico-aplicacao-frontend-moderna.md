# Projeto pratico: aplicacao frontend moderna

## Objetivo

Ao final desta aula, voce deve ser capaz de:

- integrar os principais assuntos do modulo em um projeto pequeno;
- criar uma aplicacao React com componentes, estado, rotas, API, formulario e validacao;
- revisar acessibilidade, testes e performance basica;
- preparar a base conceitual para o modulo de Next.js.

## Onde Esta Aula Entra no Curso

- Modulo: 06. Frontend Moderno
- Aula anterior: Micro-frontends
- Proxima aula: 07. Next.js e Aplicacoes Fullstack no Frontend
- Pre-requisitos: Todas as aulas anteriores do modulo 06.

## Ideia Central

O projeto final do modulo serve para transformar conceitos isolados em uma pequena aplicacao coerente. A ideia nao e criar um produto enorme. E mostrar que componentes, estado, rotas, APIs, formularios, validacao, testes e performance se conectam.

Um bom projeto pratico deve ser pequeno o suficiente para terminar e rico o bastante para revelar problemas reais de frontend moderno.

Sugestao: um catalogo de recursos de estudo, produtos ficticios ou tarefas com paginas de listagem, detalhes e cadastro.

## Topicos da Aula

- Definicao do escopo.
- Estrutura inicial do projeto.
- Arvore de componentes.
- Rotas principais.
- Dados e chamadas de API.
- Formulario e validacao.
- Testes essenciais.
- Revisao final de acessibilidade e performance.

## Exemplo Pratico

```text
Aplicacao: Catalogo de Estudos

Rotas:
- /recursos
- /recursos/:id
- /novo

Funcionalidades:
- listar recursos;
- filtrar por termo;
- ver detalhes;
- cadastrar novo recurso;
- validar titulo e URL;
- mostrar carregamento, erro e sucesso.
```

## Analogias e Intuicoes

- O projeto final e um ensaio geral: cada aula anterior aparece como uma parte do fluxo.
- Escopo pequeno bem acabado ensina mais do que um projeto grande abandonado.

## Erros Comuns

- Aumentar o escopo antes de terminar o fluxo principal.
- Criar componentes sem responsabilidade clara.
- Ignorar estados de erro e carregamento.
- Deixar validacao e testes para depois e nunca voltar.
- Otimizar antes de ter um projeto funcionando.

## Checklist de Entendimento

Antes de seguir, voce deve conseguir responder:

- Quais componentes principais a aplicacao tem?
- Quais rotas existem?
- Quais estados de API aparecem?
- Como o formulario valida dados?
- Quais fluxos merecem teste?
- O que precisa ser revisado antes de ir para Next.js?

## Pratica Recomendada

- Construa a aplicacao em etapas pequenas.
- Faca primeiro o fluxo principal funcionando.
- Depois adicione validacao, testes e melhorias de performance.
- Escreva um README curto explicando como rodar o projeto.
