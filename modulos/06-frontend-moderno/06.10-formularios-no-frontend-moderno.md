# Formularios no frontend moderno

## Objetivo

Ao final desta aula, voce deve ser capaz de:

- criar formularios controlados em React;
- conectar campos a estado;
- lidar com submissao sem recarregar a pagina;
- reconhecer cuidados de acessibilidade e UX em formularios.

## Ideia Central

Formulario continua sendo HTML. React nao elimina `label`, `input`, `button`, tipos de campo e semantica. O que muda e que, em interfaces modernas, os valores dos campos muitas vezes passam a ser controlados por estado.

Um formulario controlado deixa a UI e os dados sincronizados: o valor exibido no campo vem do estado, e cada mudanca atualiza esse estado.

Isso facilita validacao, mensagens, habilitar/desabilitar botoes e submissao para APIs.

## Topicos da Aula

- Revisao de formularios HTML.
- Campos controlados.
- `onChange` e `onSubmit`.
- `preventDefault`.
- Estado de envio.
- Labels, foco e mensagens.

## Exemplo Pratico

```tsx
function FormularioContato() {
  const [nome, setNome] = useState("");

  function enviar(evento: React.FormEvent) {
    evento.preventDefault();
    console.log({ nome });
  }

  return (
    <form onSubmit={enviar}>
      <label htmlFor="nome">Nome</label>
      <input id="nome" value={nome} onChange={(evento) => setNome(evento.target.value)} />
      <button type="submit">Enviar</button>
    </form>
  );
}
```

## Analogias e Intuicoes

- O campo mostra o estado; o evento atualiza o estado.
- Submeter formulario e transformar dados de entrada em uma acao.

## Erros Comuns

- Remover semantica HTML por estar usando React.
- Esquecer `label`.
- Usar botao sem `type`.
- Nao impedir recarregamento quando a submissao sera tratada no cliente.
- Guardar todos os campos sem pensar na estrutura dos dados.
