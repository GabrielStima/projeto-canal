## Ideia Central

React permite descrever a interface como uma arvore de componentes. Cada componente recebe dados, devolve uma descricao da UI e pode reagir a eventos ou mudancas de estado.

Em vez de manipular o DOM manualmente a cada mudanca, voce descreve como a tela deve parecer para determinado estado. React cuida da atualizacao da interface a partir dessa descricao.

JSX parece HTML, mas e uma sintaxe usada dentro do JavaScript para declarar a estrutura visual de componentes.

## Topicos da Aula

- React como biblioteca de UI.
- Componentes como funcoes que retornam UI.
- JSX como descricao da interface.
- Props como entrada de dados.
- Estado como memoria local da interface.
- O que React faz e o que ele nao faz sozinho.

## Exemplo Pratico

Um contador simples mostra a relacao entre estado, evento e renderizacao:

```tsx
import { useState } from "react";

export function Contador() {
  const [total, setTotal] = useState(0);

  return (
    <button onClick={() => setTotal(total + 1)}>
      Cliques: {total}
    </button>
  );
}
```

Quando o usuario clica, o estado muda. Quando o estado muda, a UI e renderizada novamente com o novo valor.

## Analogias e Intuicoes

- Um componente e uma funcao que transforma dados em interface.
- Estado e a memoria que faz a interface lembrar algo entre interacoes.

## Erros Comuns

- Pensar que JSX e HTML puro.
- Manipular o DOM diretamente sem necessidade.
- Colocar regra demais dentro de um unico componente.
- Achar que React resolve rotas, formularios, API e testes automaticamente.

## Checklist de Entendimento

Antes de seguir, voce deve conseguir responder:

- O que e React?
- O que e JSX?
- O que sao props?
- O que e estado?
- Que problemas React resolve e quais continuam sendo responsabilidade do projeto?

## Pratica Recomendada

- Leia um componente React simples e identifique JSX, props, estado e eventos.
- Reescreva mentalmente uma interacao DOM simples como componente.
- Explique em voz alta o que acontece quando um estado muda.
