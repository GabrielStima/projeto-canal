# Gerenciamento de estado

## Objetivo

Ao final desta aula, voce deve ser capaz de:

- explicar o que e estado em uma interface;
- diferenciar estado local, estado compartilhado e estado derivado;
- decidir onde um estado deve morar em uma arvore de componentes;
- reconhecer quando uma biblioteca de estado pode ser exagero.

## Onde Esta Aula Entra no Curso

- Proxima aula: Rotas
- Pre-requisitos: Componentes, props, renderizacao, hooks e eventos.

## Ideia Central

Estado e qualquer informacao que muda ao longo do tempo e influencia a interface. Pode ser um texto digitado, um item selecionado, uma lista carregada ou uma aba ativa.

Gerenciar estado nao significa instalar uma biblioteca. Antes disso, e preciso entender onde o dado nasce, quem precisa dele e se ele realmente precisa ser guardado.

Muitos problemas de estado sao resolvidos com estado local, props, estado derivado e levantamento de estado para um componente pai.

## Exemplo Pratico

```tsx
function FiltroProdutos() {
  const [busca, setBusca] = useState("");
  const produtosFiltrados = produtos.filter((produto) =>
    produto.nome.toLowerCase().includes(busca.toLowerCase())
  );

  return (
    <>
      <input value={busca} onChange={(evento) => setBusca(evento.target.value)} />
      <ListaProdutos produtos={produtosFiltrados} />
    </>
  );
}
```

`busca` e estado. `produtosFiltrados` e derivado desse estado e nao precisa ser guardado separadamente.

## Analogias e Intuicoes

- Estado deve ficar perto de quem usa, mas alto o suficiente para quem precisa compartilhar.
- Estado derivado e como um calculo: se voce consegue recalcular, talvez nao precise armazenar.

## Erros Comuns

- Guardar em estado algo que poderia ser calculado.
- Duplicar a mesma informacao em varios lugares.
- Subir todo estado para o topo da aplicacao sem necessidade.
- Instalar biblioteca antes de entender o problema.

## Checklist de Entendimento

Antes de seguir, voce deve conseguir responder:

- O que e estado?
- O que e estado derivado?
- Quando levantar estado para um componente pai?
- O que pode acontecer quando duplicamos estado?
- Quando uma biblioteca externa pode ajudar?

## Pratica Recomendada

- Crie uma lista filtravel com estado local.
- Identifique quais valores sao estado e quais sao derivados.
- Refatore um componente para mover estado para o pai quando dois filhos precisarem dele.
