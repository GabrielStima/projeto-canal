# Rotas

## Objetivo

Ao final desta aula, voce deve ser capaz de:

- explicar o papel de rotas em uma SPA;
- diferenciar navegacao tradicional e navegacao client-side;
- entender `Link`, rotas aninhadas e parametros em nivel introdutorio;
- organizar telas principais de uma aplicacao React.

## Ideia Central

Rotas conectam URLs a partes da interface. Em uma aplicacao tradicional, navegar para outra URL costuma pedir uma nova pagina ao servidor. Em uma SPA, o JavaScript no browser pode atualizar a interface sem recarregar tudo.

React nao define rotas sozinho. Uma biblioteca como React Router ajuda a mapear caminhos para componentes, criar links internos e lidar com parametros como `/produtos/:id`.

Rotas deixam a aplicacao navegavel, compartilhavel por URL e mais organizada em telas.

## Exemplo Pratico

```tsx
import { Link, Route, Routes } from "react-router";

export function App() {
  return (
    <>
      <nav>
        <Link to="/">Inicio</Link>
        <Link to="/produtos">Produtos</Link>
      </nav>

      <Routes>
        <Route path="/" element={<Inicio />} />
        <Route path="/produtos" element={<Produtos />} />
      </Routes>
    </>
  );
}
```

## Analogias e Intuicoes

- Rotas sao o mapa que liga enderecos a telas.
- `Link` e uma navegacao interna que conversa com o router em vez de recarregar a pagina inteira.

## Erros Comuns

- Usar `<a>` para toda navegacao interna sem pensar no recarregamento.
- Criar rotas demais antes de entender as telas reais.
- Guardar estado importante somente em memoria quando deveria estar na URL.
- Esquecer pagina de erro ou rota nao encontrada.

## Checklist de Entendimento

Antes de seguir, voce deve conseguir responder:

- O que e uma rota?
- Por que uma SPA precisa de router?
- Qual a diferenca entre `Link` e uma ancora comum em navegacao interna?
- Para que servem parametros de rota?
- Quando uma informacao deveria aparecer na URL?

## Pratica Recomendada

- Crie tres telas: inicio, listagem e detalhes.
- Adicione links internos entre elas.
- Crie uma rota de detalhes usando um identificador na URL.
