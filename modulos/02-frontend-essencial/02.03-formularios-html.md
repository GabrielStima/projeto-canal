# Formulários HTML

## Objetivo

Ao final desta aula, você deve ser capaz de:

- explicar para que formulários existem na Web;
- criar campos básicos com `label`, `input`, `select`, `textarea` e `button`;
- entender a importância de `name`, `type`, `required` e associação entre label e campo;
- reconhecer erros comuns em formulários.

## Onde Esta Aula Entra na Formação

- Módulo: Frontend Essencial
- Aula anterior: Semântica HTML
- Próxima aula: CSS
- Pré-requisitos: HTML básico e semântica HTML

## Ideia Central

Formulários permitem que a pessoa envie informações. Eles aparecem em login, cadastro, busca, checkout, filtros, comentários e praticamente qualquer fluxo interativo.

Antes de validar com JavaScript ou enviar dados para um backend, é preciso saber estruturar campos corretamente. Um bom formulário começa com HTML claro, labels conectadas e tipos adequados.

## Tópicos da Aula

- O papel da tag `form`
- Campos de texto, e-mail, senha, checkbox e seleção
- `label` e associação com `for` e `id`
- `name` como nome do dado enviado
- Botões de envio
- Validação nativa básica
- Relação futura com JavaScript e backend

## Exemplo Prático

```html
<form action="/interesse" method="post">
  <h2>Receber novidades</h2>

  <label for="nome">Nome</label>
  <input id="nome" name="nome" type="text" required>

  <label for="email">E-mail</label>
  <input id="email" name="email" type="email" required>

  <label for="area">Área de interesse</label>
  <select id="area" name="area">
    <option value="frontend">Frontend</option>
    <option value="backend">Backend</option>
  </select>

  <button type="submit">Enviar</button>
</form>
```

## Analogias e Intuições

- Um formulário é uma conversa estruturada.
- `label` é a pergunta; `input` é o espaço para resposta.
- `name` é o nome que identifica a resposta quando os dados são enviados.

## Erros Comuns

- Criar campo sem label.
- Usar `placeholder` como substituto de label.
- Esquecer o atributo `name`.
- Usar `type="text"` para tudo.
- Fazer validação importante apenas no frontend.

## Checklist de Entendimento

Antes de seguir, você deve conseguir responder:

- Para que serve um formulário?
- Por que label é importante?
- Qual é a diferença entre `id` e `name` em um campo?
- O que `required` faz?
- Por que validação no browser não substitui validação no backend?

## Prática Recomendada

- Crie um formulário de cadastro simples.
- Inclua nome, e-mail, senha, área de interesse e aceite de termos.
- Associe todos os labels aos campos.
- Teste o envio com campos vazios e observe a validação nativa.
