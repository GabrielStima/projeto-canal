# HTML

## Objetivo

Ao final desta aula, você deve ser capaz de:

- explicar o papel do HTML em uma página web;
- reconhecer a estrutura mínima de um documento HTML;
- usar elementos básicos para organizar conteúdo;
- saber o que estudar em seguida.

## Onde Esta Aula Entra na Formação

## Ideia Central

HTML é a linguagem usada para estruturar o conteúdo de uma página web. Quando o browser recebe um documento HTML, ele interpreta os elementos e monta uma representação da página.

Nesta aula, o foco é entender HTML como estrutura, não como aparência. Títulos, parágrafos, listas, links, imagens e seções existem para descrever o que o conteúdo é e como ele se relaciona.

## Tópicos da Aula

- O que é HTML
- Documento, elementos, tags e atributos
- Estrutura mínima com `html`, `head` e `body`
- Títulos, parágrafos, listas, links e imagens
- Caminhos relativos e absolutos em links e arquivos
- Diferença entre conteúdo, estrutura e estilo

## Exemplo Prático

Crie uma página simples de apresentação:

```html
<!doctype html>
<html lang="pt-BR">
  <head>
    <meta charset="utf-8">
    <meta name="viewport" content="width=device-width, initial-scale=1">
    <title>Minha primeira página</title>
  </head>
  <body>
    <h1>Minha primeira página</h1>
    <p>Estou aprendendo a estruturar conteúdo para o browser.</p>

    <h2>O que vou estudar</h2>
    <ul>
      <li>HTML</li>
      <li>CSS</li>
      <li>JavaScript no browser</li>
    </ul>
  </body>
</html>
```

## Analogias e Intuições

- HTML é o esqueleto do conteúdo.
- Tags são marcações que dizem ao browser que tipo de coisa cada parte representa.
- A página não começa pela beleza; começa por significado e organização.

## Erros Comuns

- Usar HTML como ferramenta de desenho visual.
- Esquecer `lang`, `charset` ou `viewport`.
- Pular níveis de título sem intenção.
- Usar elementos genéricos para tudo.
- Colocar estilo inline cedo demais.

## Checklist de Entendimento

Antes de seguir, você deve conseguir responder:

- O que é HTML?
- Qual é a diferença entre tag, elemento e atributo?
- Para que servem `head` e `body`?
- Por que o browser precisa interpretar HTML?
- Por que HTML não deve ser tratado como aparência?

## Prática Recomendada

- Crie uma página HTML sobre você ou sobre um projeto fictício. Se quiser se aproximar do projeto prático, experimente a estrutura inicial de um card de status do pet.
- Use pelo menos um título principal, dois subtítulos, parágrafos, uma lista e um link.
- Abra o arquivo no browser e confira o resultado.
- Depois, leia o HTML e veja se a estrutura faz sentido sem olhar a tela.
