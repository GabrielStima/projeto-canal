# Browsers e Compatibilidade

## Objetivo

Ao final desta aula, você deve ser capaz de:

- explicar por que browsers podem se comportar de formas diferentes;
- entender a ideia de padrões web e suporte de funcionalidades;
- reconhecer problemas comuns de compatibilidade;
- saber consultar documentação antes de usar recursos novos.

## Onde Esta Aula Entra no Curso

- Módulo: Frontend Essencial
- Aula anterior: Acessibilidade
- Próxima aula: Projeto Prático: Página Interativa Essencial
- Pré-requisitos: HTML, CSS, JavaScript no browser, DOM, eventos e acessibilidade

## Ideia Central

Browsers implementam tecnologias da Web: HTML, CSS, JavaScript e Web APIs. Em geral, os fundamentos são estáveis, mas recursos novos podem ter diferenças de suporte, detalhes de comportamento ou exigir cuidado.

Compatibilidade não significa parar de usar recursos modernos. Significa saber verificar suporte, testar em mais de um contexto e criar interfaces que falhem da forma menos problemática possível.

## Tópicos da Aula

- Browser como implementação de padrões
- Diferença entre especificação, documentação e suporte real
- Recursos estáveis vs recursos novos
- Progressive enhancement em alto nível
- Testes em browsers e tamanhos de tela diferentes
- Consulta de compatibilidade na documentação
- Relação futura com ferramentas de build

## Exemplo Prático

Antes de usar uma propriedade ou API nova:

```text
1. Verifique se ela resolve um problema real.
2. Consulte documentação e tabela de compatibilidade.
3. Teste no browser principal do seu público.
4. Pense no comportamento quando o recurso não existir.
5. Evite depender de novidade para uma ação essencial sem fallback.
```

## Analogias e Intuições

- O padrão é a regra; o browser é quem executa a regra.
- Compatibilidade é perguntar: "isso funciona para quem precisa usar?"
- Progressive enhancement começa por uma base funcional e melhora quando há suporte.

## Erros Comuns

- Testar apenas em um browser.
- Usar recurso novo porque parece elegante, sem necessidade real.
- Ignorar usuários em dispositivos mais antigos.
- Confundir erro de código com diferença de suporte.
- Achar que ferramenta de build resolve todos os problemas de compatibilidade.

## Checklist de Entendimento

Antes de seguir, você deve conseguir responder:

- Por que browsers podem ter diferenças?
- O que é suporte de funcionalidade?
- Como consultar compatibilidade?
- O que é progressive enhancement em alto nível?
- Por que testar manualmente ainda importa?

## Prática Recomendada

- Escolha uma propriedade CSS ou Web API que você ainda não conhece.
- Pesquise sua documentação e suporte.
- Crie um exemplo mínimo.
- Teste em pelo menos dois browsers ou contextos quando possível.
- Escreva o que aconteceria se o recurso não estivesse disponível.
