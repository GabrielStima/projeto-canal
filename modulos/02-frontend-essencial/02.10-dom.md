## Ideia Central

DOM significa Document Object Model. É uma representação do documento que o JavaScript pode acessar e modificar enquanto a página está aberta.

Quando você usa JavaScript para encontrar um botão, trocar um texto ou adicionar uma classe CSS, está interagindo com o DOM. Isso é uma das bases da interatividade no frontend.

## Tópicos da Aula

- Alteração de atributos e classes
- Cuidados ao selecionar elementos inexistentes

## Exemplo Prático

```html
<h1 id="titulo">Produto</h1>
<p class="descricao">Descrição inicial.</p>
<button id="destacar">Destacar</button>
```

```javascript
const titulo = document.querySelector("#titulo");
const descricao = document.querySelector(".descricao");

titulo.textContent = "Produto em destaque";
descricao.classList.add("ativa");
```

## Analogias e Intuições

- HTML é o texto recebido; DOM é a estrutura que o browser monta para trabalhar.
- Selecionar elemento é apontar para uma parte da página.
- Alterar o DOM é alterar a página enquanto ela está viva no browser.

## Erros Comuns

- Confundir arquivo HTML com DOM em execução.
- Selecionar um elemento que não existe e tentar usá-lo.
- Usar seletores frágeis demais.
- Reescrever HTML inteiro para mudar apenas um texto.
- Ignorar que mudanças no DOM não alteram automaticamente o arquivo original.
