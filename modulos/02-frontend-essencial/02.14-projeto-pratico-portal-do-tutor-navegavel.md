# Projeto Prático: Portal do Tutor Navegável

## Objetivo

Ao final desta aula, você deve ser capaz de:

- juntar HTML, CSS e JavaScript no browser em uma experiência navegável;
- estruturar duas telas simples do Portal do Tutor;
- aplicar layout, responsividade e acessibilidade básica;
- usar DOM e eventos para alternar entre telas sem recarregar a página;
- entender que este é o primeiro marco do PetCare OS em frontend puro.

## Onde Esta Aula Entra na Formação

- Módulo: Frontend Essencial
- Aula anterior: Browsers e Compatibilidade
- Próxima aula: Programação com TypeScript
- Pré-requisitos: todas as aulas anteriores do módulo

## Ideia Central

Esta aula junta as peças do módulo. Você vai construir a primeira versão navegável do Portal do Tutor do PetCare OS, ainda sem framework.

O objetivo não é criar um produto completo. O objetivo é provar que você consegue construir uma página simples, compreensível e interativa usando apenas fundamentos da Web.

Este projeto também serve como ponte para os próximos módulos: ao sentir limitações do JavaScript usado aqui, fica mais claro por que estudar a linguagem com mais profundidade. Em módulos futuros, você vai reconstruir essa mesma ideia com bibliotecas e frameworks de frontend.

## Tópicos da Aula

- Planejamento das telas do portal.
- Estrutura HTML semântica para duas seções.
- Navegação simples entre telas.
- Estilo e layout responsivo.
- Interação com DOM e eventos.
- Mensagens de erro e sucesso.
- Revisão de acessibilidade e compatibilidade.

## Exemplo Prático

Projeto: primeira versão do Portal do Tutor.

```text
Tela 1 — Status do Pet:
- Nome do pet e espécie.
- Nome do tutor.
- Status atual do atendimento.
- Previsão de alta.
- Linha do tempo com os dois últimos status.

Tela 2 — Orçamento Pendentes:
- Lista de itens/procedimentos.
- Valor total estimado.
- Botões "Aprovar" e "Recusar".
- Mensagem de confirmação ao aprovar.

Navegação:
- Links ou botões alternam entre as telas sem recarregar a página.
```

## Analogias e Intuições

- O projeto é um ensaio geral do módulo.
- Cada fundamento resolve uma parte pequena da interface.
- Simplicidade bem feita vale mais do que complexidade frágil.

## Erros Comuns

- Começar pelo JavaScript antes de estruturar HTML.
- Estilizar antes de saber qual conteúdo existe.
- Fazer layout apenas para uma largura de tela.
- Criar interação que só funciona com mouse.
- Transformar o projeto em algo grande demais para o objetivo da aula.

## Checklist de Entendimento

Antes de seguir, você deve conseguir responder:

- O HTML da página faz sentido sem CSS?
- O formulário ou interação tem labels e feedback adequados?
- O CSS organiza a página em telas diferentes?
- O JavaScript altera a interface usando DOM e eventos?
- A página pode ser usada com teclado?
- O projeto evita depender de framework?

## Prática Recomendada

Esta atividade é obrigatória para continuidade, porque ela consolida tudo o que você praticou no módulo e produz o primeiro artefato navegável do Portal do Tutor.

### O que você já sabe

Você já praticou:

- HTML semântico no exercício do card de status do pet.
- Formulários acessíveis no exercício de triagem.
- DOM e eventos no exercício da lista de atendimentos.
- CSS, layout, responsividade e acessibilidade nas aulas.

### Sua tarefa

Construa uma pasta `portal-do-tutor/` com três arquivos:

- `index.html`
- `styles.css`
- `script.js`

O portal deve ter pelo menos duas telas navegáveis na mesma página:

1. **Status do Pet:** card com nome do pet, espécie, tutor, status atual, previsão de alta e linha do tempo resumida.
2. **Orçamento ou Histórico:** escolha entre mostrar um orçamento pendente com botões de aprovar/recusar ou um histórico resumido de atendimentos anteriores.

A navegação entre as telas deve ser feita com JavaScript, sem recarregar a página.

### O que você vai produzir

Uma pasta com os três arquivos que formam a primeira versão navegável do Portal do Tutor. Você deve conseguir abrir o `index.html` no browser e navegar entre as telas.

### Como corrigir sua atividade

Use o prompt do arquivo de exercício para enviar seu código a uma IA e receber feedback:

### Como este trabalho continuará

Esta é a primeira versão do Portal do Tutor. Em módulos futuros de frontend moderno, você vai reconstruir essa mesma ideia com componentes reutilizáveis e rotas. Guarde esse trabalho em um lugar seguro.
