# Convenções de Nomenclatura

## Objetivo

Ao final desta aula, você deve ser capaz de:

- explicar por que nomes de classes afetam manutenção e colaboração;
- criar nomes orientados ao papel dos elementos;
- reconhecer os princípios de convenções como BEM;
- escolher uma convenção sem tratá-la como regra universal.

## Ideia Central

O browser não exige uma convenção para nomes de classes, mas projetos precisam de consistência. Bons nomes ajudam a entender a responsabilidade de um elemento, reduzem conflitos e tornam mudanças menos arriscadas.

Convenções como BEM organizam nomes em bloco, elemento e modificador. O objetivo da aula não é impor uma metodologia, mas mostrar como uma regra compartilhada pode tornar CSS mais previsível.

## Exemplo Prático

```html
<article class="pet-card pet-card--urgent">
  <h2 class="pet-card__name">Thor</h2>
  <p class="pet-card__status">Em Triagem</p>
</article>
```

```css
.pet-card {
  padding: 16px;
  border: 1px solid #ccc;
}

.pet-card__name {
  font-size: 1.25rem;
}

.pet-card--urgent {
  border-color: crimson;
}
```

O nome comunica que existe um bloco `pet-card`, um elemento `name` e uma variação `urgent`.

## Analogias e Intuições

- Uma convenção de nomes funciona como vocabulário compartilhado pelo time.
- O nome deve comunicar responsabilidade, não apenas aparência atual.
- BEM é uma ferramenta de organização, não uma exigência do CSS.

## Erros Comuns

- Usar nomes como `.red-text` ou `.left-box` que perdem sentido após mudanças visuais.
- Misturar várias convenções sem critério.
- Reproduzir toda a árvore do HTML no nome da classe.
- Criar nomes genéricos demais, como `.item` ou `.container`, sem contexto.
- Usar BEM de forma mecânica em elementos muito simples.
- Confundir classes de estado com elementos permanentes.

## Checklist de Entendimento

Antes de seguir, você deve conseguir responder:

- Por que uma convenção de nomes ajuda um projeto?
- Qual é a diferença entre nome visual e nome orientado ao papel?
- O que são bloco, elemento e modificador em BEM?
- Quando uma classe utilitária pode ser adequada?
- Por que nenhuma convenção substitui decisões claras de estrutura?
