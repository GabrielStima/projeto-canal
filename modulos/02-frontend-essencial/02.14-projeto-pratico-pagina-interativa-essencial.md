# Projeto Prático: Página Interativa Essencial

## Objetivo

Ao final desta aula, você deve ser capaz de:

- juntar HTML, CSS e JavaScript no browser em uma página simples;
- estruturar uma interface com formulário;
- aplicar layout, responsividade e acessibilidade básica;
- usar DOM e eventos para criar uma interação pequena.

## Onde Esta Aula Entra na Formação

- Módulo: Frontend Essencial
- Aula anterior: Browsers e Compatibilidade
- Próxima aula: Programação com TypeScript
- Pré-requisitos: todas as aulas anteriores do módulo

## Ideia Central

Esta aula junta as peças do módulo. O objetivo não é criar um produto completo, nem usar framework. O objetivo é provar que você consegue construir uma página simples, compreensível e interativa usando apenas fundamentos da Web.

O projeto também serve como ponte para o próximo módulo: ao sentir limitações do JavaScript usado aqui, fica mais claro por que estudar a linguagem com mais profundidade.

## Tópicos da Aula

- Planejamento da página
- Estrutura HTML
- Formulário semântico
- Estilo e layout responsivo
- Interação com DOM e eventos
- Mensagens de erro e sucesso
- Revisão de acessibilidade e compatibilidade

## Exemplo Prático

Projeto: página de interesse em uma trilha de estudos.

```text
Funcionalidades:
1. Mostrar uma apresentação curta.
2. Exibir cards com áreas de estudo.
3. Ter um formulário com nome, e-mail e área de interesse.
4. Validar preenchimento básico no browser.
5. Mostrar mensagem de sucesso sem recarregar a página.
6. Funcionar bem em tela pequena e grande.
```

## Analogias e Intuições

- O projeto é um ensaio geral do módulo.
- Cada fundamento resolve uma parte pequena da interface.
- Simplicidade bem feita vale mais do que complexidade frágil.

## Erros Comuns

- Começar pelo JavaScript antes de estruturar HTML.
- Estilizar antes de saber qual conteúdo existe.
- Fazer layout apenas para uma largura de tela.
- Criar interação que só funciona com mouse.
- Transformar o projeto em algo grande demais para o objetivo da aula.

## Checklist de Entendimento

Antes de seguir, você deve conseguir responder:

- O HTML da página faz sentido sem CSS?
- O formulário tem labels e tipos adequados?
- O CSS organiza a página em telas diferentes?
- O JavaScript altera a interface usando DOM e eventos?
- A página pode ser usada com teclado?
- O projeto evita depender de framework?

## Prática Recomendada

- Construa o projeto em três arquivos: `index.html`, `styles.css` e `script.js`.
- Faça primeiro o HTML completo.
- Depois aplique CSS para layout e responsividade.
- Por último adicione JavaScript para validação e mensagens.
- Revise o projeto usando o checklist da aula.
