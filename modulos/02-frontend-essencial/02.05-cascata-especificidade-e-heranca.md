# Cascata, Especificidade e Herança

## Objetivo

Ao final desta aula, você deve ser capaz de:

- explicar como o browser decide qual regra CSS aplicar;
- diferenciar cascata, especificidade e herança;
- identificar por que uma declaração foi sobrescrita;
- evitar seletores e correções difíceis de manter.

## Ideia Central

CSS significa Cascading Style Sheets. Quando mais de uma regra pode estilizar o mesmo elemento, o browser precisa decidir qual declaração vence.

Essa decisão considera origem, importância, camada, especificidade e ordem das regras. Algumas propriedades também são herdadas dos elementos ancestrais. Entender esse comportamento evita depender de tentativas, seletores cada vez maiores ou uso excessivo de `!important`.

## Exemplo Prático

```html
<article class="card" id="featured">
  <h2 class="card__title">Curso de CSS</h2>
</article>
```

```css
h2 {
  color: navy;
}

.card__title {
  color: green;
}

#featured .card__title {
  color: purple;
}
```

Mesmo que a primeira regra apareça antes, as regras seguintes podem vencer por terem maior especificidade.

## Analogias e Intuições

- A cascata é o processo completo de resolução de conflitos.
- Especificidade é apenas um dos critérios usados nesse processo.
- Herança permite que certos valores desçam pela árvore do documento.

## Erros Comuns

- Tratar cascata e especificidade como sinônimos.
- Resolver qualquer conflito com `!important`.
- Usar IDs para estilização sem necessidade.
- Criar seletores longos e dependentes da estrutura do HTML.
- Presumir que todas as propriedades são herdadas.
- Alterar regras sem consultar o DevTools.

## Checklist de Entendimento

Antes de seguir, você deve conseguir responder:

- O que é a cascata?
- Quando a ordem das regras decide o resultado?
- Como classes, IDs e elementos afetam a especificidade?
- Qual é a diferença entre cascata e herança?
- Por que seletores muito específicos dificultam manutenção?
- Quando `!important` pode causar problemas?

## Prática Recomendada

- Crie três regras que disputem a cor do mesmo elemento.
- Preveja qual regra vence antes de abrir o browser.
- Teste herança com `color` e compare com `margin`.
- Remova um `!important` substituindo-o por uma estrutura de regras mais clara.
- Use o DevTools para identificar a origem de uma declaração sobrescrita.
