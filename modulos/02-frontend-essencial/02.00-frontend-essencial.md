# Frontend Essencial

## Objetivo

Ao final desta aula, você deve ser capaz de:
- entender o papel de HTML, CSS e JavaScript como as três camadas da web;
- reconhecer que frameworks são abstrações sobre fundamentos que precisam ser sólidos;
- relacionar estrutura, estilo e comportamento ao que o usuário vê e interage;
- saber o que será construído ao longo do módulo e por que essa base importa.

## Ideia Central

Frameworks de frontend como React, Vue e Angular são ferramentas poderosas — mas todas elas renderizam HTML, aplicam CSS e executam JavaScript no browser. Antes de usar qualquer framework, você precisa entender essas três camadas: o que cada uma faz, como se relacionam e onde cada responsabilidade começa e termina.

Este módulo cobre o frontend sem atalhos: HTML semântico e formulários, CSS com cascata, especificidade, layout e responsividade, e JavaScript no browser com DOM e eventos. Também cobre acessibilidade e compatibilidade entre browsers — problemas que frameworks não resolvem por você.

Ao final, você terá construído telas reais sem framework, o que tornará qualquer framework que você aprender depois muito mais compreensível.

## Analogias e Intuições

- HTML é a estrutura do prédio: paredes, portas, janelas. CSS é a pintura e decoração. JavaScript é a elétrica e os sensores — o que responde quando alguém aperta um botão.
- Cascata CSS funciona como herança em uma família: propriedades descem do pai para o filho, mas o filho pode sobrescrever o que quiser com especificidade suficiente.
- O DOM é o mapa vivo do HTML: quando JavaScript muda o DOM, o browser re-renderiza automaticamente.

## Erros Comuns

- Usar `<div>` para tudo em vez de elementos semânticos (`<nav>`, `<main>`, `<article>`, `<button>`).
- Ignorar acessibilidade até o final — ela precisa ser pensada desde o HTML.
- Tentar aprender Flexbox e Grid ao mesmo tempo sem solidificar o modelo de caixa antes.
- Manipular o DOM em loop sem considerar o custo de reflow e repaint no browser.
- Confundir evento de `click` em um `<div>` com um `<button>` — o comportamento padrão e a acessibilidade são diferentes.

- O que é o DOM e como JavaScript interage com ele?
- Por que acessibilidade importa e como HTML semântico ajuda?

## Prática Recomendada

- Crie uma página HTML com apenas estrutura semântica (sem CSS) simulando um painel de atendimentos de clínica veterinária.
- Aplique Flexbox para organizar os cartões de pacientes em uma grade responsiva.
- Adicione um evento de `click` que marque um paciente como "em atendimento" sem recarregar a página.
- Abra o DevTools, inspecione o DOM e observe as mudanças em tempo real ao interagir com a página.
