# Acessibilidade

## Objetivo

Ao final desta aula, você deve ser capaz de:

- explicar o que é acessibilidade no frontend;
- reconhecer decisões de HTML, CSS e JavaScript que afetam uso real;
- aplicar cuidados básicos com semântica, labels, foco, teclado e contraste;
- entender acessibilidade como parte da qualidade, não como etapa extra.

## Onde Esta Aula Entra no Curso

- Módulo: Frontend Essencial
- Aula anterior: Eventos
- Próxima aula: Browsers e Compatibilidade
- Pré-requisitos: HTML, semântica, formulários, CSS, DOM e eventos

## Ideia Central

Acessibilidade é criar interfaces que possam ser usadas por pessoas em diferentes condições, dispositivos e necessidades. Isso inclui pessoas que usam teclado, leitores de tela, zoom, alto contraste, navegação por voz ou outras tecnologias assistivas.

Grande parte da acessibilidade começa com fundamentos: HTML semântico, labels corretas, foco visível, contraste suficiente e interações que não dependem apenas de mouse.

## Tópicos da Aula

- O que é acessibilidade
- Semântica como base
- Labels em formulários
- Foco e navegação por teclado
- Contraste e legibilidade
- Mensagens de erro compreensíveis
- Cuidados com eventos e elementos interativos

## Exemplo Prático

```html
<label for="email">E-mail</label>
<input id="email" name="email" type="email" required aria-describedby="erro-email">
<p id="erro-email">Digite um e-mail válido.</p>
```

```css
button:focus-visible,
input:focus-visible {
  outline: 3px solid #005fcc;
  outline-offset: 2px;
}
```

## Analogias e Intuições

- Acessibilidade é fazer a porta abrir para mais formas de uso.
- HTML semântico entrega informações que aparência visual sozinha não entrega.
- Teclado é um teste simples e poderoso de usabilidade.

## Erros Comuns

- Remover contorno de foco sem substituir por outro indicador.
- Usar `div` clicável no lugar de botão.
- Depender apenas de cor para informar erro.
- Criar formulário sem label.
- Tratar acessibilidade como tarefa final, depois da interface pronta.

## Checklist de Entendimento

Antes de seguir, você deve conseguir responder:

- O que é acessibilidade?
- Como semântica HTML ajuda?
- Por que foco visível importa?
- Por que label não deve ser substituída por placeholder?
- Como testar uma interação usando apenas teclado?

## Prática Recomendada

- Revise o formulário criado nas aulas anteriores.
- Confirme se todos os campos têm labels.
- Navegue pela página usando apenas teclado.
- Verifique se foco, mensagens e botões são compreensíveis.
- Ajuste contraste e legibilidade quando necessário.
