# JavaScript no Browser

## Objetivo

Ao final desta aula, você deve ser capaz de:

- explicar o papel do JavaScript no browser;
- conectar um arquivo JavaScript a uma página HTML;
- entender a diferença entre carregar HTML e executar scripts;
- criar interações pequenas sem depender de framework.

## Onde Esta Aula Entra na Formação

- Módulo: Frontend Essencial
- Aula anterior: Responsividade
- Próxima aula: DOM
- Pré-requisitos: HTML, CSS, responsividade e papel do browser

## Ideia Central

JavaScript no browser permite adicionar comportamento à página. Ele pode reagir a cliques, ler campos, alterar textos, controlar classes CSS e conversar com APIs do browser.

Neste módulo, JavaScript aparece apenas no contexto da interface. A linguagem será estudada com mais profundidade no módulo de Programação com TypeScript.

## Tópicos da Aula

- O que JavaScript faz no frontend
- Tag `script` e atributo `defer`
- Console do browser
- Variáveis e funções em nível mínimo
- Separação entre HTML, CSS e JavaScript
- Cuidados com scripts que rodam antes do HTML estar pronto

## Exemplo Prático

```html
<button id="botao-mensagem">Mostrar mensagem</button>
<p id="mensagem"></p>

<script src="script.js" defer></script>
```

```javascript
const botao = document.querySelector("#botao-mensagem");
const mensagem = document.querySelector("#mensagem");

botao.addEventListener("click", () => {
  mensagem.textContent = "JavaScript alterou a página.";
});
```

## Analogias e Intuições

- HTML estrutura, CSS apresenta e JavaScript reage.
- O browser é o ambiente onde esse JavaScript roda.
- Frameworks automatizam muita coisa, mas não eliminam os fundamentos.

## Erros Comuns

- Colocar JavaScript para resolver problema de HTML ou CSS.
- Esquecer `defer` e tentar acessar elementos antes de existirem.
- Misturar muito código dentro do HTML.
- Começar por biblioteca antes de entender interação básica.
- Achar que JavaScript no browser e JavaScript no Node.js são sempre o mesmo ambiente.

## Checklist de Entendimento

Antes de seguir, você deve conseguir responder:

- Para que serve JavaScript no browser?
- Como conectar um arquivo JavaScript a uma página?
- O que `defer` ajuda a evitar?
- Qual é o papel do console?
- Por que este módulo não aprofunda toda a linguagem JavaScript?

## Prática Recomendada

- Crie um botão que altera o texto de um parágrafo.
- Abra o console do browser e teste mensagens com `console.log`.
- Mova o script para um arquivo separado.
- Teste a diferença entre usar e não usar `defer`.
