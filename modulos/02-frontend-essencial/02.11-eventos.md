## Ideia Central

Eventos são sinais de que algo aconteceu: um clique, uma tecla pressionada, um formulário enviado, uma página carregada. O JavaScript pode escutar esses sinais e executar uma função em resposta.

Essa ideia é central no frontend. Interfaces não são apenas páginas estáticas; elas reagem às ações da pessoa e às mudanças do ambiente.

## Exemplo Prático

```html
<form id="formulario-triagem">
  <label for="pet">Nome do pet</label>
  <input id="pet" name="pet" type="text" required>
  <button type="submit">Registrar Triagem</button>
</form>

<p id="resultado"></p>
```

```javascript
const formulario = document.querySelector("#formulario-triagem");
const resultado = document.querySelector("#resultado");

formulario.addEventListener("submit", (event) => {
  event.preventDefault();
  const nomeDoPet = document.querySelector("#pet").value;
  resultado.textContent = `Triagem registrada para ${nomeDoPet}.`;
});
```

## Analogias e Intuições

- Evento é um aviso de que algo aconteceu.
- Listener é uma função esperando esse aviso.
- `preventDefault` cancela o comportamento padrão quando você precisa assumir o controle.

## Erros Comuns

- Chamar a função imediatamente em vez de passá-la para o listener.
- Esquecer `preventDefault` em demonstrações com formulário.
- Colocar regra crítica apenas no evento do frontend.
- Criar muitos listeners sem organização.
- Ignorar teclado e foco, pensando apenas em clique.
