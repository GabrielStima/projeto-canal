# Responsividade

## Objetivo

Ao final desta aula, você deve ser capaz de:

- explicar o que é responsividade;
- entender por que páginas precisam se adaptar a diferentes telas;
- usar `viewport`, unidades flexíveis e media queries em exemplos simples;
- revisar layouts pensando em conteúdo real.

## Onde Esta Aula Entra na Formação

- Módulo: Frontend Essencial
- Aula anterior: Layout com CSS
- Próxima aula: JavaScript no Browser
- Pré-requisitos: HTML, CSS e layout básico

## Ideia Central

Responsividade é a capacidade da interface de funcionar bem em diferentes tamanhos de tela, modos de entrada e contextos de uso. Não é apenas "ficar bonito no celular"; é continuar legível, navegável e usável.

Um layout responsivo começa com HTML bem estruturado, CSS flexível e decisões que não dependem de uma largura fixa perfeita.

## Tópicos da Aula

- O problema das telas diferentes
- Meta viewport
- Larguras fluidas e limites com `max-width`
- Imagens e conteúdo flexível
- Media queries
- Pensamento mobile-first em alto nível
- Testes manuais em larguras diferentes

## Exemplo Prático

```css
.container {
  width: min(100% - 32px, 960px);
  margin: 0 auto;
}

.cards {
  display: grid;
  grid-template-columns: 1fr;
  gap: 16px;
}

@media (min-width: 720px) {
  .cards {
    grid-template-columns: repeat(3, 1fr);
  }
}
```

## Analogias e Intuições

- Responsividade é uma conversa entre conteúdo e espaço disponível.
- Uma interface boa não depende de uma única tela ideal.
- Mobile-first ajuda a começar pelo essencial.

## Erros Comuns

- Criar tudo com larguras fixas.
- Esquecer a meta viewport.
- Testar apenas no monitor do desenvolvedor.
- Esconder conteúdo importante no mobile.
- Usar media query para remendar estrutura ruim.

## Checklist de Entendimento

Antes de seguir, você deve conseguir responder:

- O que é responsividade?
- Para que serve a meta viewport?
- O que é uma media query?
- Por que `max-width` pode ajudar?
- Como testar se uma página responde bem a mudanças de tela?

## Prática Recomendada

- Pegue a seção de cards da aula anterior.
- Faça funcionar em uma coluna no celular e três colunas em telas maiores.
- Ajuste espaçamentos para não depender de valores fixos grandes.
- Teste no browser reduzindo e aumentando a largura da janela.
