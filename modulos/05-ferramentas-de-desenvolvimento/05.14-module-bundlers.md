## Ideia Central

Module bundler é uma ferramenta que analisa os módulos de um projeto e gera arquivos prontos para serem usados pelo browser ou por outro ambiente.

Ele resolve problemas como juntar dependências, transformar TypeScript ou JSX, otimizar arquivos, lidar com assets e preparar o código para desenvolvimento ou produção. Nesta etapa do curso, o objetivo é entender o papel da ferramenta, não dominar configurações avançadas.

## Tópicos da Aula

- Assets.
- Exemplos de ferramentas: Vite, Webpack, Rollup e esbuild.

## Exemplo Prático

Imagine estes arquivos:

```text
src/main.ts
src/ui/renderTask.ts
src/domain/task.ts
```

O bundler parte de `src/main.ts`, segue os imports, entende quais arquivos entram no grafo e gera uma saída que o ambiente alvo consegue carregar.

## Analogias e Intuições

- Bundler é uma linha de montagem: recebe várias peças e entrega um pacote utilizável.
- Grafo de dependências é o mapa de quem importa quem.

## Erros Comuns

- Achar que bundler é necessário para todo script pequeno.
- Confundir TypeScript com bundler.
- Mexer em configuração avançada sem entender o problema.
- Esquecer que desenvolvimento e produção podem ter builds diferentes.

## Checklist de Entendimento

- Compare um script único em HTML com um projeto modular.
