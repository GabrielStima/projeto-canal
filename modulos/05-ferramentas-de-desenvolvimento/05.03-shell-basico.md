## Ideia Central

Terminal é a janela. Shell é o programa que interpreta o que você digita nessa janela. Bash, zsh e PowerShell são exemplos de shells.

O shell recebe comandos, expande caminhos, le argumentos, executa programas e mostra a saída. Para desenvolvimento web, você não precisa virar especialista em shell no início, mas precisa conseguir executar tarefas comuns sem medo.

## Tópicos da Aula

- Terminal vs shell.
- Prompt, comando, argumento e flag.
- Variáveis temporarias no shell.
- Saída padrão e mensagens de erro.
- Histórico de comandos.
- Cuidados ao copiar comandos destrutivos.

## Exemplo Prático

Compare comandos com e sem argumentos:

```bash
ls
ls -la
cd ..
pwd
```

Depois use o histórico do shell para repetir um comando anterior.

## Analogias e Intuições

- Terminal é o balcão; shell é a pessoa que entende o pedido e chama a ferramenta certa.
- Flags são ajustes finos do comando, como escolher modo detalhado ou resumido.

## Erros Comuns

- Achar que todo terminal usa o mesmo shell.
- Rodar comandos destrutivos sem entender o que fazem.
- Confundir flag com argumento.
- Fechar o terminal achando que isso apaga o projeto.

## Checklist de Entendimento

Antes de seguir, a pessoa deve conseguir responder:

- O que e shell?
- Qual a diferença entre terminal e shell?

- Execute `pwd`, `ls`, `ls -la` e compare as saidas.
- Use seta para cima para recuperar comandos anteriores.
- Escreva uma explicacao curta de terminal vs shell.
