# Git

## Objetivo

Ao final desta aula, voce deve ser capaz de:

- explicar o papel do Git;
- criar commits pequenos;
- usar comandos basicos com seguranca;
- saber o que estudar em seguida.

## Onde Esta Aula Entra no Curso

- Modulo: Ferramentas de Desenvolvimento
- Aula anterior: VCS: controle de versao
- Proxima aula: VCS hosting
- Pre-requisitos: VCS em alto nivel, terminal e projetos com multiplos arquivos.

## Ideia Central

Git e um sistema de controle de versao distribuido. Ele registra mudancas, permite comparar estados do projeto, criar branches e integrar trabalho de varias pessoas.

No inicio, o foco deve ser o fluxo essencial: ver estado, preparar arquivos, criar commit, criar branch, trocar de branch, integrar mudancas e conversar com um repositorio remoto.

## Topicos da Aula

- Repositorio Git.
- Working tree, staging area e commit.
- `git status`, `git add` e `git commit`.
- Branches com `git branch` e `git switch`.
- Merge.
- Remotos, `push` e `pull`.
- Historico com `git log`.

## Exemplo Pratico

Fluxo basico:

```bash
git init
git status
git add .
git commit -m "cria estrutura inicial"
git switch -c minha-feature
git status
git log
```

Depois de configurar um remoto, o fluxo passa a incluir:

```bash
git push origin minha-feature
git pull
```

## Analogias e Intuicoes

- Working tree e sua mesa atual de trabalho.
- Staging area e a bandeja do que vai entrar no proximo registro.
- Commit e o registro salvo com mensagem.

## Erros Comuns

- Fazer `git add .` sem olhar `git status`.
- Escrever mensagens de commit vagas.
- Misturar muitas mudancas sem relacao no mesmo commit.
- Dar `pull` ou `merge` sem ler conflitos com calma.

## Checklist de Entendimento

Antes de seguir, a pessoa deve conseguir responder:

- O que Git registra?
- Qual a diferenca entre arquivo modificado, staged e commitado?
- Para que serve uma branch?
- O que `push` e `pull` fazem em alto nivel?

## Pratica Recomendada

- Inicialize um repositorio em um projeto pequeno.
- Faca tres commits pequenos.
- Crie uma branch, altere um arquivo e commit.
- Volte para a branch principal e observe a diferenca.
