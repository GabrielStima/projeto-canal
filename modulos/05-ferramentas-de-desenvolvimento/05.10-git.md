## Ideia Central

Git é um sistema de controle de versão distribuído. Ele registra mudanças, permite comparar estados do projeto, criar branches e integrar trabalho de várias pessoas.

No início, o foco deve ser o fluxo essencial: ver estado, preparar arquivos, criar commit, criar branch, trocar de branch, integrar mudanças e conversar com um repositório remoto.

## Tópicos da Aula

- Repositório Git.
- Working tree, staging area e commit.
- `git status`, `git add` e `git commit`.
- Branches com `git branch` e `git switch`.
- Merge.
- Remotos, `push` e `pull`.
- Histórico com `git log`.

## Exemplo Prático

Fluxo básico:

```bash
git init
git status
git add .
git commit -m "cria estrutura inicial"
git switch -c minha-feature
git status
git log
```

Depois de configurar um remoto, o fluxo passa a incluir:

```bash
git push origin minha-feature
git pull
```

## Analogias e Intuições

- Working tree é sua mesa atual de trabalho.
- Staging area é a bandeja do que vai entrar no próximo registro.
- Commit é o registro salvo com mensagem.

## Erros Comuns

- Fazer `git add .` sem olhar `git status`.
- Escrever mensagens de commit vagas.
- Misturar muitas mudanças sem relação no mesmo commit.
- Dar `pull` ou `merge` sem ler conflitos com calma.

## Checklist de Entendimento

Antes de seguir, a pessoa deve conseguir responder:

- O que Git registra?
- Qual a diferença entre arquivo modificado, staged e commitado?
- Para que serve uma branch?
- O que `push` e `pull` fazem em alto nível?

## Prática Recomendada

- Inicialize um repositório em um projeto pequeno.
- Faça três commits pequenos.
- Crie uma branch, altere um arquivo e commit.
- Volte para a branch principal e observe a diferença.
