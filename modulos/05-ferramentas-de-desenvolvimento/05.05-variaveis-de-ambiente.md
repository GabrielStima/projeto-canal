# Variáveis de Ambiente

## Objetivo

Ao final desta aula, você deve ser capaz de:

- explicar o que são variáveis de ambiente;
- reconhecer quando uma configuração não deve ficar fixa no código;
- diferenciar `.env` de `.env.example`;
- saber o que estudar em seguida.

## Ideia Central

Variáveis de ambiente são valores fornecidos ao programa pelo ambiente onde ele roda. Elas servem para configurar comportamento sem alterar o código.

Exemplos comuns: porta do servidor, URL de API, modo de execução, chaves de serviço e credenciais. Valores sensíveis não devem ser publicados no repositório.

## Tópicos da Aula

- Configuração vs código.
- Variáveis temporarias no terminal.
- Arquivo `.env`.
- Arquivo `.env.example`.
- Segredos e cuidado com Git.
- Diferença entre desenvolvimento e produção.

## Exemplo Prático

Um projeto pode ter um arquivo `.env.example` assim:

```env
APP_PORT=3000
API_URL=http://localhost:3000
```

Cada pessoa copia esse arquivo para `.env` e preenche os valores reais do próprio ambiente.

## Analogias e Intuições

- Código é a receita; variável de ambiente é o ingrediente escolhido para aquele preparo.
- `.env.example` é uma lista de campos esperados, não o cofre dos valores reais.

## Erros Comuns

- Commitar `.env` com credenciais.
- Espalhar URLs fixas pelo código.
- Não documentar quais variáveis o projeto espera.
- Usar o mesmo valor sensível em desenvolvimento e produção.

## Checklist de Entendimento

Antes de seguir, a pessoa deve conseguir responder:

- O que é uma variável de ambiente?
- Por que não colocar segredo direto no código?
- Para que serve `.env.example`?
- Que tipo de valor pode variar entre ambientes?

## Prática Recomendada

- Crie um `.env.example` para um projeto fictício.
- Liste três variáveis que fariam sentido nesse projeto.
- Escreva no README como criar o `.env`.
- Explique por que o `.env` real não deve ser versionado.
