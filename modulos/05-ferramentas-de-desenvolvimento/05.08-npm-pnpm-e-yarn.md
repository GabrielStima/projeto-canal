# npm, pnpm e yarn

## Objetivo

Ao final desta aula, voce deve ser capaz de:

- reconhecer npm, pnpm e yarn como gerenciadores de pacote;
- executar comandos basicos de instalacao e scripts;
- entender que cada projeto deve escolher uma ferramenta principal;
- saber o que estudar em seguida.

## Onde Esta Aula Entra no Curso

- Modulo: Ferramentas de Desenvolvimento
- Aula anterior: Gerenciadores de pacote
- Proxima aula: VCS: controle de versao
- Pre-requisitos: `package.json`, dependencias, scripts e semver em nivel introdutorio.

## Ideia Central

npm, pnpm e yarn resolvem o mesmo problema central: gerenciar pacotes em projetos JavaScript/TypeScript. Eles tem diferencas de implementacao, desempenho, lockfile e comandos, mas no inicio o mais importante e entender o fluxo comum: inicializar projeto, instalar dependencia, instalar dependencia de desenvolvimento e rodar scripts.

Em um projeto real, evite misturar gerenciadores sem motivo. O lockfile indica qual ferramenta foi usada.

## Topicos da Aula

- npm como gerenciador muito comum no ecossistema Node.js.
- pnpm e yarn como alternativas populares.
- `npm init`, instalacao e scripts.
- Dependencia normal vs dependencia de desenvolvimento.
- Lockfiles diferentes.
- Como seguir a escolha do projeto.

## Exemplo Pratico

Com npm, um fluxo introdutorio pode ser:

```bash
npm init -y
npm install date-fns
npm install -D typescript
npm run build
```

Se o projeto usa pnpm ou yarn, procure os comandos equivalentes e mantenha consistencia com o lockfile existente.

## Analogias e Intuicoes

- npm, pnpm e yarn sao caminhos diferentes para administrar a mesma despensa de pacotes.
- O lockfile e uma foto das versoes que aquele projeto resolveu usar.

## Erros Comuns

- Rodar `npm install`, `yarn install` e `pnpm install` no mesmo projeto sem necessidade.
- Ignorar o lockfile versionado.
- Instalar tudo como dependencia de producao.
- Achar que o gerenciador de pacote entende a intencao do projeto sem scripts claros.
