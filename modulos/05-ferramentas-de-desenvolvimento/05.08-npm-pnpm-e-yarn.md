# npm, pnpm e yarn

## Objetivo

Ao final desta aula, você deve ser capaz de:

- reconhecer npm, pnpm e yarn como gerenciadores de pacote;
- executar comandos básicos de instalação e scripts;
- entender que cada projeto deve escolher uma ferramenta principal;
- saber o que estudar em seguida.

## Onde Esta Aula Entra no Curso

- Módulo: Ferramentas de Desenvolvimento
- Aula anterior: Gerenciadores de pacote
- Próxima aula: VCS: controle de versão
- Pré-requisitos: `package.json`, dependências, scripts e semver em nível introdutório.

## Ideia Central

npm, pnpm e yarn resolvem o mesmo problema central: gerenciar pacotes em projetos JavaScript/TypeScript. Eles tem diferenças de implementacao, desempenho, lockfile e comandos, mas no início o mais importante e entender o fluxo comum: inicializar projeto, instalar dependência, instalar dependência de desenvolvimento e rodar scripts.

Em um projeto real, evite misturar gerenciadores sem motivo. O lockfile indica qual ferramenta foi usada.

## Tópicos da Aula

- npm como gerenciador muito comum no ecossistema Node.js.
- pnpm e yarn como alternativas populares.
- `npm init`, instalação e scripts.
- Dependência normal vs dependência de desenvolvimento.
- Lockfiles diferentes.
- Como seguir a escolha do projeto.

## Exemplo Prático

Com npm, um fluxo introdutório pode ser:

```bash
npm init -y
npm install date-fns
npm install -D typescript
npm run build
```

Se o projeto usa pnpm ou yarn, procure os comandos equivalentes e mantenha consistência com o lockfile existente.

## Analogias e Intuições

- npm, pnpm e yarn são caminhos diferentes para administrar a mesma despensa de pacotes.
- O lockfile é uma foto das versões que aquele projeto resolveu usar.

## Erros Comuns

- Rodar `npm install`, `yarn install` e `pnpm install` no mesmo projeto sem necessidade.
- Ignorar o lockfile versionado.
- Instalar tudo como dependência de produção.
- Achar que o gerenciador de pacote entende a intencao do projeto sem scripts claros.
