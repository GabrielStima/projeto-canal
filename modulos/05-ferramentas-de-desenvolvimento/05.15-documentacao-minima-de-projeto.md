# Documentação Mínima de Projeto

## Objetivo

Ao final desta aula, você deve ser capaz de:

- explicar por que documentação mínima importa;
- escrever um README útil para rodar um projeto;
- documentar scripts, dependências e variáveis de ambiente;
- saber o que estudar em seguida.

## Onde Esta Aula Entra no Curso

- Módulo: Ferramentas de Desenvolvimento
- Aula anterior: Module bundlers
- Próxima aula: Code review
- Pré-requisitos: ambiente local, scripts, dependências e variáveis de ambiente.

## Ideia Central

Documentação mínima é o conjunto de informações que permite outra pessoa entender o que o projeto faz e como rodar o básico sem adivinhar.

Ela não precisa ser longa. Precisa responder as perguntas certas: o que é o projeto, quais pré-requisitos existem, como instalar, como configurar, como rodar, como testar ou validar, e onde ficam as partes principais.

## Tópicos da Aula

- README como porta de entrada.
- Pré-requisitos.
- Instalação.
- Variáveis de ambiente.
- Scripts disponiveis.
- Estrutura básica de pastas.
- Problemas conhecidos e próximos passos.

## Exemplo Prático

Estrutura mínima:

~~~markdown
# Nome do Projeto

## Sobre

Explique em poucas linhas.

## Pré-requisitos

- Node.js
- Gerenciador de pacote usado pelo projeto

## Como rodar

```bash
npm install
npm run dev
```
~~~

## Analogias e Intuições

- README é a placa na entrada do projeto.
- Boa documentação mínima evita que o setup vire investigacao arqueológica.

## Erros Comuns

- Escrever README bonito, mas sem comandos reais.
- Não documentar variáveis de ambiente.
- Esquecer qual gerenciador de pacote o projeto usa.
- Deixar instruções antigas depois de mudar scripts.

## Checklist de Entendimento

Antes de seguir, a pessoa deve conseguir responder:

- Que perguntas um README mínimo deve responder?
- Como documentar variáveis de ambiente sem expor segredos?
- Por que comandos precisam ser testados?
- Quando documentação fica desatualizada?

## Prática Recomendada

- Escreva um README mínimo para um projeto TypeScript.
- Inclua pré-requisitos, instalação, scripts e variáveis.
- Peca para outra pessoa seguir as instruções.
- Corrija o README com base nas duvidas que aparecerem.
