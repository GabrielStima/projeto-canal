# Documentacao Minima de Projeto

## Objetivo

Ao final desta aula, voce deve ser capaz de:

- explicar por que documentacao minima importa;
- escrever um README util para rodar um projeto;
- documentar scripts, dependencias e variaveis de ambiente;
- saber o que estudar em seguida.

## Onde Esta Aula Entra no Curso

- Modulo: Ferramentas de Desenvolvimento
- Aula anterior: Module bundlers
- Proxima aula: Code review
- Pre-requisitos: ambiente local, scripts, dependencias e variaveis de ambiente.

## Ideia Central

Documentacao minima e o conjunto de informacoes que permite outra pessoa entender o que o projeto faz e como rodar o basico sem adivinhar.

Ela nao precisa ser longa. Precisa responder as perguntas certas: o que e o projeto, quais pre-requisitos existem, como instalar, como configurar, como rodar, como testar ou validar, e onde ficam as partes principais.

## Topicos da Aula

- README como porta de entrada.
- Pre-requisitos.
- Instalacao.
- Variaveis de ambiente.
- Scripts disponiveis.
- Estrutura basica de pastas.
- Problemas conhecidos e proximos passos.

## Exemplo Pratico

Estrutura minima:

~~~markdown
# Nome do Projeto

## Sobre

Explique em poucas linhas.

## Pre-requisitos

- Node.js
- Gerenciador de pacote usado pelo projeto

## Como rodar

```bash
npm install
npm run dev
```
~~~

## Analogias e Intuicoes

- README e a placa na entrada do projeto.
- Boa documentacao minima evita que o setup vire investigacao arqueologica.

## Erros Comuns

- Escrever README bonito, mas sem comandos reais.
- Nao documentar variaveis de ambiente.
- Esquecer qual gerenciador de pacote o projeto usa.
- Deixar instrucoes antigas depois de mudar scripts.

## Checklist de Entendimento

Antes de seguir, a pessoa deve conseguir responder:

- Que perguntas um README minimo deve responder?
- Como documentar variaveis de ambiente sem expor segredos?
- Por que comandos precisam ser testados?
- Quando documentacao fica desatualizada?

## Pratica Recomendada

- Escreva um README minimo para um projeto TypeScript.
- Inclua pre-requisitos, instalacao, scripts e variaveis.
- Peca para outra pessoa seguir as instrucoes.
- Corrija o README com base nas duvidas que aparecerem.
