# Ambientes de Desenvolvimento

## Objetivo

Ao final desta aula, você deve ser capaz de:

- explicar o que é um ambiente de desenvolvimento;
- reconhecer diferenças entre ambiente local, teste, homologacao e produção;
- preparar um projeto local com ferramentas básicas;
- saber o que estudar em seguida.

## Onde Esta Aula Entra na Formação

- Módulo: Ferramentas de Desenvolvimento
- Aula anterior: Shell básico
- Próxima aula: Variáveis de ambiente
- Pré-requisitos: editor, terminal e shell básico.

## Ideia Central

Ambiente de desenvolvimento é o conjunto de sistema operacional, editor, runtime, dependências, configurações e comandos usados para trabalhar em um projeto.

Quando duas pessoas rodam o mesmo projeto em máquinas diferentes, pequenas diferenças podem causar comportamentos diferentes. Por isso projetos profissionais tentam documentar versões, comandos e configurações importantes.

## Tópicos da Aula

- Ambiente local.
- Ambiente de teste, homologacao e produção em alto nível.
- Runtime, dependências e scripts.
- Arquivos de configuração.
- Documentação de setup.
- Reprodutibilidade.

## Exemplo Prático

Liste o que um projeto TypeScript simples precisa para rodar:

```text
- Node.js instalado
- Gerenciador de pacote
- Dependências instaladas
- Script de desenvolvimento
- Arquivo README com instruções
- Variáveis de ambiente quando houver configuração sensível
```

## Analogias e Intuições

- Ambiente é a cozinha onde a receita será feita: ingredientes, ferramentas e temperatura importam.
- Produzir o mesmo resultado exige mais do que ter o mesmo código.

## Erros Comuns

- Dizer "na minha máquina funciona" sem investigar diferenças de ambiente.
- Esquecer de documentar versões e comandos.
- Misturar configuração local com código do projeto.
- Usar valores de produção durante desenvolvimento.

## Checklist de Entendimento

Antes de seguir, a pessoa deve conseguir responder:

- O que compoe um ambiente de desenvolvimento?
- Por que ambiente local e produção não são a mesma coisa?
- O que deve ser documentado para outra pessoa rodar o projeto?
- Como diferenças de versão podem quebrar um projeto?

## Prática Recomendada

- Escolha um projeto local e escreva uma lista do que ele precisa para rodar.
- Identifique quais partes são código e quais partes são ambiente.
- Peca para outra pessoa tentar seguir suas instruções.
- Ajuste a documentação depois do teste.
