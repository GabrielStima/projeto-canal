# VCS: Controle de Versão

- entender por que histórico de mudanças importa;

## Ideia Central

VCS significa Version Control System, ou sistema de controle de versão. Ele registra mudanças no projeto ao longo do tempo.

Sem controle de versão, cada alteracao importante vira uma aposta: se quebrar algo, fica difícil voltar; se duas pessoas editam o mesmo arquivo, fica difícil combinar; se uma decisão foi tomada semanas atras, fica difícil lembrar o motivo. Controle de versão cria histórico, comparação e colaboração.

## Exemplo Prático

Compare estes dois fluxos:

```text
projeto-final.zip
projeto-final-agora-vai.zip
projeto-final-corrigido-2.zip
```

versus:

```text
commit: cria tela inicial
commit: adiciona validação do formulario
commit: corrige erro ao salvar tarefa
```

O segundo fluxo conta uma historia técnica mais clara.

## Analogias e Intuições

- Commit é uma fotografia comentada do projeto em um momento importante.
- Branch é uma mesa separada para trabalhar sem bagunçar a linha principal.

## Erros Comuns

- Fazer commits gigantes com várias ideias misturadas.
- Versionar arquivos gerados sem necessidade.
- Usar branch como copia permanente sem integrar mudanças.
- Achar que VCS substitui backup e revisão cuidadosa.
