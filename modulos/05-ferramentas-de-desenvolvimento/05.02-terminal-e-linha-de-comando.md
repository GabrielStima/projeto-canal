# Terminal e Linha de Comando

## Objetivo

Ao final desta aula, você deve ser capaz de:

- explicar o que e terminal e linha de comando;
- navegar por pastas usando comandos simples;
- executar comandos de projeto;
- saber o que estudar em seguida.

## Onde Esta Aula Entra na Formação

- Módulo: Ferramentas de Desenvolvimento
- Aula anterior: Editor/IDE e produtividade
- Próxima aula: Shell básico
- Pré-requisitos: entender arquivos, pastas e projetos locais.

## Ideia Central

Terminal é uma interface de texto para conversar com o sistema operacional e com ferramentas de desenvolvimento. Em vez de clicar em botões, você digita comandos.

Linha de comando assusta no início porque parece menos visual, mas ela é uma das formas mais diretas de trabalhar com projetos. Muitas ferramentas profissionais partem do terminal: Git, npm, servidores locais, testes, formatadores, geradores de projeto e scripts.

## Tópicos da Aula

- Terminal como interface.
- Comando, argumento, opção e saída.
- Diretório atual.
- Caminhos relativos e absolutos.
- Comandos básicos: `pwd`, `ls`, `cd`, `mkdir`, `touch`.
- Erros comuns ao executar comandos na pasta errada.

## Exemplo Prático

Crie uma pasta de estudo pelo terminal:

```bash
pwd
mkdir estudo-terminal
cd estudo-terminal
touch anotacoes.md
ls
```

Depois abra a pasta no editor e confirme que o arquivo existe.

## Analogias e Intuições

- Terminal é uma conversa objetiva: você pede uma ação e recebe uma resposta.
- Diretório atual é o "lugar onde você está" dentro do sistema de arquivos.

## Erros Comuns

- Executar comandos sem conferir a pasta atual.
- Copiar comandos sem entender os argumentos.
- Achar que toda mensagem no terminal e erro.
- Ignorar diferenças entre caminhos relativos e absolutos.

## Checklist de Entendimento

Antes de seguir, a pessoa deve conseguir responder:

- O que é um terminal?
- O que é um comando?
- Como descubro a pasta atual?
- Como entro e saio de pastas?
- Por que a pasta atual muda o resultado de um comando?

## Prática Recomendada

- Crie uma árvore de pastas pelo terminal.
- Navegue entre elas usando `cd`.
- Liste arquivos com `ls`.
- Execute um comando em uma pasta errada de proposito e explique o erro.
