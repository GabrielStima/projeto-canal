# Exercício 02 — Versionando uma Mudança do PetCare OS

## O que você já sabe

Você estudou que Git registra mudanças em commits, que branches permitem trabalhar sem alterar a linha principal e que commits pequenos contam uma história mais clara.

## Cenário

Você está trabalhando no PetCare OS. Hoje você precisa ajustar duas coisas no repositório local:

1. adicionar um `.gitignore` que ignore `node_modules`, arquivos `.env` e a pasta de build `dist/`;
2. corrigir o nome de uma função no arquivo `src/calculaTotal.ts` de `calc()` para `calcularTotalDoOrcamento()`.

## Tarefa

Escreva a sequência de comandos Git que você usaria para fazer essas mudanças com commits separados e bem descritos. Depois, justifique por que você separou as mudanças em dois commits.

Você pode assumir que já existe um repositório Git inicializado e que você está na branch principal.

## Critérios de Correção

- Dois commits distintos, um para cada mudança.
- Mensagens de commit descritivas em português ou inglês consistente.
- `.gitignore` com `node_modules`, `.env` e `dist/`.
- Comandos na ordem correta: status, add, commit, branch opcional.
- Justificativa clara para separar as mudanças.
