# Editor/IDE e Produtividade

## Objetivo

Ao final desta aula, você deve ser capaz de:

- explicar o papel de um editor ou IDE no desenvolvimento;
- reconhecer recursos que ajudam sem esconder o raciocínio;
- configurar um ambiente simples para JavaScript/TypeScript;
- saber o que estudar em seguida.

## Ideia Central

Editor e IDE são ferramentas para escrever, navegar, executar e entender código. Um editor pode ser leve e extensível. Uma IDE costuma trazer mais recursos integrados. Na prática, o mais importante não é o nome da ferramenta, mas a qualidade do fluxo: abrir arquivos, buscar código, receber feedback de erro, formatar, refatorar e usar o terminal sem fricção.

Produtividade não significa instalar muitas extensões. Significa reduzir tarefas repetitivas e aumentar clareza. O editor deve ajudar você a pensar melhor, não virar uma muleta que impede entender o projeto.

## Exemplo Prático

Abra um pequeno projeto TypeScript e faça um passeio guiado:

1. encontre um arquivo pelo nome;
2. busque uma função pelo texto;
3. abra o terminal integrado;
4. corrija um erro de tipo indicado pelo editor;
5. formate o arquivo;
6. navegue até a definição de uma função.

## Analogias e Intuições

- Editor é a bancada de trabalho: quanto menos bagunça, mais fácil pensar.
- Autocomplete é como sugestão de caminho: ajuda, mas você ainda precisa saber para onde está indo.

## Erros Comuns

- Instalar extensões demais antes de entender a necessidade.
- Confundir aviso do editor com verdade absoluta.
- Depender de autocomplete sem ler a assinatura da função.
- Ignorar mensagens de erro porque "o código parece certo".

## Checklist de Entendimento

Antes de seguir, a pessoa deve conseguir responder:

- O que um editor/IDE resolve?
- Quais recursos ajudam mais no início?
- Quando uma extensão atrapalha?
- Por que o editor não substitui entendimento de código?

## Prática Recomendada

- Configure um editor para abrir uma pasta de projeto.
- Crie três arquivos TypeScript e navegue entre eles sem usar o mouse o tempo todo.
- Provoque um erro de tipo e leia a mensagem exibida.
- Remova uma extensão que você não sabe explicar para que serve.
