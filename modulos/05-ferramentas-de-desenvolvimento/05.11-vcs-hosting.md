# VCS Hosting

## Objetivo

Ao final desta aula, você deve ser capaz de:

- explicar o que e VCS hosting;
- diferenciar Git local de repositório hospedado;
- reconhecer recursos comuns de colaboração;
- saber o que estudar em seguida.

## Onde Esta Aula Entra na Formação

- Módulo: Ferramentas de Desenvolvimento
- Aula anterior: Git
- Próxima aula: GitHub
- Pré-requisitos: Git local, commits e branches.

## Ideia Central

VCS hosting é um serviço que hospeda repositórios de controle de versão. Ele permite que um projeto saia da sua máquina e vire um ponto de colaboração.

Git funciona localmente. Serviços como GitHub, GitLab e Bitbucket adicionam hospedagem, interface web, pull requests, revisoes, permissões, issues, integracoes e automações.

## Tópicos da Aula

- Repositório local vs remoto.
- `origin` como nome comum de remoto.
- Push e pull em alto nível.
- Permissões e visibilidade.
- Issues, pull requests e revisoes.
- Diferenças entre ferramenta Git e plataforma de colaboração.

## Exemplo Prático

Fluxo conceitual do PetCare OS:

```text
1. Você cria commits localmente na branch feature/notificacao-tutor.
2. Você envia a branch para o repositório remoto.
3. Outras pessoas do time conseguem ver a branch.
4. Um pull request organiza discussão e revisão da notificação.
5. A mudança aprovada entra na branch principal.
```

## Analogias e Intuições

- Git local é seu caderno de histórico.
- VCS hosting é a biblioteca compartilhada onde o time consulta e revisa esse histórico.

## Erros Comuns

- Achar que Git e GitHub são a mesma coisa.
- Esquecer de enviar a branch para o remoto.
- Colocar segredo em repositório público.
- Fazer mudanças direto na branch principal sem combinar o fluxo do time.

## Checklist de Entendimento
