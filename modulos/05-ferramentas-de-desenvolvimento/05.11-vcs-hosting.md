# VCS Hosting

## Objetivo

Ao final desta aula, voce deve ser capaz de:

- explicar o que e VCS hosting;
- diferenciar Git local de repositorio hospedado;
- reconhecer recursos comuns de colaboracao;
- saber o que estudar em seguida.

## Onde Esta Aula Entra no Curso

- Modulo: Ferramentas de Desenvolvimento
- Aula anterior: Git
- Proxima aula: GitHub
- Pre-requisitos: Git local, commits e branches.

## Ideia Central

VCS hosting e um servico que hospeda repositorios de controle de versao. Ele permite que um projeto saia da sua maquina e vire um ponto de colaboracao.

Git funciona localmente. Servicos como GitHub, GitLab e Bitbucket adicionam hospedagem, interface web, pull requests, revisoes, permissoes, issues, integracoes e automacoes.

## Topicos da Aula

- Repositorio local vs remoto.
- `origin` como nome comum de remoto.
- Push e pull em alto nivel.
- Permissoes e visibilidade.
- Issues, pull requests e revisoes.
- Diferencas entre ferramenta Git e plataforma de colaboracao.

## Exemplo Pratico

Fluxo conceitual:

```text
1. Voce cria commits localmente.
2. Voce envia a branch para o remoto.
3. Outras pessoas conseguem ver a branch.
4. Um pull request organiza discussao e revisao.
5. A mudanca aprovada entra na branch principal.
```

## Analogias e Intuicoes

- Git local e seu caderno de historico.
- VCS hosting e a biblioteca compartilhada onde o time consulta e revisa esse historico.

## Erros Comuns

- Achar que Git e GitHub sao a mesma coisa.
- Esquecer de enviar a branch para o remoto.
- Colocar segredo em repositorio publico.
- Fazer mudancas direto na branch principal sem combinar o fluxo do time.

## Checklist de Entendimento

Antes de seguir, a pessoa deve conseguir responder:

- O que um servico de VCS hosting adiciona ao Git?
- O que e um repositorio remoto?
- Por que pull request ajuda na colaboracao?
- Qual a diferenca entre ferramenta local e plataforma online?

## Pratica Recomendada

- Explique Git vs GitHub em cinco linhas.
- Desenhe o fluxo local -> remoto -> revisao -> merge.
- Liste tres cuidados antes de publicar um repositorio.
