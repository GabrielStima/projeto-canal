# Introdução e Visão Geral do Projeto

## Objetivo

Ao final desta aula, você deve ser capaz de:

- entender a proposta e os objetivos da formação;
- identificar as fases e os módulos que compõem a trilha;
- navegar pelo repositório com autonomia;
- entender o papel do PetCare OS como fio condutor da formação;
- configurar o ambiente mínimo para acompanhar as primeiras aulas.

## Ideia Central

Esta formação foi construída com uma premissa simples: aprender ensinando.

Cada aula transforma um assunto técnico em um material claro, consultável e útil — tanto para quem acompanha a trilha completa quanto para quem busca um tema específico. A sequência foi planejada para ir do zero absoluto até tópicos avançados de arquitetura, infraestrutura e inteligência artificial, sem pular os fundamentos que sustentam qualquer coisa que vem depois.

O repositório não é apenas um arquivo de slides. Ele é o seu mapa de navegação, material de revisão e ponto de partida para a prática.

## O Que É Esta Formação e Para Quem Ela É

A Formação Engenharia FullStack Gratuita é uma trilha de aprendizado aberta e sem custo, focada em JavaScript e TypeScript do lado do cliente ao lado do servidor, passando por bancos de dados, infraestrutura em nuvem, segurança, system design e IA aplicada ao desenvolvimento.

Ela é indicada para:

- quem quer entrar na área como desenvolvedor fullstack;
- quem já programa, mas tem lacunas em fundamentos ou temas mais avançados;
- quem quer um material de referência organizado para revisão e consulta.

Ela não é um bootcamp de resultado rápido. O objetivo é construir maturidade técnica real — o tipo de base que permite entender o que está acontecendo em produção e tomar decisões com critério.

## Como Navegar pelo Repositório

A estrutura principal do projeto é:

```text
.
├── README.md                    # Visão geral pública do projeto
├── curriculo.md                 # Matriz curricular completa
├── estrutura-do-repositorio.md  # Como os arquivos estão organizados e por quê
├── templates/                   # Modelos de aula, módulo e prompt de IA
├── desafio/                     # Especificação do PetCare OS
└── modulos/                     # Pastas com o conteúdo da formação
    ├── 00-introducao-e-metodo-de-estudo/
    ├── 01-fundamentos-da-internet-e-web/
    └── ...
```

Dentro de cada módulo:

```text
modulos/00-introducao-e-metodo-de-estudo/
├── README.md              # Apresentação do módulo, lista de aulas e status
├── 00.00-introducao-...   # Aula guarda-chuva (esta aula)
└── 00.01-metodo-...       # Aulas específicas
```

A numeração das aulas segue o padrão `módulo.sequência`: `00.01`, `01.03`, `04.06`. Isso mantém a ordem da formação mesmo quando você navega por módulos separados.

Para encontrar um assunto específico, use:

- `curriculo.md` — visão macro de todos os tópicos;
- o `README.md` de cada módulo — lista de aulas com links e status;
- a busca do GitHub ou do seu editor — por nome do arquivo ou por palavra-chave.

## O Projeto Prático: PetCare OS

Toda a formação tem um fio condutor: o **PetCare OS**.

Em vez de criar pequenos projetos descartáveis em cada módulo, você vai construir um sistema real de ponta a ponta. O PetCare OS é uma plataforma para clínicas veterinárias e petshops, com módulos de triagem clínica, controle de estoque, faturamento e portal do tutor.

O projeto começa simples: nas primeiras aulas, você escreverá telas estáticas em HTML e CSS. Com o tempo, ele vai evoluir para uma aplicação completa com backend em Node.js, banco de dados relacional, autenticação, filas, WebSockets, infraestrutura em nuvem via Terraform, containers no Kubernetes e integração com IA generativa.

Ao final da formação, o PetCare OS é um projeto de portfólio que demonstra profundidade técnica real.

Você não precisa entender tudo isso agora. O ponto é saber que cada módulo técnico da formação tem uma aplicação prática direta no projeto.

Veja a especificação completa em: [`desafio/README.md`](../../desafio/README.md)

## Analogias e Intuições

- Pense no `curriculo.md` como o sumário de um livro técnico.
- Pense em cada módulo como um capítulo, com suas seções (aulas específicas).
- Pense no PetCare OS como o projeto do livro — um sistema que vai sendo construído capítulo a capítulo.
- Pense no `README.md` de cada módulo como o cabeçalho do capítulo: ele diz o que está lá antes de você mergulhar.

## Erros Comuns

- **Tentar pular a sequência cedo demais.** Módulos mais avançados assumem que os anteriores foram estudados. Atalhos no início criam lacunas que aparecem depois.
- **Ignorar o `curriculo.md`.** Ele é o mapa. Consultar a visão macro antes de entrar em um módulo novo ajuda a entender onde você está e o que vem depois.
- **Achar que precisa instalar tudo no primeiro dia.** Você não precisa de Node.js, Git ou dominar o terminal para começar. Essas ferramentas são ensinadas no módulo 05. No início, basta um editor de código e um navegador.
- **Ver o PetCare OS como algo obrigatório e separado.** Ele é integrado à formação. Não é um projeto paralelo — é o contexto prático de cada módulo técnico.
