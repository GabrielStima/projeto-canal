# Algoritmos de Hashing

## Objetivo

Ao final desta aula, voce deve ser capaz de:

- diferenciar hash generico de algoritmo para senha;
- explicar por que velocidade pode ser ruim em senhas;
- reconhecer criterios para escolher um algoritmo;
- evitar decisoes perigosas como criar criptografia propria.

## Onde Esta Aula Entra no Curso

- Modulo: 12. Autenticacao, Autorizacao e Seguranca
- Aula anterior: Hashing de Senhas
- Proxima aula: Autenticacao Basica
- Pre-requisitos: Hashing de senhas, backend, banco de dados e nocao de performance.

## Ideia Central

Nem todo hash serve para senha. Alguns algoritmos sao feitos para integridade e velocidade. Senhas precisam do contrario: uma funcao com salt e custo ajustavel, para dificultar ataques offline quando hashes vazam.

Em projetos reais, a escolha deve seguir bibliotecas maduras e recomendacoes atuais. O desenvolvedor precisa entender o criterio, nao reinventar o algoritmo.

## Topicos da Aula

- Hash para integridade.
- Hash para senha.
- Ataque offline.
- Salt unico por senha.
- Custo computacional.
- Algoritmos adaptativos.
- Migracao de hashes antigos.
- Bibliotecas confiaveis.

## Exemplo Pratico

```text
Cenario:
- usuario antigo tem hash com custo baixo;
- usuario faz login com sucesso;
- backend verifica o hash;
- backend percebe que o parametro esta antigo;
- backend recalcula o hash com parametros atuais.
```

## Analogias e Intuicoes

- Hash rapido e bom para conferir arquivo, mas ruim para proteger senha.
- Custo e uma trava: deve incomodar o atacante sem derrubar o usuario legitimo.

## Erros Comuns

- Usar MD5 ou SHA simples para senha.
- Reutilizar o mesmo salt para todos os usuarios.
- Salvar parametros do algoritmo fora do hash de forma confusa.
- Nunca revisar custo com o passar dos anos.
- Misturar segredo de aplicacao com senha sem entender o objetivo.

## Checklist de Entendimento

Antes de seguir, voce deve conseguir responder:

- Por que velocidade e perigosa em hash de senha?
- Qual a diferenca entre salt e segredo?
- O que e ataque offline?
- Como migrar hashes antigos com menos atrito?
- Por que bibliotecas maduras importam?

## Pratica Recomendada

- Compare hash para arquivo e hash para senha.
- Escreva criterios para escolher uma biblioteca.
- Desenhe uma estrategia de migracao de custo.
- Liste sinais de configuracao insegura.
- Explique por que criar algoritmo proprio e uma ma ideia.
