# Segurança na Web

## Objetivo

Ao final desta aula, você deve ser capaz de:

- explicar por que segurança web não é apenas "ter login";
- reconhecer dados, ações e superfícies de ataque;
- diferenciar autenticação, autorização e proteção contra ataques;
- preparar o estudo das aulas específicas do módulo.

## Onde Esta Aula Entra no Curso

- Módulo: 12. Autenticação, Autorização e Segurança
- Aula anterior: Autenticação, Autorização e Segurança
- Próxima aula: Hashing de Senhas
- Pré-requisitos: HTTP, browsers, APIs, backend, bancos de dados, validação, contratos de API e failure modes.

## Ideia Central

Segurança na web é o conjunto de decisões que reduz a chance e o impacto de uso indevido de uma aplicação. Ela envolve identidade, permissão, armazenamento de dados, entrada de usuário, configuração de browser, rede, banco, logs e infraestrutura.

Uma aplicação segura não é uma aplicação impossível de atacar. E uma aplicação que entende seus riscos, evita falhas comuns, limita danos e torna problemas investigáveis.

## Tópicos da Aula

- Ativos: dados, contas, tokens, sessões e permissões.
- Ameaças: roubo de conta, vazamento, abuso e manipulação.
- Superfície de ataque.
- Confiança entre cliente, servidor e banco.
- Defesa em profundidade.
- Mínimo privilégio.
- Falhar de forma segura.

## Analogias e Intuições

- Segurança é uma cadeia: uma proteção forte não compensa todas as outras fracas.
- O cliente é uma interface, não uma fonte confiável de verdade.

## Erros Comuns

- Achar que esconder botão no frontend é autorização.
- Guardar segredo no código do cliente.
- Confiar em `userId` vindo do payload.
- Tratar segurança apenas no fim do projeto.
- Expor detalhes internos em mensagens de erro.

## Checklist de Entendimento

Antes de seguir, você deve conseguir responder:

- O que uma aplicação precisa proteger?
- Qual a diferença entre autenticar e autorizar?
- Por que validação no backend continua obrigatória?
- O que significa mínimo privilégio?
- Como uma falha pequena pode virar falha grave?

## Prática Recomendada

- Escolha três funcionalidades do painel de estudos.
- Liste dados sensíveis envolvidos.
- Liste quem pode executar cada ação.
- Liste duas formas de abuso para cada funcionalidade.
- Defina uma proteção inicial para cada abuso.
