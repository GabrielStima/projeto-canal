## Ideia Central

OAuth é um protocolo de autorização delegada. Ele permite que uma aplicação acesse recursos em nome de um usuário sem receber a senha desse usuário. Em vez de entregar credenciais para a aplicação, o usuário autoriza um provedor a emitir tokens com escopo limitado.

Quando OAuth é usado para login, normalmente entra junto com OpenID Connect, que adiciona identidade do usuário ao fluxo.

## Exemplo Prático

```text
Fluxo simplificado:
1. usuário clica em entrar com provedor;
2. app redireciona para authorization server;
3. usuário consente;
4. provedor redireciona de volta com code;
5. backend troca code por tokens;
6. app usa token conforme escopo permitido.
```

## Analogias e Intuições

- OAuth é uma autorização temporária e limitada, não a entrega da chave principal.
- Scope é o limite do que a aplicação pode fazer.

## Erros Comuns

- Tratar OAuth como protocolo de senha.
- Ignorar validação de redirect URI.
- Guardar access token sem cuidado.
- Pedir escopos amplos demais.
- Confundir OAuth puro com autenticação de usuário sem OpenID Connect.
