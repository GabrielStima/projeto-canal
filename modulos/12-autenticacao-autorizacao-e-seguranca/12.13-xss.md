## Ideia Central

XSS acontece quando conteúdo controlado por atacante é executado como JavaScript dentro da página de outro usuário. O problema não é apenas "aparecer um alert". Um script executado no contexto da aplicação pode ler dados expostos, disparar ações, manipular tela e tentar roubar credenciais acessíveis.

A mitigação combina escaping, sanitização quando houver HTML permitido, frameworks usados corretamente, `HttpOnly` para cookies sensíveis e Content Security Policy em cenários mais avançados.

## Exemplo Prático

```html
<p>Comentário: <script>fetch('/me')</script></p>
```

Se a aplicação renderiza comentário como HTML confiável, um atacante pode transformar dado em código executável.

## Analogias e Intuições

- XSS é deixar comentário de usuário virar comando dentro da sua página.
- Escapar conteúdo é tratar texto como texto, não como HTML.

## Erros Comuns

- Confiar em "usuários não vão digitar isso".
- Montar HTML com string sem escapar conteúdo.
- Usar HTML rico sem sanitização.
- Guardar token em local fácil de acessar por script.
- Achar que framework elimina todos os riscos automaticamente.
