# Autenticacao via Token

## Objetivo

Ao final desta aula, voce deve ser capaz de:

- explicar autenticacao baseada em token;
- diferenciar token opaco de token com conteudo;
- entender envio por header `Authorization`;
- reconhecer cuidados de armazenamento e revogacao.

## Onde Esta Aula Entra no Curso

- Modulo: 12. Autenticacao, Autorizacao e Seguranca
- Aula anterior: CSRF
- Proxima aula: JWT
- Pre-requisitos: HTTP, headers, autenticacao basica, sessoes, cookies e APIs.

## Ideia Central

Na autenticacao via token, o cliente apresenta um valor que representa uma autenticacao valida. Em APIs, esse valor frequentemente vai no header `Authorization`, em vez de ser enviado automaticamente como cookie.

Token nao e permissao magica. O servidor ainda precisa validar o token, identificar o usuario, checar expiracao, lidar com revogacao e aplicar autorizacao.

## Topicos da Aula

- Token de acesso.
- Token opaco.
- Header `Authorization`.
- Bearer token.
- Expiracao.
- Refresh token.
- Revogacao.
- Armazenamento no cliente.

## Exemplo Pratico

```http
GET /me
Authorization: Bearer token_aleatorio
```

O servidor recebe o token, valida se ele existe ou se sua assinatura e confiavel, verifica expiracao e associa a requisicao a um usuario.

## Analogias e Intuicoes

- Token e uma credencial temporaria que acompanha cada requisicao.
- Bearer token funciona como "quem possui, usa"; por isso roubo de token e grave.

## Erros Comuns

- Criar token sem expiracao.
- Guardar token em local exposto sem avaliar risco de XSS.
- Achar que token elimina autorizacao.
- Nao ter estrategia de revogacao.
- Enviar token em query string.

## Checklist de Entendimento

Antes de seguir, voce deve conseguir responder:

- O que o token prova?
- Como o cliente envia o token?
- O que e token opaco?
- O que acontece se o token vazar?
- Como expiracao e revogacao entram no desenho?

## Pratica Recomendada

- Desenhe um login que retorna access token.
- Escreva uma requisicao autenticada com `Authorization`.
- Liste onde voce nao guardaria o token.
- Defina expiracao para access token.
- Explique quando refresh token faria sentido.
