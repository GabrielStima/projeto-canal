# Autenticação via Token

## Objetivo

Ao final desta aula, você deve ser capaz de:

- explicar autenticação baseada em token;
- diferenciar token opaco de token com conteúdo;
- entender envio por header `Authorization`;
- reconhecer cuidados de armazenamento e revogação.

## Onde Esta Aula Entra na Formação

- Módulo: 12. Autenticação, Autorização e Segurança
- Aula anterior: CSRF
- Próxima aula: JWT
- Pré-requisitos: HTTP, headers, autenticação básica, sessões, cookies e APIs.

## Ideia Central

Na autenticação via token, o cliente apresenta um valor que representa uma autenticação válida. Em APIs, esse valor frequentemente vai no header `Authorization`, em vez de ser enviado automaticamente como cookie.

Token não é permissão mágica. O servidor ainda precisa validar o token, identificar o usuário, checar expiração, lidar com revogação e aplicar autorização.

## Tópicos da Aula

- Token de acesso.
- Token opaco.
- Header `Authorization`.
- Bearer token.
- Expiração.
- Refresh token.
- Revogação.
- Armazenamento no cliente.

## Exemplo Prático

API de Estoque via Aplicativo Coletor:

```http
GET /api/estoque/lotes/1004
Authorization: Bearer token_secreto_789
```

O aplicativo móvel do funcionário do estoque anexa o token de acesso no cabeçalho `Authorization`. O servidor valida o token e responde. Diferente do cookie, o aplicativo precisa gerenciar explicitamente onde guarda esse token e como o anexa às requisições.

## Analogias e Intuições

- O Token é um passe VIP temporário. Ele diz ao segurança que você foi autorizado, mas o segurança (servidor) ainda precisa olhar para o passe e decidir se ele é genuíno ou se foi revogado.
*Limite: tokens opacos dependem do servidor acessar o banco para verificar a validade, ao contrário dos JWTs.*

## Erros Comuns

- Criar token sem expiração.
- Guardar token em local exposto sem avaliar risco de XSS.
- Achar que token elimina autorização.
- Não ter estratégia de revogação.
- Enviar token em query string.

## Checklist de Entendimento

Antes de seguir, você deve conseguir responder:

- O que o token prova?
- Como o cliente envia o token?
- O que é token opaco?
- O que acontece se o token vazar?
- Como expiração e revogação entram no desenho?

## Prática Recomendada

- Desenhe o fluxo onde o coletor de estoque faz login e recebe um access token.
- Escreva o formato da requisição HTTP que usa esse token para consultar o catálogo.
- Liste os problemas de segurança se o desenvolvedor do app salvar o token na memória desprotegida do dispositivo.
- Defina uma rotina para revogar esse token se o aparelho do estoque for perdido.
