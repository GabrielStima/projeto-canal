## Ideia Central

SQL injection acontece quando entrada controlada pelo usuario altera a estrutura de uma query. O atacante deixa de enviar apenas dado e passa a influenciar o comando executado pelo banco.

A defesa principal e usar queries parametrizadas ou APIs seguras do ORM. Validacao ajuda, mas nao substitui parametros. Permissoes minimas no banco reduzem impacto caso uma falha exista.

## Exemplo Pratico

```ts
// Perigoso
const sql = `SELECT * FROM users WHERE email = '${email}'`;

// Melhor ideia: parametros
const sql = "SELECT * FROM users WHERE email = ?";
const params = [email];
```

O objetivo e impedir que `email` mude a estrutura do comando SQL.

## Analogias e Intuicoes

- Parametro e colocar dado em um compartimento proprio, nao misturar com o comando.
- Validar entrada e bom; parametrizar query e obrigatorio.

## Erros Comuns

- Concatenar SQL com entrada do usuario.
- Achar que ORM torna qualquer uso automaticamente seguro.
- Validar apenas no frontend.
- Usar usuario do banco com permissao ampla demais.
- Exibir erro SQL completo para o cliente.
