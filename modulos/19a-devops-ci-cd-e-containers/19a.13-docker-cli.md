## Ideia Central

A Docker CLI é a interface prática para construir, executar, listar, inspecionar, acompanhar logs e remover imagens e containers. O objetivo não é decorar comandos, mas entender quais perguntas cada comando responde sobre o estado do ambiente.

## Tópicos da Aula

- `docker build`
- `docker images`
- `docker run`
- `docker ps`
- `docker logs`
- `docker exec`
- `docker stop` e `docker rm`
- `docker inspect`

## Exemplo Prático

O Controle da Caixa (CLI):

O desenvolvedor júnior da clínica digita `docker run -d -p 8080:3000 petcare-api`.
O `-d` (detach) joga o serviço pro fundo e não trava o terminal.
O `-p 8080:3000` mapeia a porta 8080 da máquina host para a porta 3000 dentro do container isolado.

## Analogias e Intuições

- O CLI do Docker é o controle remoto do porto. Você aperta o botão para o guindaste pegar a caixa (run), parar ela no ar (stop), ou abrir para inspecionar os logs (logs).

## Erros Comuns

- Remover container sem olhar logs.
- Confundir imagem parada com container parado.
- Executar comandos interativos como solução permanente.
