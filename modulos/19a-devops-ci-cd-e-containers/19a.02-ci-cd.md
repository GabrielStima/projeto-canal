## Ideia Central

CI/CD transforma mudanças pequenas em um fluxo verificável. A cada alteração, o projeto pode instalar dependências, rodar testes, validar qualidade, gerar artefatos e preparar uma entrega. O objetivo não é apertar menos botões; é descobrir problemas cedo e tornar a entrega repetível.

## Exemplo Prático

A Esteira do PetCare (CI/CD):

O estagiário fez um Pull Request alterando a regra de Custo do Raio-X.
CI (Continuous Integration): O GitHub Actions roda os testes unitários (`npm test`) sozinho. Ele detecta que o código do estagiário quebrou a conta de multiplicar e bloqueia o botão de Merge.
CD (Continuous Deployment/Delivery): O sênior arruma o código e aprova. Automaticamente, a AWS pega esse código validado e atualiza as clínicas (Prod) sem ninguém digitar comandos manuais.

## Analogias e Intuições

- CI/CD é como o raio-X de bagagens no aeroporto. CI (integração) verifica se há problemas no código. CD (entrega) leva o código validado até a produção.

## Erros Comuns

- Misturar CI, CD e deploy como se fossem a mesma coisa.
- Rodar testes lentos demais em toda mudança sem estratégia.
- Gerar artefatos diferentes para cada ambiente.
