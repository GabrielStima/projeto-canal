# DevOps, CI/CD e Containers

## Objetivo

Ao final desta aula, você deve ser capaz de:

- explicar como DevOps, CI/CD e containers participam da entrega de software;
- reconhecer os riscos de uma entrega manual e não rastreável;
- relacionar validação, empacotamento e publicação de artefatos;
- entender o que será aprofundado nas próximas aulas do módulo.

## Onde Esta Aula Entra na Formação

- Módulo: 19A. DevOps, CI/CD e Containers
- Aula anterior: Projeto Prático: Provisionando a Infraestrutura de uma Aplicação
- Próxima aula: O que é DevOps na Prática
- Pré-requisitos: Git, GitHub, terminal, variáveis de ambiente e code review; Backend com Node.js, APIs, bancos de dados, segurança e observabilidade; Cloud Fundamentals e Infrastructure as Code; Terraform, state, plan, apply, ambientes e segredos.

## Ideia Central

Aplicações modernas não terminam quando o código compila. Elas precisam ser empacotadas, entregues, configuradas, observadas e operadas em ambientes que mudam o tempo todo. Este módulo organiza esse caminho: primeiro a entrega, depois containers, depois Kubernetes.

## Tópicos da Aula

- O problema entre código pronto e aplicação rodando
- DevOps como colaboração, automação e responsabilidade compartilhada
- CI/CD como caminho de entrega
- Containers como unidade de empacotamento
- Kubernetes como orquestração de containers
- O que será aprofundado nas próximas aulas

## Exemplo Prático

O PetCare automatizado ao extremo:

Nos módulos anteriores, você criou a API do PetCare, configurou banco, usou Nuvem (Cloud) e versionou a infraestrutura (Terraform).
Mas como o código novo chega no servidor toda sexta-feira? Sem DevOps, o programador acessa a máquina por SSH, dá `git pull`, instala pacotes e cruza os dedos.
Com DevOps e Containers, o Dev faz `git push`. A nuvem empacota o código numa 'caixa inquebrável' (Docker), testa tudo e joga essa caixa dentro de um Orquestrador (Kubernetes) que gerencia 50 hospitais sozinho sem o desenvolvedor nem abrir o terminal.

## Analogias e Intuições

- DevOps é a linha de montagem da montadora de carros (CI/CD). O Docker é o container marítimo que leva o carro sem amassar. O Kubernetes é o porto automatizado de Singapura que gerencia 10 mil containers ao mesmo tempo sem bater nenhum.

## Erros Comuns

- Achar que DevOps é apenas cargo ou ferramenta.
- Pular CI/CD e ir direto para Kubernetes.
- Tratar container como máquina virtual pequena.
- Usar Kubernetes antes de entender deploy, configuração, rede e logs.
