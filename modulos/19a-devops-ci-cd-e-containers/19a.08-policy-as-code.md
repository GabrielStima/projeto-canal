# Policy as Code

## Objetivo

Ao final desta aula, você deve ser capaz de:

- explicar o conceito principal da aula;
- reconhecer quando ele aparece em projetos reais;
- aplicar a ideia em um exemplo simples de entrega, container ou Kubernetes;
- saber o que estudar em seguida.

## Onde Esta Aula Entra na Formação

- Módulo: 19A. DevOps, CI/CD e Containers
- Aula anterior: Credenciais Temporárias para Automação

## Ideia Central

Policy as Code transforma regras de engenharia, segurança e operação em verificações versionadas. Em vez de depender apenas de memória ou revisão manual, o pipeline pode bloquear padrões inseguros, custos inesperados, permissões amplas e configurações fora do combinado.

## Tópicos da Aula

- O que é Policy as Code
- Regras preventivas
- Regras detectivas
- Exemplos em IaC e containers
- Onde colocar políticas no pipeline
- Limites da automação

## Exemplo Prático

A Polícia da Nuvem (OPA):

A clínica definiu a regra financeira: 'Ninguém pode criar servidores M5.XLarge de U$ 500 no Terraform'.
Um Dev submete um PR criando a máquina gigante. O pipeline trava. Uma ferramenta de Policy as Code (ex: OPA/Conftest ou Checkov) analisa o código do Terraform antes do Apply e bloqueia o deploy automaticamente: *'Violação da Política FinOps: Tamanho EC2 não permitido'.*

## Analogias e Intuições

- Policy as Code é a catraca do metrô. Você não depende do segurança ficar olhando para impedir quem não pagou; o próprio sistema lê o bilhete e tranca a roleta de forma passiva e infalível.

## Erros Comuns

- Criar regras demais e travar o time.
- Escrever política sem explicar a razão.
- Usar exceções permanentes.
