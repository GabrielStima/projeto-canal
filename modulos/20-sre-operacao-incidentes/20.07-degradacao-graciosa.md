# Degradacao Graciosa

## Objetivo

Ao final desta aula, voce deve ser capaz de:

- explicar degradacao graciosa;
- reconhecer quando entregar valor parcial e melhor que falhar completamente;
- propor estrategias simples de degradacao em uma aplicacao;
- saber o que estudar em seguida.

## Onde Esta Aula Entra no Curso

- Modulo: 20. SRE, Operacao e Incidentes
- Aula anterior: Readiness e Liveness
- Proxima aula: Rollback
- Pre-requisitos: Failure modes, APIs, filas, dependencias externas, cache, feature flags, observabilidade e confiabilidade.

## Ideia Central

Degradacao graciosa e a capacidade de um sistema continuar oferecendo parte do valor quando alguma dependencia, recurso ou funcionalidade falha.

Em vez de responder "tudo ou nada", o sistema pode reduzir escopo: esconder um recurso secundario, usar cache, enfileirar trabalho, limitar uma operacao pesada, desligar uma feature temporariamente ou mostrar uma mensagem honesta para o usuario.

O ponto nao e mascarar problema. O ponto e reduzir dano enquanto a equipe investiga, mitiga ou recupera o sistema.

## Topicos da Aula

- Falha total versus valor parcial
- Dependencias criticas e secundarias
- Cache, fallback e filas
- Feature flags e kill switches
- Comunicacao com usuario durante degradacao

## Exemplo Pratico

Uma plataforma de estudos usa uma API externa para gerar sugestoes personalizadas. Se essa API externa falha, o sistema ainda pode:

- mostrar a lista de aulas normalmente;
- esconder a sugestao personalizada;
- usar uma sugestao em cache;
- registrar o erro com contexto;
- avisar que recomendacoes estao temporariamente indisponiveis.

Assim, a parte principal do produto continua funcionando.

## Analogias e Intuicoes

- Degradar graciosamente e manter o essencial funcionando quando o ideal nao esta disponivel.
- Nem toda falha precisa virar tela branca.

## Erros Comuns

- Tratar dependencia secundaria como obrigatoria.
- Criar fallback que retorna dado incorreto sem aviso.
- Nao medir quando o sistema esta em modo degradado.
- Esconder falhas a ponto de ninguem investigar.
- Degradar funcionalidade critica sem criterio.

## Checklist de Entendimento

Antes de seguir, a pessoa deve conseguir responder:

- O que e degradacao graciosa?
- Quando ela ajuda o usuario?
- Quando ela pode ser perigosa?
- Qual a diferenca entre fallback e mascarar erro?
- Como feature flags podem ajudar em incidentes?

## Pratica Recomendada

- Escolha uma funcionalidade com dependencia externa e proponha um fallback.
- Liste quais sinais indicariam que o sistema esta degradado.
- Escreva uma mensagem simples para usuario em caso de recurso temporariamente indisponivel.
