# CDN e Entrega de Conteúdo

## Objetivo

- identificar cuidados com invalidação, segurança e conteúdo dinâmico.

## Ideia Central

CDN significa Content Delivery Network. A ideia é aproximar conteúdo dos usuários usando pontos de presença distribuídos. Em vez de todo acesso buscar arquivos na origem, a CDN pode entregar uma cópia cacheada mais próxima do usuário.

Isso é especialmente útil para arquivos estáticos: JavaScript, CSS, imagens, vídeos, fontes, documentos públicos e frontends estáticos. Também pode ajudar em alguns fluxos dinâmicos, mas cachear conteúdo personalizado exige muito mais cuidado.

CDN não é só performance. Ela também pode reduzir carga da origem, absorver picos, centralizar TLS, aplicar algumas regras de segurança e controlar entrega de conteúdo.

## Exemplo Prático

```text
Fluxo: imagem de capa de aula

Sem CDN:
- usuário pede a imagem;
- origem responde de uma região distante;
- muitos usuários aumentam carga na origem.

Com CDN:
- primeiro acesso busca na origem;
- próximos acessos podem vir do ponto de presença mais próximo;
- origem recebe menos tráfego repetido.
```

## Analogias e Intuições

- CDN é uma rede de prateleiras próximas dos usuários.
- A origem é o estoque principal.
- Invalidação é avisar que uma cópia antiga não deve mais ser servida.

## Erros Comuns

- Cachear resposta personalizada como se fosse pública.
- Não versionar assets e depender de invalidação manual para tudo.
- Usar TTL longo em conteúdo que muda o tempo todo.
- Esquecer headers HTTP de cache.
- Achar que CDN corrige API lenta por milagre.

## Checklist de Entendimento

- Esse conteúdo é público ou personalizado?
- Ele muda com frequência?
- Posso versionar o nome do arquivo?
- Qual TTL faz sentido?
- O que acontece se uma versão antiga for servida por alguns minutos?

## Prática Recomendada

- Escolha um frontend com imagens, CSS e JavaScript.
- Separe quais arquivos poderiam ir para CDN.
- Defina quais podem ter cache longo e quais precisam de cache curto.
- Liste um erro perigoso de cache.
- Explique como object storage e CDN trabalhariam juntos.
