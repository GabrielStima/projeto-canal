## Ideia Central

Object storage é um serviço para guardar objetos: arquivos acompanhados de metadados e identificados por uma chave. Ele é muito usado para imagens, vídeos, PDFs, anexos, backups, logs, relatórios, exports e assets públicos.

Ele não substitui banco de dados relacional. Um banco organiza registros e consultas estruturadas. Object storage guarda blobs e permite recuperar objetos por chave. Em aplicações reais, os dois costumam trabalhar juntos: o arquivo fica no storage, enquanto o banco guarda metadados, dono, status e regras de acesso.

O cuidado principal é segurança. Nem todo arquivo deve ser público, e nem toda URL deve durar para sempre.

Upload via Cloud: A API envia (ou o cliente envia direto via Presigned URL) pro Amazon S3 (Object Storage). O S3 devolve um link e você salva SÓ o link (string) no banco de dados. Armazenamento virtualmente infinito e barato.

## Analogias e Intuições

- Object Storage é um guarda-volumes gigantesco. Você entrega a mala pesada (arquivo), ele te dá um ticket (URL). Você guarda o ticket pequenininho na carteira (Banco de Dados).

## Erros Comuns

- Guardar arquivos grandes dentro do banco sem necessidade.
- Deixar bucket público inteiro por engano.
- Confiar apenas no nome do arquivo enviado pelo usuário.
- Não validar tipo, tamanho e permissão de upload.
- Esquecer política de retenção e limpeza.

## Checklist de Entendimento

- O objeto deve ser público ou privado?
- Quem pode fazer upload?
- Quem pode baixar?
- Quanto tempo esse objeto precisa existir?
