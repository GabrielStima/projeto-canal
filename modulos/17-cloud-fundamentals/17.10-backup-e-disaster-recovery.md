# Backup e Disaster Recovery

## Objetivo

Ao final desta aula, você deve ser capaz de:

- explicar a diferença entre backup, restore e disaster recovery;
- entender RPO e RTO em nível fundamental;
- reconhecer que backup sem teste de restauração é só esperança;
- desenhar uma estratégia inicial simples de recuperação.

## Ideia Central

Backup é uma cópia de dados para recuperação. Restore é o ato de recuperar. Disaster recovery é o plano para continuar ou voltar a operar depois de uma falha séria, como perda de banco, região indisponível, erro humano ou exclusão acidental.

Dois termos ajudam a pensar: RPO e RTO. RPO indica quanto dado você aceita perder. RTO indica quanto tempo você aceita ficar fora do ar. Quanto menores esses números, maior tende a ser a complexidade e o custo.

O ponto principal é que backup precisa ser testado. Se ninguém sabe restaurar, não existe plano real.

## Exemplo Prático

```text
Aplicação: plataforma de cursos

Plano inicial:
- backup automático diário do banco;
- retenção de 30 dias;
- teste mensal de restore em ambiente separado;
- object storage com versionamento para arquivos importantes;
- documentação do passo a passo de recuperação;
- alerta para falhas de backup.
```

## Analogias e Intuições

- Backup é a cópia.
- Restore é provar que a cópia funciona.
- Disaster recovery é o plano para voltar a operar sob pressão.
- RPO é quanto passado você aceita perder.
- RTO é quanto tempo você aceita demorar para voltar.

## Erros Comuns

- Nunca testar restore.
- Guardar backup na mesma falha que destruiria o original.
- Não definir retenção.
- Não monitorar falhas de backup.
- Confundir alta disponibilidade com backup.
- Não planejar recuperação de arquivos no object storage.

## Checklist de Entendimento

- Quais dados são críticos?
- Quanto dado posso perder?
- Quanto tempo posso ficar indisponível?
- Onde o backup fica?
- Quem executa o restore?
- Quando foi o último teste de restauração?
