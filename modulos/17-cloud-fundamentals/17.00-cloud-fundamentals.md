# Cloud Fundamentals

## Objetivo

Ao final desta aula, você deve ser capaz de:

- explicar por que cloud é mais do que hospedar código em um servidor remoto;
- reconhecer os principais blocos de construção de uma infraestrutura cloud;
- relacionar cloud com backend, bancos, segurança, observabilidade e arquitetura;
- entender o que será aprofundado nas próximas aulas do módulo.

## Ideia Central

Uma aplicação em produção precisa de muito mais do que código. Ela precisa rodar em algum lugar, receber tráfego, acessar dados, guardar arquivos, proteger segredos, controlar permissões, escalar, recuperar falhas, gerar logs, entregar conteúdo e não destruir o orçamento.

Cloud organiza esses recursos como serviços consumíveis. Em vez de comprar servidores, instalar tudo manualmente e operar cada peça do zero, você monta uma infraestrutura combinando serviços de computação, rede, identidade, dados, armazenamento, mensageria, segurança e operação.

O ponto principal deste módulo não é decorar nomes de produtos. É entender os blocos, os problemas que eles resolvem e os trade-offs que aparecem quando uma aplicação sai da máquina local.

## Tópicos da Aula

- O que cloud resolve.
- Diferença entre infraestrutura, plataforma e aplicação.
- Regiões, zonas e disponibilidade.

- Custos, backup, disaster recovery e FinOps.

## Exemplo Prático

```text
Aplicação: plataforma de cursos

Precisa de:
- frontend publicado para alunos;
- API para autenticação, progresso e pagamentos;
- banco de dados para usuários e aulas;
- object storage para imagens, certificados e anexos;
- CDN para entregar assets com baixa latência;
- fila para processar certificados e e-mails;
- secrets para credenciais;
- backup para recuperar dados;
- controle de custos para evitar surpresa.
```

## Analogias e Intuições

- Cloud é um conjunto de blocos de infraestrutura alugáveis sob demanda.
- IAM é a portaria e o crachá de acesso.
- VPC é o bairro privado onde seus recursos conversam.
- Serviços gerenciados são responsabilidades operacionais que você terceiriza parcialmente.

## Erros Comuns

- Achar que cloud elimina operação.
- Escolher serviço por moda, não por problema.
- Ignorar IAM e rede no começo.
- Tratar custo como assunto só financeiro.
- Usar serverless, fila ou CDN sem entender o fluxo da aplicação.

## Checklist de Entendimento

- Que problema cloud resolve para uma aplicação web?
- Quais blocos básicos uma aplicação precisa em produção?
- O que muda quando um serviço é gerenciado?
- Que riscos aparecem em identidade, rede, dados e custo?
- Por que estudar cloud antes de Terraform, Docker e Kubernetes?
