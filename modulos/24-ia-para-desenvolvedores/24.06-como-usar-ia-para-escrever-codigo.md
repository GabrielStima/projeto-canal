# Como Usar IA Para Escrever Código

## Objetivo

Ao final desta aula, você deve ser capaz de:

- usar IA para gerar rascunhos de código com segurança;
- transformar requisitos em pedidos pequenos;
- pedir alternativas e trade-offs;
- validar código gerado antes de integrar ao projeto.

## Onde Esta Aula Entra no Curso

- Módulo: 24. IA para Desenvolvedores
- Aula anterior: [Prompt Engineering](24.05-prompt-engineering.md)
- Próxima aula: [Como Revisar Código Gerado por IA](24.07-como-revisar-codigo-gerado-por-ia.md)
- Pré-requisitos: TypeScript, testes, Git, requisitos e code review.

## Ideia Central

IA pode acelerar a escrita de código quando a tarefa está bem delimitada. Ela funciona melhor quando recebe contexto pequeno, contrato claro, exemplos e critérios de aceite.

O melhor uso não é pedir "faça tudo". É quebrar o problema, pedir uma proposta, revisar a direção, gerar uma parte, testar, comparar e integrar com cuidado.

## Tópicos da Aula

- Transformar requisito em tarefa pequena.
- Pedir plano antes de código.
- Gerar funções, testes, tipos e exemplos.
- Comparar alternativas.
- Integrar com diff pequeno.

## Exemplo Prático

Tarefa: criar uma função que normaliza filtros de query string.

Peça:

```text
Crie uma função TypeScript chamada normalizeOrderFilters.
Entrada: Record<string, string | string[] | undefined>.
Saída: objeto com status opcional, page numérico e pageSize limitado a 100.
Inclua casos de borda e testes unitários.
```

Depois revise tipos, limites, nomes e testes.

## Analogias e Intuições

- IA como par de programação que escreve rápido.
- Código gerado como pull request: precisa de review.

## Erros Comuns

- Pedir implementação grande sem contrato.
- Aceitar mudança que não entende.
- Ignorar estilo do projeto.
- Gerar código sem testes.
- Pedir para IA corrigir erro sem reproduzir o erro.

## Checklist de Entendimento

- A tarefa está pequena?
- O contrato de entrada e saída está claro?
- O código segue o estilo do projeto?
- Existem testes?
- Eu consigo explicar a solução?

## Prática Recomendada

- Escolha uma função pequena.
- Escreva requisitos e casos de borda antes do prompt.
- Gere uma solução com IA.
- Reescreva pelo menos uma parte manualmente.
- Rode ou descreva testes.

## Prompt Para Estudar Com IA

```text
Estou estudando Como Usar IA Para Escrever Código em uma trilha de desenvolvimento fullstack JavaScript/TypeScript.

Meu objetivo é entender o conceito, praticar e revisar sem receber respostas prontas cedo demais.

Contexto da aula:
- Descrição da aula: Esta aula estuda Como Usar IA Para Escrever Código dentro do contexto de IA para desenvolvedores, conectando o conceito com uso prático, validação, segurança e aplicação em projetos JavaScript/TypeScript.
- Descrição do módulo: O módulo IA para Desenvolvedores ensina fundamentos de IA generativa, uso de IA no trabalho de desenvolvimento, coding agents, tool calling, agentes, RAG, avaliação, evals, guardrails, segurança, privacidade, observabilidade, LLMOps e custos.
- Itens já estudados: Aulas anteriores do módulo 24: LLMs, Tokens, Contexto e Visão Geral de Embeddings; Limitações, Alucinações e Validação; Privacidade, Dados Sensíveis e Segredos em IA; Prompt Engineering. Também entram como base TypeScript, APIs, testes, segurança, privacidade, observabilidade e system design.

Crie uma sessão de estudo com:
1. uma explicação curta do conceito;
2. 5 perguntas progressivas;
3. 3 exercícios práticos;
4. 1 desafio um pouco mais difícil;
5. exemplos de aplicação em projetos reais;
6. um checklist para eu avaliar se entendi.

Regras:
- Não entregue respostas completas antes de eu tentar responder.
- Corrija minhas respostas com explicações.
- Se eu errar, dê uma dica antes de revelar a solução.
- Ao final, sugira o próximo assunto que devo estudar.
```
