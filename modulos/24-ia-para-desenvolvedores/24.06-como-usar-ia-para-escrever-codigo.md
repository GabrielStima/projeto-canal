# Como Usar IA Para Escrever Codigo

## Objetivo

Ao final desta aula, voce deve ser capaz de:

- usar IA para gerar rascunhos de codigo com seguranca;
- transformar requisitos em pedidos pequenos;
- pedir alternativas e trade-offs;
- validar codigo gerado antes de integrar ao projeto.

## Onde Esta Aula Entra no Curso

- Modulo: 24. IA para Desenvolvedores
- Aula anterior: [Prompt Engineering](24.05-prompt-engineering.md)
- Proxima aula: [Como Revisar Codigo Gerado por IA](24.07-como-revisar-codigo-gerado-por-ia.md)
- Pre-requisitos: TypeScript, testes, Git, requisitos e code review.

## Ideia Central

IA pode acelerar a escrita de codigo quando a tarefa esta bem delimitada. Ela funciona melhor quando recebe contexto pequeno, contrato claro, exemplos e criterios de aceite.

O melhor uso nao e pedir "faca tudo". E quebrar o problema, pedir uma proposta, revisar a direcao, gerar uma parte, testar, comparar e integrar com cuidado.

## Topicos da Aula

- Transformar requisito em tarefa pequena.
- Pedir plano antes de codigo.
- Gerar funcoes, testes, tipos e exemplos.
- Comparar alternativas.
- Integrar com diff pequeno.

## Exemplo Pratico

Tarefa: criar uma funcao que normaliza filtros de query string.

Peça:

```text
Crie uma funcao TypeScript chamada normalizeOrderFilters.
Entrada: Record<string, string | string[] | undefined>.
Saida: objeto com status opcional, page numerico e pageSize limitado a 100.
Inclua casos de borda e testes unitarios.
```

Depois revise tipos, limites, nomes e testes.

## Analogias e Intuicoes

- IA como par de programacao que escreve rapido.
- Codigo gerado como pull request: precisa de review.

## Erros Comuns

- Pedir implementacao grande sem contrato.
- Aceitar mudanca que nao entende.
- Ignorar estilo do projeto.
- Gerar codigo sem testes.
- Pedir para IA corrigir erro sem reproduzir o erro.

## Checklist de Entendimento

- A tarefa esta pequena?
- O contrato de entrada e saida esta claro?
- O codigo segue o estilo do projeto?
- Existem testes?
- Eu consigo explicar a solucao?

## Pratica Recomendada

- Escolha uma funcao pequena.
- Escreva requisitos e casos de borda antes do prompt.
- Gere uma solucao com IA.
- Reescreva pelo menos uma parte manualmente.
- Rode ou descreva testes.

## Prompt Para Estudar Com IA

```text
Estou estudando Como Usar IA Para Escrever Codigo em uma trilha de desenvolvimento fullstack JavaScript/TypeScript.

Meu objetivo é entender o conceito, praticar e revisar sem receber respostas prontas cedo demais.

Contexto da aula:
- Descrição da aula: Esta aula estuda Como Usar IA Para Escrever Codigo dentro do contexto de IA para desenvolvedores, conectando o conceito com uso pratico, validacao, seguranca e aplicacao em projetos JavaScript/TypeScript.
- Descrição do modulo: O modulo IA para Desenvolvedores ensina fundamentos de IA generativa, uso de IA no trabalho de desenvolvimento, coding agents, tool calling, agentes, RAG, avaliacao, evals, guardrails, seguranca, privacidade, observabilidade, LLMOps e custos.
- Itens já estudados: Aulas anteriores do modulo 24: LLMs, Tokens, Contexto e Visao Geral de Embeddings; Limitacoes, Alucinacoes e Validacao; Privacidade, Dados Sensiveis e Segredos em IA; Prompt Engineering. Tambem entram como base TypeScript, APIs, testes, seguranca, privacidade, observabilidade e system design.

Crie uma sessão de estudo com:
1. uma explicação curta do conceito;
2. 5 perguntas progressivas;
3. 3 exercícios práticos;
4. 1 desafio um pouco mais difícil;
5. exemplos de aplicação em projetos reais;
6. um checklist para eu avaliar se entendi.

Regras:
- Não entregue respostas completas antes de eu tentar responder.
- Corrija minhas respostas com explicações.
- Se eu errar, dê uma dica antes de revelar a solução.
- Ao final, sugira o próximo assunto que devo estudar.
```
