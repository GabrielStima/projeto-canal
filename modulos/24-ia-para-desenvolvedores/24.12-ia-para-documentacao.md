# IA Para Documentacao

## Objetivo

Ao final desta aula, voce deve ser capaz de:

- usar IA para rascunhar documentacao tecnica;
- transformar codigo, requisitos e decisoes em texto claro;
- revisar documentacao gerada por IA;
- evitar documentacao bonita, mas falsa.

## Onde Esta Aula Entra no Curso

- Modulo: 24. IA para Desenvolvedores
- Aula anterior: [IA Para Testes](24.11-ia-para-testes.md)
- Proxima aula: [Coding Agents](24.13-coding-agents.md)
- Pre-requisitos: README, documentacao minima de projeto, requisitos, APIs e revisao tecnica.

## Ideia Central

IA ajuda a documentar porque consegue reorganizar informacao, resumir mudancas, sugerir estrutura e adaptar tom. Mas documentacao precisa ser fiel ao sistema real.

O desenvolvedor deve fornecer contexto verificavel e revisar nomes, contratos, exemplos, comandos, limites, versoes e decisoes. Uma documentacao clara e errada e pior que uma documentacao incompleta.

## Topicos da Aula

- README, guias, ADRs, changelogs e docs de API.
- Resumo de pull requests.
- Documentacao de comandos e configuracao.
- Revisao de exemplos.
- Ligacao com o modulo 25.

## Exemplo Pratico

Peça para IA criar uma secao de README:

```text
Crie uma secao "Como rodar localmente" para um projeto Node.js.
Use apenas estes comandos:
- pnpm install
- pnpm dev
- pnpm test
Inclua variaveis obrigatorias sem inventar valores.
Se faltar informacao, liste perguntas.
```

O pedido limita invencao e deixa claro que comandos nao devem ser inventados.

## Analogias e Intuicoes

- IA como editora tecnica.
- Documentacao como contrato social do projeto.

## Erros Comuns

- Deixar IA inventar comandos.
- Documentar endpoint que nao existe.
- Omitir limitacoes.
- Copiar explicacao sem testar exemplo.
- Usar texto generico que nao ajuda o time.

## Checklist de Entendimento

- A documentacao corresponde ao sistema real?
- Exemplos foram testados?
- Comandos existem?
- O texto explica limites e requisitos?
- Faltou citar decisao importante?

## Pratica Recomendada

- Escolha um pequeno projeto.
- Gere uma secao de README com IA.
- Verifique cada comando e exemplo.
- Reescreva trechos genericos com contexto real.

## Prompt Para Estudar Com IA

```text
Estou estudando IA Para Documentacao em uma trilha de desenvolvimento fullstack JavaScript/TypeScript.

Meu objetivo é entender o conceito, praticar e revisar sem receber respostas prontas cedo demais.

Contexto da aula:
- Descrição da aula: Esta aula estuda IA Para Documentacao dentro do contexto de IA para desenvolvedores, conectando o conceito com uso pratico, validacao, seguranca e aplicacao em projetos JavaScript/TypeScript.
- Descrição do modulo: O modulo IA para Desenvolvedores ensina fundamentos de IA generativa, uso de IA no trabalho de desenvolvimento, coding agents, tool calling, agentes, RAG, avaliacao, evals, guardrails, seguranca, privacidade, observabilidade, LLMOps e custos.
- Itens já estudados: Aulas anteriores do modulo 24: Seguranca em Codigo Gerado por IA; Quality Gates Para Codigo Gerado por IA; IA no Debug; IA Para Testes. Tambem entram como base TypeScript, APIs, testes, seguranca, privacidade, observabilidade e system design.

Crie uma sessão de estudo com:
1. uma explicação curta do conceito;
2. 5 perguntas progressivas;
3. 3 exercícios práticos;
4. 1 desafio um pouco mais difícil;
5. exemplos de aplicação em projetos reais;
6. um checklist para eu avaliar se entendi.

Regras:
- Não entregue respostas completas antes de eu tentar responder.
- Corrija minhas respostas com explicações.
- Se eu errar, dê uma dica antes de revelar a solução.
- Ao final, sugira o próximo assunto que devo estudar.
```
