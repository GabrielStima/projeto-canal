# IA Para Documentação

## Objetivo

Ao final desta aula, você deve ser capaz de:

- usar IA para rascunhar documentação técnica;
- transformar código, requisitos e decisões em texto claro;
- revisar documentação gerada por IA;
- evitar documentação bonita, mas falsa.

## Onde Esta Aula Entra no Curso

- Módulo: 24. IA para Desenvolvedores
- Aula anterior: [IA Para Testes](24.11-ia-para-testes.md)
- Próxima aula: [Coding Agents](24.13-coding-agents.md)
- Pré-requisitos: README, documentação mínima de projeto, requisitos, APIs e revisão técnica.

## Ideia Central

IA ajuda a documentar porque consegue reorganizar informação, resumir mudanças, sugerir estrutura e adaptar tom. Mas documentação precisa ser fiel ao sistema real.

O desenvolvedor deve fornecer contexto verificável e revisar nomes, contratos, exemplos, comandos, limites, versões e decisões. Uma documentação clara e errada é pior que uma documentação incompleta.

## Tópicos da Aula

- README, guias, ADRs, changelogs e docs de API.
- Resumo de pull requests.
- Documentação de comandos e configuração.
- Revisão de exemplos.
- Ligação com o módulo 25.

## Exemplo Prático

Peça para IA criar uma seção de README:

```text
Crie uma seção "Como rodar localmente" para um projeto Node.js.
Use apenas estes comandos:
- pnpm install
- pnpm dev
- pnpm test
Inclua variáveis obrigatórias sem inventar valores.
Se faltar informação, liste perguntas.
```

O pedido limita invenção e deixa claro que comandos não devem ser inventados.

## Analogias e Intuições

- IA como editora técnica.
- Documentação como contrato social do projeto.

## Erros Comuns

- Deixar IA inventar comandos.
- Documentar endpoint que não existe.
- Omitir limitações.
- Copiar explicação sem testar exemplo.
- Usar texto genérico que não ajuda o time.

## Checklist de Entendimento

- A documentação corresponde ao sistema real?
- Exemplos foram testados?
- Comandos existem?
- O texto explica limites e requisitos?
- Faltou citar decisão importante?

## Prática Recomendada

- Escolha um pequeno projeto.
- Gere uma seção de README com IA.
- Verifique cada comando e exemplo.
- Reescreva trechos genéricos com contexto real.

```text
- Descrição da aula: Esta aula estuda IA Para Documentação dentro do contexto de IA para desenvolvedores, conectando o conceito com uso prático, validação, segurança e aplicação em projetos JavaScript/TypeScript.
- Descrição do módulo: O módulo IA para Desenvolvedores ensina fundamentos de IA generativa, uso de IA no trabalho de desenvolvimento, coding agents, tool calling, agentes, RAG, avaliação, evals, guardrails, segurança, privacidade, observabilidade, LLMOps e custos.
- Itens já estudados: Aulas anteriores do módulo 24: Segurança em Código Gerado por IA; Quality Gates Para Código Gerado por IA; IA no Debug; IA Para Testes. Também entram como base TypeScript, APIs, testes, segurança, privacidade, observabilidade e system design.

Crie uma sessão de estudo com:
1. uma explicação curta do conceito;
2. 5 perguntas progressivas;
3. 3 exercícios práticos;
4. 1 desafio um pouco mais difícil;
5. exemplos de aplicação em projetos reais;
6. um checklist para eu avaliar se entendi.

Regras:
- Não entregue respostas completas antes de eu tentar responder.
- Corrija minhas respostas com explicações.
- Se eu errar, dê uma dica antes de revelar a solução.
- Ao final, sugira o próximo assunto que devo estudar.
```
