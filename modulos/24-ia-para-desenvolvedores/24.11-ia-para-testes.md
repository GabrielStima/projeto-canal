# IA Para Testes

## Objetivo

Ao final desta aula, você deve ser capaz de:

- usar IA para sugerir casos de teste;
- revisar testes gerados por IA;
- evitar testes que apenas confirmam a implementação;
- criar prompts para unitários, integração e bordas.

## Onde Esta Aula Entra na Formação

- Módulo: 24. IA para Desenvolvedores
- Aula anterior: [IA no Debug](24.10-ia-no-debug.md)
- Próxima aula: [IA Para Documentação](24.12-ia-para-documentacao.md)
- Pré-requisitos: estratégia de testes, pirâmide de testes, mocks, stubs, fakes e CI.

## Ideia Central

IA é boa para listar casos que você esqueceu, gerar rascunhos de teste e explicar cenários. Mas ela também pode criar testes fracos, acoplados à implementação ou cheios de mocks irreais.

O melhor uso é pedir testes a partir do comportamento esperado, não a partir do código como verdade absoluta.

## Tópicos da Aula

- Testes por comportamento.
- Casos felizes, bordas e erro.
- Testes unitários e integração.
- Mocks realistas.
- Revisão de testes gerados.

## Exemplo Prático

Para uma função `calculateDiscount`, peça:

```text
Liste casos de teste para esta regra:
- cupom percentual aplica desconto limitado a 30%;
- cupom expirado deve ser ignorado;
- pedido com total zero não recebe desconto;
- valores inválidos devem gerar erro.
Não escreva código ainda. Primeiro organize os cenários.
```

Depois gere os testes e revise se cada regra está coberta.

## Analogias e Intuições

- IA como checklist de casos esquecidos.
- Teste como especificação executável.

## Erros Comuns

- Gerar testes depois do código e aceitar cobertura superficial.
- Testar detalhes internos.
- Mockar tudo e não testar integração real.
- Não incluir casos de erro.
- Confiar que cobertura alta significa qualidade.

## Checklist de Entendimento

- O teste valida comportamento?
- Existem casos de borda?
- Existem caminhos de erro?
- Mocks são necessários e realistas?
- O teste falharia se o requisito fosse quebrado?

## Prática Recomendada

- Escolha uma regra de negócio.
- Peça primeiro uma matriz de casos de teste.
- Revise a matriz.
- Gere testes e ajuste manualmente.

```text
Estou estudando IA Para Testes em uma trilha de desenvolvimento fullstack JavaScript/TypeScript.

Meu objetivo é entender o conceito, praticar e revisar sem receber respostas prontas cedo demais.

Contexto da aula:
- Descrição da aula: Esta aula estuda IA Para Testes dentro do contexto de IA para desenvolvedores, conectando o conceito com uso prático, validação, segurança e aplicação em projetos JavaScript/TypeScript.
- Descrição do módulo: O módulo IA para Desenvolvedores ensina fundamentos de IA generativa, uso de IA no trabalho de desenvolvimento, coding agents, tool calling, agentes, RAG, avaliação, evals, guardrails, segurança, privacidade, observabilidade, LLMOps e custos.
- Itens já estudados: Aulas anteriores do módulo 24: Como Revisar Código Gerado por IA; Segurança em Código Gerado por IA; Quality Gates Para Código Gerado por IA; IA no Debug. Também entram como base TypeScript, APIs, testes, segurança, privacidade, observabilidade e system design.

Crie uma sessão de estudo com:
1. uma explicação curta do conceito;
2. 5 perguntas progressivas;
3. 3 exercícios práticos;
4. 1 desafio um pouco mais difícil;
5. exemplos de aplicação em projetos reais;
6. um checklist para eu avaliar se entendi.

Regras:
- Não entregue respostas completas antes de eu tentar responder.
- Corrija minhas respostas com explicações.
- Se eu errar, dê uma dica antes de revelar a solução.
- Ao final, sugira o próximo assunto que devo estudar.
```
