# Integração com APIs de IA em TypeScript

## Objetivo

Ao final desta aula, você deve ser capaz de:

- isolar um provedor atrás de uma interface;
- configurar autenticação e segredos sem expô-los;
- normalizar resposta, uso, erro e metadados;
- testar a integração com um adapter falso.

## Ideia Central

Chamar um modelo é uma integração externa: tem autenticação, contrato, latência, limites, custo e falhas. O domínio não deve conhecer detalhes do SDK escolhido.

Uma interface pequena permite trocar provedor, usar modelo local ou executar testes determinísticos sem alterar a regra de negócio.

## Tópicos da Aula

- Adapter de modelo.
- Configuração e secrets.
- Request, resposta e metadados de uso.
- Erros de transporte, provedor e validação.
- Test doubles.
- Versionamento de modelo e prompt.

## Exemplo Prático

```ts
type Geracao = {
  texto: string;
  modelo: string;
  tokensEntrada?: number;
  tokensSaida?: number;
};

interface ModeloTexto {
  gerar(entrada: {
    instrucao: string;
    contexto: string;
  }): Promise<Geracao>;
}
```

O serviço de domínio depende de `ModeloTexto`. Um adapter real traduz esse contrato para a API selecionada; um fake retorna cenários previsíveis para testar sucesso, recusa, timeout e resposta inválida.

## Analogias e Intuições

**Tomada e adaptador:** o aparelho usa uma interface estável; o adaptador resolve o padrão específico da rede elétrica.  
*Limite: provedores podem oferecer capacidades diferentes que não cabem em uma abstração única.*

## Erros Comuns

- Espalhar chamadas do SDK por controllers e regras de negócio.
- Registrar chave, prompt ou resposta sensível.
- Ignorar versão do modelo na telemetria.
- Fazer testes dependerem de uma API paga e instável.
- Criar uma abstração tão genérica que esconde capacidades necessárias.

## Checklist de Entendimento

- Onde fica o adapter?
- Como o segredo é fornecido?
- Que metadados são normalizados?
- Quais erros chegam ao domínio?
- Como testar sem chamada externa?

## Prática Recomendada

- Implemente a interface com um fake determinístico e prove que o domínio não depende de um SDK específico.
