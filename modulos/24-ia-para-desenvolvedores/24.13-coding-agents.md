# Coding Agents

## Objetivo

Ao final desta aula, você deve ser capaz de:

- explicar o que diferencia um coding agent de um chat comum;
- entender como agentes planejam, editam arquivos e executam comandos;
- definir limites de permissão e escopo;
- revisar o trabalho de um agente com critério.

## Onde Esta Aula Entra na Formação

- Módulo: 24. IA para Desenvolvedores
- Aula anterior: [IA Para Documentação](24.12-ia-para-documentacao.md)

## Ideia Central

Um coding agent é uma ferramenta de IA capaz de agir sobre um ambiente de desenvolvimento: ler arquivos, propor planos, editar código, rodar testes, analisar erros e iterar. Ele não apenas responde; ele executa parte do fluxo.

Isso aumenta produtividade e também risco. O agente precisa de tarefa pequena, contexto claro, permissão limitada, verificação real e revisão humana.

## Tópicos da Aula

- Chat vs agente.
- Ferramentas: filesystem, terminal, busca, browser e testes.
- Planejamento, edição, verificação e resumo.
- Escopo de tarefa.
- Riscos de autonomia.

## Exemplo Prático

Tarefa boa para agente:

```text
Adicione validação para page e pageSize nesta rota.
Mantenha o padrão atual do projeto.
Crie ou ajuste testes.
Não altere autenticação, banco de dados ou estrutura de pastas.
Mostre o diff e os comandos de verificação usados.
```

O pedido delimita o que pode mudar e como validar.

## Analogias e Intuições

- Coding agent como par que também mexe no teclado.
- Permissões do agente como permissões de produção: devem ser mínimas.

## Erros Comuns

- Dar tarefa grande demais.
- Não limitar arquivos ou comportamento esperado.
- Deixar agente rodar comandos destrutivos sem revisão.
- Aceitar resumo sem olhar diff.
- Confundir testes rodados com testes suficientes.

## Checklist de Entendimento

- Qual é a diferença entre chat e agente?
- Que ferramentas o agente pode usar?
- Qual é o escopo da tarefa?
- Como revisar as alterações?
- Que comandos provam que a mudança funciona?

## Prática Recomendada

- Pegue uma tarefa pequena de manutenção.
- Escreva um prompt com escopo, restrições e verificação.
- Liste comandos que o agente poderia rodar.
- Defina o que exigiria aprovação humana.

## Prompt Para Estudar Com IA

```text
Estou estudando Coding Agents em uma trilha de desenvolvimento fullstack JavaScript/TypeScript.

Meu objetivo é entender o conceito, praticar e revisar sem receber respostas prontas cedo demais.

Contexto da aula:
- Descrição da aula: Esta aula estuda Coding Agents dentro do contexto de IA para desenvolvedores, conectando o conceito com uso prático, validação, segurança e aplicação em projetos JavaScript/TypeScript.
- Descrição do módulo: O módulo IA para Desenvolvedores ensina fundamentos de IA generativa, uso de IA no trabalho de desenvolvimento, coding agents, tool calling, agentes, RAG, avaliação, evals, guardrails, segurança, privacidade, observabilidade, LLMOps e custos.
- Itens já estudados: Aulas anteriores do módulo 24: Quality Gates Para Código Gerado por IA; IA no Debug; IA Para Testes; IA Para Documentação. Também entram como base TypeScript, APIs, testes, segurança, privacidade, observabilidade e system design.

Crie uma sessão de estudo com:
1. uma explicação curta do conceito;
2. 5 perguntas progressivas;
3. 3 exercícios práticos;
4. 1 desafio um pouco mais difícil;
5. exemplos de aplicação em projetos reais;
6. um checklist para eu avaliar se entendi.

Regras:
- Não entregue respostas completas antes de eu tentar responder.
- Corrija minhas respostas com explicações.
- Se eu errar, dê uma dica antes de revelar a solução.
- Ao final, sugira o próximo assunto que devo estudar.
```
