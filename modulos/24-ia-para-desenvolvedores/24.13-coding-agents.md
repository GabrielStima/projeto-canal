# Coding Agents

## Objetivo

Ao final desta aula, você deve ser capaz de:

- explicar o que diferencia um coding agent de um chat comum;
- entender como agentes planejam, editam arquivos e executam comandos;
- definir limites de permissão e escopo;
- revisar o trabalho de um agente com critério.

## Ideia Central

Um coding agent é uma ferramenta de IA capaz de agir sobre um ambiente de desenvolvimento: ler arquivos, propor planos, editar código, rodar testes, analisar erros e iterar. Ele não apenas responde; ele executa parte do fluxo.

Isso aumenta produtividade e também risco. O agente precisa de tarefa pequena, contexto claro, permissão limitada, verificação real e revisão humana.

## Tópicos da Aula

- Ferramentas: filesystem, terminal, busca, browser e testes.
- Planejamento, edição, verificação e resumo.
- Escopo de tarefa.
- Riscos de autonomia.

## Exemplo Prático

O Agente Engenheiro (Devin / Cursor):

O próximo passo não é autocompletar. É *Autonomia*.
O Dev manda no terminal pro Coding Agent: *'Temos um bug no botão de Agendar. Navegue nos nossos 300 arquivos do repositório, ache o componente React, edite a cor pra vermelho, faça o commit, rode os testes de integração local e, se passar, crie um Pull Request no GitHub.'*
O Coding Agent age em Loop (Thought/Action/Observation) sozinho, navegando no computador do dev, abrindo arquivos, rodando bash e abrindo a PR. É um engenheiro virtual vivo.

## Analogias e Intuições

- Copilot autocompletando é um GPS no painel do carro dizendo 'vire à direita'. Um Coding Agent (Ex: Devin) é um Tesla no piloto automático 100% Full-Self-Driving rodando pela rodovia e estacionando por você.

## Erros Comuns

- Dar tarefa grande demais.
- Não limitar arquivos ou comportamento esperado.
- Deixar agente rodar comandos destrutivos sem revisão.
- Aceitar resumo sem olhar diff.
- Confundir testes rodados com testes suficientes.

## Checklist de Entendimento

- Qual é a diferença entre chat e agente?
- Que ferramentas o agente pode usar?
- Qual é o escopo da tarefa?
- Como revisar as alterações?
- Que comandos provam que a mudança funciona?

## Prática Recomendada

- Descreva a barreira de *Trust (Confiança)* num ecossistema onde Coding Agents possuem permissões de execução (RCE) no seu terminal nativo de desenvolvedor.

```text
Estou estudando Coding Agents (Engenharia Autônoma) em uma trilha de desenvolvimento fullstack JavaScript/TypeScript.

Meu objetivo é entender o conceito, praticar e revisar sem receber respostas prontas cedo demais.

- Descrição da aula: A aula introduz a Era Agentic. IAs deixam de ser papagaios passivos de texto e viram braços robóticos com loop recursivo (Thought, Action) no mundo real.
- Descrição do modulo: O módulo introduz as Redes Neurais generativas, RAG, Tool Callings e Proteções (Guardrails) de LLMs de uma ótica pragmática e Arquitetural.
- Itens já estudados: System Design, CI/CD, Observabilidade, Segurança (Secure SDLC) e Prompting Básico.

Crie uma sessão de estudo com:
1. uma explicação curta do conceito;
2. 5 perguntas progressivas;
3. 3 exercícios práticos;
4. 1 desafio um pouco mais difícil;
5. exemplos de aplicação em projetos reais;
6. um checklist para eu avaliar se entendi.

Regras:
- Não entregue respostas completas antes de eu tentar responder.
- Corrija minhas respostas com explicações.

- Ao final, sugira o próximo assunto que devo estudar.
```
