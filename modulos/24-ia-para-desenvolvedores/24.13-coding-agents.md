# Coding Agents

## Objetivo

Ao final desta aula, voce deve ser capaz de:

- explicar o que diferencia um coding agent de um chat comum;
- entender como agentes planejam, editam arquivos e executam comandos;
- definir limites de permissao e escopo;
- revisar o trabalho de um agente com criterio.

## Onde Esta Aula Entra no Curso

- Modulo: 24. IA para Desenvolvedores
- Aula anterior: [IA Para Documentacao](24.12-ia-para-documentacao.md)
- Proxima aula: [Workflows com Agentes](24.14-workflows-com-agentes.md)
- Pre-requisitos: Git, terminal, testes, code review, seguranca e quality gates.

## Ideia Central

Um coding agent e uma ferramenta de IA capaz de agir sobre um ambiente de desenvolvimento: ler arquivos, propor planos, editar codigo, rodar testes, analisar erros e iterar. Ele nao apenas responde; ele executa parte do fluxo.

Isso aumenta produtividade e tambem risco. O agente precisa de tarefa pequena, contexto claro, permissao limitada, verificacao real e revisao humana.

## Topicos da Aula

- Chat vs agente.
- Ferramentas: filesystem, terminal, busca, browser e testes.
- Planejamento, edicao, verificacao e resumo.
- Escopo de tarefa.
- Riscos de autonomia.

## Exemplo Pratico

Tarefa boa para agente:

```text
Adicione validacao para page e pageSize nesta rota.
Mantenha o padrao atual do projeto.
Crie ou ajuste testes.
Nao altere autenticacao, banco de dados ou estrutura de pastas.
Mostre o diff e os comandos de verificacao usados.
```

O pedido delimita o que pode mudar e como validar.

## Analogias e Intuicoes

- Coding agent como par que tambem mexe no teclado.
- Permissoes do agente como permissoes de producao: devem ser minimas.

## Erros Comuns

- Dar tarefa grande demais.
- Nao limitar arquivos ou comportamento esperado.
- Deixar agente rodar comandos destrutivos sem revisao.
- Aceitar resumo sem olhar diff.
- Confundir testes rodados com testes suficientes.

## Checklist de Entendimento

- Qual e a diferenca entre chat e agente?
- Que ferramentas o agente pode usar?
- Qual e o escopo da tarefa?
- Como revisar as alteracoes?
- Que comandos provam que a mudanca funciona?

## Pratica Recomendada

- Pegue uma tarefa pequena de manutencao.
- Escreva um prompt com escopo, restricoes e verificacao.
- Liste comandos que o agente poderia rodar.
- Defina o que exigiria aprovacao humana.

## Prompt Para Estudar Com IA

```text
Estou estudando Coding Agents em uma trilha de desenvolvimento fullstack JavaScript/TypeScript.

Meu objetivo é entender o conceito, praticar e revisar sem receber respostas prontas cedo demais.

Contexto da aula:
- Descrição da aula: Esta aula estuda Coding Agents dentro do contexto de IA para desenvolvedores, conectando o conceito com uso pratico, validacao, seguranca e aplicacao em projetos JavaScript/TypeScript.
- Descrição do modulo: O modulo IA para Desenvolvedores ensina fundamentos de IA generativa, uso de IA no trabalho de desenvolvimento, coding agents, tool calling, agentes, RAG, avaliacao, evals, guardrails, seguranca, privacidade, observabilidade, LLMOps e custos.
- Itens já estudados: Aulas anteriores do modulo 24: Quality Gates Para Codigo Gerado por IA; IA no Debug; IA Para Testes; IA Para Documentacao. Tambem entram como base TypeScript, APIs, testes, seguranca, privacidade, observabilidade e system design.

Crie uma sessão de estudo com:
1. uma explicação curta do conceito;
2. 5 perguntas progressivas;
3. 3 exercícios práticos;
4. 1 desafio um pouco mais difícil;
5. exemplos de aplicação em projetos reais;
6. um checklist para eu avaliar se entendi.

Regras:
- Não entregue respostas completas antes de eu tentar responder.
- Corrija minhas respostas com explicações.
- Se eu errar, dê uma dica antes de revelar a solução.
- Ao final, sugira o próximo assunto que devo estudar.
```
