# IA no Debug

## Objetivo

Ao final desta aula, você deve ser capaz de:

- usar IA para levantar hipóteses de debug;
- fornecer contexto seguro e suficiente;
- validar hipóteses com reprodução e testes;
- evitar que IA esconda a causa raiz.

## Onde Esta Aula Entra na Formação

- Módulo: 24. IA para Desenvolvedores
- Aula anterior: [Quality Gates Para Código Gerado por IA](24.09-quality-gates-para-codigo-gerado-por-ia.md)
- Próxima aula: [IA Para Testes](24.11-ia-para-testes.md)
- Pré-requisitos: tratamento de erros, logs, testes, observabilidade e debugging básico.

## Ideia Central

IA ajuda muito no debug quando você oferece sintomas, contexto, hipóteses já testadas e limites. O objetivo não é pedir uma resposta mágica, mas acelerar investigação.

Debug bom continua sendo sistemático: reproduzir, isolar, observar, formular hipótese, testar, corrigir e prevenir regressão.

## Tópicos da Aula

- Descrever sintomas.
- Separar fato de hipótese.
- Reduzir contexto com segurança.
- Pedir plano de investigação.
- Confirmar causa raiz.

## Exemplo Prático

Prompt útil:

```text
Estou investigando um erro intermitente em uma API Node.js.
Fatos:
- endpoint: POST /orders
- ocorre apenas quando há cupom
- erro: Cannot read properties of undefined (reading 'amount')
- teste unitário de order total passa
- teste de integração falha com cupom expirado

Liste hipóteses ordenadas por probabilidade e diga como testar cada uma.
```

## Analogias e Intuições

- IA como parceira de investigação.
- Debug como método científico aplicado a software.

## Erros Comuns

- Colar log com segredos.
- Pedir correção antes de reproduzir.
- Aceitar a primeira hipótese.
- Corrigir sintoma e não causa.
- Não criar teste de regressão.

## Checklist de Entendimento

- O bug foi reproduzido?
- Quais fatos foram observados?
- Quais hipóteses foram testadas?
- A causa raiz foi confirmada?
- Existe teste para evitar regressão?

## Prática Recomendada

- Pegue um bug fictício.
- Escreva fatos, hipóteses e experimentos.
- Use IA para sugerir mais hipóteses.
- Escolha como confirmar ou descartar cada uma.
