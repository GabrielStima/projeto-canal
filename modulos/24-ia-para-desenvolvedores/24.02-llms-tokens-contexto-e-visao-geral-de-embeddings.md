# LLMs, Tokens, Contexto e Visão Geral de Embeddings

## Objetivo

Ao final desta aula, você deve ser capaz de:

- explicar o que é um LLM;
- entender tokens e janela de contexto;
- reconhecer por que contexto influencia qualidade, custo e latência;
- ter uma primeira intuição sobre embeddings.

## Ideia Central

LLM significa Large Language Model: um modelo treinado para processar e gerar linguagem. Para interagir com ele, você envia texto, instruções, histórico e dados relevantes. O modelo transforma esse material em unidades chamadas tokens e produz uma continuação provável.

A janela de contexto é o limite de material que o modelo consegue considerar em uma interação. Contexto demais aumenta custo e pode diluir o foco. Contexto de menos deixa o modelo sem informação importante.

Embeddings são representações numéricas de significado. Neste momento, basta entender que eles permitem comparar textos por proximidade semântica. O aprofundamento vem mais adiante.

## Exemplo Prático

Compare dois prompts:

```text
Explique este erro.
```

```text
Explique este erro em uma API Node.js.
Contexto:
- rota: POST /orders
- erro: Cannot read properties of undefined (reading 'id')
- trecho relevante: const userId = req.user.id
- autenticação usa middleware antes da rota
```

O segundo prompt tem mais contexto útil. Mas se você colar o projeto inteiro, o modelo pode ficar mais caro, lento e menos focado.

## Analogias e Intuições

- Tokens como pedaços de texto que o modelo consegue processar.
- Janela de contexto como a mesa de trabalho: grande ajuda, mas bagunça atrapalha.
- Embeddings como coordenadas aproximadas de significado.

## Erros Comuns

- Achar que mais contexto sempre melhora a resposta.
- Colar segredos no prompt.
- Confundir histórico da conversa com memória confiável.
- Usar embeddings como se fossem busca exata.

## Checklist de Entendimento

- O que é um LLM?
- O que são tokens?
- Por que contexto custa dinheiro?
- Quando contexto demais atrapalha?
- Qual é a intuição inicial de embeddings?

## Prática Recomendada

- Pegue uma dúvida técnica real.
- Escreva um prompt sem contexto e outro com contexto mínimo suficiente.
- Compare as respostas.
- Identifique quais informações melhoraram a resposta.
