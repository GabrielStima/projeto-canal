# Vector Databases

## Objetivo

Ao final desta aula, voce deve ser capaz de:

- explicar o papel de um vector database;
- entender busca por similaridade;
- relacionar embeddings, metadados e filtros;
- reconhecer trade-offs de armazenamento, custo e atualizacao.

## Onde Esta Aula Entra no Curso

- Modulo: 24. IA para Desenvolvedores
- Aula anterior: [Embeddings](24.19-embeddings.md)
- Proxima aula: [RAG](24.21-rag.md)
- Pre-requisitos: bancos de dados, indexes, busca, embeddings e governanca de dados.

## Ideia Central

Vector database e um banco ou recurso especializado para armazenar vetores e buscar itens proximos por similaridade. Em aplicacoes com IA, ele costuma guardar embeddings de documentos, trechos, produtos, mensagens ou codigo.

Na pratica, a busca vetorial quase sempre precisa de metadados: origem, permissao, data, idioma, versao, tipo de documento e tenant. Similaridade sozinha nao basta para uma aplicacao segura.

## Topicos da Aula

- Armazenamento de vetores.
- Indices de similaridade.
- Metadados e filtros.
- Atualizacao e reindexacao.
- Multitenancy e permissao.

## Exemplo Pratico

Registro indexado:

```json
{
  "id": "doc_123_chunk_04",
  "embedding": "[...]",
  "text": "Para renovar um token expirado...",
  "metadata": {
    "module": "auth",
    "visibility": "internal",
    "updatedAt": "2026-06-01"
  }
}
```

Na busca, filtre por permissao antes de entregar contexto ao modelo.

## Analogias e Intuicoes

- Vector database como indice semantico.
- Metadados como porteiro da busca.

## Erros Comuns

- Usar busca vetorial sem controle de acesso.
- Ignorar versao dos documentos.
- Retornar muitos chunks irrelevantes.
- Nao medir qualidade da recuperacao.
- Tratar vector database como substituto completo de banco relacional.

## Checklist de Entendimento

- O que sera vetorizado?
- Quais metadados sao obrigatorios?
- Como permissao sera aplicada?
- Como documentos serao atualizados?
- Como medir se a busca encontrou bons trechos?

## Pratica Recomendada

- Modele uma colecao vetorial para aulas do curso.
- Defina metadados minimos.
- Escreva tres consultas e filtros esperados.
- Liste riscos de privacidade e permissao.
