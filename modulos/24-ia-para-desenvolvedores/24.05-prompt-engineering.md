# Prompt Engineering

## Objetivo

Ao final desta aula, você deve ser capaz de:

- escrever prompts claros para tarefas técnicas;
- separar objetivo, contexto, restrições e formato de saída;
- usar exemplos e critérios de aceite;
- revisar prompts quando a resposta vier ruim.

## Ideia Central

Prompt engineering é a prática de formular uma interação para aumentar a chance de uma resposta útil. Um bom prompt não é um truque mágico. Ele deixa explícito o que você quer, qual contexto importa, quais limites existem e como a resposta será avaliada.

Para desenvolvimento, bons prompts parecem bons tickets: objetivo claro, contexto mínimo, restrições, exemplos, critérios de aceite e pedido de incertezas.

## Tópicos da Aula

- Objetivo da tarefa.
- Contexto suficiente e seguro.
- Restrições técnicas.
- Formato de resposta.
- Exemplos, contraexemplos e critérios de aceite.
- Iteração: melhorar prompt com base na resposta.

## Exemplo Prático

O Comando Certo:

Um Dev Junior manda: 'Faz um código pra cadastrar cachorro'. A IA cospe uma massaroca ilegível em Python.
O Dev Sênior (Prompt Engineering) manda:
```
Contexto: Você é um Sênior de TypeScript Node.js.
Tarefa: Crie uma função assíncrona para inserir cães no Prisma ORM.
Regras:
- Não gere explicações inúteis, me dê só o código.
- Use Padrões SOLID e Early Returns.
- Tipagem estrita é obrigatória.
```
A IA devolve uma obra de arte cirúrgica, pronta para ir pra produção.

## Analogias e Intuições

- Prompt Engineering é delegar tarefas pro pedreiro. Se você disser 'Faz uma casa', ele faz de qualquer jeito. Se você der a planta, a marca do cimento e proibir o uso de tijolo furado, a casa sai um castelo.

## Erros Comuns

- Pedir uma tarefa ampla demais.
- Omitir restrições importantes.
- Misturar várias tarefas independentes.
- Pedir resposta pronta quando o objetivo é estudar.
- Não explicar como a resposta será usada.

## Checklist de Entendimento

- Qual é o objetivo?
- Que contexto é realmente necessário?
- Quais limites a IA deve respeitar?
- Que formato ajuda a revisar a resposta?
- Como vou validar o resultado?

## Prática Recomendada

- Defina o poder prático das táticas *Few-Shot Prompting* (Dar 2 exemplos antes de pedir) e *Chain-of-Thought* (Pedir para a IA pensar passo a passo).

## Prompt Para Estudar Com IA

```text
Estou estudando Prompt Engineering em uma trilha de desenvolvimento fullstack JavaScript/TypeScript.

Meu objetivo é entender o conceito, praticar e revisar sem receber respostas prontas cedo demais.

Contexto da aula:
- Descrição da aula: A aula transforma o desenvolvedor num diretor implacável de orquestras (Prompt Engineer), utilizando técnicas documentadas para extrair o máximo do LLM no mínimo de Tokens.
- Descrição do modulo: O módulo introduz as Redes Neurais generativas, RAG, Tool Callings e Proteções (Guardrails) de LLMs de uma ótica pragmática e Arquitetural.
- Itens já estudados: System Design, CI/CD, Observabilidade, Segurança (Secure SDLC) e Prompting Básico.

Crie uma sessão de estudo com:
1. uma explicação curta do conceito;
2. 5 perguntas progressivas;
3. 3 exercícios práticos;
4. 1 desafio um pouco mais difícil;
5. exemplos de aplicação em projetos reais;
6. um checklist para eu avaliar se entendi.

Regras:
- Não entregue respostas completas antes de eu tentar responder.
- Corrija minhas respostas com explicações.
- Se eu errar, dê uma dica antes de revelar a solução.
- Ao final, sugira o próximo assunto que devo estudar.
```
