# Function Calling e Tool Calling

## Objetivo

Ao final desta aula, voce deve ser capaz de:

- explicar function calling e tool calling;
- entender por que modelos precisam de ferramentas;
- desenhar contratos simples para ferramentas;
- validar entradas, saidas e permissoes.

## Onde Esta Aula Entra no Curso

- Modulo: 24. IA para Desenvolvedores
- Aula anterior: [Workflows com Agentes](24.14-workflows-com-agentes.md)
- Proxima aula: [Agentes com Ferramentas](24.16-agentes-com-ferramentas.md)
- Pre-requisitos: APIs, contratos, validacao, autorizacao e tratamento de erros.

## Ideia Central

Modelos de linguagem geram texto. Quando precisam buscar dados, calcular, consultar sistemas ou executar acoes, eles dependem de ferramentas. Function calling ou tool calling e o padrao em que o modelo escolhe uma ferramenta e produz argumentos estruturados para ela.

A ferramenta deve ser tratada como API: contrato claro, validacao forte, autorizacao, logs seguros e limites de uso.

## Topicos da Aula

- Modelo decide ou sugere chamada.
- Ferramenta executa fora do modelo.
- Schema de parametros.
- Validacao de argumentos.
- Permissoes, auditoria e erros.

## Exemplo Pratico

Ferramenta conceitual:

```ts
type GetOrderInput = {
  orderId: string;
};

type GetOrderOutput = {
  id: string;
  status: "pending" | "paid" | "cancelled";
};
```

Antes de executar, valide `orderId`, confira permissao do usuario, limite campos retornados e registre a chamada sem dados sensiveis.

## Analogias e Intuicoes

- Tool calling como uma API que a IA pode solicitar.
- Schema como trilho para reduzir ambiguidade.

## Erros Comuns

- Deixar a ferramenta aceitar argumentos livres demais.
- Executar acao sem confirmacao quando ha risco.
- Retornar dados demais para o modelo.
- Confiar que o modelo validou entrada.
- Omitir auditoria.

## Checklist de Entendimento

- Qual ferramenta existe?
- Quem pode chama-la?
- Quais argumentos sao aceitos?
- Que dados ela retorna?
- Como erro, auditoria e limite sao tratados?

## Pratica Recomendada

- Defina tres ferramentas para um assistente de pedidos.
- Escreva entrada, saida, permissao e riscos de cada uma.
- Identifique qual ferramenta exige confirmacao humana.
