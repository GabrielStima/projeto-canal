# Function Calling e Tool Calling

## Objetivo

Ao final desta aula, você deve ser capaz de:

- explicar function calling e tool calling;
- entender por que modelos precisam de ferramentas;
- desenhar contratos simples para ferramentas;
- validar entradas, saídas e permissões.

## Ideia Central

Modelos de linguagem geram texto. Quando precisam buscar dados, calcular, consultar sistemas ou executar ações, eles dependem de ferramentas. Function calling ou tool calling é o padrão em que o modelo escolhe uma ferramenta e produz argumentos estruturados para ela.

A ferramenta deve ser tratada como API: contrato claro, validação forte, autorização, logs seguros e limites de uso.

## Tópicos da Aula

- Modelo decide ou sugere chamada.
- Ferramenta executa fora do modelo.
- Schema de parâmetros.
- Validação de argumentos.
- Permissões, auditoria e erros.

## Exemplo Prático

Ferramenta conceitual:

```ts
type GetOrderInput = {
  orderId: string;
};

type GetOrderOutput = {
  id: string;
  status: "pending" | "paid" | "cancelled";
};
```

Antes de executar, valide `orderId`, confira permissão do usuário, limite campos retornados e registre a chamada sem dados sensíveis.

## Analogias e Intuições

- Tool calling como uma API que a IA pode solicitar.
- Schema como trilho para reduzir ambiguidade.

## Erros Comuns

- Deixar a ferramenta aceitar argumentos livres demais.
- Executar ação sem confirmação quando há risco.
- Retornar dados demais para o modelo.
- Confiar que o modelo validou entrada.
- Omitir auditoria.

## Checklist de Entendimento

- Qual ferramenta existe?
- Quem pode chamá-la?
- Quais argumentos são aceitos?
- Que dados ela retorna?
- Como erro, auditoria e limite são tratados?

## Prática Recomendada

- Defina três ferramentas para um assistente de pedidos.
- Escreva entrada, saída, permissão e riscos de cada uma.
- Identifique qual ferramenta exige confirmação humana.
