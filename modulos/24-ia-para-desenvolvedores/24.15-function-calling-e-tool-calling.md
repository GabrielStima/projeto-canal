# Function Calling e Tool Calling

## Objetivo

Ao final desta aula, você deve ser capaz de:

- explicar function calling e tool calling;
- entender por que modelos precisam de ferramentas;
- desenhar contratos simples para ferramentas;
- validar entradas, saídas e permissões.

- Módulo: 24. IA para Desenvolvedores

- Próxima aula: [Agentes com Ferramentas](24.16-agentes-com-ferramentas.md)

## Ideia Central

Modelos de linguagem geram texto. Quando precisam buscar dados, calcular, consultar sistemas ou executar ações, eles dependem de ferramentas. Function calling ou tool calling é o padrão em que o modelo escolhe uma ferramenta e produz argumentos estruturados para ela.

A ferramenta deve ser tratada como API: contrato claro, validação forte, autorização, logs seguros e limites de uso.

## Tópicos da Aula

- Modelo decide ou sugere chamada.
- Ferramenta executa fora do modelo.
- Schema de parâmetros.
- Validação de argumentos.
- Permissões, auditoria e erros.

## Exemplo Prático

Dando Braços ao Cérebro (Tool Calling):

Você faz um chatbot pra clínica: 'Qual a temperatura hoje em São Paulo?'
O ChatGPT é preso no passado (Treinamento de 2023). Ele não sabe.
Mas usando `Function Calling` na sua API Node, você programa ele: 'Se alguém perguntar de clima, chame essa função X'.
O ChatGPT recebe a pergunta, responde pra sua API: 'Eu não sei, por favor chame a sua função externa X com parâmetro (São Paulo)'. A sua API vai numa API Meteorológica, pega os graus e devolve pro ChatGPT montar o texto final bonito. A IA acabou de interagir fisicamente com o mundo externo.

## Analogias e Intuições

- IA pura é um gênio amarrado numa cadeira num quarto fechado sem internet. Tool Calling é dar um Walkie-Talkie pro gênio mandar os seguranças dele lá fora buscarem informações frescas pra ele.

## Erros Comuns

- Deixar a ferramenta aceitar argumentos livres demais.
- Executar ação sem confirmação quando há risco.
- Retornar dados demais para o modelo.
- Confiar que o modelo validou entrada.
- Omitir auditoria.

## Checklist de Entendimento

- Qual ferramenta existe?
- Quem pode chamá-la?
- Quais argumentos são aceitos?
- Que dados ela retorna?
- Como erro, auditoria e limite são tratados?

## Prática Recomendada

- Detalhe o formato estrutural que a OpenAI exige (JSON Schema) para que você consiga 'explicar' as suas funções de Node.js corporativas para o LLM aprender a chamá-las.
