# Workflows com Agentes

## Objetivo

Ao final desta aula, voce deve ser capaz de:

- desenhar fluxos de trabalho com agentes;
- separar planejamento, implementacao, verificacao e revisao;
- escolher quando usar agente e quando trabalhar manualmente;
- definir pontos de controle no fluxo.

## Onde Esta Aula Entra no Curso

- Modulo: 24. IA para Desenvolvedores
- Aula anterior: [Coding Agents](24.13-coding-agents.md)
- Proxima aula: [Function Calling e Tool Calling](24.15-function-calling-e-tool-calling.md)
- Pre-requisitos: Git, CI, testes, code review, requisitos e quality gates.

## Ideia Central

Um bom workflow com agente nao e "solte o agente e espere". E uma sequencia controlada: entender tarefa, planejar, editar em passos pequenos, verificar, revisar diff, registrar riscos e decidir o proximo passo.

Agentes funcionam melhor quando o fluxo cria feedback rapido e evita autonomia invisivel.

## Topicos da Aula

- Fluxo para bugfix.
- Fluxo para feature pequena.
- Fluxo para refatoracao.
- Checkpoints humanos.
- Registro de verificacao.

## Exemplo Pratico

Workflow para bug:

1. reproduzir o erro;
2. pedir ao agente hipoteses;
3. escolher uma hipotese;
4. criar teste de regressao;
5. implementar mudanca pequena;
6. rodar teste;
7. revisar diff;
8. documentar causa raiz.

## Analogias e Intuicoes

- Workflow como trilho: acelera sem deixar sair do caminho.
- Checkpoint como code review antecipado.

## Erros Comuns

- Pular reproducao do bug.
- Pedir feature inteira em uma unica rodada.
- Deixar agente refatorar fora do escopo.
- Nao registrar comandos executados.
- Ignorar falhas de verificacao.

## Checklist de Entendimento

- Qual e a etapa atual do trabalho?
- O agente recebeu contexto suficiente?
- O escopo esta limitado?
- Existe verificacao em cada etapa?
- A revisao humana acontece antes da integracao?

## Pratica Recomendada

- Desenhe um workflow para bugfix com agente.
- Desenhe um workflow para nova rota de API.
- Marque onde entram testes, revisao e aprovacao.
