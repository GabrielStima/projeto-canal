# Workflows com Agentes

## Objetivo

Ao final desta aula, você deve ser capaz de:

- desenhar fluxos de trabalho com agentes;
- separar planejamento, implementação, verificação e revisão;
- escolher quando usar agente e quando trabalhar manualmente;
- definir pontos de controle no fluxo.

## Onde Esta Aula Entra na Formação

- Módulo: 24. IA para Desenvolvedores
- Aula anterior: [Coding Agents](24.13-coding-agents.md)
- Próxima aula: [Function Calling e Tool Calling](24.15-function-calling-e-tool-calling.md)
- Pré-requisitos: Git, CI, testes, code review, requisitos e quality gates.

## Ideia Central

Um bom workflow com agente não é "solte o agente e espere". É uma sequência controlada: entender tarefa, planejar, editar em passos pequenos, verificar, revisar diff, registrar riscos e decidir o próximo passo.

Agentes funcionam melhor quando o fluxo cria feedback rápido e evita autonomia invisível.

## Tópicos da Aula

- Fluxo para bugfix.
- Fluxo para feature pequena.
- Fluxo para refatoração.
- Checkpoints humanos.
- Registro de verificação.

## Exemplo Prático

Workflow para bug:

1. reproduzir o erro;
2. pedir ao agente hipóteses;
3. escolher uma hipótese;
4. criar teste de regressão;
5. implementar mudança pequena;
6. rodar teste;
7. revisar diff;
8. documentar causa raiz.

## Analogias e Intuições

- Workflow como trilho: acelera sem deixar sair do caminho.
- Checkpoint como code review antecipado.

## Erros Comuns

- Pular reprodução do bug.
- Pedir feature inteira em uma única rodada.
- Deixar agente refatorar fora do escopo.
- Não registrar comandos executados.
- Ignorar falhas de verificação.

## Checklist de Entendimento

- Qual é a etapa atual do trabalho?
- O agente recebeu contexto suficiente?
- O escopo está limitado?
- Existe verificação em cada etapa?
- A revisão humana acontece antes da integração?

## Prática Recomendada

- Desenhe um workflow para bugfix com agente.
- Desenhe um workflow para nova rota de API.
- Marque onde entram testes, revisão e aprovação.
