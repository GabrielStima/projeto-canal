# Load Balancers

## Objetivo

Ao final desta aula, você deve ser capaz de:

- explicar o papel de um load balancer;
- entender distribuição de tráfego entre instâncias;
- reconhecer health checks e remoção de instâncias ruins;
- diferenciar balanceamento simples de alta disponibilidade.

## Ideia Central

Load balancer distribui tráfego entre múltiplas instâncias ou destinos. Ele permite escalar horizontalmente, retirar instâncias com problema, fazer deploys graduais e reduzir o risco de depender de um único processo.

Em system design, load balancer costuma ficar entre a borda do sistema e a camada de aplicação. Ele não torna o sistema automaticamente resiliente, mas cria uma base para redundância.

O desenho precisa considerar protocolo, health checks, sessões, sticky sessions, TLS, timeouts e comportamento em falhas.

## Exemplo Prático

Uma API roda em três instâncias. O load balancer envia requisições para as três. Se uma instância falha no health check, ela sai da rotação.

Se a aplicação guarda sessão em memória local, o usuário pode perder estado ao cair em outra instância. Por isso, aplicações stateless ou sessões externas costumam escalar melhor.

## Analogias e Intuições

- Load balancer é uma portaria que distribui pessoas entre atendentes disponíveis.
- Health check é perguntar se o atendente ainda consegue trabalhar.
- Escalar instâncias não resolve gargalo no banco.

## Erros Comuns

- Manter estado local em aplicação que precisa escalar.
- Health check superficial que responde mesmo com dependência crítica quebrada.
- Timeouts maiores que o aceitável para o usuário.
- Achar que load balancer protege contra todos os gargalos.
- Não testar retirada e volta de instâncias.

## Checklist de Entendimento

Antes de seguir, a pessoa deve conseguir responder:

- O que acontece quando uma instância cai?
- O health check mede o que importa?
- A aplicação é stateless?
- Existe dependência compartilhada que continua sendo gargalo?
- Como deploys passam pelo load balancer?

## Prática Recomendada

- Desenhe uma API com três instâncias atrás de um load balancer.
- Marque onde ficam sessões, logs e conexão com banco.
- Explique o que acontece se uma instância começar a responder lentamente.
