# PoSA e Padroes Alem do GoF

## Objetivo

Ao final desta aula, voce deve ser capaz de:

- entender que existem padroes alem do GoF;
- reconhecer PoSA como literatura de padroes arquiteturais e de sistemas;
- diferenciar padroes de design local e padroes arquiteturais;
- preparar a transicao para arquitetura de software.

## Onde Esta Aula Entra no Curso

- Modulo: 14. Design de Software
- Aula anterior: GoF: Padroes Criacionais, Estruturais e Comportamentais
- Proxima aula: Projeto Pratico: Refatorando uma API para Melhor Design
- Pre-requisitos: Padroes de design, principios de design, backend, mensageria introdutoria e testabilidade.

## Ideia Central

PoSA e outras familias de padroes ampliam a conversa para estruturas maiores: camadas, brokers, pipes, microkernel, concorrencia e sistemas distribuidos. Esses assuntos encostam no modulo 15, mas aqui entram apenas como mapa.

O limite importante: neste modulo, padroes servem para design de codigo e transicao conceitual. Arquitetura sera aprofundada depois.

## Topicos da Aula

- GoF vs PoSA.
- Padroes locais.
- Padroes arquiteturais.
- Sistemas e infraestrutura.
- Vocabulario para o modulo 15.
- Cuidado com salto de escopo.

## Exemplo Pratico

```text
Design local:
Adapter para uma API externa.

Arquitetura:
camadas, filas, brokers ou microkernel para organizar partes maiores do sistema.
```

## Analogias e Intuicoes

- GoF costuma olhar pecas menores; PoSA ajuda a olhar estruturas maiores.
- O tamanho do problema muda o tipo de padrao.

## Erros Comuns

- Chamar decisao local de arquitetura completa.
- Usar padrao arquitetural para problema pequeno.
- Misturar design de classe com estilo arquitetural.
- Entrar em microservicos cedo demais.

## Checklist de Entendimento

- O problema e local ou sistemico?
- O padrao organiza objetos ou partes maiores?
- O modulo 14 deve aprofundar isso ou apenas apresentar?
- Que assunto fica para arquitetura?
- O exemplo ainda cabe em codigo pequeno?

## Pratica Recomendada

- Liste tres padroes locais que voce ja viu.
- Liste tres padroes arquiteturais que serao estudados depois.
- Explique a diferenca de escala.
