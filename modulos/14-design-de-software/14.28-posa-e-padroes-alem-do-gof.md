# POSA e Padrões Além do GoF

## Objetivo

Ao final desta aula, você deve ser capaz de:

- entender que existem padrões além do GoF;
- reconhecer POSA como literatura de padrões arquiteturais e de sistemas;
- diferenciar padrões de design local e padrões arquiteturais;
- preparar a transição para arquitetura de software.

## Onde Esta Aula Entra no Curso

- Módulo: 14. Design de Software
- Aula anterior: GoF: Padrões Criacionais, Estruturais e Comportamentais
- Próxima aula: Projeto Prático: Refatorando uma API para Melhor Design
- Pré-requisitos: Padrões de design, princípios de design, backend, mensageria introdutória e testabilidade.

## Ideia Central

POSA e outras famílias de padrões ampliam a conversa para estruturas maiores: camadas, brokers, pipes, microkernel, concorrência e sistemas distribuídos. Esses assuntos encostam no módulo 15, mas aqui entram apenas como mapa.

O limite importante: neste módulo, padrões servem para design de código e transição conceitual. Arquitetura será aprofundada depois.

## Tópicos da Aula

- GoF vs POSA.
- Padrões locais.
- Padrões arquiteturais.
- Sistemas e infraestrutura.
- Vocabulário para o módulo 15.
- Cuidado com salto de escopo.

## Exemplo Prático

```text
Design local:
Adapter para uma API externa.

Arquitetura:
camadas, filas, brokers ou microkernel para organizar partes maiores do sistema.
```

## Analogias e Intuições

- GoF costuma olhar peças menores; POSA ajuda a olhar estruturas maiores.
- O tamanho do problema muda o tipo de padrão.

## Erros Comuns

- Chamar decisão local de arquitetura completa.
- Usar padrão arquitetural para problema pequeno.
- Misturar design de classe com estilo arquitetural.
- Entrar em microsserviços cedo demais.

## Checklist de Entendimento

- O problema é local ou sistêmico?
- O padrão organiza objetos ou partes maiores?
- O módulo 14 deve aprofundar isso ou apenas apresentar?
- Que assunto fica para arquitetura?
- O exemplo ainda cabe em código pequeno?

## Prática Recomendada

- Liste três padrões locais que você já viu.
- Liste três padrões arquiteturais que serão estudados depois.
- Explique a diferença de escala.
