# Program Against Abstractions

## Objetivo

Ao final desta aula, você deve ser capaz de:

- explicar o que significa programar contra abstrações;
- criar contratos pequenos e expressivos;
- diferenciar abstração útil de abstração vazia;
- conectar este princípio com DIP e testes.

## Onde Esta Aula Entra na Formação

- Módulo: 14. Design de Software
- Aula anterior: Desafio: Aplicando SOLID no PetCare OS
- Próxima aula: Composition Over Inheritance
- Pré-requisitos: Interfaces, DIP, testes, backend e acoplamento.

## Ideia Central

Programar contra abstrações significa depender de capacidades e contratos, não de detalhes concretos. O caso de uso precisa de algo que salve progresso, não necessariamente de uma classe específica do ORM.

Abstração boa tem nome claro, tamanho pequeno e nasce de uma necessidade real. Abstração ruim apenas troca uma dependência concreta por uma interface sem significado.

## Tópicos da Aula

- Capacidade vs implementação.
- Contratos pequenos.
- Interfaces de entrada e saída.
- Testes com fakes.
- Abstração prematura.
- Relação com DIP.

## Exemplo Prático

Dependendo do Que Importa:

`function gerarReceita(impressora: ImpressoraEpson)` <-- Errado, depende da marca concreta.
`function gerarReceita(impressora: ImpressoraTextual)` <-- Certo. Depende do que o PetCare OS quer que a impressora faça.

## Analogias e Intuições

- Se você programar contra implementações, quando o modelo sair de linha você vai falir. Se você programa contra a interface USB, não importa a marca da câmera, ela vai funcionar no computador.

## Erros Comuns

- Criar interface com o mesmo nome da classe concreta.
- Expor métodos demais no contrato.
- Abstrair detalhe estável sem motivo.
- Usar abstração para esconder design confuso.

## Checklist de Entendimento

- Que capacidade o consumidor precisa?
- O contrato é menor que a implementação?
- O nome comunica papel?
- Existe ganho de teste ou evolução?
- A abstração tem motivo real?

## Prática Recomendada

- Defina uma Abstração em TypeScript (`interface ArmazenamentoFotos`) para o uploader de fotos de Pets. Ela não deve citar AWS S3 nem Azure.
