# Open Closed Principle

## Objetivo

Ao final desta aula, você deve ser capaz de:

- explicar aberto para extensão e fechado para modificação;
- reconhecer pontos de variação reais;
- usar polimorfismo ou composição quando fizer sentido;
- evitar abstrair variações hipotéticas.

## Onde Esta Aula Entra na Formação

- Módulo: 14. Design de Software
- Aula anterior: Single Responsibility Principle
- Próxima aula: Liskov Substitution Principle
- Pré-requisitos: SOLID, POO, interfaces, composição, YAGNI e testes.

## Ideia Central

OCP procura reduzir modificações arriscadas em código estável quando novas variações surgem. Em vez de editar uma cadeia de condicionais toda vez, você pode adicionar uma nova implementação em um ponto controlado.

Mas OCP depende de variação real. Sem evidência, ele pode virar excesso de abstração.

## Tópicos da Aula

- Pontos de extensão.
- Condicionais crescentes.
- Estratégias.
- Registro de implementações.
- Testes de contrato.
- Limite com YAGNI.

## Exemplo Prático

OCP no Cálculo de Descontos:

Se criarmos um switch/case infinito `switch (tipoDeTutor) { case 'VIP': ... case 'STANDARD': ... }`, teremos que alterar a mesma função sempre que o hospital inventar o tutor 'PREMIUM'.

Com OCP (via Polimorfismo e interfaces): `interface RegraDesconto { aplicar(total: number) }`. Para novos tutores, você cria um novo arquivo `PremiumDesconto.ts`. Você "abriu para extensão" (adicionou classe) e "fechou para modificação" (a função central que roda os cálculos nunca mais é editada).

## Analogias e Intuições

- OCP é o conector USB do seu computador. Você pode adicionar impressoras, mouses e pendrives (extensão) sem precisar desmontar a placa-mãe (modificação).

## Erros Comuns

- Criar estratégia para uma única variação.
- Esconder regra simples atrás de muitas classes.
- Transformar todo `if` em polimorfismo.
- Não testar o contrato comum.

## Checklist de Entendimento

- A variação já existe?
- O código muda sempre no mesmo ponto?
- A abstração tem contrato claro?
- A extensão reduz risco?
- YAGNI ainda permite esperar?

## Prática Recomendada

- Escreva uma mini-função que lide com Tipos de Consulta (Rotina, Cirurgia, Emergência).
- Use uma Interface `CalculadoraPrecoConsulta` para que a adição de 'Consulta Oftalmológica' não exija abrir o arquivo principal.
