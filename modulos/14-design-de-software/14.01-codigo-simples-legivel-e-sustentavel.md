# Código Simples, Legível e Sustentável

## Objetivo

Ao final desta aula, você deve ser capaz de:

- explicar simplicidade no contexto de código;
- diferenciar legibilidade de gosto pessoal;
- reconhecer sinais de código difícil de sustentar;
- avaliar trade-offs pequenos de design.

## Onde Esta Aula Entra na Formação

- Módulo: 14. Design de Software
- Aula anterior: Design de Software
- Próxima aula: Princípios do Clean Code
- Pré-requisitos: TypeScript, funções, módulos, testes e manutenção básica de APIs.

## Ideia Central

Código simples não é código curto a qualquer custo. É código com poucas ideias misturadas, fluxo compreensível, nomes honestos e dependências visíveis.

Legibilidade não é apenas "eu gostei". Ela aparece quando outra pessoa consegue prever o comportamento, encontrar a regra importante e alterar uma parte sem medo excessivo.

## Tópicos da Aula

- Simplicidade acidental vs essencial.
- Leitura antes da escrita.
- Código localmente compreensível.
- Mudança como critério de design.
- Sustentabilidade e custo futuro.

## Exemplo Prático

Sustentabilidade no Cadastro de Pets:

Código ruim: A função `addPet(p)` checa raça, checa dono, grava no banco e cospe `res.send()`. Daqui a 6 meses, pedem pra adicionar validação de peso. Você tem medo de quebrar.

Código sustentável: A validação fica no `PetValidator`, a gravação no `PetRepository`. Adicionar a regra de peso afeta apenas o Validator. Você confia na mudança.

## Analogias e Intuições

- Código legível é como o prontuário de um paciente bem preenchido: qualquer veterinário no plantão consegue ler, entender o histórico e assumir o caso sem precisar ligar para quem atendeu ontem.

## Erros Comuns

- Achar que menos linhas sempre significa mais simples.
- Esconder comportamento em nomes genéricos.
- Otimizar para escrever rápido e ignorar leitura futura.
- Criar helper para tudo sem ganhar clareza.

## Checklist de Entendimento

- Qual ideia principal este trecho comunica?
- Quantos motivos ele tem para mudar?
- O nome revela a intenção?
- A dependência está clara?
- Um teste simples consegue exercitar a regra?

## Prática Recomendada

- Encontre uma função na sua API que passe de 50 linhas.
- Tente explicar para si mesmo o que ela faz em uma única frase curta.
- Se usar 'e', 'ou', 'mas' na frase, ela faz mais de uma coisa.
