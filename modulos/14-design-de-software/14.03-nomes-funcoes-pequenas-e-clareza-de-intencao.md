# Nomes, Funções Pequenas e Clareza de Intenção

## Objetivo

Ao final desta aula, você deve ser capaz de:

- escolher nomes que revelam intenção;
- dividir funções por responsabilidade;
- reconhecer abstrações pequenas demais;
- melhorar leitura sem esconder o fluxo.

## Onde Esta Aula Entra no Curso

- Módulo: 14. Design de Software
- Aula anterior: Princípios do Clean Code
- Próxima aula: Paradigmas da Programação
- Pré-requisitos: TypeScript, funções, objetos, módulos e testes unitários.

## Ideia Central

Nomes bons contam a história do código. Funções pequenas ajudam quando cada função representa uma decisão ou uma etapa compreensível.

O risco é fragmentar demais. Uma função pequena que só troca uma linha por um nome ruim não melhora o design. A pergunta central é: esta divisão revelou intenção?

## Tópicos da Aula

- Nomes de domínio.
- Verbos para ações.
- Predicados com `is`, `has`, `can` ou equivalentes.
- Funções com uma ideia principal.
- Parâmetros demais como sinal de alerta.

## Exemplo Prático

```ts
function canCompleteLesson(progress: Progress, lesson: Lesson) {
  return !progress.completedLessonIds.includes(lesson.id)
}
```

## Analogias e Intuições

- Um bom nome é uma placa: ele reduz a necessidade de abrir tudo para descobrir.
- Função pequena boa tem motivo próprio para existir.

## Erros Comuns

- Usar nomes genéricos como `data`, `item`, `handle` e `process`.
- Criar funções pequenas sem vocabulário melhor.
- Colocar detalhes técnicos em nomes de regra de negócio.
- Aceitar muitos parâmetros sem perceber acoplamento.

## Checklist de Entendimento

- O nome revela o resultado esperado?
- A função tem uma ideia central?
- Os parâmetros formam um conceito escondido?
- A extração melhorou o teste?
- O fluxo ainda é fácil de seguir?

## Prática Recomendada

- Renomeie três variáveis genéricas.
- Extraia uma condicional para uma função com nome de pergunta.
- Agrupe parâmetros relacionados em um objeto tipado.
