# Nomes, Funcoes Pequenas e Clareza de Intencao

## Objetivo

Ao final desta aula, voce deve ser capaz de:

- escolher nomes que revelam intencao;
- dividir funcoes por responsabilidade;
- reconhecer abstracoes pequenas demais;
- melhorar leitura sem esconder o fluxo.

## Onde Esta Aula Entra no Curso

- Modulo: 14. Design de Software
- Aula anterior: Principios do Clean Code
- Proxima aula: Paradigmas da Programacao
- Pre-requisitos: TypeScript, funcoes, objetos, modulos e testes unitarios.

## Ideia Central

Nomes bons contam a historia do codigo. Funcoes pequenas ajudam quando cada funcao representa uma decisao ou uma etapa compreensivel.

O risco e fragmentar demais. Uma funcao pequena que so troca uma linha por um nome ruim nao melhora o design. A pergunta central e: esta divisao revelou intencao?

## Topicos da Aula

- Nomes de dominio.
- Verbos para acoes.
- Predicados com `is`, `has`, `can` ou equivalentes.
- Funcoes com uma ideia principal.
- Parametros demais como sinal de alerta.

## Exemplo Pratico

```ts
function canCompleteLesson(progress: Progress, lesson: Lesson) {
  return !progress.completedLessonIds.includes(lesson.id)
}
```

## Analogias e Intuicoes

- Um bom nome e uma placa: ele reduz a necessidade de abrir tudo para descobrir.
- Funcao pequena boa tem motivo proprio para existir.

## Erros Comuns

- Usar nomes genericos como `data`, `item`, `handle` e `process`.
- Criar funcoes pequenas sem vocabulario melhor.
- Colocar detalhes tecnicos em nomes de regra de negocio.
- Aceitar muitos parametros sem perceber acoplamento.

## Checklist de Entendimento

- O nome revela o resultado esperado?
- A funcao tem uma ideia central?
- Os parametros formam um conceito escondido?
- A extracao melhorou o teste?
- O fluxo ainda e facil de seguir?

## Pratica Recomendada

- Renomeie tres variaveis genericas.
- Extraia uma condicional para uma funcao com nome de pergunta.
- Agrupe parametros relacionados em um objeto tipado.
