# Encapsulamento, Estado e Invariantes

## Objetivo

Ao final desta aula, voce deve ser capaz de:

- explicar encapsulamento alem de `private`;
- identificar invariantes de uma regra;
- proteger estado contra mudancas invalidas;
- escrever testes para comportamento, nao para detalhe interno.

## Onde Esta Aula Entra no Curso

- Modulo: 14. Design de Software
- Aula anterior: Principios Primarios de POO
- Proxima aula: Acoplamento e Coesao no Codigo
- Pre-requisitos: Classes, objetos, testes unitarios, regras de negocio e tratamento de erros.

## Ideia Central

Encapsular e controlar como o estado pode mudar. Uma invariante e uma regra que precisa continuar verdadeira para o objeto fazer sentido.

Quando qualquer parte do sistema altera dados diretamente, a regra fica espalhada. Quando o objeto oferece operacoes significativas, ele protege seu proprio estado.

## Topicos da Aula

- Estado valido.
- Invariantes.
- Metodos de comportamento.
- Evitar setters automaticos.
- Erros de dominio.
- Testes de comportamento.

## Exemplo Pratico

```ts
class Enrollment {
  cancel() {
    if (this.status === "completed") throw new Error("Cannot cancel")
    this.status = "cancelled"
  }
}
```

## Analogias e Intuicoes

- Invariante e uma promessa que o objeto nunca deve quebrar.
- Encapsulamento bom reduz lugares onde uma regra pode vazar.

## Erros Comuns

- Criar setter para todo campo.
- Validar regra apenas no endpoint.
- Expor arrays internos para alteracao externa.
- Testar campo privado em vez de comportamento publico.

## Checklist de Entendimento

- Que estados sao validos?
- Quem pode mudar o estado?
- Que operacao representa a mudanca?
- Que erro deve impedir estado invalido?
- O teste protege a regra?

## Pratica Recomendada

- Escolha um objeto com status.
- Liste estados validos e transicoes proibidas.
- Implemente metodos para mudar estado.
- Teste uma transicao invalida.
