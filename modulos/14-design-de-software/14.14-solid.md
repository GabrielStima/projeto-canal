# SOLID

## Objetivo

Ao final desta aula, você deve ser capaz de:

- explicar SOLID como conjunto de princípios de design;
- entender o problema central de cada letra;
- reconhecer que SOLID não exige POO pesada;
- preparar as aulas específicas de cada princípio.

## Onde Esta Aula Entra na Formação

- Módulo: 14. Design de Software
- Aula anterior: YAGNI
- Próxima aula: Single Responsibility Principle
- Pré-requisitos: POO, acoplamento, coesão, DRY, YAGNI, testes e TypeScript.

## Ideia Central

SOLID ajuda a criar código mais fácil de mudar quando responsabilidades, extensões, substituições, contratos e dependências ficam problemáticas.

Ele não deve ser aplicado como ornamento. Cada princípio responde a uma dor: classes com motivos demais para mudar, modificações arriscadas, subtipos que quebram contrato, interfaces grandes demais e dependências em detalhes instáveis.

## Tópicos da Aula

- S: responsabilidade única.
- O: aberto para extensão.
- L: substituição segura.
- I: interfaces específicas.
- D: dependência de abstrações.
- Limites de SOLID.

## Exemplo Prático

Uma Visão Geral do SOLID no PetCare OS:

Um código mal desenhado exige abrir arquivos gigantes para alterar detalhes minúsculos. O SOLID visa criar arquiteturas onde adicionar a feature 'Pagamento via PIX' cria *novos* arquivos, em vez de modificar *velhos* arquivos que podem quebrar o Cartão de Crédito.

## Analogias e Intuições

- SOLID é como lego. Você encaixa novas funcionalidades nas portas abertas sem precisar derreter e remodelar a peça principal.

## Erros Comuns

- Criar uma interface para cada classe sem necessidade.
- Usar SOLID para justificar excesso de camadas.
- Aplicar em CRUD simples sem dor real.
- Decorar definições sem reconhecer sintomas.

## Checklist de Entendimento

- Que letra responde a qual problema?
- O problema existe neste código?
- A solução adiciona custo?
- O teste ficou mais simples?
- A mudança ficou mais localizada?

## Prática Recomendada

- Resuma mentalmente: Quais são as 5 letras do S.O.L.I.D. e o que elas significam, em uma linha cada?
