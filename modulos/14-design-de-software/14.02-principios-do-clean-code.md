# Principios do Clean Code

## Objetivo

Ao final desta aula, voce deve ser capaz de:

- explicar Clean Code sem transformar em dogma;
- reconhecer principios praticos de clareza;
- aplicar melhorias pequenas em codigo existente;
- evitar exageros comuns em nome de "codigo limpo".

## Onde Esta Aula Entra no Curso

- Modulo: 14. Design de Software
- Aula anterior: Codigo Simples, Legivel e Sustentavel
- Proxima aula: Nomes, Funcoes Pequenas e Clareza de Intencao
- Pre-requisitos: Funcoes, objetos, modulos, TypeScript, testes e code review.

## Ideia Central

Clean Code e um conjunto de praticas para tornar codigo mais compreensivel e modificavel. Ele ajuda quando lembra que codigo e lido muitas vezes e alterado sob pressao.

Mas principio nao e lei absoluta. Codigo limpo tambem precisa respeitar contexto, equipe, dominio, performance, risco e custo de mudanca.

## Topicos da Aula

- Intencao clara.
- Funcoes focadas.
- Baixa surpresa.
- Comentarios uteis.
- Tratamento de erro legivel.
- Limites do Clean Code.

## Exemplo Pratico

```text
Antes:
if (u.p && u.s !== "blocked") save(u)

Depois:
if (canActivateUser(user)) {
  saveUser(user)
}
```

## Analogias e Intuicoes

- Clean Code e higiene: importante todo dia, perigoso quando vira ritual vazio.
- Clareza boa reduz perguntas durante code review.

## Erros Comuns

- Transformar toda linha em uma funcao.
- Remover comentarios uteis por regra cega.
- Preferir nomes longos demais em vez de bons limites.
- Usar Clean Code para impor gosto pessoal.

## Checklist de Entendimento

- A intencao esta clara?
- A funcao faz coisas demais?
- O comentario explica por que, ou repete o que?
- O erro e tratado perto do contexto?
- A refatoracao reduziu complexidade real?

## Pratica Recomendada

- Escolha um trecho com condicional confusa.
- Extraia uma funcao de predicado com nome claro.
- Escreva um teste para proteger o comportamento.
- Explique o trade-off da mudanca.
