# Princípios do Clean Code

## Objetivo

Ao final desta aula, você deve ser capaz de:

- explicar Clean Code sem transformar em dogma;
- reconhecer princípios práticos de clareza;
- aplicar melhorias pequenas em código existente;
- evitar exageros comuns em nome de "código limpo".

## Onde Esta Aula Entra na Formação

- Módulo: 14. Design de Software
- Aula anterior: Código Simples, Legível e Sustentável
- Próxima aula: Nomes, Funções Pequenas e Clareza de Intenção
- Pré-requisitos: Funções, objetos, módulos, TypeScript, testes e code review.

## Ideia Central

Clean Code é um conjunto de práticas para tornar código mais compreensível e modificável. Ele ajuda quando lembra que código é lido muitas vezes e alterado sob pressão.

Mas princípio não é lei absoluta. Código limpo também precisa respeitar contexto, equipe, domínio, performance, risco e custo de mudança.

## Tópicos da Aula

- Intenção clara.
- Funções focadas.
- Baixa surpresa.
- Comentários úteis.
- Tratamento de erro legível.
- Limites do Clean Code.

## Exemplo Prático

Clean Code no Módulo Clínico:

Um if terrível: `if (p.st == 1 && t > 0)`
Um if Clean Code: `if (pet.isReadyForDischarge() && balanceDue > 0)`

A máquina compila os dois da mesma forma, mas o segundo não exige que o desenvolvedor pergunte no Slack "o que significa st == 1?".

## Analogias e Intuições

- Clean Code é manter o balcão da clínica organizado: seringas numa gaveta, remédios na outra. Se tudo ficar solto, você gasta mais tempo procurando material do que operando.

## Erros Comuns

- Transformar toda linha em uma função.
- Remover comentários úteis por regra cega.
- Preferir nomes longos demais em vez de bons limites.
- Usar Clean Code para impor gosto pessoal.

## Checklist de Entendimento

- A intenção está clara?
- A função faz coisas demais?
- O comentário explica por que, ou repete o que?
- O erro é tratado perto do contexto?
- A refatoração reduziu complexidade real?
