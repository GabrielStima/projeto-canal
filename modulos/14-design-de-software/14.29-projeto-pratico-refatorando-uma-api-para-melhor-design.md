# Projeto Pratico: Refatorando uma API para Melhor Design

## Objetivo

Ao final desta aula, voce deve ser capaz de:

- diagnosticar problemas de design em uma API pequena;
- refatorar com passos seguros;
- proteger comportamento com testes;
- explicar trade-offs de design.

## Onde Esta Aula Entra no Curso

- Modulo: 14. Design de Software
- Aula anterior: PoSA e Padroes Alem do GoF
- Proxima aula: Principios Arquiteturais
- Pre-requisitos: Todo o modulo 14, backend com Node.js, bancos de dados, API design, testes e testabilidade.

## Ideia Central

O projeto do modulo junta os conceitos em uma refatoracao realista. A meta nao e criar a arquitetura perfeita. E pegar um fluxo que ja funciona e melhorar nomes, responsabilidades, dependencias, testes e clareza.

Refatorar bem significa mudar estrutura sem mudar comportamento observado. Por isso os testes sao parte do design, nao uma etapa decorativa no final.

## Topicos da Aula

- Escolha do fluxo.
- Diagnostico de responsabilidades.
- Testes de protecao.
- Separacao de regra e infraestrutura.
- Abstracoes pequenas.
- Registro de decisoes.

## Exemplo Pratico

```text
Fluxo sugerido:
POST /lessons/:id/complete

Refatorar:
- handler HTTP;
- validacao de entrada;
- caso de uso;
- modelo de progresso;
- repositorio;
- resposta padronizada.
```

## Analogias e Intuicoes

- Refatoracao boa troca a estrutura enquanto mantem a promessa externa.
- Teste e o cinto de seguranca da mudanca.

## Erros Comuns

- Refatorar e adicionar feature ao mesmo tempo.
- Criar camadas demais.
- Nao escrever teste antes de mexer.
- Trocar nomes sem melhorar responsabilidade.
- Nao registrar trade-offs.

## Checklist de Entendimento

- O comportamento externo continuou igual?
- Que responsabilidades foram separadas?
- Que dependencia ficou explicita?
- Que teste protege a regra principal?
- Que decisao de design foi tomada?

## Pratica Recomendada

- Escolha um fluxo da API do curso.
- Escreva testes do comportamento atual.
- Liste problemas de design.
- Refatore em passos pequenos.
- Compare antes e depois.
- Escreva uma nota curta explicando trade-offs.
