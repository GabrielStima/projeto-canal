# Projeto Prático: Refatorando uma API para Melhor Design

## Objetivo

Ao final desta aula, você deve ser capaz de:

- diagnosticar problemas de design em uma API pequena;
- refatorar com passos seguros;
- proteger comportamento com testes;
- explicar trade-offs de design.

## Onde Esta Aula Entra na Formação

- Módulo: 14. Design de Software
- Aula anterior: POSA e Padrões Além do GoF
- Próxima aula: Princípios Arquiteturais
- Pré-requisitos: Todo o módulo 14, backend com Node.js, bancos de dados, API design, testes e testabilidade.

## Ideia Central

O projeto do módulo junta os conceitos em uma refatoração realista. A meta não é criar a arquitetura perfeita. É pegar um fluxo que já funciona e melhorar nomes, responsabilidades, dependências, testes e clareza.

Refatorar bem significa mudar estrutura sem mudar comportamento observado. Por isso os testes são parte do design, não uma etapa decorativa no final.

## Tópicos da Aula

- Escolha do fluxo.
- Diagnóstico de responsabilidades.
- Testes de proteção.
- Separação de regra e infraestrutura.
- Abstrações pequenas.
- Registro de decisões.

## Exemplo Prático

```text
Fluxo sugerido:
POST /lessons/:id/complete

Refatorar:
- handler HTTP;
- validação de entrada;
- caso de uso;
- modelo de progresso;
- repositório;
- resposta padronizada.
```

## Analogias e Intuições

- Refatoração boa troca a estrutura enquanto mantém a promessa externa.
- Teste é o cinto de segurança da mudança.

## Erros Comuns

- Refatorar é adicionar feature ao mesmo tempo.
- Criar camadas demais.
- Não escrever teste antes de mexer.
- Trocar nomes sem melhorar responsabilidade.
- Não registrar trade-offs.

## Checklist de Entendimento

- O comportamento externo continuou igual?
- Que responsabilidades foram separadas?
- Que dependência ficou explícita?
- Que teste protege a regra principal?
- Que decisão de design foi tomada?

## Prática Recomendada

- Escolha um fluxo da API da formação.
- Escreva testes do comportamento atual.
- Liste problemas de design.
- Refatore em passos pequenos.
- Compare antes e depois.
- Escreva uma nota curta explicando trade-offs.
