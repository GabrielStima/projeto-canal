# Model-Driven Design

## Objetivo

Ao final desta aula, você deve ser capaz de:

- explicar model-driven design em nível de código;
- diferenciar modelo de domínio de modelo de banco;
- usar linguagem do problema nos nomes;
- evitar modelos que só refletem tabela.

## Onde Esta Aula Entra na Formação

- Módulo: 14. Design de Software
- Aula anterior: Acoplamento e Coesão no Código
- Próxima aula: Princípios de Design de Software
- Pré-requisitos: Requisitos, modelagem de dados, APIs, POO, TypeScript e testes.

## Ideia Central

Model-driven design organiza o código em torno dos conceitos do problema. Em vez de deixar framework, banco ou payload mandarem em todos os nomes, o modelo tenta expressar o domínio: matrícula, progresso, aula, conclusão, assinatura, permissão.

Isso não significa criar uma camada complexa. Significa evitar que as regras importantes fiquem escondidas em objetos genéricos, tabelas ou handlers.

## Tópicos da Aula

- Linguagem do domínio.
- Modelo de domínio vs DTO.
- Modelo de banco vs comportamento.
- Regras perto dos conceitos.
- Limites do modelo.
- Ponte com DDD no módulo 15.

## Exemplo Prático

A Linguagem Ubíqua no PetCare OS:

O Veterinário não diz "Fiz o update do row na table animal status = 2". Ele diz "Dei Alta Médica ao Paciente".
Seu código reflete isso: em vez de `db.pets.update({status: 2})`, você tem `paciente.darAlta()`. O modelo do código é guiado (driven) pelo modelo do negócio.

## Analogias e Intuições

- MDD é fazer o código falar o mesmo idioma do cliente. Se o médico usa a palavra 'Prontuário', o código jamais deveria chamar a classe de 'MedicalRecordSheetData'.

## Erros Comuns

- Tratar entidade de banco como modelo completo.
- Criar modelo rico para CRUD simples demais.
- Usar nomes técnicos onde existe nome de negócio.
- Misturar DTO, entidade, view model e regra sem critério.

## Checklist de Entendimento

- Quais conceitos existem no problema?
- Que regras pertencem a cada conceito?
- O nome veio do domínio ou da tecnologia?
- O banco está mandando no design?
- O modelo ajuda a escrever testes?
