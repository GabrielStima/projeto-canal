# Features de Paradigmas

## Objetivo

Ao final desta aula, voce deve ser capaz de:

- reconhecer features associadas a paradigmas;
- separar recurso de linguagem de estilo de design;
- usar funcoes, closures, classes e interfaces com criterio;
- evitar confundir ferramenta com arquitetura.

## Onde Esta Aula Entra no Curso

- Modulo: 14. Design de Software
- Aula anterior: Paradigmas da Programacao
- Proxima aula: Programacao Orientada a Objetos no Design
- Pre-requisitos: JavaScript moderno, TypeScript, funcoes, classes, interfaces e modulos.

## Ideia Central

Linguagens oferecem features: funcoes de primeira classe, closures, classes, interfaces, generics, composicao de objetos e modulos. Paradigmas usam essas features para criar estilos de organizacao.

Saber que uma feature existe nao significa que ela deve aparecer em todo lugar. Design bom usa o recurso que torna a intencao mais clara.

## Topicos da Aula

- Funcoes de primeira classe.
- Closures.
- Imutabilidade.
- Classes.
- Interfaces e contratos.
- Generics.
- Modulos.

## Exemplo Pratico

```ts
function createProgressCalculator(requiredLessons: number) {
  return (completedLessons: number) => completedLessons / requiredLessons
}
```

## Analogias e Intuicoes

- Features sao ferramentas; paradigma e o jeito de montar a bancada.
- Um recurso poderoso pode deixar o codigo menos claro quando usado sem necessidade.

## Erros Comuns

- Usar classe apenas porque existe `class`.
- Usar generics para parecer avancado.
- Confundir imutabilidade com copiar objetos sem pensar.
- Criar closures que escondem dependencia demais.

## Checklist de Entendimento

- Que feature esta sendo usada?
- Ela melhora a clareza?
- Ela esconde estado?
- Ela facilita teste?
- Ela encaixa no restante do modulo?

## Pratica Recomendada

- Pegue uma regra com configuracao.
- Implemente com closure.
- Implemente com classe.
- Compare dependencias, teste e leitura.
