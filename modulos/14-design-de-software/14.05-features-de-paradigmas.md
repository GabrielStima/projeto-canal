# Features de Paradigmas

## Objetivo

Ao final desta aula, você deve ser capaz de:

- reconhecer features associadas a paradigmas;
- separar recurso de linguagem de estilo de design;
- usar funções, closures, classes e interfaces com critério;
- evitar confundir ferramenta com arquitetura.

## Onde Esta Aula Entra na Formação

- Módulo: 14. Design de Software
- Aula anterior: Paradigmas da Programação
- Próxima aula: Programação Orientada a Objetos no Design
- Pré-requisitos: JavaScript moderno, TypeScript, funções, classes, interfaces e módulos.

## Ideia Central

Linguagens oferecem features: funções de primeira classe, closures, classes, interfaces, generics, composição de objetos e módulos. Paradigmas usam essas features para criar estilos de organização.

Saber que uma feature existe não significa que ela deve aparecer em todo lugar. Design bom usa o recurso que torna a intenção mais clara.

## Tópicos da Aula

- Funções de primeira classe.
- Closures.
- Imutabilidade.
- Classes.
- Interfaces e contratos.
- Generics.
- Módulos.

## Exemplo Prático

Mecanismos nas Linguagens:

O JavaScript e TypeScript permitem mesclar paradigmas no PetCare OS.
Você cria classes para as dependências do Controller (POO), mas usa `map`, `filter`, e `reduce` manipulando arrays de pets sem sofrer efeitos colaterais (Funcional).

## Analogias e Intuições

- Features de paradigma são temperos. Você pode cozinhar (programar) usando vários, dependendo do prato (módulo do sistema) que está servindo.

## Erros Comuns

- Usar classe apenas porque existe `class`.
- Usar generics para parecer avançado.
- Confundir imutabilidade com copiar objetos sem pensar.
- Criar closures que escondem dependência demais.

## Checklist de Entendimento

- Que feature está sendo usada?
- Ela melhora a clareza?
- Ela esconde estado?
- Ela facilita teste?
- Ela encaixa no restante do módulo?

## Prática Recomendada
