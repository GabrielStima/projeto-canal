# Permissões Mínimas

## Objetivo

Ao final desta aula, você deve ser capaz de:

- explicar o princípio do menor privilégio;
- aplicar permissões mínimas em repositórios, pipelines, cloud e registries;
- diferenciar permissão necessária de permissão conveniente;
- revisar acessos com foco em impacto.

- Próxima aula: 21.06 Segurança em CI/CD

## Ideia Central

Permissões mínimas significam dar a cada pessoa, serviço ou automação apenas o acesso necessário para cumprir sua função.

Isso não impede todos os ataques, mas reduz o dano quando algo dá errado. Se um token de teste vaza, ele não deve conseguir apagar produção. Se um job só precisa ler dependências, ele não deve publicar pacotes. Se uma pessoa revisa código, talvez não precise administrar secrets.

Menor privilégio é uma prática de desenho, não um detalhe de configuração.

## Exemplo Prático

Um pipeline tem três etapas:

- rodar testes;
- construir imagem;
- publicar imagem;
- fazer deploy.

Todas as etapas não precisam do mesmo acesso. Testes podem não precisar de token de cloud. Build pode não precisar de acesso a produção. Publicação pode acessar apenas o registry. Deploy pode usar uma credencial temporária e restrita.

Separar permissões por etapa reduz impacto se uma etapa for comprometida.

## Analogias e Intuições

- Menor privilégio é entregar a chave da sala certa, não o molho inteiro.
- Permissão temporária é um passe de visita, não um crachá permanente.

## Erros Comuns

- Dar acesso administrador para "resolver rápido".
- Usar a mesma credencial para todos os jobs.
- Manter permissões antigas depois que a necessidade acabou.
- Não revisar acesso de pessoas que mudaram de função.
- Ignorar permissões de leitura que expõem dados sensíveis.

## Checklist de Entendimento

Antes de seguir, a pessoa deve conseguir responder:

- Quem precisa acessar o quê?
- Por quanto tempo esse acesso é necessário?
- Qual é o impacto se essa credencial vazar?
- Existe uma permissão menor que ainda resolve o problema?
- Como revisar acessos periodicamente?

## Prática Recomendada

- Escolha um pipeline e liste permissões por etapa.
- Remova permissões que não são necessárias para aquela etapa.
- Escreva uma justificativa curta para cada permissão restante.
