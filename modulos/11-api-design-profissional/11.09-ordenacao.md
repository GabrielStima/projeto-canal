# Ordenação

## Objetivo

Ao final desta aula, você deve ser capaz de:

- explicar por que ordenação faz parte do contrato de listagem;
- desenhar parâmetros de ordenação;
- evitar resultados instáveis;
- combinar ordenação com paginação e filtros.

## Ideia Central

Ordenação define em que sequência uma coleção será devolvida. Sem ordenação clara, a mesma requisição pode retornar itens em ordem diferente, o que quebra paginação, confunde usuários e dificulta testes.

Uma API profissional precisa documentar campos ordenáveis, direção, padrão e critério de desempate. Isso é especialmente importante quando há paginação.

Na `petcare-api`, ordenar atendimentos por horário de chegada ou por prioridade de triagem afeta diretamente o trabalho da recepção e dos clínicos.

## Exemplo Prático

```text
GET /appointments?sort=scheduledAt:desc
```

Resposta ordenada por agendamento mais recente. Se dois atendimentos tiverem o mesmo `scheduledAt`, a API pode usar `id` como desempate para manter ordem estável.

Outro exemplo: ordenar itens de estoque por data de validade, do mais próximo do vencimento para o mais distante.

```text
GET /inventory?sort=expiresAt:asc
```

## Analogias e Intuições

- Ordenação é a regra da fila.
- Um critério de desempate evita que dois itens troquem de lugar sem motivo aparente.
- Ordenação instável é como uma fila de triagem onde pets com a mesma gravidade trocam de lugar a cada chamada: causa erro e retrabalho.

## Erros Comuns

- Permitir ordenar por qualquer campo.
- Não definir ordenação padrão.
- Paginar sem ordem estável.
- Não validar direção.
- Ignorar o custo de ordenar campos sem index.
