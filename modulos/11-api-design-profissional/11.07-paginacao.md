## Ideia Central

Paginação evita que uma listagem devolva dados demais de uma vez. Ela protege o servidor, melhora a experiência do consumidor e torna o contrato de listagem mais previsível.

Existem várias estratégias. Página e tamanho são fáceis de entender. Offset e limit são comuns em consultas. Cursor costuma funcionar melhor em listas grandes ou que mudam com frequência, mas exige mais cuidado de implementação e contrato.

Na `petcare-api`, paginar a lista de atendimentos do dia é essencial para o painel administrativo continuar rápido mesmo em clínicas grandes.

## Exemplo Prático

```text
GET /appointments?page=2&pageSize=20
```

```json
{
  "data": [
    {
      "id": "apt-21",
      "petName": "Thor",
      "status": "in_triagem",
      "scheduledAt": "2026-06-05T09:00:00Z"
    }
  ],
  "pagination": {
    "page": 2,
    "pageSize": 20,
    "hasNextPage": true
  }
}
```

O contrato não retorna apenas a lista. Ele também informa como continuar navegando.

Outro exemplo: paginação por cursor para o histórico de atendimentos de um pet, onde a ordem pode mudar com menos frequência e a lista pode ser grande.

```text
GET /pets/{petId}/appointments?cursor=apt_42&limit=20
```

```json
{
  "data": [...],
  "pagination": {
    "nextCursor": "apt_63",
    "hasNextPage": true
  }
}
```

## Analogias e Intuições

- Paginação é entregar um livro por capítulos em vez de jogar todas as páginas na mesa.
- Cursor é um marcador de posição na coleção.
- Listagem sem paginação é como tentar atender todos os pets da clínica ao mesmo tempo: ninguém recebe atenção de qualidade.

## Erros Comuns

- Não definir limite máximo.
- Retornar metadados diferentes em cada endpoint.
- Paginar sem ordenação estável.
- Usar offset em listas enormes sem medir impacto.
- Esquecer que filtros mudam o resultado paginado.
