# API Governance

## Objetivo

Ao final desta aula, você deve ser capaz de:

- explicar API governance em nível prático;
- entender por que times precisam de padrões comuns;
- criar um checklist simples de revisão de API;
- relacionar governança com documentação, segurança, testes e evolução.

## Onde Esta Aula Entra no Curso

- Próxima aula: Contract Testing

## Ideia Central

API governance é o conjunto de regras, processos e revisões que mantém APIs consistentes ao longo do tempo. Não é burocracia por si só. É uma forma de evitar que cada endpoint tenha um estilo diferente, um formato de erro diferente e uma regra de evolução diferente.

Em times pequenos, governança pode ser um checklist. Em organizações maiores, pode envolver catálogo de APIs, padrões obrigatórios, revisão de contrato, lint de OpenAPI, versionamento, ownership e ciclo de depreciação.

## Tópicos da Aula

- Padrões de design.
- Ownership de API.

- Versionamento e depreciação.
- Relação com segurança e testes.

## Exemplo Prático

Checklist mínimo para publicar um endpoint:

```text
- O recurso tem nome consistente?
- Métodos HTTP e status codes fazem sentido?
- Erros seguem o padrão?
- Paginação, filtros e ordenação estão documentados?
- A OpenAPI está atualizada?
- A mudança é backward compatible?
- Existe teste de contrato para resposta principal?
- O owner da API está claro?
```

## Analogias e Intuições

- Governança é o combinado que impede cada rota de falar uma língua diferente.
- Um checklist bom economiza discussão repetida.

## Erros Comuns

- Confundir governança com excesso de aprovação.
- Criar padrão que ninguém consegue aplicar.
- Revisar só o código e esquecer o contrato.
- Não definir dono da API.
- Não automatizar verificações simples.

## Checklist de Entendimento

Antes de seguir, você deve conseguir responder:

- Quais padrões a API precisa seguir?
- Quem aprova mudanças de contrato?
- Como detectar quebra de compatibilidade?
- Onde a API fica documentada?
- Que parte pode ser automatizada?

## Prática Recomendada

- Crie um checklist de API governance para o painel de estudos.
- Aplique o checklist em três endpoints.
- Encontre inconsistências.
- Decida quais verificações podem se transformar em testes ou lint.
- Escreva uma regra de depreciação.
