# Filtros

## Objetivo

Ao final desta aula, você deve ser capaz de:

- explicar o papel de filtros em listagens;
- desenhar query params claros;
- diferenciar filtros simples de busca textual;
- reconhecer riscos de flexibilidade excessiva.

## Onde Esta Aula Entra no Curso

- Módulo: 11. API Design Profissional
- Aula anterior: Paginação
- Próxima aula: Ordenação
- Pré-requisitos: Paginação, query params, modelagem de dados e indexes.

## Ideia Central

Filtros permitem que o consumidor reduza uma coleção para o subconjunto que importa. Em uma API profissional, filtros precisam ser explícitos, documentados e previsíveis.

O perigo é transformar filtros em uma linguagem de consulta improvisada. Quanto mais flexível o contrato, maior o custo de validação, performance, segurança e compatibilidade.

## Tópicos da Aula

- Filtros como query params.
- Filtros por igualdade.
- Filtros por intervalo.
- Filtros por status.
- Busca textual vs filtro.
- Validação de filtros.
- Combinação com paginação.
- Impacto em indexes.

## Exemplo Prático

```text
GET /lessons?module=11&status=published&createdAfter=2026-01-01
```

Esse contrato permite filtrar aulas por módulo, status e data mínima de criação. Cada parâmetro precisa ter tipo, regra e comportamento documentados.

## Analogias e Intuições

- Filtro é uma peneira: ele reduz o conjunto, mas precisa ter critério claro.
- Um filtro mal definido vira uma consulta escondida.

## Erros Comuns

- Aceitar qualquer campo como filtro sem validação.
- Misturar filtro com ordenação no mesmo parâmetro.
- Não documentar formato de datas.
- Criar filtros que o banco não suporta bem.
- Deixar filtros mudarem de significado com o tempo.

## Checklist de Entendimento

Antes de seguir, você deve conseguir responder:

- Quais filtros são suportados?
- Qual é o tipo de cada filtro?
- O filtro é obrigatório ou opcional?
- Como filtros combinam com paginação?
- O banco consegue executar isso com eficiência aceitável?

## Prática Recomendada

- Defina cinco filtros para `GET /lessons`.
- Escreva exemplos válidos e inválidos.
- Decida quais filtros precisam de index.
- Documente erros de filtro inválido.
- Diferencie filtro por título de busca textual.
