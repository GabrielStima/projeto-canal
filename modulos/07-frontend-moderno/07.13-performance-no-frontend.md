# Performance no frontend

## Objetivo

Ao final desta aula, você deve ser capaz de:

- reconhecer gargalos comuns de performance no frontend;
- diferenciar carregamento inicial, renderização e interação;
- entender por que medir vem antes de otimizar;
- aplicar cuidados basicos sem exagerar em micro-otimizações.

## Onde Esta Aula Entra no Curso

- Módulo: 07. Frontend Moderno
- Aula anterior: Testes em frontend
- Próxima aula: Web Components
- Pré-requisitos: Browser, cache, React, renderização, estado, chamadas de API e bundlers.

## Ideia Central

Performance no frontend não é apenas "deixar o código mais rápido". Ela envolve quanto tempo a aplicação demora para carregar, quando o usuário consegue interagir, quanto trabalho o browser precisa fazer e como a UI responde a mudanças.

Antes de otimizar, é preciso medir ou pelo menos identificar o gargalo. Otimizar sem entender o problema pode deixar o código mais complexo sem melhorar a experiência.

No contexto React, performance passa por renderizações desnecessárias, listas grandes, cálculos caros, imagens pesadas, bundles grandes e chamadas de API mal planejadas.

## Tópicos da Aula

- Carregamento inicial.
- Bundle e assets.
- Renderização e re-renderizações.
- Listas grandes.
- Imagens e rede.
- Medição antes de otimização.
- Performance percebida.

## Exemplo Prático

```text
Problema: uma página de produtos parece lenta.

Possíveis causas:
- imagem grande demais;
- chamada de API lenta;
- lista enorme renderizada de uma vez;
- calculo pesado em toda renderização;
- bundle inicial muito grande.
```

Cada causa pede uma solução diferente.

## Analogias e Intuições

- Performance é diagnóstico: primeiro descubra onde doi, depois trate.
- O usuário sente tempo de espera, travamento e falta de feedback.

## Erros Comuns

- Otimizar tudo antes de medir.
- Culpar React por problemas de rede ou imagens.
- Usar memoização sem entender o custo.
- Ignorar estados de carregamento e feedback visual.

## Checklist de Entendimento

Antes de seguir, você deve conseguir responder:

- O que pode deixar uma página lenta?
- Qual a diferença entre carregamento e interação?
- Por que medir antes de otimizar?
- Que problemas podem causar renderizações desnecessárias?
- Como feedback visual melhora performance percebida?

## Prática Recomendada

- Analise uma tela e liste possíveis gargalos.
- Reduza uma imagem grande e observe a diferença.
- Crie uma lista grande e pense como renderiza-la com cuidado.
