# Formulários no frontend moderno

## Objetivo

Ao final desta aula, você deve ser capaz de:

- criar formulários controlados em React;
- conectar campos a estado;
- lidar com submissão sem recarregar a página;
- reconhecer cuidados de acessibilidade e UX em formulários.

## Ideia Central

Formulário continua sendo HTML. React não élimina `label`, `input`, `button`, tipos de campo e semântica. O que muda é que, em interfaces modernas, os valores dos campos muitas vezes passam a ser controlados por estado.

Um formulário controlado deixa a UI e os dados sincronizados: o valor exibido no campo vem do estado, e cada mudança atualiza esse estado.

Isso facilita validação, mensagens, habilitar/desabilitar botões e submissão para APIs.

## Tópicos da Aula

- Labels, foco e mensagens.

## Exemplo Prático

```tsx
function FormularioContato() {
  const [nome, setNome] = useState("");

  function enviar(evento: React.FormEvent) {
    evento.preventDefault();
    console.log({ nome });
  }

  return (
    <form onSubmit={enviar}>
      <label htmlFor="nome">Nome</label>
      <input id="nome" value={nome} onChange={(evento) => setNome(evento.target.value)} />
      <button type="submit">Enviar</button>
    </form>
  );
}
```

## Analogias e Intuições

- O campo mostra o estado; o evento atualiza o estado.
- Submeter formulário e transformar dados de entrada em uma ação.

## Erros Comuns

- Remover semântica HTML por estar usando React.
- Esquecer `label`.
- Usar botão sem `type`.
- Não impedir recarregamento quando a submissão será tratada no cliente.
- Guardar todos os campos sem pensar na estrutura dos dados.
