# Componentes

## Objetivo

Ao final desta aula, você deve ser capaz de:

- explicar o que e um componente;
- dividir uma interface em componentes menores;
- passar dados com props;
- reconhecer quando um componente está fazendo coisa demais.

## Onde Esta Aula Entra no Curso

- Módulo: 07. Frontend Moderno
- Aula anterior: React
- Próxima aula: Renderização
- Pré-requisitos: React introdutório, funções, objetos, arrays, HTML e CSS.

## Ideia Central

Componentes são blocos reutilizáveis de interface. Eles ajudam a transformar uma tela grande em partes menores, cada uma com uma responsabilidade mais clara.

Um bom componente recebe dados, renderiza uma parte da UI e deixa claro o que espera de fora. Props funcionam como entradas: quem usa o componente fornece valores, e o componente decide como mostrar esses valores.

Nem todo pedaço visual precisa virar componente imediatamente. A divisão deve ajudar leitura, reutilização ou organização.

## Tópicos da Aula

- Componentes como funções de UI.
- Props como entrada.
- Composição de componentes.
- Componentes pequenos, mas não fragmentados demais.
- Separação entre estrutura, dados e estilo.

## Exemplo Prático

Uma lista pode ser separada em componentes:

```tsx
type Produto = {
  nome: string;
  preco: number;
};

function ProdutoCard({ produto }: { produto: Produto }) {
  return <li>{produto.nome} - R$ {produto.preco}</li>;
}

export function ListaProdutos({ produtos }: { produtos: Produto[] }) {
  return (
    <ul>
      {produtos.map((produto) => (
        <ProdutoCard key={produto.nome} produto={produto} />
      ))}
    </ul>
  );
}
```

## Analogias e Intuições

- Componentes são peças de montagem: cada peca tem formato e função.
- Props são os dados que você entrega para a peca ser renderizada corretamente.

## Erros Comuns

- Criar um componente enorme chamado `App` com tudo dentro.
- Criar componentes minusculos sem ganho real.
- Alterar props dentro do componente.
- Usar o índice do array como `key` sem pensar nas consequências.

## Checklist de Entendimento

Antes de seguir, você deve conseguir responder:

- O que torna algo um bom candidato a componente?
- O que são props?
- Por que componentes ajudam a organizar uma aplicação?
- Quando a divisão em componentes atrapalha?
- Por que listas precisam de `key`?

## Prática Recomendada

- Pegue uma tela simples e desenhe sua árvore de componentes.
- Crie um componente de card que recebe título, descrição e status.
- Renderize uma lista usando esse card.
