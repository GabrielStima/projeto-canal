## Ideia Central

React permite descrever a interface como uma árvore de componentes. Cada componente recebe dados, devolve uma descrição da UI e pode reagir a eventos ou mudanças de estado.

Em vez de manipular o DOM manualmente a cada mudança, você descreve como a tela deve parecer para determinado estado. React cuida da atualização da interface a partir dessa descrição.

JSX parece HTML, mas é uma sintaxe usada dentro do JavaScript para declarar a estrutura visual de componentes.

## Tópicos da Aula

- React como biblioteca de UI.
- Componentes como funções que retornam UI.
- JSX como descrição da interface.
- Props como entrada de dados.
- Estado como memória local da interface.
- O que React faz e o que ele não faz sozinho.

## Exemplo Prático

Um contador simples mostra a relação entre estado, evento e renderização:

```tsx
import { useState } from "react";

export function Contador() {
  const [total, setTotal] = useState(0);

  return (
    <button onClick={() => setTotal(total + 1)}>
      Cliques: {total}
    </button>
  );
}
```

Quando o usuário clica, o estado muda. Quando o estado muda, a UI é renderizada novamente com o novo valor.

## Analogias e Intuições

- Um componente é uma função que transforma dados em interface.
- Estado é a memória que faz a interface lembrar algo entre interações.

## Erros Comuns

- Pensar que JSX é HTML puro.
- Manipular o DOM diretamente sem necessidade.
- Colocar regra demais dentro de um único componente.
- Achar que React resolve rotas, formulários, API e testes automaticamente.
