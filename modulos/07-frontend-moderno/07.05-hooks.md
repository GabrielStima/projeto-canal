## Ideia Central

Hooks são funções especiais que permitem usar recursos do React dentro de componentes. O primeiro contato costuma ser com `useState`, que guarda memória local da interface, e `useEffect`, que permite sincronizar o componente com algo externo, como uma API, um timer ou uma assinatura de evento.

O ponto importante e não tratar `useEffect` como lugar para qualquer código. Muitàs vezes, valores podem ser calculados durante a renderização ou derivados de estado existente.

Hooks devem ser chamados no topo do componente ou de outro hook, não dentro de condicionais ou loops.

## Exemplo Prático

```tsx
import { useEffect, useState } from "react";

export function Relogio() {
  const [agora, setAgora] = useState(new Date());

  useEffect(() => {
    const id = window.setInterval(() => setAgora(new Date()), 1000);
    return () => window.clearInterval(id);
  }, []);

  return <time>{agora.toLocaleTimeString()}</time>;
}
```

O efeito sincroniza o componente com um timer externo e limpa esse timer quando não for mais necessário.

## Analogias e Intuições

- `useState` é a memória de curto prazo do componente.
- `useEffect` é uma ponte controlada entre React e algo de fora da renderização.

## Erros Comuns

- Chamar hooks dentro de `if`, `for` ou funções comuns.
- Usar `useEffect` para calcular algo que poderia ser derivado diretamente.
- Esquecer limpeza de timers, subscriptions ou listeners.
- Criar loop infinito atualizando estado dentro de efeito sem cuidado.
