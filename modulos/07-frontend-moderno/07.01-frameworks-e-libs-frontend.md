# Frameworks e libs frontend

## Objetivo

Ao final desta aula, você deve ser capaz de:

- explicar por que frameworks e bibliotecas frontend existem;
- diferenciar biblioteca, framework e ferramenta de build em alto nível;
- reconhecer problemas que aparecem quando uma interface cresce;
- saber por que React será usado como ferramenta principal neste módulo.

## Onde Esta Aula Entra no Curso

- Módulo: 07. Frontend Moderno
- Aula anterior: Frontend Moderno
- Próxima aula: React
- Pré-requisitos: HTML, CSS, JavaScript no browser, TypeScript básico, módulos e bundlers em nível introdutório.

## Ideia Central

Uma página simples pode funcionar bem com HTML, CSS e um pouco de JavaScript. O problema aparece quando a interface passa a ter muitas partes que dependem de dados, estado, eventos, validação, navegação e comunicação com APIs.

Bibliotecas e frameworks frontend surgem para organizar essa complexidade. Eles oferecem modelos para dividir a interface, atualizar a tela quando dados mudam e manter o código mais previsível.

React será usado aqui como ferramenta principal porque seu modelo de componentes e muito presente no mercado e prepara bem o caminho para Next.js.

## Tópicos da Aula

- O que muda entre página simples e aplicação frontend.
- Biblioteca, framework, runtime e ferramenta de build.
- Por que componentes ajudam.
- Trade-offs: produtividade, curva de aprendizado e dependência de ferramenta.
- Onde React entra na sequência do curso.

## Exemplo Prático

Compare uma lista de produtos feita com DOM manual e a mesma lista pensada como componentes:

```text
App
- FiltroDeProdutos
- ListaDeProdutos
  - ProdutoCard
```

O objetivo não é codar tudo ainda. É perceber que a interface tem partes com responsabilidades diferentes.

## Analogias e Intuições

- HTML solto e como montar uma vitrine item por item; componentes são como criar peças reutilizáveis da vitrine.
- Frameworks não fazem o trabalho por você; eles definem trilhos para repetir decisões comuns.

## Erros Comuns

- Comecar por framework sem entender HTML, CSS e eventos.
- Achar que toda página precisa de React.
- Confundir framework com linguagem.
- Usar ferramenta para esconder uma falta de entendimento do browser.

## Checklist de Entendimento

Antes de seguir, você deve conseguir responder:

- Que problema frameworks e libs frontend tentam resolver?
- Quando JavaScript puro ainda pode ser suficiente?
- O que significa dividir uma interface em componentes?
- Por que este módulo usa React?
- O que fica para o módulo de Next.js?

## Prática Recomendada

- Escolha uma página que você já criou e liste suas partes visuais.
- Separe essas partes em possíveis componentes.
- Marque quais partes dependem de dados ou interação.
