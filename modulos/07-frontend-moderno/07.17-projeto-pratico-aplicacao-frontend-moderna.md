# Projeto prático: aplicação frontend moderna

## Objetivo

Ao final desta aula, você deve ser capaz de:

- integrar os principais assuntos do módulo em um projeto pequeno;
- criar uma aplicação React com componentes, estado, rotas, API, formulário e validação;
- revisar acessibilidade, testes e performance básica;
- preparar a base conceitual para o módulo de Next.js.

## Onde Esta Aula Entra na Formação

- Módulo: 07. Frontend Moderno
- Aula anterior: Micro-frontends
- Próxima aula: 08. Next.js e Aplicações Fullstack no Frontend
- Pré-requisitos: Todas as aulas anteriores do módulo 07.

## Ideia Central

O projeto final do módulo serve para transformar conceitos isolados em uma pequena aplicação coerente. A ideia não é criar um produto enorme. É mostrar que componentes, estado, rotas, APIs, formulários, validação, testes e performance se conectam.

Um bom projeto prático deve ser pequeno o suficiente para terminar e rico o bastante para revelar problemas reais de frontend moderno.

Sugestão: um catálogo de recursos de estudo, produtos fictícios ou tarefas com páginas de listagem, detalhes e cadastro.

## Tópicos da Aula

- Definição do escopo.
- Estrutura inicial do projeto.
- Árvore de componentes.
- Rotas principais.
- Dados e chamadas de API.
- Formulário e validação.
- Testes essenciais.
- Revisão final de acessibilidade e performance.

## Exemplo Prático

```text
Aplicação: Catálogo de Estudos

Rotas:
- /recursos
- /recursos/:id
- /novo

Funcionalidades:
- listar recursos;
- filtrar por termo;
- ver detalhes;
- cadastrar novo recurso;
- validar título e URL;
- mostrar carregamento, erro e sucesso.
```

## Analogias e Intuições

- O projeto final é um ensaio geral: cada aula anterior aparece como uma parte do fluxo.
- Escopo pequeno bem acabado ensina mais do que um projeto grande abandonado.

## Erros Comuns

- Aumentar o escopo antes de terminar o fluxo principal.
- Criar componentes sem responsabilidade clara.
- Ignorar estados de erro e carregamento.
- Deixar validação e testes para depois e nunca voltar.
- Otimizar antes de ter um projeto funcionando.

## Checklist de Entendimento

Antes de seguir, você deve conseguir responder:

- Quais componentes principais a aplicação tem?
- Quais rotas existem?
- Quais estados de API aparecem?
- Como o formulário valida dados?
- Quais fluxos merecem teste?
- O que precisa ser revisado antes de ir para Next.js?

## Prática Recomendada

- Construa a aplicação em etapas pequenas.
- Faça primeiro o fluxo principal funcionando.
- Depois adicione validação, testes e melhorias de performance.
- Escreva um README curto explicando como rodar o projeto.
