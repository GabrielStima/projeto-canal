# Renderização

## Objetivo

Ao final desta aula, você deve ser capaz de:

- explicar renderização em React em nível prático;
- entender que componentes descrevem UI para um estado;
- reconhecer por que mudanças de estado causam nova renderização;
- evitar confundir renderização com manipulação manual do DOM.

## Ideia Central

Renderizar e calcular como a interface deve aparecer naquele momento. Em React, um componente e executado para produzir uma descrição da UI. Quando estado ou props mudam, React pode renderizar novamente para refletir esses novos dados.

Isso não significa recriar a página inteira manualmente. O programador descreve o resultado esperado; React compara e aplica as mudanças necessarias na interface.

Entender renderização ajuda a evitar bugs comuns, como calcular valores no lugar errado, atualizar estado durante renderização ou esperar que uma variável comum preserve memória entre interações.

## Tópicos da Aula

- Renderização inicial.
- Nova renderização por mudança de estado ou props.
- UI como resultado de dados.
- Estado preservado pela posição na árvore.
- Renderização condicional e listas.

## Exemplo Prático

```tsx
function Saudacao({ nome }: { nome: string }) {
  if (!nome) {
    return <p>Informe seu nome.</p>;
  }

  return <p>Ola, {nome}!</p>;
}
```

O componente descreve dois possíveis resultados. A UI depende do valor recebido em `nome`.

## Analogias e Intuições

- Renderização e como recalcular uma vitrine a partir do estoque atual.
- Estado é props são os ingredientes; JSX e a receita visual.

## Erros Comuns

- Atualizar estado diretamente durante a renderização.
- Guardar dados importantes em variáveis comuns dentro do componente.
- Achar que toda nova renderização e um bug.
- Fazer cálculos caros sem necessidade em toda renderização.

## Checklist de Entendimento

## Prática Recomendada

- Altere props e observe como a UI mudaria.
- Explique por que uma variável local comum não substitui estado.
