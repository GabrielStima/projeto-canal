## Ideia Central

Web Components sao um conjunto de tecnologias da plataforma Web para criar elementos customizados e reutilizaveis. Diferente de React, eles fazem parte do browser e nao de uma biblioteca especifica.

O objetivo aqui nao e trocar React por Web Components. E entender que o browser tambem oferece mecanismos para componentes, com outro modelo mental e outros trade-offs.

Em projetos reais, Web Components podem aparecer em design systems, widgets reutilizaveis, integracoes entre stacks diferentes ou componentes que precisam funcionar fora de um framework especifico.

## Exemplo Pratico

```js
class CanalBadge extends HTMLElement {
  connectedCallback() {
    this.innerHTML = "<strong>Projeto Canal</strong>";
  }
}

customElements.define("canal-badge", CanalBadge);
```

Depois de registrado, o elemento pode ser usado no HTML:

```html
<canal-badge></canal-badge>
```

## Analogias e Intuicoes

- Web Components criam novas pecas reconhecidas pelo browser.
- React organiza UI com JavaScript e componentes; Web Components usam APIs nativas da plataforma.

## Erros Comuns

- Achar que Web Components sao "React nativo".
- Ignorar diferencas de estado, eventos e renderizacao.
- Usar Shadow DOM sem entender impacto em estilos e testes.
- Escolher Web Components apenas por parecer mais "puro".
