## Ideia Central

Web Components são um conjunto de tecnologias da plataforma Web para criar elementos customizados e reutilizáveis. Diferente de React, eles fazem parte do browser e não de uma biblioteca específica.

O objetivo aqui não é trocar React por Web Components. É entender que o browser também oferece mecanismos para componentes, com outro modelo mental e outros trade-offs.

Em projetos reais, Web Components podem aparecer em design systems, widgets reutilizáveis, integrações entre stacks diferentes ou componentes que precisam funcionar fora de um framework específico.

## Exemplo Prático

```js
class CanalBadge extends HTMLElement {
  connectedCallback() {
    this.innerHTML = "<strong>Projeto Canal</strong>";
  }
}

customElements.define("canal-badge", CanalBadge);
```

Depois de registrado, o elemento pode ser usado no HTML:

```html
<canal-badge></canal-badge>
```

## Analogias e Intuições

- Web Components criam novas peças reconhecidas pelo browser.
- React organiza UI com JavaScript e componentes; Web Components usam APIs nativas da plataforma.

## Erros Comuns

- Achar que Web Components são "React nativo".
- Ignorar diferenças de estado, eventos e renderização.
- Usar Shadow DOM sem entender impacto em estilos e testes.
- Escolher Web Components apenas por parecer mais "puro".
