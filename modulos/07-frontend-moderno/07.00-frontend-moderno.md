## Ideia Central

Frontend essencial te ensina a construir páginas. Frontend moderno te ensina a construir aplicações. A diferença está na complexidade do estado, na quantidade de interações e na necessidade de manter o código organizado à medida que a interface cresce.

Frameworks como React não substituem HTML, CSS e JavaScript — eles organizam o trabalho: dividem a interface em componentes, gerenciam o estado da aplicação e sincronizam a tela com os dados. O resultado é uma forma de construir interfaces que escala.

Este módulo cobre desde o porquê dos frameworks até formulários, testes e performance, passando por React, gerenciamento de estado, rotas e chamadas de API. Cada tema prepara o terreno para o seguinte.

## Analogias e Intuições

- Um componente React é como um bloco de LEGO: encapsula aparência e comportamento, e pode ser reutilizado em qualquer lugar.
- Gerenciar estado global é como ter uma lousa central na clínica: qualquer tela lê e atualiza o mesmo dado sem precisar repassar informação de mão em mão.
- Rotas no frontend são como corredores de um hospital: cada URL leva a uma área diferente, mas o paciente (usuário) não precisa sair do prédio.

## Erros Comuns

- Usar estado global para tudo, inclusive dados locais de um único componente.
- Ignorar acessibilidade ao construir componentes interativos.
- Fazer chamadas de API diretamente dentro de componentes sem abstrair a camada de serviço.
- Não tratar estados de loading e erro na interface.
- Criar componentes grandes demais que misturam apresentação com lógica de negócio.
