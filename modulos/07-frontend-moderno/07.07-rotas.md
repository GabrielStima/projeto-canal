# Rotas

## Ideia Central

Rotas conectam URLs a partes da interface. Em uma aplicação tradicional, navegar para outra URL costuma pedir uma nova página ao servidor. Em uma SPA, o JavaScript no browser pode atualizar a interface sem recarregar tudo.

React não define rotas sozinho. Uma biblioteca como React Router ajuda a mapear caminhos para componentes, criar links internos e lidar com parametros como `/produtos/:id`.

Rotas deixam a aplicação navegável, compartilhável por URL e mais organizada em telas.

## Exemplo Prático

```tsx
import { Link, Route, Routes } from "react-router";

export function App() {
  return (
    <>
      <nav>
        <Link to="/">Dashboard</Link>
        <Link to="/atendimentos">Atendimentos</Link>
        <Link to="/pets">Pets</Link>
      </nav>

      <Routes>
        <Route path="/" element={<Dashboard />} />
        <Route path="/atendimentos" element={<Atendimentos />} />
        <Route path="/pets/:id" element={<DetalhesPet />} />
      </Routes>
    </>
  );
}
```

## Analogias e Intuições

- Rotas são o mapa que liga enderecos a telas.
- `Link` e uma navegação interna que conversa com o router em vez de recarregar a página inteira.

## Erros Comuns

- Usar `<a>` para toda navegação interna sem pensar no recarregamento.
- Criar rotas demais antes de entender as telas reais.
- Guardar estado importante somente em memória quando deveria estar na URL.
- Esquecer página de erro ou rota não encontrada.

## Prática Recomendada

- Crie três telas: inicio, listagem e detalhes.
- Adicione links internos entre elas.
- Crie uma rota de detalhes usando um identificador na URL.
