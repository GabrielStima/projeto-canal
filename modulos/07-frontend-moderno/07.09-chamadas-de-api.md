# Chamadas de API

## Objetivo

Ao final desta aula, você deve ser capaz de:

- buscar dados em uma API a partir do frontend;
- representar carregamento, erro e sucesso na interface;
- entender cuidados basicos com `useEffect` e `fetch`;
- separar dados remotos de estado puramente visual.

## Onde Esta Aula Entra no Curso

- Módulo: 07. Frontend Moderno
- Aula anterior: SPA vs PWA
- Próxima aula: Formulários no frontend moderno
- Pré-requisitos: HTTP, APIs em alto nível, promises, `async/await`, React, hooks e estado.

## Ideia Central

Aplicações frontend modernas raramente vivem sozinhas. Elas precisam buscar dados, enviar formulários, lidar com falhas de rede e mostrar ao usuário o que está acontecendo.

Uma chamada de API não deve ser pensada apenas como "pegar JSON". Ela cria estados de interface: carregando, erro, vazio, sucesso e, às vezes, atualizando.

Nesta etapa, o objetivo é entender o fluxo básico. Bibliotecas especializadas para data fetching podem aparecer depois, mas primeiro o aluno precisa dominar o problema.

## Tópicos da Aula

- `fetch` e promises.
- Estado de carregamento.
- Estado de erro.
- Estado de sucesso.
- Efeito para buscar dados.
- Cancelamento e cuidados introdutórios.

## Exemplo Prático

```tsx
function Produtos() {
  const [status, setStatus] = useState("carregando");
  const [produtos, setProdutos] = useState<Produto[]>([]);

  useEffect(() => {
    async function carregar() {
      try {
        const resposta = await fetch("/api/produtos");
        const dados = await resposta.json();
        setProdutos(dados);
        setStatus("sucesso");
      } catch {
        setStatus("erro");
      }
    }

    carregar();
  }, []);

  if (status === "carregando") return <p>Carregando...</p>;
  if (status === "erro") return <p>Erro ao carregar produtos.</p>;
  return <ListaProdutos produtos={produtos} />;
}
```

## Analogias e Intuições

- Chamada de API é uma conversa com outro sistema.
- A interface precisa mostrar o estado da conversa, não apenas o resultado final.

## Erros Comuns

- Não tratar erro.
- Mostrar tela vazia enquanto carrega.
- Fazer chamada em loop por dependência errada no efeito.
- Misturar dados remotos, estado visual e regra de negócio sem organização.

## Checklist de Entendimento

Antes de seguir, você deve conseguir responder:

- Quais estados uma chamada de API pode ter?
- Por que `async/await` não élimina tratamento de erro?
- Por que buscar dados afeta a UI?
- Quando usar `useEffect` para buscar dados?
- O que pode causar chamadas repetidas sem querer?

## Prática Recomendada

- Busque uma lista de dados de uma API simples.
- Mostre carregamento, erro e sucesso.
- Adicione um estado de lista vazia.
