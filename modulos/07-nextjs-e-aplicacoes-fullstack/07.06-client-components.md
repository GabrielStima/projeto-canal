# Client Components

## Objetivo

Ao final desta aula, voce deve ser capaz de:

- explicar o que e um Client Component;
- usar a diretiva `'use client'`;
- identificar quando uma tela precisa de interatividade no browser;
- evitar transformar a aplicacao inteira em cliente sem necessidade.

## Onde Esta Aula Entra no Curso

- Modulo: 07. Next.js e Aplicacoes Fullstack no Frontend
- Aula anterior: Server Components
- Proxima aula: Composicao entre Server e Client Components
- Pre-requisitos: React, hooks, estado, eventos, componentes e Server Components.

## Ideia Central

Client Components sao componentes que rodam no navegador e podem usar recursos interativos do React, como estado, efeitos e manipuladores de evento.

Eles continuam sendo importantes. O cuidado e usar Client Components nos pontos que precisam de interatividade, mantendo o restante da arvore como Server Component quando possivel.

## Topicos da Aula

- Diretiva `'use client'`.
- Estado local com `useState`.
- Eventos do browser.
- Hooks de cliente.
- Custo de JavaScript no navegador.
- Limite entre servidor e cliente.

## Exemplo Pratico

```tsx
"use client";

import { useState } from "react";

export function Contador() {
  const [valor, setValor] = useState(0);

  return <button onClick={() => setValor(valor + 1)}>{valor}</button>;
}
```

## Analogias e Intuicoes

- Client Component e a parte da tela que precisa responder ao toque do usuario.
- Nem toda informacao exibida precisa virar codigo interativo no browser.

## Erros Comuns

- Colocar `'use client'` no layout raiz sem necessidade.
- Usar Client Component apenas para renderizar texto estatico.
- Buscar todo dado com `useEffect` por costume.
- Esquecer que tudo abaixo de um Client Component entra no mundo do cliente.

## Checklist de Entendimento

Antes de seguir, voce deve conseguir responder:

- Para que serve `'use client'`?
- Quais recursos exigem Client Component?
- Qual o custo de enviar JavaScript ao browser?
- Quando `useState` e necessario?
- Como evitar Client Components grandes demais?

## Pratica Recomendada

- Pegue uma pagina com lista de dados.
- Mantenha a listagem no servidor.
- Crie apenas um filtro ou botao interativo como Client Component.
