# Route Handlers e APIs no Next.js

## Objetivo

Ao final desta aula, voce deve ser capaz de:

- explicar o que e um Route Handler;
- criar um endpoint simples no App Router;
- retornar JSON a partir de uma rota;
- entender o limite entre API interna e backend completo.

## Onde Esta Aula Entra no Curso

- Modulo: 07. Next.js e Aplicacoes Fullstack no Frontend
- Aula anterior: Chamadas de API e busca de dados
- Proxima aula: Formularios em Next.js
- Pre-requisitos: HTTP, metodos HTTP, JSON, rotas, `fetch`, TypeScript e tratamento de erros.

## Ideia Central

Route Handlers permitem criar endpoints dentro da estrutura do App Router. Eles sao uteis para casos de backend-for-frontend: adaptar dados, esconder detalhes de uma API externa, receber formularios simples ou criar pequenas rotas de suporte para a interface.

Eles nao eliminam a necessidade de aprender backend. O modulo de Node.js vai aprofundar APIs, arquitetura, banco de dados, seguranca e operacao.

## Topicos da Aula

## Exemplo Pratico

```tsx
export async function GET() {
  return Response.json([
    { id: "1", titulo: "Estudar Server Components" },
    { id: "2", titulo: "Praticar formularios" },
  ]);
}
```

Crie um endpoint `/api/estudos` para alimentar uma pagina do painel.

## Analogias e Intuicoes

- Route Handler e uma pequena janela de backend dentro do projeto Next.js.
- Backend-for-frontend adapta o servidor para as necessidades da interface.

## Erros Comuns

- Colocar regra de negocio grande demais em Route Handler.
- Ignorar status HTTP.
- Expor segredo ou token para o cliente.
- Confundir API interna com arquitetura backend completa.

## Checklist de Entendimento

Antes de seguir, voce deve conseguir responder:

- Para que serve `route.ts`?
- Qual a diferenca entre `page.tsx` e `route.ts`?
- Quando retornar JSON?
- Por que status HTTP importa?
- Quando mover logica para um backend dedicado?

## Pratica Recomendada

- Crie um Route Handler `GET`.
- Consuma esse endpoint em uma pagina.
- Adicione retorno de erro para um caso simulado.
