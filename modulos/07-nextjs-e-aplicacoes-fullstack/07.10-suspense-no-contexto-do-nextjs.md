# Suspense no contexto do Next.js

## Objetivo

Ao final desta aula, voce deve ser capaz de:

- explicar Suspense em alto nivel;
- entender sua relacao com carregamento assincro;
- usar Suspense para isolar partes da interface;
- diferenciar Suspense de tratamento de erro.

## Onde Esta Aula Entra no Curso

- Modulo: 07. Next.js e Aplicacoes Fullstack no Frontend
- Aula anterior: Error boundaries
- Proxima aula: Chamadas de API e busca de dados
- Pre-requisitos: React, componentes, assincronismo, loading states, streaming e error boundaries.

## Ideia Central

Suspense permite declarar uma interface alternativa enquanto uma parte da arvore ainda nao esta pronta. No contexto do Next.js, ele se conecta com streaming e com a forma como partes assincro da tela podem carregar de maneira independente.

Suspense nao substitui tratamento de erro. Ele lida com espera; error boundaries lidam com falha.

## Topicos da Aula

- Espera em interfaces React.
- `Suspense` e `fallback`.
- Relacao com streaming.
- Componentes assincros.
- Suspense vs error boundary.
- Granularidade de carregamento.

## Exemplo Pratico

```tsx
import { Suspense } from "react";

export default function Page() {
  return (
    <>
      <h1>Painel</h1>
      <Suspense fallback={<p>Carregando resumo...</p>}>
        <ResumoAssincrono />
      </Suspense>
    </>
  );
}
```

## Analogias e Intuicoes

- Suspense e uma reserva de espaco enquanto uma parte da tela chega.
- Loading responde "ainda nao"; erro responde "nao deu certo".

## Erros Comuns

- Usar Suspense como se fosse `try/catch`.
- Criar fallbacks demais e fragmentar a experiencia.
- Mostrar fallback que muda o layout de forma brusca.
- Nao pensar em qual parte realmente precisa carregar separadamente.

## Checklist de Entendimento

Antes de seguir, voce deve conseguir responder:

- O que Suspense espera?
- Para que serve `fallback`?
- Como Suspense se relaciona com streaming?
- Por que Suspense nao trata erro?
- Quando isolar uma parte da tela?

## Pratica Recomendada

- Crie uma pagina com duas areas assincro.
- Coloque Suspense apenas em uma delas.
- Compare a percepcao de carregamento da pagina.
