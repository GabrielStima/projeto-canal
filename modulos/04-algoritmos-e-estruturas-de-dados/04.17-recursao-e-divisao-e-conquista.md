## Ideia Central

Recursão acontece quando uma função resolve um problema chamando a si mesma para uma versão menor do mesmo problema. Para funcionar, ela precisa de um caso base que encerra as chamadas.

Divisao e conquista usa essa ideia para dividir um problema, resolver partes menores e combinar os resultados. Essa estratégia aparece em algoritmos importantes de ordenação e em varias operações com árvores.

## Tópicos da Aula

- Caso base.
- Passo recursivo.
- Pilha de chamadas em alto nível.
- Dividir, resolver e combinar.

## Exemplo Prático

```ts
function sum(values: number[]): number {
  if (values.length === 0) {
    return 0;
  }

  const [first, ...rest] = values;
  return first + sum(rest);
}
```

Este exemplo é didático. Em JavaScript, para arrays grandes, uma versão iterativa pode ser mais segura.

## Analogias e Intuições

- Resolver uma escada olhando um degrau por vez.
- Abrir caixas dentro de caixas até achar uma vazia.
- Dividir uma pilha de provas entre pessoas e juntar os resultados.

## Erros Comuns

- Esquecer o caso base.
- Criar uma chamada recursiva que não reduz o problema.
- Usar recursão quando um loop seria mais simples.
- Ignorar limite de pilha de chamadas.

## Checklist de Entendimento

Antes de seguir, você deve conseguir responder:

- Qual é o caso base?
- O problema fica menor a cada chamada?
- O resultado precisa ser combinado?
- Um loop seria mais claro aqui?

## Prática Recomendada

- Implemente contagem regressiva recursiva.
