# Árvore binária

## Objetivo

Ao final desta aula, você deve ser capaz de:

- explicar o que torna uma árvore binária;
- modelar nós com filho esquerdo e direito;
- percorrer uma árvore binária em nível introdutório;
- preparar a árvore de busca binária.

## Onde Esta Aula Entra na Formação

- Módulo: 04. Algoritmos e Estruturas de Dados
- Aula anterior: Árvores
- Próxima aula: Árvore de busca binária
- Pré-requisitos: árvores, objetos, tipos genéricos e recursão.

## Ideia Central

Árvore binária é uma árvore em que cada nó tem no máximo dois filhos: esquerdo e direito. Essa restrição cria uma estrutura simples para explicar percursos, buscas e outras árvores especializadas.

Ela ainda não define uma regra de ordenação. Isso vem na árvore de busca binária.

## Tópicos da Aula

- Filho esquerdo e filho direito.
- Nó vazio.
- Percurso recursivo.
- Diferença entre árvore binária e árvore de busca binária (BST).

## Exemplo Prático

```ts
type BinaryNode<T> = {
  value: T;
  left: BinaryNode<T> | null;
  right: BinaryNode<T> | null;
};

const root: BinaryNode<string> = {
  value: "A",
  left: { value: "B", left: null, right: null },
  right: { value: "C", left: null, right: null },
};
```

## Analogias e Intuições

- Uma decisão que sempre se divide em duas possibilidades.
- Uma árvore genealógica simplificada.
- Uma estrutura com lado esquerdo e direito.

## Erros Comuns

- Achar que esquerda sempre significa menor.
- Esquecer que filhos podem ser `null`.
- Confundir árvore binária com árvore balanceada.
- Ignorar o caso de árvore vazia.

## Checklist de Entendimento

Antes de seguir, você deve conseguir responder:

- Quantos filhos um nó pode ter?
- O que diferencia árvore binária de árvore comum?
- Uma árvore binária precisa estar ordenada?
- Como a recursão percorre os nós?

## Prática Recomendada

- Modele uma árvore binária com números.
- Escreva uma função que conte nós.
- Escreva uma função que imprima os valores em pré-ordem.
