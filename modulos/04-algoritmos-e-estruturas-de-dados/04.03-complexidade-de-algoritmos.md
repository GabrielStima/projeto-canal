# Complexidade de algoritmos

## Objetivo

Ao final desta aula, voce deve ser capaz de:

- explicar por que medimos complexidade;
- diferenciar tempo de execucao real e crescimento do custo;
- reconhecer ideias como constante, linear e quadratica;
- preparar a leitura dos proximos algoritmos.

## Onde Esta Aula Entra no Curso

- Modulo: 04. Algoritmos e Estruturas de Dados
- Aula anterior: Arrays
- Proxima aula: Tempo vs espaco
- Pre-requisitos: arrays, funcoes, repeticoes e condicionais.

## Ideia Central

Complexidade descreve como o custo de um algoritmo cresce quando a entrada aumenta. Nao e uma promessa de quantos milissegundos o codigo vai levar, mas uma forma de comparar solucoes.

Uma solucao pode ser rapida com 10 itens e ruim com 1 milhao. Complexidade ajuda a perceber esse risco antes que o problema apareca em producao.

## Topicos da Aula

- Entrada como tamanho do problema.
- Custo de operacoes.
- Crescimento: constante, linear, logaritmico, quadratico.
- Big O como linguagem de comparacao.

## Exemplo Pratico

```ts
function hasNumber(numbers: number[], target: number): boolean {
  for (const number of numbers) {
    if (number === target) {
      return true;
    }
  }

  return false;
}
```

Se o numero estiver no final ou nao existir, a funcao pode olhar todos os itens. O custo cresce junto com o tamanho do array.

## Analogias e Intuicoes

- Procurar uma pagina folheando uma a uma.
- Conferir uma lista de presenca.
- Comparar todos contra todos em um campeonato.

## Erros Comuns

- Confundir Big O com tempo exato.
- Analisar apenas o melhor caso.
- Ignorar o tamanho da entrada.
- Decorar nomes sem entender o crescimento.

## Checklist de Entendimento

Antes de seguir, voce deve conseguir responder:

- Qual e a entrada do algoritmo?
- O custo cresce quando a entrada cresce?
- Existe repeticao? Existe repeticao dentro de repeticao?
- Estou analisando melhor caso, pior caso ou caso medio?

## Pratica Recomendada

- Analise uma funcao que percorre um array uma vez.
- Analise uma funcao com dois loops aninhados.
- Explique por que buscar pelo id em um array tende a ser linear.
