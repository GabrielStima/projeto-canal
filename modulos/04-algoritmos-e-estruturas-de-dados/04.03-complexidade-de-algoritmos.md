# Complexidade de algoritmos

## Objetivo

Ao final desta aula, você deve ser capaz de:

- explicar por que medimos complexidade;
- diferenciar tempo de execução real e crescimento do custo;
- reconhecer ideias como constante, linear e quadrática;
- preparar a leitura dos próximos algoritmos.

## Ideia Central

Complexidade descreve como o custo de um algoritmo cresce quando a entrada aumenta. Não é uma promessa de quantos milissegundos o código vai levar, mas uma forma de comparar soluções.

Uma solução pode ser rápida com 10 itens e ruim com 1 milhão. Complexidade ajuda a perceber esse risco antes que o problema apareça em produção.

## Tópicos da Aula

- Entrada como tamanho do problema.
- Custo de operações.
- Crescimento: constante, linear, logarítmico, quadrático.
- Big O como linguagem de comparação.

## Exemplo Prático

```ts
function hasNumber(numbers: number[], target: number): boolean {
  for (const number of numbers) {
    if (number === target) {
      return true;
    }
  }

  return false;
}
```

Se o número estiver no final ou não existir, a função pode olhar todos os itens. O custo cresce junto com o tamanho do array.

## Analogias e Intuições

- Procurar uma página folheando uma a uma.
- Conferir uma lista de presença.
- Comparar todos contra todos em um campeonato.

## Erros Comuns

- Confundir Big O com tempo exato.
- Analisar apenas o melhor caso.
- Ignorar o tamanho da entrada.
- Decorar nomes sem entender o crescimento.

## Checklist de Entendimento

Antes de seguir, você deve conseguir responder:

- Qual é a entrada do algoritmo?
- O custo cresce quando a entrada cresce?
- Existe repetição? Existe repetição dentro de repetição?
- Estou analisando melhor caso, pior caso ou caso médio?

## Prática Recomendada

- Analise uma função que percorre um array uma vez.
- Analise uma função com dois loops aninhados.
- Explique por que buscar pelo id em um array tende a ser linear.
