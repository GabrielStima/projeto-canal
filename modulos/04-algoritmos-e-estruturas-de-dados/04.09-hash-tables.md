## Ideia Central

Hash table e uma estrutura que associa chaves a valores. Em vez de procurar item por item, usamos uma chave para chegar rapidamente ao valor.

Em JavaScript, objetos, `Map` e `Set` resolvem muitos casos praticos. O importante aqui e entender a ideia: trocar memoria e organizacao por acesso rapido.

## Topicos da Aula

- Chave e valor.

- `Map`, objeto e `Set`.

## Exemplo Pratico

```ts
type User = {
  id: string;
  name: string;
};

const usersById = new Map<string, User>();

usersById.set("u1", { id: "u1", name: "Ana" });

const user = usersById.get("u1");
```

Buscar pelo id em um `Map` evita percorrer uma lista inteira de usuarios na maioria dos casos praticos.

## Analogias e Intuicoes

- Um dicionario: palavra como chave, significado como valor.
- Um armario com etiquetas.
- Uma agenda por numero de telefone ou nome.

## Erros Comuns

- Usar objeto comum para todo caso sem considerar `Map`.
- Esquecer que chaves precisam ser consistentes.
- Ignorar o custo de memoria.
- Achar que hash table mantem sempre uma ordem relevante.
