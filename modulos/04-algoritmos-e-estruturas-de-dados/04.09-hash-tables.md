## Ideia Central

Hash table é uma estrutura que associa chaves a valores. Em vez de procurar item por item, usamos uma chave para chegar rapidamente ao valor.

Em JavaScript, objetos, `Map` e `Set` resolvem muitos casos práticos. O importante aqui e entender a ideia: trocar memória e organização por acesso rápido.

## Tópicos da Aula

- `Map`, objeto e `Set`.

## Exemplo Prático

```ts
type User = {
  id: string;
  name: string;
};

const usersById = new Map<string, User>();

usersById.set("u1", { id: "u1", name: "Ana" });

const user = usersById.get("u1");
```

Buscar pelo id em um `Map` evita percorrer uma lista inteira de usuários na maioria dos casos práticos.

## Analogias e Intuições

- Um dicionário: palavra como chave, significado como valor.
- Um armário com etiquetas.
- Uma agenda por número de telefone ou nome.

## Erros Comuns

- Usar objeto comum para todo caso sem considerar `Map`.
- Esquecer que chaves precisam ser consistentes.
- Ignorar o custo de memória.
- Achar que hash table mantem sempre uma ordem relevante.
