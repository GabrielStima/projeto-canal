# Algoritmos de busca

## Objetivo

Ao final desta aula, você deve ser capaz de:

- explicar o problema de busca;
- diferenciar busca em dados desordenados e ordenados;
- reconhecer pré-condições de algoritmos;
- preparar as aulas de busca linear e busca binária.

## Ideia Central

Buscar e encontrar um item que satisfaz uma condicao. Parece simples, mas a melhor estratégia depende de como os dados estão organizados.

Se os dados estão desordenados, talvez seja necessário olhar item por item. Se estão ordenados, podemos usar essa propriedade para eliminar partes da busca.

## Tópicos da Aula

- O que significa buscar.
- Chave de busca e critério.
- Dados ordenados vs desordenados.
- Pre-condicoes.

## Exemplo Prático

```ts
type Contact = {
  name: string;
  email: string;
};

function findContactByEmail(contacts: Contact[], email: string): Contact | undefined {
  return contacts.find((contact) => contact.email === email);
}
```

Essa busca não assume ordenação. Ela apenas verifica cada contato até encontrar o email.

## Analogias e Intuições

- Procurar um nome em uma lista sem ordem.
- Procurar uma palavra em um dicionário ordenado.
- Usar um índice para não varrer tudo.

## Erros Comuns

- Aplicar busca binária em dados não ordenados.
- Não definir claramente o critério de busca.
- Otimizar busca pequena sem necessidade.
- Ignorar alternativas como `Map` quando a busca por chave e frequente.

## Checklist de Entendimento

Antes de seguir, você deve conseguir responder:

- O que estou procurando?
- Os dados estão ordenados?
- A busca acontece uma vez ou muitas vezes?
- Existe estrutura melhor para esse tipo de acesso?

## Prática Recomendada

- Identifique quais buscas poderiam usar `Map`.
- Explique por que ordenação muda a estratégia.
