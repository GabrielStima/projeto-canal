# Algoritmos de busca

## Objetivo

Ao final desta aula, voce deve ser capaz de:

- explicar o problema de busca;
- diferenciar busca em dados desordenados e ordenados;
- reconhecer pre-condicoes de algoritmos;
- preparar as aulas de busca linear e busca binaria.

## Onde Esta Aula Entra no Curso

- Modulo: 04. Algoritmos e Estruturas de Dados
- Aula anterior: Hash tables
- Proxima aula: Busca linear

## Ideia Central

Buscar e encontrar um item que satisfaz uma condicao. Parece simples, mas a melhor estrategia depende de como os dados estao organizados.

Se os dados estao desordenados, talvez seja necessario olhar item por item. Se estao ordenados, podemos usar essa propriedade para eliminar partes da busca.

## Topicos da Aula

- O que significa buscar.
- Chave de busca e criterio.
- Dados ordenados vs desordenados.
- Pre-condicoes.

## Exemplo Pratico

```ts
type Contact = {
  name: string;
  email: string;
};

function findContactByEmail(contacts: Contact[], email: string): Contact | undefined {
  return contacts.find((contact) => contact.email === email);
}
```

Essa busca nao assume ordenacao. Ela apenas verifica cada contato ate encontrar o email.

## Analogias e Intuicoes

- Procurar um nome em uma lista sem ordem.
- Procurar uma palavra em um dicionario ordenado.
- Usar um indice para nao varrer tudo.

## Erros Comuns

- Aplicar busca binaria em dados nao ordenados.
- Nao definir claramente o criterio de busca.
- Otimizar busca pequena sem necessidade.
- Ignorar alternativas como `Map` quando a busca por chave e frequente.

## Checklist de Entendimento

Antes de seguir, voce deve conseguir responder:

- O que estou procurando?
- Os dados estao ordenados?
- A busca acontece uma vez ou muitas vezes?
- Existe estrutura melhor para esse tipo de acesso?

## Pratica Recomendada

- Escreva buscas por id, nome e status em arrays de objetos.
- Identifique quais buscas poderiam usar `Map`.
- Explique por que ordenacao muda a estrategia.
