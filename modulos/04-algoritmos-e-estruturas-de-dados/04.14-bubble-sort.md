## Ideia Central

Bubble Sort compara pares vizinhos e troca quando estão fora de ordem. A cada passada, um valor "sobe" para sua posição correta, como uma bolha.

Ele é fácil de visualizar, mas costuma ser ineficiente para entradas grandes porque faz muitas comparações e trocas.

## Tópicos da Aula

- Comparar vizinhos.
- Trocar valores.
- Passadas pelo array.
- Complexidade quadrática.

## Exemplo Prático

```ts
function bubbleSort(values: number[]): number[] {
  const result = [...values];

  for (let i = 0; i < result.length; i++) {
    for (let j = 0; j < result.length - 1 - i; j++) {
      if (result[j] > result[j + 1]) {
        const temp = result[j];
        result[j] = result[j + 1];
        result[j + 1] = temp;
      }
    }
  }

  return result;
}
```

## Analogias e Intuições

- Comparar pessoas lado a lado em uma fila.
- Empurrar o maior valor para o final.
- Ajustar a ordem por pequenas trocas locais.

## Erros Comuns

- Esquecer o limite `result.length - 1 - i`.
- Alterar o array original sem perceber.
- Achar que Bubble Sort é bom para uso geral.
- Errar a troca e perder um valor.

## Checklist de Entendimento

Antes de seguir, você deve conseguir responder:

- O que é comparado em cada passo?
- Por que ha dois loops?
- Qual valor fica correto ao final de uma passada?
- Por que o custo tende a ser `O(n²)`?

## Prática Recomendada

- Rode Bubble Sort manualmente em `[5, 1, 4, 2]`.
- Adicione um contador de comparações.
- Implemente uma otimização que pare se nenhuma troca acontecer.
