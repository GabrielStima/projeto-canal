## Ideia Central

Bubble Sort compara pares vizinhos e troca quando estao fora de ordem. A cada passada, um valor "sobe" para sua posicao correta, como uma bolha.

Ele e facil de visualizar, mas costuma ser ineficiente para entradas grandes porque faz muitas comparacoes e trocas.

## Topicos da Aula

- Comparar vizinhos.
- Trocar valores.
- Passadas pelo array.
- Complexidade quadratica.

## Exemplo Pratico

```ts
function bubbleSort(values: number[]): number[] {
  const result = [...values];

  for (let i = 0; i < result.length; i++) {
    for (let j = 0; j < result.length - 1 - i; j++) {
      if (result[j] > result[j + 1]) {
        const temp = result[j];
        result[j] = result[j + 1];
        result[j + 1] = temp;
      }
    }
  }

  return result;
}
```

## Analogias e Intuicoes

- Comparar pessoas lado a lado em uma fila.
- Empurrar o maior valor para o final.
- Ajustar a ordem por pequenas trocas locais.

## Erros Comuns

- Esquecer o limite `result.length - 1 - i`.
- Alterar o array original sem perceber.
- Achar que Bubble Sort e bom para uso geral.
- Errar a troca e perder um valor.

## Checklist de Entendimento

Antes de seguir, voce deve conseguir responder:

- O que e comparado em cada passo?
- Por que ha dois loops?
- Qual valor fica correto ao final de uma passada?
- Por que o custo tende a ser `O(n²)`?

## Pratica Recomendada

- Rode Bubble Sort manualmente em `[5, 1, 4, 2]`.
- Adicione um contador de comparacoes.
- Implemente uma otimizacao que pare se nenhuma troca acontecer.
