# Algoritmos de ordenacao

## Objetivo

Ao final desta aula, voce deve ser capaz de:

- explicar o problema de ordenacao;
- reconhecer criterios de comparacao;
- entender por que existem varios algoritmos de ordenacao;
- preparar as aulas de Bubble, Selection, Insertion, Merge, Quick e Heap Sort.

## Onde Esta Aula Entra no Curso

- Modulo: 04. Algoritmos e Estruturas de Dados
- Aula anterior: Busca binaria
- Proxima aula: Bubble Sort
- Pre-requisitos: arrays, busca, complexidade e funcoes de comparacao.

## Ideia Central

Ordenar e reorganizar dados segundo um criterio: menor para maior, data mais recente, nome alfabetico, prioridade ou qualquer regra comparavel.

Algoritmos de ordenacao existem porque diferentes estrategias tem custos, vantagens e dificuldades diferentes. Alguns sao didaticos, outros sao melhores para entradas grandes, outros brilham em casos quase ordenados.

## Topicos da Aula

- Criterio de ordenacao.
- Comparacao e troca.
- Estabilidade em alto nivel.
- In-place vs memoria auxiliar.
- Por que estudar algoritmos que raramente usamos diretamente.

## Exemplo Pratico

```ts
type Task = {
  title: string;
  priority: number;
};

const tasks: Task[] = [
  { title: "Estudar", priority: 2 },
  { title: "Entregar projeto", priority: 1 },
];

tasks.sort((a, b) => a.priority - b.priority);
```

Mesmo usando `sort`, entender ordenacao ajuda a compreender custo, criterio e efeitos sobre a colecao.

## Analogias e Intuicoes

- Organizar livros por titulo.
- Ordenar tarefas por prioridade.
- Colocar cartas em ordem na mao.

## Erros Comuns

- Ordenar sem definir criterio claro.
- Achar que todo sort tem o mesmo custo.
- Esquecer que ordenar pode alterar o array original.
- Confundir ordenacao com busca.

## Checklist de Entendimento

Antes de seguir, voce deve conseguir responder:

- Qual criterio define a ordem?
- A ordenacao altera a entrada original?
- O algoritmo usa memoria auxiliar?
- Por que ordenacao pode ajudar buscas futuras?

## Pratica Recomendada

- Ordene numeros, strings e objetos por criterios diferentes.
- Explique quando a ordem importa para o usuario.
- Compare ordenar uma copia e ordenar o array original.
