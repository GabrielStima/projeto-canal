# Algoritmos de ordenação

## Objetivo

Ao final desta aula, você deve ser capaz de:

- explicar o problema de ordenação;
- reconhecer critérios de comparação;
- entender por que existem vários algoritmos de ordenação;
- preparar as aulas de Bubble, Selection, Insertion, Merge, Quick e Heap Sort.

## Onde Esta Aula Entra no Curso

- Aula anterior: Busca binária

- Pré-requisitos: arrays, busca, complexidade e funções de comparação.

## Ideia Central

Ordenar e reorganizar dados segundo um critério: menor para maior, data mais recente, nome alfabético, prioridade ou qualquer regra comparável.

Algoritmos de ordenação existem porque diferentes estratégias tem custos, vantagens e dificuldades diferentes. Alguns são didáticos, outros são melhores para entradas grandes, outros brilham em casos quase ordenados.

## Tópicos da Aula

- Criterio de ordenação.
- Comparação e troca.
- Estabilidade em alto nível.
- In-place vs memória auxiliar.
- Por que estudar algoritmos que raramente usamos diretamente.

## Exemplo Prático

```ts
type Task = {
  title: string;
  priority: number;
};

const tasks: Task[] = [
  { title: "Estudar", priority: 2 },
  { title: "Entregar projeto", priority: 1 },
];

tasks.sort((a, b) => a.priority - b.priority);
```

Mesmo usando `sort`, entender ordenação ajuda a compreender custo, critério e efeitos sobre a coleção.

## Analogias e Intuições

- Organizar livros por titulo.
- Ordenar tarefas por prioridade.
- Colocar cartas em ordem na mão.

## Erros Comuns

- Ordenar sem definir critério claro.
- Achar que todo sort tem o mesmo custo.
- Esquecer que ordenar pode alterar o array original.
- Confundir ordenação com busca.

## Checklist de Entendimento

Antes de seguir, você deve conseguir responder:

- Qual critério define a ordem?
- A ordenação altera a entrada original?
- O algoritmo usa memória auxiliar?
- Por que ordenação pode ajudar buscas futuras?

## Prática Recomendada

- Ordene números, strings e objetos por critérios diferentes.
- Explique quando a ordem importa para o usuário.
- Compare ordenar uma copia e ordenar o array original.
