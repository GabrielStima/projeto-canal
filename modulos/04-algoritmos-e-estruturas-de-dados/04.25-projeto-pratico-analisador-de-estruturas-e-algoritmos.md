# Projeto prático: analisador de estruturas e algoritmos

## Objetivo

Ao final desta aula, você deve ser capaz de:

- integrar estruturas e algoritmos do módulo;
- implementar comparações simples de passos;
- explicar trade-offs entre soluções;
- revisar o módulo com uma prática concreta em TypeScript.

## Onde Esta Aula Entra na Formação

- Módulo: 04. Algoritmos e Estruturas de Dados
- Aula anterior: Heap Sort
- Próxima aula: 05. Ferramentas de Desenvolvimento
- Pré-requisitos: arrays, complexidade, estruturas lineares, hash tables, busca, ordenação, árvores e heaps.

## Ideia Central

O projeto fecha o módulo colocando as ideias em contato. Em vez de implementar algoritmos isolados, você vai criar um pequeno analisador que executa soluções, registra passos e compara escolhas.

O objetivo não é criar benchmark profissional. O objetivo é treinar leitura: qual estrutura foi usada, qual operação domina, que custo aparece e que trade-off foi feito.

## Tópicos da Aula

- Organizacao do projeto em TypeScript.
- Dados de entrada controlados.
- Contadores de passos.
- Comparação entre busca, ordenação e estruturas.
- Relatorio final em texto.

## Exemplo Prático

```ts
type AlgorithmResult<T> = {
  name: string;
  result: T;
  steps: number;
};

function measuredLinearSearch(values: number[], target: number): AlgorithmResult<number> {
  let steps = 0;

  for (let index = 0; index < values.length; index++) {
    steps++;

    if (values[index] === target) {
      return { name: "linearSearch", result: index, steps };
    }
  }

  return { name: "linearSearch", result: -1, steps };
}
```

## Analogias e Intuições

- Um laboratorio pequeno para testar hipoteses.
- Uma ficha de avaliação para cada solução.
- Um diario de bordo explicando decisões tecnicas.

## Erros Comuns

- Transformar o projeto em ferramenta grande demais.
- Comparar algoritmos com entradas diferentes.
- Medir apenas tempo real e ignorar passos.
- Copiar implementações sem explicar o raciocínio.

## Checklist de Entendimento

Antes de seguir, você deve conseguir responder:

- O que cada algoritmo está tentando resolver?
- Qual estrutura de dados ele usa?
- Qual operação domina o custo?
- Que trade-off aparece entre tempo, espaço e clareza?

## Prática Recomendada

- Implemente busca linear e busca binária com contador de passos.
- Implemente pelo menos três ordenações do módulo.
- Modele stack, queue, hash table com `Map` é uma árvore de busca binária (BST) simples.
- Escreva um relatório comparando os resultados.
