# Projeto prático: analisador de estruturas e algoritmos

## Objetivo

Ao final desta aula, você deve ser capaz de:

- integrar estruturas e algoritmos do módulo;
- implementar comparações simples de passos;
- explicar trade-offs entre soluções;
- revisar o módulo com uma prática concreta em TypeScript.

## Onde Esta Aula Entra na Formação

- Módulo: 04. Algoritmos e Estruturas de Dados
- Aula anterior: Heap Sort
- Próxima aula: 05. Ferramentas de Desenvolvimento
- Pré-requisitos: arrays, complexidade, estruturas lineares, hash tables, busca, ordenação, árvores e heaps.

## Ideia Central

O projeto fecha o módulo colocando as ideias em contato. Em vez de implementar algoritmos isolados, você vai criar um pequeno analisador que executa soluções, registra passos e compara escolhas.

O objetivo não é criar benchmark profissional. O objetivo é treinar leitura: qual estrutura foi usada, qual operação domina, que custo aparece e que trade-off foi feito.

Você vai aplicar essas ideias em um contexto de clínica veterinária, usando dados de atendimentos, estoque e triagem. Não é preciso ter feito o projeto do módulo anterior: os tipos e dados necessários serão fornecidos aqui.

## Tópicos da Aula

- Organização do projeto em TypeScript.
- Dados de entrada controlados.
- Contadores de passos.
- Comparação entre busca, ordenação e estruturas.
- Relatório final em texto.

## Exemplo Prático

```ts
type AlgorithmResult<T> = {
  name: string;
  result: T;
  steps: number;
};

function measuredLinearSearch(values: number[], target: number): AlgorithmResult<number> {
  let steps = 0;

  for (let index = 0; index < values.length; index++) {
    steps++;

    if (values[index] === target) {
      return { name: "linearSearch", result: index, steps };
    }
  }

  return { name: "linearSearch", result: -1, steps };
}
```

## Analogias e Intuições

- Um laboratório pequeno para testar hipóteses.
- Uma ficha de avaliação para cada solução.
- Um diário de bordo explicando decisões técnicas.

## Erros Comuns

- Transformar o projeto em ferramenta grande demais.
- Comparar algoritmos com entradas diferentes.
- Medir apenas tempo real e ignorar passos.
- Copiar implementações sem explicar o raciocínio.

## Checklist de Entendimento

Antes de seguir, você deve conseguir responder:

- O que cada algoritmo está tentando resolver?
- Qual estrutura de dados ele usa?
- Qual operação domina o custo?
- Que trade-off aparece entre tempo, espaço e clareza?

## Prática Recomendada

Nesta prática de síntese do módulo, você vai construir um analisador de estruturas e algoritmos usando dados de uma clínica veterinária. **Essa atividade é recomendada para consolidar o módulo, mas você pode estudar as aulas e os exercícios anteriores sem fazê-la.**

**O que você já sabe/possui:**

- TypeScript com arrays, objetos, funções, classes e tipos.
- Arrays, linked lists, stacks, queues, hash tables, busca linear, busca binária, algoritmos de ordenação, árvores, árvore de busca binária, heaps e Heap Sort.

**Tarefa atual:**

Crie um pequeno programa em TypeScript puro que analise diferentes estruturas e algoritmos usando dados de uma clínica veterinária. O programa deve:

1. Representar atendimentos com `id`, `petName`, `arrivalOrder` e `urgency`.
2. Representar itens de estoque com `code`, `name` e `quantity`.
3. Implementar busca linear e busca binária por id de atendimento, contando passos.
4. Implementar pelo menos três algoritmos de ordenação e ordenar atendimentos por urgência.
5. Modelar uma stack, uma queue, uma hash table com `Map` e uma árvore de busca binária simples.
6. Escrever um relatório comparando tempo, espaço e clareza de cada solução.

**Exercício/resultado a produzir:**

Um arquivo `analisador.ts` (ou vários arquivos organizados em módulos) com:

- tipos de dados;
- funções de busca com contador de passos;
- funções de ordenação;
- modelos de stack, queue, hash table e BST;
- um relatório em texto com as comparações.
