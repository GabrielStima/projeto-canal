# Projeto pratico: analisador de estruturas e algoritmos

## Objetivo

Ao final desta aula, voce deve ser capaz de:

- integrar estruturas e algoritmos do modulo;
- implementar comparacoes simples de passos;
- explicar trade-offs entre solucoes;
- revisar o modulo com uma pratica concreta em TypeScript.

## Onde Esta Aula Entra no Curso

- Modulo: 04. Algoritmos e Estruturas de Dados
- Aula anterior: Heap Sort
- Proxima aula: 05. Ferramentas de Desenvolvimento
- Pre-requisitos: arrays, complexidade, estruturas lineares, hash tables, busca, ordenacao, arvores e heaps.

## Ideia Central

O projeto fecha o modulo colocando as ideias em contato. Em vez de implementar algoritmos isolados, voce vai criar um pequeno analisador que executa solucoes, registra passos e compara escolhas.

O objetivo nao e criar benchmark profissional. O objetivo e treinar leitura: qual estrutura foi usada, qual operacao domina, que custo aparece e que trade-off foi feito.

## Topicos da Aula

- Organizacao do projeto em TypeScript.
- Dados de entrada controlados.
- Contadores de passos.
- Comparacao entre busca, ordenacao e estruturas.
- Relatorio final em texto.

## Exemplo Pratico

```ts
type AlgorithmResult<T> = {
  name: string;
  result: T;
  steps: number;
};

function measuredLinearSearch(values: number[], target: number): AlgorithmResult<number> {
  let steps = 0;

  for (let index = 0; index < values.length; index++) {
    steps++;

    if (values[index] === target) {
      return { name: "linearSearch", result: index, steps };
    }
  }

  return { name: "linearSearch", result: -1, steps };
}
```

## Analogias e Intuicoes

- Um laboratorio pequeno para testar hipoteses.
- Uma ficha de avaliacao para cada solucao.
- Um diario de bordo explicando decisoes tecnicas.

## Erros Comuns

- Transformar o projeto em ferramenta grande demais.
- Comparar algoritmos com entradas diferentes.
- Medir apenas tempo real e ignorar passos.
- Copiar implementacoes sem explicar o raciocinio.

## Checklist de Entendimento

Antes de seguir, voce deve conseguir responder:

- O que cada algoritmo esta tentando resolver?
- Qual estrutura de dados ele usa?
- Qual operacao domina o custo?
- Que trade-off aparece entre tempo, espaco e clareza?

## Pratica Recomendada

- Implemente busca linear e busca binaria com contador de passos.
- Implemente pelo menos tres ordenacoes do modulo.
- Modele stack, queue, hash table com `Map` e uma árvore de busca binária (BST) simples.
- Escreva um relatorio comparando os resultados.
