# Árvore de busca binária

## Objetivo

Ao final desta aula, você deve ser capaz de:

- implementar inserção e busca em nível introdutório;
- relacionar árvore de busca binária (BST) com busca binária;
- reconhecer o risco de árvore desbalanceada.

## Ideia Central

Árvore de busca binária (BST) é uma árvore binária com regra de ordenação: valores menores ficam de um lado e valores maiores ficam do outro.

Essa regra permite buscar descartando partes da árvore, parecido com a ideia da busca binária. O desempenho depende muito do formato da árvore.

## Tópicos da Aula

- Regra esquerda/menor e direita/maior.
- Inserção.
- Busca.
- Árvore balanceada vs desbalanceada em alto nível.

## Exemplo Prático

```ts
type AppointmentNode = {
  id: number;
  petName: string;
  left: AppointmentNode | null;
  right: AppointmentNode | null;
};

function search(root: AppointmentNode | null, targetId: number): AppointmentNode | null {
  if (root === null) {
    return null;
  }

  if (root.id === targetId) {
    return root;
  }

  if (targetId < root.id) {
    return search(root.left, targetId);
  }

  return search(root.right, targetId);
}
```

Neste exemplo, cada nó representa um atendimento identificado por `id`. A busca decide ir para a esquerda ou para a direita comparando ids. Se os atendimentos forem inseridos em ordem crescente, a árvore fica desbalanceada e perde a vantagem da busca rápida.

## Analogias e Intuições

- Um jogo de maior ou menor em forma de árvore.
- Um dicionário com caminhos de decisão.
- Um índice que elimina metade idealmente, mas nem sempre perfeitamente.

## Erros Comuns

- Quebrar a regra de ordenação ao inserir.
- Achar que toda árvore de busca binária (BST) é automaticamente eficiente.
- Ignorar valores duplicados.
- Confundir árvore de busca binária (BST) com heap.

- Implemente inserção de números.
- Teste inserir valores já ordenados e observe o formato resultante.
