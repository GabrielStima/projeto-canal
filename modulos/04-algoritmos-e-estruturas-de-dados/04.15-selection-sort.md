## Ideia Central

Selection Sort divide o array em uma parte ordenada e outra nao ordenada. A cada passo, ele procura o menor item da parte nao ordenada e coloca na proxima posicao correta.

Ele reduz o numero de trocas em relacao ao Bubble Sort, mas ainda precisa fazer muitas comparacoes.

## Topicos da Aula

- Parte ordenada e parte nao ordenada.
- Encontrar o menor valor.
- Trocar com a posicao atual.
- Complexidade quadratica.

## Exemplo Pratico

```ts
function selectionSort(values: number[]): number[] {
  const result = [...values];

  for (let i = 0; i < result.length; i++) {
    let minIndex = i;

    for (let j = i + 1; j < result.length; j++) {
      if (result[j] < result[minIndex]) {
        minIndex = j;
      }
    }

    const temp = result[i];
    result[i] = result[minIndex];
    result[minIndex] = temp;
  }

  return result;
}
```

## Analogias e Intuicoes

- Escolher a menor carta e colocar no inicio.
- Montar uma fila do menor para o maior.
- Separar o que ja esta ordenado do que ainda falta.

## Erros Comuns

- Trocar dentro do loop interno antes de terminar a busca.
- Nao inicializar `minIndex` corretamente.
- Achar que menos trocas elimina o custo quadratico.
- Confundir Selection Sort com Insertion Sort.

## Checklist de Entendimento

- O que o algoritmo seleciona a cada rodada?
- Qual parte ja esta ordenada?
- Quantas buscas pelo menor acontecem?
- Por que o custo ainda e `O(n²)`?
