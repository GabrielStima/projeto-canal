# Tempo vs espaço

## Objetivo

Ao final desta aula, você deve ser capaz de:

- diferenciar complexidade de tempo e de espaço;
- reconhecer trade-offs entre velocidade e memória;
- explicar quando vale guardar dados auxiliares;
- analisar exemplos simples em TypeScript.

## Onde Esta Aula Entra na Formação

- Módulo: 04. Algoritmos e Estruturas de Dados
- Aula anterior: Complexidade de algoritmos
- Próxima aula: Como calcular complexidade

## Ideia Central

Tempo mede o quanto de trabalho o algoritmo faz. Espaco mede quanta memória adicional ele usa. Muitas soluções melhoram um lado piorando o outro.

Guardar um mapa de ids pode acelerar buscas, mas consome memória. Ordenar uma copia de um array preserva o original, mas duplica parte do uso de memória.

## Tópicos da Aula

- Custo de tempo.
- Custo de espaço.
- Memória auxiliar.
- Trade-offs comuns em projetos.

## Exemplo Prático

```ts
function hasDuplicate(values: string[]): boolean {
  const seen = new Set<string>();

  for (const value of values) {
    if (seen.has(value)) {
      return true;
    }

    seen.add(value);
  }

  return false;
}
```

O `Set` usa memória extra, mas evita comparar cada item com todos os outros.

## Analogias e Intuições

- Usar uma anotacao para não repetir uma conferencia.
- Separar itens em caixas para encontrar mais rápido.
- Fazer copia de seguranca antes de mexer no original.

## Erros Comuns

- Achar que a solução mais rápida sempre e melhor.
- Ignorar memória porque o exemplo e pequeno.
- Contar a entrada como memória auxiliar sem critério.
- Trocar clareza por micro-otimização cedo demais.

## Checklist de Entendimento

Antes de seguir, você deve conseguir responder:

- O algoritmo usa estrutura auxiliar?
- Essa estrutura cresce com a entrada?
- O ganho de tempo compensa o custo de memória?
- O problema real precisa dessa otimização?

## Prática Recomendada

- Resolva deteccao de duplicados com dois loops e depois com `Set`.
- Compare as duas soluções em tempo, espaço e clareza.
- Explique quando você escolheria a solução mais simples.
