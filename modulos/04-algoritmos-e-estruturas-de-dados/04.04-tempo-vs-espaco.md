# Tempo vs espaco

## Objetivo

Ao final desta aula, voce deve ser capaz de:

- diferenciar complexidade de tempo e de espaco;
- reconhecer trade-offs entre velocidade e memoria;
- explicar quando vale guardar dados auxiliares;
- analisar exemplos simples em TypeScript.

## Onde Esta Aula Entra no Curso

- Modulo: 04. Algoritmos e Estruturas de Dados
- Aula anterior: Complexidade de algoritmos
- Proxima aula: Como calcular complexidade

## Ideia Central

Tempo mede o quanto de trabalho o algoritmo faz. Espaco mede quanta memoria adicional ele usa. Muitas solucoes melhoram um lado piorando o outro.

Guardar um mapa de ids pode acelerar buscas, mas consome memoria. Ordenar uma copia de um array preserva o original, mas duplica parte do uso de memoria.

## Topicos da Aula

- Custo de tempo.
- Custo de espaco.
- Memoria auxiliar.
- Trade-offs comuns em projetos.

## Exemplo Pratico

```ts
function hasDuplicate(values: string[]): boolean {
  const seen = new Set<string>();

  for (const value of values) {
    if (seen.has(value)) {
      return true;
    }

    seen.add(value);
  }

  return false;
}
```

O `Set` usa memoria extra, mas evita comparar cada item com todos os outros.

## Analogias e Intuicoes

- Usar uma anotacao para nao repetir uma conferencia.
- Separar itens em caixas para encontrar mais rapido.
- Fazer copia de seguranca antes de mexer no original.

## Erros Comuns

- Achar que a solucao mais rapida sempre e melhor.
- Ignorar memoria porque o exemplo e pequeno.
- Contar a entrada como memoria auxiliar sem criterio.
- Trocar clareza por micro-otimizacao cedo demais.

## Checklist de Entendimento

Antes de seguir, voce deve conseguir responder:

- O algoritmo usa estrutura auxiliar?
- Essa estrutura cresce com a entrada?
- O ganho de tempo compensa o custo de memoria?
- O problema real precisa dessa otimizacao?

## Pratica Recomendada

- Resolva deteccao de duplicados com dois loops e depois com `Set`.
- Compare as duas solucoes em tempo, espaco e clareza.
- Explique quando voce escolheria a solucao mais simples.
