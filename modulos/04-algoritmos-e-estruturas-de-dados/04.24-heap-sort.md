## Ideia Central

Heap Sort usa um heap para transformar o maior item em algo facil de acessar. Em uma estrategia com max-heap, o maior valor fica no topo; entao ele pode ser colocado no final da area ordenada.

Depois de remover o topo, o heap precisa ser reorganizado. Repetindo esse processo, o array fica ordenado.

## Topicos da Aula

- Construir um heap.
- Remover ou trocar o topo.
- Restaurar a propriedade de heap.
- Complexidade `O(n log n)`.

## Exemplo Pratico

```ts
function swap(values: number[], first: number, second: number): void {
  const temp = values[first];
  values[first] = values[second];
  values[second] = temp;
}
```

Heap Sort completo exige operacoes auxiliares como `heapify`. Nesta aula, o foco e entender o fluxo: criar heap, mover maior para o final e restaurar heap.

## Analogias e Intuicoes

- Sempre pegar o item de maior prioridade.
- Colocar o maior item na posicao final correta.
- Reorganizar a fila de prioridade apos cada remocao.

## Erros Comuns

- Tentar entender Heap Sort sem entender heap.
- Confundir propriedade de heap com array totalmente ordenado.
- Esquecer de reduzir a area do heap apos cada troca.
- Perder indices de pai e filhos.

## Checklist de Entendimento

Antes de seguir, voce deve conseguir responder:

- Por que usamos max-heap para ordenar crescente?
- O que acontece apos mover o topo para o final?
- Por que precisamos restaurar o heap?
- Como Heap Sort difere de Merge Sort e Quick Sort?
