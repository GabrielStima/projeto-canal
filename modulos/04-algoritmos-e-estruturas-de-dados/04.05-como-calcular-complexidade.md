# Como calcular complexidade

## Objetivo

Ao final desta aula, voce deve ser capaz de:

- identificar a entrada principal de um algoritmo;
- contar operacoes relevantes em exemplos simples;
- simplificar a analise para Big O;
- evitar erros comuns ao analisar loops e chamadas auxiliares.

## Ideia Central

Calcular complexidade e observar como o codigo trabalha conforme a entrada cresce. Nao precisamos contar cada detalhe com precisao absoluta; queremos entender o comportamento dominante.

Constantes e detalhes pequenos costumam ser descartados em Big O. O que importa e o padrao de crescimento: uma vez, varias vezes, metade a cada passo, todos contra todos.

## Topicos da Aula

- Definir `n`.
- Analisar sequencias de comandos.
- Analisar loops simples e aninhados.
- Simplificar termos dominantes.

## Exemplo Pratico

```ts
function printPairs(values: string[]): void {
  for (const first of values) {
    for (const second of values) {
      console.log(first, second);
    }
  }
}
```

Se o array tem `n` itens, o loop externo roda `n` vezes e o interno tambem roda `n` vezes para cada rodada externa. O custo cresce como `n * n`.

## Analogias e Intuicoes

- Conferir uma lista uma vez.
- Comparar cada pessoa com todas as outras.
- Cortar um problema pela metade repetidamente.

## Erros Comuns

- Chamar todo loop de `O(n)` sem olhar aninhamento.
- Somar custos e esquecer o termo dominante.
- Ignorar funcoes chamadas dentro do loop.
- Analisar com base em um exemplo pequeno demais.

## Checklist de Entendimento

Antes de seguir, voce deve conseguir responder:

- O que representa `n`?
- Quantas vezes o trecho principal roda?
- Existe loop dentro de loop?
- Alguma estrutura auxiliar cresce com a entrada?

## Pratica Recomendada

- Analise 5 funcoes pequenas com loops diferentes.
- Reescreva uma solucao quadratica usando uma estrutura auxiliar.
- Explique em voz alta por que constantes sao descartadas em Big O.
