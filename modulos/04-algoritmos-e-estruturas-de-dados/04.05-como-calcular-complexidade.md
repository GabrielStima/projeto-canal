# Como calcular complexidade

## Objetivo

- simplificar a análise para Big O;
- evitar erros comuns ao analisar loops e chamadas auxiliares.

## Ideia Central

Calcular complexidade e observar como o código trabalha conforme a entrada cresce. Não precisamos contar cada detalhe com precisao absoluta; queremos entender o comportamento dominante.

Constantes e detalhes pequenos costumam ser descartados em Big O. O que importa e o padrao de crescimento: uma vez, varias vezes, metade a cada passo, todos contra todos.

## Tópicos da Aula

- Definir `n`.
- Analisar sequencias de comandos.
- Analisar loops simples e aninhados.
- Simplificar termos dominantes.

## Exemplo Prático

```ts
function printPairs(values: string[]): void {
  for (const first of values) {
    for (const second of values) {
      console.log(first, second);
    }
  }
}
```

Se o array tem `n` itens, o loop externo roda `n` vezes e o interno também roda `n` vezes para cada rodada externa. O custo cresce como `n * n`.

## Analogias e Intuições

- Conferir uma lista uma vez.
- Comparar cada pessoa com todas as outras.
- Cortar um problema pela metade repetidamente.

## Erros Comuns

- Chamar todo loop de `O(n)` sem olhar aninhamento.
- Somar custos e esquecer o termo dominante.
- Ignorar funções chamadas dentro do loop.
- Analisar com base em um exemplo pequeno demais.

## Checklist de Entendimento

Antes de seguir, você deve conseguir responder:

- O que representa `n`?
- Quantas vezes o trecho principal roda?
- Existe loop dentro de loop?
- Alguma estrutura auxiliar cresce com a entrada?

## Prática Recomendada

- Analise 5 funções pequenas com loops diferentes.
- Reescreva uma solução quadrática usando uma estrutura auxiliar.
- Explique em voz alta por que constantes são descartadas em Big O.
