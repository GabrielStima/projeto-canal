- entender por que ela reduz o problema pela metade;

## Ideia Central

Busca binaria funciona quando os dados estao ordenados. Em vez de olhar item por item, ela compara com o elemento do meio e descarta metade da busca.

Esse descarte repetido torna a busca muito eficiente, mas apenas quando a pre-condicao e verdadeira. Em dados desordenados, o algoritmo pode retornar resultado errado.

## Topicos da Aula

- Array ordenado.
- Inicio, meio e fim.
- Descartar metade.
- Complexidade logaritmica.

## Exemplo Pratico

```ts
function binarySearch(values: number[], target: number): number {
  let start = 0;
  let end = values.length - 1;

  while (start <= end) {
    const middle = Math.floor((start + end) / 2);

    if (values[middle] === target) {
      return middle;
    }

    if (values[middle] < target) {
      start = middle + 1;
    } else {
      end = middle - 1;
    }
  }

  return -1;
}
```

## Analogias e Intuicoes

- Procurar uma palavra no dicionario abrindo no meio.
- Adivinhar um numero recebendo dicas de maior ou menor.
- Cortar o espaco de busca pela metade.

## Erros Comuns

- Usar busca binaria em array desordenado.
- Errar a atualizacao de `start` e `end`.
- Criar loop infinito quando o meio nao avanca.
- Esquecer de tratar alvo inexistente.

## Checklist de Entendimento

Antes de seguir, voce deve conseguir responder:

- Qual pre-condicao a busca binaria exige?
- Por que ela e mais eficiente que busca linear em arrays grandes?
- O que acontece quando o alvo nao existe?
- Por que o custo e logaritmico?

## Pratica Recomendada

- Implemente busca binaria iterativa.
- Teste com alvo no inicio, meio, fim e inexistente.
- Explique por que ordenar antes de uma unica busca pode nao compensar.
