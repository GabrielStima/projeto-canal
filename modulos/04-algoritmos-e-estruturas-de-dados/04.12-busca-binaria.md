- entender por que ela reduz o problema pela metade;
- analisar sua complexidade.

## Ideia Central

Busca binária funciona quando os dados estão ordenados. Em vez de olhar item por item, ela compara com o elemento do meio e descarta metade da busca.

Esse descarte repetido torna a busca muito eficiente, mas apenas quando a pré-condição é verdadeira. Em dados desordenados, o algoritmo pode retornar resultado errado.

## Tópicos da Aula

- Array ordenado.
- Início, meio e fim.
- Descartar metade.
- Complexidade logarítmica.

## Exemplo Prático

```ts
function binarySearch(values: number[], target: number): number {
  let start = 0;
  let end = values.length - 1;

  while (start <= end) {
    const middle = Math.floor((start + end) / 2);

    if (values[middle] === target) {
      return middle;
    }

    if (values[middle] < target) {
      start = middle + 1;
    } else {
      end = middle - 1;
    }
  }

  return -1;
}
```

## Analogias e Intuições

- Procurar uma palavra no dicionário abrindo no meio.
- Adivinhar um número recebendo dicas de maior ou menor.
- Cortar o espaço de busca pela metade.

## Erros Comuns

- Usar busca binária em array desordenado.
- Errar a atualização de `start` e `end`.
- Criar loop infinito quando o meio não avança.
- Esquecer de tratar alvo inexistente.

## Checklist de Entendimento

- Qual pré-condição a busca binária exige?
- Por que ela é mais eficiente que busca linear em arrays grandes?
- O que acontece quando o alvo não existe?
- Por que o custo é logarítmico?

## Prática Recomendada

- Teste com alvo no início, meio, fim e inexistente.
- Explique por que ordenar antes de uma única busca pode não compensar.
