## Ideia Central

Heap é uma estrutura baseada em uma árvore binária quase completa que respeita uma regra de prioridade. Em um max-heap, o maior valor fica no topo. Em um min-heap, o menor valor fica no topo.

Heap não é uma árvore de busca binária (BST). A regra do heap garante prioridade no topo, não uma ordenação total para busca arbitrária.

Uma fila de triagem é um exemplo clássico de uso de heap. O próximo atendimento a ser chamado não é necessariamente o mais antigo, mas o mais urgente. O heap mantém o caso mais prioritário acessível rapidamente.

## Exemplo Prático

```ts
function parentIndex(index: number): number {
  return Math.floor((index - 1) / 2);
}

function leftChildIndex(index: number): number {
  return index * 2 + 1;
}

function rightChildIndex(index: number): number {
  return index * 2 + 2;
}
```

Em muitos heaps, a árvore e representada em array. Os índices permitem navegar entre pai e filhos.

## Analogias e Intuições

- Uma fila de prioridade.
- O item mais urgente sempre no topo.
- Uma hierarquia que garante prioridade local, não ordem completa.

## Erros Comuns

- Confundir heap com memória heap.
- Confundir heap com árvore de busca binária (BST).
- Achar que todos os itens ficam totalmente ordenados.
- Ignorar que a prioridade precisa ser restaurada após inserir/remover.
