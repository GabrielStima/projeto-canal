## Ideia Central

Heap e uma estrutura baseada em uma arvore binaria quase completa que respeita uma regra de prioridade. Em um max-heap, o maior valor fica no topo. Em um min-heap, o menor valor fica no topo.

Heap nao e uma árvore de busca binária (BST). A regra do heap garante prioridade no topo, nao uma ordenacao total para busca arbitraria.

## Topicos da Aula

- Min-heap e max-heap.
- Propriedade de prioridade.
- Arvore quase completa.
- Representacao em array em alto nivel.

## Exemplo Pratico

```ts
function parentIndex(index: number): number {
  return Math.floor((index - 1) / 2);
}

function leftChildIndex(index: number): number {
  return index * 2 + 1;
}

function rightChildIndex(index: number): number {
  return index * 2 + 2;
}
```

Em muitos heaps, a arvore e representada em array. Os indices permitem navegar entre pai e filhos.

## Analogias e Intuicoes

- Uma fila de prioridade.
- O item mais urgente sempre no topo.
- Uma hierarquia que garante prioridade local, nao ordem completa.

## Erros Comuns

- Confundir heap com memoria heap.
- Confundir heap com árvore de busca binária (BST).
- Achar que todos os itens ficam totalmente ordenados.
- Ignorar que a prioridade precisa ser restaurada apos inserir/remover.

## Checklist de Entendimento

Antes de seguir, voce deve conseguir responder:

- Qual valor fica no topo de um max-heap?
- Heap serve para buscar qualquer valor rapidamente?
- Como um array pode representar uma arvore?
- Por que Heap Sort precisa dessa estrutura?

## Pratica Recomendada

- Desenhe um max-heap pequeno.
- Represente o mesmo heap em array.
- Compare a regra de heap com a regra de árvore de busca binária (BST).
