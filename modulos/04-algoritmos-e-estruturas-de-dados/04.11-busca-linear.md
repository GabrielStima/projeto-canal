## Ideia Central

Busca linear percorre os itens um por um até encontrar o alvo ou acabar a entrada. Ela não exige que os dados estejam ordenados.

Essa simplicidade é uma vantagem. Para listas pequenas, buscas ocasionais ou dados sem organização especial, busca linear pode ser a escolha mais clara.

## Tópicos da Aula

- Procurar item por item.
- Melhor caso, pior caso e caso médio.
- Busca em arrays de primitivos.
- Busca em arrays de objetos.

## Exemplo Prático

```ts
type Appointment = {
  id: number;
  petName: string;
};

function findAppointmentIndex(appointments: Appointment[], targetId: number): number {
  for (let index = 0; index < appointments.length; index++) {
    if (appointments[index].id === targetId) {
      return index;
    }
  }

  return -1;
}
```

Entrada: array de atendimentos e o id procurado. Processamento: comparar o id de cada atendimento. Saída: índice encontrado ou `-1`.

Essa busca funciona mesmo que os atendimentos não estejam ordenados. O custo cresce conforme o tamanho da lista.

## Analogias e Intuições

- Procurar uma chave olhando gaveta por gaveta.
- Conferir nomes em uma lista sem ordem.
- Ler mensagens até achar uma palavra.

## Erros Comuns

- Chamar busca linear de ruim em todos os casos.
- Esquecer de retornar algo quando não encontra.
- Misturar índice com valor encontrado.
- Ignorar que o melhor caso pode ser o primeiro item.

- A busca linear precisa de dados ordenados?
