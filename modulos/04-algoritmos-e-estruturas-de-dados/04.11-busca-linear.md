# Busca linear

## Objetivo

Ao final desta aula, voce deve ser capaz de:

- explicar como a busca linear funciona;
- implementar busca linear em arrays;
- analisar sua complexidade;
- reconhecer quando ela e aceitavel.

## Onde Esta Aula Entra no Curso

- Modulo: 04. Algoritmos e Estruturas de Dados
- Aula anterior: Algoritmos de busca
- Proxima aula: Busca binaria
- Pre-requisitos: arrays, funcoes, repeticoes e complexidade linear.

## Ideia Central

Busca linear percorre os itens um por um ate encontrar o alvo ou acabar a entrada. Ela nao exige que os dados estejam ordenados.

Essa simplicidade e uma vantagem. Para listas pequenas, buscas ocasionais ou dados sem organizacao especial, busca linear pode ser a escolha mais clara.

## Topicos da Aula

- Procurar item por item.
- Melhor caso, pior caso e caso medio.
- Busca em arrays de primitivos.
- Busca em arrays de objetos.

## Exemplo Pratico

```ts
function linearSearch(values: number[], target: number): number {
  for (let index = 0; index < values.length; index++) {
    if (values[index] === target) {
      return index;
    }
  }

  return -1;
}
```

Entrada: array e alvo. Processamento: comparar cada item. Saida: indice encontrado ou `-1`.

## Analogias e Intuicoes

- Procurar uma chave olhando gaveta por gaveta.
- Conferir nomes em uma lista sem ordem.
- Ler mensagens ate achar uma palavra.

## Erros Comuns

- Chamar busca linear de ruim em todos os casos.
- Esquecer de retornar algo quando nao encontra.
- Misturar indice com valor encontrado.
- Ignorar que o melhor caso pode ser o primeiro item.

## Checklist de Entendimento

Antes de seguir, voce deve conseguir responder:

- A busca linear precisa de dados ordenados?
- Qual e o pior caso?
- Quando ela e suficiente?
- O que muda ao buscar em array de objetos?

## Pratica Recomendada

- Implemente busca por numero.
- Implemente busca por `id` em um array de objetos.
- Conte quantas comparacoes foram feitas antes de encontrar o alvo.
