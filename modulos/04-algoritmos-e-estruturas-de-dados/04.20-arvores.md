# Árvores

## Objetivo

Ao final desta aula, você deve ser capaz de:

- explicar o que é uma árvore em estruturas de dados;
- reconhecer termos como raiz, nó, filho, folha, altura e profundidade;
- entender por que árvores representam hierarquias;
- preparar árvore binária, árvore de busca binária (BST) e heaps.

## Ideia Central

Árvore é uma estrutura formada por nós conectados de forma hierárquica. Existe uma raiz, e a partir dela os dados se ramificam em filhos.

Árvores aparecem quando os dados têm relação de hierarquia ou quando queremos organizar buscas e prioridades de modo mais inteligente do que uma lista simples.

## Tópicos da Aula

- Raiz, nó, aresta, filho e pai.
- Folha, altura e profundidade.
- Hierarquias.
- Percursos em alto nível.

## Exemplo Prático

```ts
type TreeNode<T> = {
  value: T;
  children: TreeNode<T>[];
};

const root: TreeNode<string> = {
  value: "Serviços",
  children: [
    {
      value: "Clínico",
      children: [
        { value: "Consulta", children: [] },
        { value: "Vacina", children: [] },
      ],
    },
    {
      value: "Estética",
      children: [
        { value: "Banho", children: [] },
        { value: "Tosa", children: [] },
      ],
    },
  ],
};
```

Árvores são naturais para representar hierarquias. Um catálogo de serviços de uma clínica veterinária, por exemplo, pode ter categorias principais e subcategorias.

## Analogias e Intuições

- Pastas dentro de pastas.
- Organograma de uma empresa.
- Comentários com respostas aninhadas.

## Erros Comuns

- Pensar que toda árvore é binária.
- Ignorar a raiz da estrutura.
- Confundir altura com quantidade total de nós.
- Tentar percorrer árvore como array sem estratégia.

## Checklist de Entendimento

- Como uma árvore representa hierarquia?
