# Arvores

## Objetivo

Ao final desta aula, voce deve ser capaz de:

- explicar o que e uma arvore em estruturas de dados;
- reconhecer termos como raiz, no, filho, folha, altura e profundidade;
- entender por que arvores representam hierarquias;
- preparar arvore binaria, árvore de busca binária (BST) e heaps.

## Onde Esta Aula Entra no Curso

- Modulo: 04. Algoritmos e Estruturas de Dados
- Aula anterior: Quick Sort
- Proxima aula: Arvore binaria
- Pre-requisitos: objetos, referencias, recursao e complexidade basica.

## Ideia Central

Arvore e uma estrutura formada por nos conectados de forma hierarquica. Existe uma raiz, e a partir dela os dados se ramificam em filhos.

Arvores aparecem quando os dados tem relacao de hierarquia ou quando queremos organizar buscas e prioridades de modo mais inteligente do que uma lista simples.

## Topicos da Aula

- Raiz, no, aresta, filho e pai.
- Folha, altura e profundidade.
- Hierarquias.
- Percursos em alto nivel.

## Exemplo Pratico

```ts
type TreeNode<T> = {
  value: T;
  children: TreeNode<T>[];
};

const root: TreeNode<string> = {
  value: "src",
  children: [
    { value: "components", children: [] },
    { value: "pages", children: [] },
  ],
};
```

## Analogias e Intuicoes

- Pastas dentro de pastas.
- Organograma de uma empresa.
- Comentarios com respostas aninhadas.

## Erros Comuns

- Pensar que toda arvore e binaria.
- Ignorar a raiz da estrutura.
- Confundir altura com quantidade total de nos.
- Tentar percorrer arvore como array sem estrategia.

## Checklist de Entendimento

Antes de seguir, voce deve conseguir responder:

- O que e a raiz?
- O que e uma folha?
- Como uma arvore representa hierarquia?
- Por que recursao combina com arvores?

## Pratica Recomendada

- Modele uma arvore de categorias.
- Conte quantos nos existem usando recursao.
- Liste exemplos reais de dados hierarquicos em aplicacoes web.
