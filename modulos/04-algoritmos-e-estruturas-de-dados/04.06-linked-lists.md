## Ideia Central

Uma linked list organiza dados em nós. Cada nó guarda um valor e uma referência para o próximo nó. Diferente de um array, os itens não precisam ser acessados por índice contínuo.

Isso facilita certas inserções e remoções quando já temos a referência correta, mas torna o acesso por posição mais caro, porque precisamos caminhar pela lista.

## Tópicos da Aula

- Nó, valor e ponteiro para o próximo.
- Cabeça da lista.
- Inserção e remoção.
- Comparação com arrays.

## Exemplo Prático

```ts
type ListNode<T> = {
  value: T;
  next: ListNode<T> | null;
};

const first: ListNode<string> = {
  value: "A",
  next: null,
};

const second: ListNode<string> = {
  value: "B",
  next: null,
};

first.next = second;
```

## Analogias e Intuições

- Uma corrente, onde cada elo aponta para o próximo.
- Uma caça ao tesouro, onde cada pista leva a outra.
- Uma fila de pessoas em que cada pessoa sabe quem vem depois.

## Erros Comuns

- Tentar acessar uma lista ligada como se fosse array.
- Perder a referência do restante da lista ao alterar `next`.
- Esquecer de tratar lista vazia.
- Supor que linked list é sempre mais eficiente.
