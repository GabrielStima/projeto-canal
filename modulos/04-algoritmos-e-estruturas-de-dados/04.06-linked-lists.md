# Linked lists

## Objetivo

Ao final desta aula, voce deve ser capaz de:

- explicar o que e uma linked list;
- diferenciar lista ligada de array;
- implementar uma lista simples em TypeScript;
- reconhecer vantagens e custos dessa estrutura.

## Ideia Central

Uma linked list organiza dados em nos. Cada no guarda um valor e uma referencia para o proximo no. Diferente de um array, os itens nao precisam ser acessados por indice continuo.

Isso facilita certas insercoes e remocoes quando ja temos a referencia correta, mas torna o acesso por posicao mais caro, porque precisamos caminhar pela lista.

## Topicos da Aula

- No, valor e ponteiro para o proximo.
- Cabeca da lista.
- Insercao e remocao.
- Comparacao com arrays.

## Exemplo Pratico

```ts
type ListNode<T> = {
  value: T;
  next: ListNode<T> | null;
};

const first: ListNode<string> = {
  value: "A",
  next: null,
};

const second: ListNode<string> = {
  value: "B",
  next: null,
};

first.next = second;
```

## Analogias e Intuicoes

- Uma corrente, onde cada elo aponta para o proximo.
- Uma caca ao tesouro, onde cada pista leva a outra.
- Uma fila de pessoas em que cada pessoa sabe quem vem depois.

## Erros Comuns

- Tentar acessar uma lista ligada como se fosse array.
- Perder a referencia do restante da lista ao alterar `next`.
- Esquecer de tratar lista vazia.
- Supor que linked list e sempre mais eficiente.
