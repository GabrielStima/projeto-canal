# Estruturas de dados: dados, operacoes e escolhas

## Objetivo

Ao final desta aula, voce deve ser capaz de:

- explicar o que e uma estrutura de dados;
- diferenciar dado, operacao e regra de organizacao;
- reconhecer que cada estrutura favorece algumas operacoes e dificulta outras;
- preparar o terreno para arrays, listas, pilhas, filas e tabelas hash.

## Onde Esta Aula Entra no Curso

- Modulo: 04. Algoritmos e Estruturas de Dados
- Aula anterior: Algoritmos e Estruturas de Dados
- Proxima aula: Arrays
- Pre-requisitos: arrays, objetos, funcoes e TypeScript basico.

## Ideia Central

Uma estrutura de dados e uma forma de organizar informacao para facilitar certas operacoes. O ponto nao e apenas guardar valores; e guardar de um jeito que combine com o problema.

Se voce precisa acessar pelo indice, um array pode ser suficiente. Se precisa sempre remover o ultimo item inserido, uma pilha combina melhor. Se precisa associar uma chave a um valor, uma tabela hash pode ser mais adequada.

## Topicos da Aula

- Dados como valores e relacoes.
- Operacoes comuns: inserir, remover, buscar, atualizar e percorrer.
- Como uma estrutura favorece algumas operacoes.
- Como escolher pensando no uso real.

## Exemplo Pratico

Imagine uma lista de tarefas. Voce pode querer:

- listar todas as tarefas;
- buscar uma tarefa pelo id;
- adicionar uma nova tarefa;
- remover a tarefa mais antiga;
- desfazer a ultima acao.

Cada necessidade aponta para uma estrutura diferente ou para uma combinacao delas.

```ts
type Task = {
  id: string;
  title: string;
  done: boolean;
};

const tasks: Task[] = [];
```

## Analogias e Intuicoes

- Um armario organiza roupas para encontrar e guardar mais rapido.
- Uma fila de atendimento organiza quem chega primeiro.
- Um historico de desfazer lembra a ultima acao antes das anteriores.

## Erros Comuns

- Escolher uma estrutura porque ela parece mais sofisticada.
- Ignorar quais operacoes acontecem com mais frequencia.
- Confundir estrutura de dados com algoritmo.
- Pensar apenas em velocidade e esquecer clareza.

## Checklist de Entendimento

Antes de seguir, voce deve conseguir responder:

- Que operacoes meu problema precisa fazer?
- Qual estrutura facilita essas operacoes?
- O que fica mais caro ou mais complexo com essa escolha?
- A solucao mais simples ja resolve?

## Pratica Recomendada

- Liste tres funcionalidades de um aplicativo de tarefas e diga quais operacoes de dados aparecem nelas.
- Compare uma lista de compras, uma fila de suporte e um historico de navegacao.
- Explique quando um array simples e suficiente.
