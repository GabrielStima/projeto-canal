# Estruturas de dados: dados, operações e escolhas

## Objetivo

Ao final desta aula, você deve ser capaz de:

- explicar o que é uma estrutura de dados;
- diferenciar dado, operação e regra de organização;
- reconhecer que cada estrutura favorece algumas operações e dificulta outras;
- preparar o terreno para arrays, listas, pilhas, filas e tabelas hash.

## Ideia Central

Uma estrutura de dados é uma forma de organizar informação para facilitar certas operações. O ponto não é apenas guardar valores; é guardar de um jeito que combine com o problema.

Se você precisa acessar pelo índice, um array pode ser suficiente. Se precisa sempre remover o último item inserido, uma pilha combina melhor. Se precisa associar uma chave a um valor, uma tabela hash pode ser mais adequada.

## Tópicos da Aula

- Dados como valores e relações.
- Operações comuns: inserir, remover, buscar, atualizar e percorrer.
- Como uma estrutura favorece algumas operações.
- Como escolher pensando no uso real.

## Exemplo Prático

Imagine uma lista de tarefas. Você pode querer:

- listar todas as tarefas;
- buscar uma tarefa pelo id;
- adicionar uma nova tarefa;
- remover a tarefa mais antiga;
- desfazer a última ação.

Cada necessidade aponta para uma estrutura diferente ou para uma combinação delas.

```ts
type Task = {
  id: number;
  title: string;
  done: boolean;
};

const tasks: Task[] = [];
```

Outro exemplo: o cadastro de atendimentos de uma clínica veterinária. Dependendo da operação, a estrutura muda:

- listar os atendimentos do dia → array, porque a ordem importa;
- buscar um atendimento pelo id → hash table ou `Map`, porque a busca por chave é frequente;
- atender na ordem de chegada → queue, porque o primeiro a chegar deve ser atendido primeiro;
- desfazer a última alteração de status → stack, porque a última ação deve ser revertida primeiro.

## Analogias e Intuições

- Um armário organiza roupas para encontrar e guardar mais rápido.
- Uma fila de atendimento organiza quem chega primeiro.
- Um histórico de desfazer lembra a última ação antes das anteriores.

## Erros Comuns

- Escolher uma estrutura porque ela parece mais sofisticada.
- Ignorar quais operações acontecem com mais frequência.
- Confundir estrutura de dados com algoritmo.
- Pensar apenas em velocidade e esquecer clareza.

## Checklist de Entendimento

Antes de seguir, você deve conseguir responder:

- Que operações meu problema precisa fazer?
- Qual estrutura facilita essas operações?
- O que fica mais caro ou mais complexo com essa escolha?
- A solução mais simples já resolve?

## Prática Recomendada

- Liste três funcionalidades de um aplicativo de tarefas e diga quais operações de dados aparecem nelas.
- Compare uma lista de compras, uma fila de suporte é um histórico de navegação.
- Explique quando um array simples é suficiente.
