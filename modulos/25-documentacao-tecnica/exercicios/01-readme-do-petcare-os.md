# Exercício 01 — README do PetCare OS

## O Que Você Já Possui

Você concluiu toda a trilha prática do PetCare OS: backend TypeScript, frontend, banco de dados, filas, cache, testes, infraestrutura, deploy e observabilidade. Agora você precisa documentar o projeto para que outras pessoas consigam rodar e entender o sistema sem depender de você.

## A Tarefa

Crie `docs/README.md` com a porta de entrada do repositório do PetCare OS. O documento deve permitir que uma pessoa nova rode o projeto localmente em até 30 minutos.

Inclua:

1. uma descrição de duas a quatro linhas sobre o que o sistema faz;
2. pré-requisitos com versões mínimas: runtime, banco, cache, fila, Docker, Node.js e qualquer ferramenta obrigatória;
3. instruções de clone, instalação de dependências e execução local;
4. comando para subir dependências externas com Docker Compose, se aplicável;
5. variáveis de ambiente obrigatórias, com exemplo em `.env.example` (sem valores reais de segredos);
6. scripts principais: dev, build, test, lint, migrations e seed;
7. links para documentação de arquitetura, ADRs, runbooks, API e contribuição;
8. uma seção de troubleshoot com os três erros mais comuns de quem tenta rodar pela primeira vez;
9. critérios de quando o README deve ser atualizado.

Não inclua senhas, tokens ou credenciais de produção. Use marcadores como `sua_chave_aqui`.

## O Que Você Vai Produzir

O README principal do projeto, que será o ponto de partida para qualquer pessoa que chegue ao repositório.

Nos próximos exercícios você criará os documentos para os quais o README aponta.
