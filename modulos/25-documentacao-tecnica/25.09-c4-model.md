# C4 Model

## Objetivo

Ao final desta aula, voce deve ser capaz de:

- explicar o objetivo do C4 Model;
- diferenciar contexto, containers, componentes e codigo;
- usar C4 para comunicar arquitetura em niveis;
- evitar diagramas detalhados demais para o publico errado.

## Ideia Central

C4 Model e uma forma de descrever arquitetura de software em quatro niveis: contexto, containers, componentes e codigo. A ideia e permitir que pessoas diferentes vejam o sistema no nivel de detalhe certo.

O nivel de contexto mostra o sistema em relacao a usuarios e sistemas externos. O nivel de containers mostra aplicacoes, bancos, filas e servicos executaveis. O nivel de componentes mostra partes internas importantes de um container. O nivel de codigo pode mostrar classes ou estruturas proximas da implementacao quando isso realmente ajuda.

C4 nao e sobre desenhar bonito. E sobre controlar zoom arquitetural.

## Exemplo Pratico

Para uma plataforma de cursos:

- Contexto: aluno, administrador, sistema de pagamento e plataforma de email.
- Containers: frontend web, API backend, banco PostgreSQL, fila e worker.
- Componentes da API: modulo de cursos, modulo de matriculas, modulo de pagamentos e adaptador do gateway.
- Codigo: classes ou funcoes de um fluxo especifico, se necessario.

Uma pessoa de produto talvez precise apenas do contexto. Uma pessoa backend provavelmente precisa de containers e componentes. Codigo so entra quando a discussao exige detalhe fino.

## Analogias e Intuicoes

- C4 e zoom progressivo.
- O mesmo sistema pode ser explicado em varias distancias.
- Detalhe demais cedo demais atrapalha a conversa.

## Erros Comuns

- Misturar todos os niveis no mesmo diagrama.
- Chamar banco, API e classe de "componentes" sem criterio.
- Desenhar codigo quando o problema ainda e arquitetura.
- Omitir sistemas externos no contexto.
- Criar diagrama sem legenda ou descricao.

## Checklist de Entendimento

Antes de seguir, a pessoa deve conseguir responder:

- Quem ou o que aparece no nivel de contexto?
- O que e um container no C4?
- Quando faz sentido detalhar componentes?
- Quando o nivel de codigo e util?
- Que publico precisa de cada nivel?

## Pratica Recomendada

- Escolha uma aplicacao fullstack.
- Desenhe uma versao textual dos niveis de contexto, containers e componentes.
- Explique o que voce deixaria fora de cada nivel.
- Compare sua estrutura com um HLD simples.
