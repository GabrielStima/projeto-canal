# C4 Model

## Objetivo

Ao final desta aula, você deve ser capaz de:

- explicar o objetivo do C4 Model;
- diferenciar contexto, containers, componentes e código;
- usar C4 para comunicar arquitetura em níveis;
- evitar diagramas detalhados demais para o público errado.

## Ideia Central

C4 Model é uma forma de descrever arquitetura de software em quatro níveis: contexto, containers, componentes e código. A ideia é permitir que pessoas diferentes vejam o sistema no nível de detalhe certo.

O nível de contexto mostra o sistema em relação a usuários e sistemas externos. O nível de containers mostra aplicações, bancos, filas e serviços executáveis. O nível de componentes mostra partes internas importantes de um container. O nível de código pode mostrar classes ou estruturas próximas da implementação quando isso realmente ajuda.

C4 não é sobre desenhar bonito. É sobre controlar zoom arquitetural.

## Exemplo Prático

Para uma plataforma de cursos:

- Contexto: aluno, administrador, sistema de pagamento e plataforma de e-mail.
- Containers: frontend web, API backend, banco PostgreSQL, fila e worker.
- Componentes da API: módulo de cursos, módulo de matrículas, módulo de pagamentos e adaptador do gateway.
- Código: classes ou funções de um fluxo específico, se necessário.

Uma pessoa de produto talvez precise apenas do contexto. Uma pessoa backend provavelmente precisa de containers e componentes. Código só entra quando a discussão exige detalhe fino.

## Analogias e Intuicoes

- C4 é zoom progressivo.
- O mesmo sistema pode ser explicado em várias distâncias.
- Detalhe demais cedo demais atrapalha a conversa.

## Erros Comuns

- Misturar todos os níveis no mesmo diagrama.
- Chamar banco, API e classe de "componentes" sem critério.
- Desenhar código quando o problema ainda é arquitetura.
- Omitir sistemas externos no contexto.
- Criar diagrama sem legenda ou descrição.

## Checklist de Entendimento

Antes de seguir, a pessoa deve conseguir responder:

- Quem ou o que aparece no nível de contexto?
- O que é um container no C4?
- Quando faz sentido detalhar componentes?
- Quando o nível de código é útil?
- Que público precisa de cada nível?

## Prática Recomendada

- Escolha uma aplicação fullstack.
- Desenhe uma versão textual dos níveis de contexto, containers e componentes.
- Explique o que você deixaria fora de cada nível.
- Compare sua estrutura com um HLD simples.
