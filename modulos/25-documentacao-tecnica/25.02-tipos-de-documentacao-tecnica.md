# Tipos de Documentacao Tecnica

## Objetivo

Ao final desta aula, voce deve ser capaz de:

- diferenciar os principais tipos de documentacao tecnica;
- escolher um formato de documento pelo problema que ele resolve;
- evitar usar README, ADR, RFC, HLD e LLD como se fossem a mesma coisa;
- montar um pequeno mapa de documentacao para um projeto.

## Ideia Central

Documentacao tecnica nao e um arquivo unico chamado "docs". Existem documentos diferentes porque existem necessidades diferentes.

Um README responde como entender e rodar o projeto. Um ADR registra uma decisao ja tomada. Uma RFC propoe uma mudanca e abre espaco para discussao. Um HLD mostra a visao de alto nivel de um sistema. Um LLD detalha uma parte da implementacao. Diagramas ajudam a comunicar relacoes, fluxos e limites. Documentacao interna ajuda o time a manter o sistema. Documentacao externa ajuda consumidores a usar algo corretamente.

Escolher o tipo certo de documento evita dois problemas: documento grande demais para tudo e informacao importante escondida no lugar errado.

## Topicos da Aula

- README como porta de entrada.
- ADR como registro de decisao.
- RFC como proposta de mudanca.
- HLD e LLD como documentacao de arquitetura em niveis diferentes.
- Diagramas como suporte visual.
- Documentacao interna e externa.

## Exemplo Pratico

Pergunta: "Por que usamos fila para processar envio de emails?"

Possiveis documentos:

- README: pode mencionar que existe uma fila e como rodar o worker.
- ADR: deve registrar por que fila foi escolhida, quais alternativas foram consideradas e quais consequencias existem.
- HLD: deve mostrar onde a fila entra na arquitetura.
- LLD: deve detalhar payload, retries, idempotencia e tratamento de erros.
- Runbook interno: deve explicar como investigar falhas na fila.

A mesma decisao aparece em varios lugares, mas cada documento responde uma pergunta diferente.

## Analogias e Intuicoes

- Documentos sao ferramentas, nao trofeus.
- O formato certo reduz atrito de leitura.
- A mesma arquitetura pode precisar de mapa, historico e manual de operacao.

## Erros Comuns

- Colocar todas as decisoes no README.
- Usar ADR para discutir proposta ainda aberta.
- Usar RFC depois que a decisao ja foi tomada sem debate.
- Criar HLD detalhado demais e LLD abstrato demais.
- Criar diagrama sem texto explicando decisoes.

## Checklist de Entendimento

Antes de seguir, a pessoa deve conseguir responder:

- Que pergunta cada tipo de documento responde?
- Quando um README deixa de ser suficiente?
- Qual a diferenca entre ADR e RFC?
- Qual a diferenca entre HLD e LLD?
- Como diagramas complementam texto?

## Pratica Recomendada

- Escolha uma funcionalidade de uma API.
- Liste cinco perguntas sobre ela.
- Para cada pergunta, escolha o tipo de documento que melhor responderia.
- Explique por que os outros formatos seriam menos adequados.
