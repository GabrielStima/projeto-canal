# Tipos de Documentação Técnica

## Objetivo

Ao final desta aula, você deve ser capaz de:

- diferenciar os principais tipos de documentação técnica;
- escolher um formato de documento pelo problema que ele resolve;
- evitar usar README, ADR, RFC, HLD e LLD como se fossem a mesma coisa;
- montar um pequeno mapa de documentação para um projeto.

## Ideia Central

Documentação técnica não é um arquivo único chamado "docs". Existem documentos diferentes porque existem necessidades diferentes.

Um README responde como entender e rodar o projeto. Um ADR registra uma decisão já tomada. Uma RFC propõe uma mudança e abre espaço para discussão. Um HLD mostra a visão de alto nível de um sistema. Um LLD detalha uma parte da implementação. Diagramas ajudam a comunicar relações, fluxos e limites. Documentação interna ajuda o time a manter o sistema. Documentação externa ajuda consumidores a usar algo corretamente.

Escolher o tipo certo de documento evita dois problemas: documento grande demais para tudo e informação importante escondida no lugar errado.

## Tópicos da Aula

- README como porta de entrada.
- ADR como registro de decisão.
- RFC como proposta de mudança.
- HLD e LLD como documentação de arquitetura em níveis diferentes.
- Diagramas como suporte visual.
- Documentação interna e externa.

## Exemplo Prático

Pergunta: "Por que usamos fila para processar envio de e-mails?"

Possíveis documentos:

- README: pode mencionar que existe uma fila e como rodar o worker.
- ADR: deve registrar por que fila foi escolhida, quais alternativas foram consideradas e quais consequências existem.
- HLD: deve mostrar onde a fila entra na arquitetura.
- LLD: deve detalhar payload, retries, idempotência e tratamento de erros.
- Runbook interno: deve explicar como investigar falhas na fila.

A mesma decisão aparece em vários lugares, mas cada documento responde uma pergunta diferente.

## Analogias e Intuições

- Documentos são ferramentas, não troféus.
- O formato certo reduz atrito de leitura.
- A mesma arquitetura pode precisar de mapa, histórico e manual de operação.

## Erros Comuns

- Colocar todas as decisões no README.
- Usar ADR para discutir proposta ainda aberta.
- Usar RFC depois que a decisão já foi tomada sem debate.
- Criar HLD detalhado demais e LLD abstrato demais.
- Criar diagrama sem texto explicando decisões.

## Checklist de Entendimento

Antes de seguir, a pessoa deve conseguir responder:

- Que pergunta cada tipo de documento responde?
- Quando um README deixa de ser suficiente?
- Qual a diferença entre ADR e RFC?
- Qual a diferença entre HLD e LLD?
- Como diagramas complementam texto?

## Prática Recomendada

- Escolha uma funcionalidade de uma API.
- Liste cinco perguntas sobre ela.
- Para cada pergunta, escolha o tipo de documento que melhor responderia.
- Explique por que os outros formatos seriam menos adequados.
