# Projeto Prático: Persistência de Dados em uma Aplicação Fullstack

## Objetivo

Ao final desta aula, você deve ser capaz de:

- integrar os principais conceitos do módulo em um projeto pequeno;
- modelar dados em uma versão relacional;
- modelar dados em uma versão documental;
- comparar decisões entre SQL e MongoDB;
- preparar a transição para backend com Node.js.

## Onde Esta Aula Entra na Formação

- Módulo: 09. Bancos de Dados
- Aula anterior: Security em NoSQL e MongoDB
- Próxima aula: 10. Backend com Node.js
- Pré-requisitos: Todas as aulas anteriores do módulo 09.

## Ideia Central

O projeto final não precisa conectar uma aplicação real a um banco em produção. O foco é consolidar raciocínio: entender dados, modelar, consultar, proteger integridade, pensar performance, planejar migrations, discutir backup e comparar SQL com MongoDB.

Sugestão: evoluir o painel de estudos do módulo de Next.js para ter uma camada de persistência planejada.

## Tópicos da Aula

- Escopo do projeto.
- Entidades principais.
- Modelo relacional.
- Consultas SQL.
- Constraints.
- Índices.
- Migration inicial.
- Backup e restore.
- Modelo MongoDB.
- Queries e aggregation.
- Segurança básica.
- Comparação final.

## Analogias e Intuições

- Este projeto é uma ponte entre aprender sobre bancos e conectá-los ao backend.
- O objetivo é tomar decisões e justificar, não decorar a resposta perfeita.

## Erros Comuns

- Querer usar ORM antes de entender o banco.
- Criar modelo relacional sem regras de integridade.
- Criar documento MongoDB sem pensar nas consultas.
- Esquecer backup, segurança e migrations.
- Comparar SQL e NoSQL como se um anulasse o outro.

## Checklist de Entendimento

Antes de seguir, você deve conseguir responder:

- Quais dados a aplicação precisa persistir?
- Qual é o modelo relacional?
- Quais consultas principais a aplicação faz?
- Quais constraints protegem regras importantes?
- Que índices parecem necessários?
- Como o schema evoluiria?
- Qual seria o modelo documental?
- Que riscos de segurança existem?
- O que preciso revisar antes de conectar isso a Node.js?

## Prática Recomendada

- Modele o projeto primeiro em texto.
- Escreva as tabelas e relações.
- Escreva cinco queries SQL essenciais.
- Planeje duas migrations.
- Escreva um plano simples de backup e restore.
- Modele a mesma aplicação em MongoDB.
- Compare as duas versões em uma tabela de trade-offs.
