## Ideia Central

SQL permite conversar com bancos relacionais usando comandos declarativos. Em vez de dizer passo a passo como percorrer dados, você descreve o que deseja: buscar linhas, filtrar registros, inserir dados, atualizar valores ou remover informações.

Nesta aula, o foco é o vocabulário essencial: `SELECT`, `FROM`, `WHERE`, `ORDER BY`, `INSERT`, `UPDATE` e `DELETE`.

## Tópicos da Aula

- Tabelas, linhas e colunas.
- `SELECT` e `FROM`.
- Filtros com `WHERE`.
- Ordenação com `ORDER BY`.
- Inserção com `INSERT`.
- Atualização com `UPDATE`.
- Remoção com `DELETE`.

## Exemplo Prático

```sql
SELECT id, titulo, status
FROM aulas
WHERE status = 'publicada'
ORDER BY titulo;
```

Essa consulta pede três colunas da tabela `aulas`, filtra apenas aulas publicadas e ordena pelo título.

## Analogias e Intuições

- `SELECT` escolhe o que você quer ver.
- `WHERE` funciona como uma peneira.
- `ORDER BY` organiza a saída para leitura.

## Erros Comuns

- Esquecer `WHERE` em `UPDATE` ou `DELETE`.
- Buscar todas as colunas sem necessidade.
- Confundir nome da tabela com nome da entidade do código.
- Achar que SQL básico é pouco importante.

## Checklist de Entendimento

Antes de seguir, você deve conseguir responder:

- Para que serve `SELECT`?
- Qual é a diferença entre `WHERE` e `ORDER BY`?
- O que acontece em um `DELETE` sem filtro?
- Por que buscar menos colunas pode ser melhor?
- Como uma tabela representa uma entidade?

## Prática Recomendada

- Crie mentalmente uma tabela `tarefas`.
- Escreva uma consulta para listar tarefas abertas.
- Escreva uma inserção de nova tarefa.
- Escreva uma atualização de status.
- Explique por que um `DELETE` precisa de cuidado.
