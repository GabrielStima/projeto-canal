## Ideia Central

MongoDB é um banco de dados orientado a documentos. Em vez de organizar tudo em tabelas e linhas, ele armazena documentos flexíveis, parecidos com objetos JSON, agrupados em collections.

Essa proximidade com estruturas usadas em JavaScript ajuda no aprendizado, mas não elimina decisões importantes de modelagem, consulta, consistência e segurança.

Esse documento guarda dados simples, array e objeto embutido na mesma estrutura.

## Analogias e Intuições

- Collection lembra uma pasta de documentos parecidos.
- Documento lembra um objeto persistido, mas com responsabilidade de banco.

## Erros Comuns

- Achar que MongoDB é apenas JSON salvo em disco.
- Não modelar porque o schema é flexível.
- Embutir tudo em um documento enorme.
- Criar documentos com formatos inconsistentes sem necessidade.
- Ignorar índices e padrões de consulta.
