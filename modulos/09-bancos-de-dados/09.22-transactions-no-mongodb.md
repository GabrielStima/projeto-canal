# Transactions no MongoDB

## Objetivo

Ao final desta aula, você deve ser capaz de:

- entender que MongoDB também oferece transactions;
- comparar transactions em MongoDB com o modelo relacional;
- reconhecer quando documentos bem modelados reduzem necessidade de transaction;
- evitar usar transaction para compensar modelagem ruim.

## Onde Esta Aula Entra no Curso

- Módulo: 09. Bancos de Dados
- Aula anterior: Aggregation no MongoDB
- Próxima aula: Scaling em Bancos de Dados
- Pré-requisitos: Transactions em SQL, MongoDB, embedding, referencing e operações de escrita.

## Ideia Central

MongoDB suporta transactions para operações que precisam envolver múltiplos documentos ou collections com consistência. Mas, em bancos de documentos, uma boa modelagem muitas vezes tenta manter juntos os dados que precisam mudar juntos.

O importante é entender a decisão: se uma regra de negócio exige múltiplas escritas coerentes, transaction pode fazer sentido. Se isso acontece o tempo todo, talvez o modelo precise ser revisto.

## Tópicos da Aula

- Transactions fora do mundo relacional.
- Operações em um documento.
- Múltiplos documentos.
- Múltiplas collections.
- Custo e complexidade.
- Relação com embedding e referencing.

## Exemplo Prático

```text
Cenário:
- criar pedido;
- registrar pagamento;
- atualizar crédito do usuário.

Se esses dados estiverem em documentos/collections diferentes e precisarem mudar juntos, transaction pode ser necessária.
```

## Analogias e Intuições

- Transaction continua sendo "tudo ou nada".
- Em MongoDB, modelagem documental pode evitar algumas costuras frequentes.

## Erros Comuns

- Achar que MongoDB não possui transactions.
- Usar transaction para toda operação simples.
- Modelar documentos separados demais e depender de transactions constantes.
- Ignorar custo operacional.
- Comparar com SQL sem olhar o padrão de acesso.

## Checklist de Entendimento

Antes de seguir, você deve conseguir responder:

- A operação mexe em quantos documentos?
- Esses dados deveriam estar juntos?
- O que acontece se a escrita falhar no meio?
- A transaction é exceção ou rotina?
- A modelagem poderia simplificar essa regra?

## Prática Recomendada

- Escolha uma operação de compra.
- Modele em documentos MongoDB.
- Marque quais dados mudam juntos.
- Decida se usaria embedding, referencing ou transaction.
