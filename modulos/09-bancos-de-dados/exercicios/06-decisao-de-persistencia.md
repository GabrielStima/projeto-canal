# Exercício 06 — Decisão de Persistência

Esta atividade é obrigatória para a continuidade. Você vai registrar a escolha que o módulo de backend receberá, sem apresentar múltiplos bancos como destino inevitável.

## O Que Você Já Possui

Você construiu uma versão relacional para os dados principais e analisou uma alternativa documental para auditoria. Também estudou segurança, transactions, performance, backup e escala.

## Sua Tarefa

Crie `database/decisao-de-persistencia.md` respondendo:

1. quais dados precisam de relações e integridade fortes;
2. quais consultas são mais importantes para o Portal do Tutor;
3. quais operações exigem transaction;
4. quais índices são hipóteses iniciais;
5. como migrations, backup e restore serão tratados;
6. quais riscos de segurança precisam ser controlados;
7. se um banco relacional atende a primeira versão sozinho;
8. em que condição uma solução documental seria reavaliada no futuro.

Finalize com uma decisão principal para a próxima etapa. Você pode escolher somente o banco relacional e manter o modelo documental como estudo comparativo.

## Critérios de Conclusão

- a decisão parte de requisitos e consultas;
- SQL e NoSQL não são tratados como rivais absolutos;
- a primeira versão evita complexidade operacional desnecessária;
- a tecnologia escolhida sustenta o fluxo que será implementado no backend;
- condições futuras são mensuráveis, não baseadas em moda;
- limitações e dúvidas ficam registradas.

Guarde o documento. A atividade final vai reunir essa decisão com os arquivos SQL produzidos.
