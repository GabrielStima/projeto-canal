# Scaling em Bancos de Dados

## Objetivo

Ao final desta aula, você deve ser capaz de:

- explicar por que bancos precisam escalar;
- diferenciar escala vertical e horizontal em nível introdutório;
- entender réplicas e sharding conceitualmente;
- reconhecer que escala adiciona complexidade.

## Onde Esta Aula Entra no Curso

- Módulo: 09. Bancos de Dados
- Aula anterior: Transactions no MongoDB
- Próxima aula: Security em NoSQL e MongoDB
- Pré-requisitos: SQL, NoSQL, indexes, performance, transactions e MongoDB.

## Ideia Central

Escalar um banco de dados é aumentar a capacidade de atender leituras, escritas, armazenamento ou disponibilidade. Isso pode envolver uma máquina maior, réplicas, particionamento, sharding, cache e mudanças de arquitetura.

O objetivo aqui é vocabulário e intuição. System design profundo vem depois. Por enquanto, o mais importante é entender que escala resolve gargalos específicos e cria novos custos.

## Tópicos da Aula

- Gargalos de leitura.

- Escala horizontal.

- Complexidade operacional.

## Exemplo Prático

```text
Cenários:
- muitas leituras: réplicas podem ajudar;
- dados grandes demais para uma máquina: particionamento pode ajudar;
- muitas escritas distribuídas: sharding pode entrar na conversa;
- query ruim: escalar hardware talvez só esconda o problema.
```

## Analogias e Intuições

- Escalar verticalmente é trocar por uma máquina mais forte.
- Escalar horizontalmente é dividir trabalho entre mais máquinas.

## Erros Comuns

- Falar de sharding antes de medir gargalos.
- Usar escala para esconder query ruim.
- Ignorar backup e restore em arquitetura maior.
- Achar que distribuição simplifica o sistema.
- Esquecer que consistência fica mais difícil.

## Checklist de Entendimento

Antes de seguir, você deve conseguir responder:

- O gargalo é leitura, escrita, armazenamento ou disponibilidade?
- Já otimizei query e índice?
- Escala vertical resolveria por enquanto?
- Réplica ajuda esse problema?
- Que complexidade a solução adiciona?

- Explique quando réplica ajuda e quando não ajuda.
- Explique por que sharding não deve ser primeira resposta.
