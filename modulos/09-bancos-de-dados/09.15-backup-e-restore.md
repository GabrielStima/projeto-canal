# Backup e Restore

## Objetivo

Ao final desta aula, você deve ser capaz de:

- explicar por que backup não é opcional;
- diferenciar backup de restore;
- reconhecer riscos de nunca testar restauração;
- pensar recuperação como parte do sistema.

## Ideia Central

Backup é criar cópias recuperáveis dos dados. Restore é conseguir usar essas cópias para voltar o sistema a um estado útil.

Backup sem teste de restore é esperança, não garantia. A parte importante não é apenas ter arquivo salvo, mas saber se ele funciona, quanto tempo leva para recuperar e quantos dados podem ser perdidos.

## Exemplo Prático

```text
Cenário:
- uma migration remove uma coluna errada;
- dados importantes desaparecem;
- a equipe precisa restaurar backup;
- a pergunta real é: quanto tempo demora e quanto dado será perdido?
```

## Analogias e Intuições

- Backup é o cinto de segurança dos dados.
- Restore testado é saber que o cinto realmente fecha.

## Erros Comuns

- Fazer backup e nunca testar restore.
- Guardar backup no mesmo lugar vulnerável.
- Não saber a frequência de backup.
- Não saber quem tem permissão para restaurar.
- Confundir migration com mecanismo de recuperação de dados.

## Checklist de Entendimento

Antes de seguir, você deve conseguir responder:

- Onde o backup fica?

- Como restaurar?
- Quando foi o último teste de restore?
- Quanto dado a aplicação aceita perder?
