## Ideia Central

Uma view é uma consulta salva com nome. Ela permite representar uma leitura útil do banco como se fosse uma tabela virtual.

Views ajudam a reduzir repetição, padronizar leituras e esconder complexidade. Mas elas não substituem modelagem, não resolvem todo problema de performance e precisam ser entendidas por quem mantém o banco.

## Exemplo Prático

```sql
CREATE VIEW progresso_por_usuario AS
SELECT usuarios.id, usuarios.nome, COUNT(progresso.id) AS total_registros
FROM usuarios
LEFT JOIN progresso ON progresso.usuario_id = usuarios.id
GROUP BY usuarios.id, usuarios.nome;
```

Depois disso, outras consultas podem ler `progresso_por_usuario` sem repetir toda a query.

## Analogias e Intuições

- View é um atalho nomeado para uma consulta importante.
- Ela funciona como uma janela organizada para uma parte dos dados.

## Erros Comuns

- Achar que toda view melhora performance.
- Criar views demais para esconder queries mal pensadas.
- Esquecer que alguém precisará manter a view.
- Usar view como desculpa para não entender o modelo.
