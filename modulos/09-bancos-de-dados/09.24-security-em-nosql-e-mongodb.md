# Security em NoSQL e MongoDB

## Objetivo

Ao final desta aula, você deve ser capaz de:

- reconhecer riscos de segurança em bancos NoSQL;
- entender autenticação e autorização em nível conceitual;
- proteger strings de conexão e credenciais;
- relacionar validação de entrada com queries NoSQL.

## Ideia Central

Security em NoSQL e MongoDB segue o mesmo princípio geral: dados são sensíveis, entrada externa não é confiável e permissões devem ser mínimas.

Além disso, bancos documentais trazem cuidados próprios: filtros construídos a partir de objetos, operadores recebidos indevidamente, exposição de campos sensíveis, validação de formato e proteção de conexão.

## Tópicos da Aula

- Autenticação no banco.
- Autorização e papéis.
- String de conexão.
- Variáveis de ambiente.
- Validação de entrada.
- Riscos em queries montadas com objetos.
- Exposição de campos sensíveis.
- Backup e logs.

## Exemplo Prático

```text
Risco conceitual:
- aplicação recebe um objeto de filtro vindo do usuário;
- esse objeto é repassado diretamente para o banco;
- operadores indesejados podem alterar a consulta.

Defesa:
- validar formato permitido;
- montar filtros no backend com campos conhecidos;
- usar permissões mínimas;
- não retornar campos sensíveis.
```

## Analogias e Intuições

- Flexibilidade de documento aumenta a importância de validar formato.
- Query montada com objeto externo precisa do mesmo respeito que SQL montado com string externa.

## Erros Comuns

- Expor string de conexão no repositório.
- Aceitar filtros arbitrários do cliente.
- Retornar documentos completos com campos sensíveis.
- Usar usuário com permissões amplas demais.
- Não separar ambientes de desenvolvimento e produção.

## Checklist de Entendimento

Antes de seguir, você deve conseguir responder:

- Quem pode se conectar ao banco?
- Quais permissões a aplicação realmente precisa?
- Quais campos não devem ser retornados?
- O filtro vindo do cliente é validado?
- Onde ficam os segredos de conexão?

## Prática Recomendada

- Liste campos sensíveis de um documento de usuário.
- Defina quais campos podem aparecer em uma API pública.
- Escreva uma regra para montar filtros permitidos.
- Explique como armazenaria a string de conexão.
