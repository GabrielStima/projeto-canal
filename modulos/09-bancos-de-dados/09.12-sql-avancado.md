# SQL Avançado

## Objetivo

Ao final desta aula, você deve ser capaz de:

- enxergar SQL avançado como uma caixa de ferramentas;
- reconhecer recursos que vão além do CRUD;
- saber quando aprofundar cada recurso;
- não transformar SQL avançado em complexidade gratuita.

## Onde Esta Aula Entra no Curso

- Módulo: 09. Bancos de Dados
- Aula anterior: Performance Optimization em SQL
- Próxima aula: Stored Procedures e Functions
- Pré-requisitos: SQL básico, JOIN queries, subqueries, agregações, views, transactions e indexes.

## Ideia Central

SQL avançado reúne recursos para resolver consultas, regras, relatórios e transformações mais complexas. Isso pode incluir expressões condicionais, CTEs, window functions, constraints mais elaboradas, views materializadas e recursos específicos do banco escolhido.

O objetivo desta aula é organizar o mapa, não esgotar tudo. Em projetos reais, você aprofunda o recurso conforme a necessidade aparece.

## Tópicos da Aula

- CTEs como consultas nomeadas temporárias.
- Expressões condicionais.
- Window functions em nível conceitual.
- Views materializadas em nível conceitual.
- Recursos específicos de cada banco.
- Cuidado com complexidade escondida.

## Exemplo Prático

```sql
WITH aulas_publicadas AS (
  SELECT id, titulo
  FROM aulas
  WHERE status = 'publicada'
)
SELECT *
FROM aulas_publicadas
ORDER BY titulo;
```

Uma CTE pode deixar uma consulta maior mais legível ao nomear uma etapa intermediária.

## Analogias e Intuições

- SQL avançado é uma bancada de ferramentas: útil quando o problema pede, exagerado quando usado por vaidade.
- CTE é dar nome a uma parte da consulta para melhorar leitura.

## Erros Comuns

- Usar recurso avançado sem necessidade.
- Escrever queries que só o autor entende.
- Misturar regra de negócio demais dentro do banco.
- Ignorar diferenças entre bancos SQL.
- Copiar queries avançadas sem entender o plano.

## Checklist de Entendimento

Antes de seguir, você deve conseguir responder:

- Que problema um recurso avançado está resolvendo?
- A query ficou mais clara ou mais obscura?
- Esse recurso existe no banco que estou usando?
- Existe uma solução mais simples?
- O time conseguiria manter essa consulta?

## Prática Recomendada

- Pegue uma query com subquery.
- Reescreva usando uma CTE.
- Compare legibilidade.
- Liste quais recursos avançados você estudaria apenas quando surgirem no projeto.
