# Subqueries

## Objetivo

Ao final desta aula, você deve ser capaz de:

- explicar o que é uma subquery;
- reconhecer consultas dentro de consultas;
- usar subqueries para filtros e cálculos simples;
- decidir quando uma subquery pode deixar a consulta mais clara ou mais confusa.

## Ideia Central

Subquery é uma consulta SQL usada dentro de outra consulta. Ela ajuda quando uma pergunta depende do resultado de outra pergunta.

Esse recurso é poderoso, mas deve ser usado com cuidado. Às vezes uma subquery deixa a intenção clara. Em outros casos, um `JOIN`, uma view ou uma consulta reorganizada pode ser melhor.

## Tópicos da Aula

- Consulta interna e consulta externa.
- Subquery em `WHERE`.
- Subquery com `IN`.
- Subquery com agregação.
- Legibilidade.
- Comparação com `JOIN`.

## Exemplo Prático

```sql
SELECT titulo
FROM aulas
WHERE id IN (
  SELECT aula_id
  FROM progresso
  WHERE concluida = true
);
```

A consulta interna encontra aulas concluídas. A consulta externa busca os títulos dessas aulas.

## Analogias e Intuições

- Subquery é responder uma pergunta menor para conseguir responder a pergunta maior.
- Ela funciona como uma etapa intermediária dentro do próprio SQL.

## Erros Comuns

- Criar subqueries difíceis de ler.
- Usar subquery quando um `JOIN` simples resolveria melhor.
- Não testar a consulta interna separadamente.
- Esquecer que a subquery pode retornar muitos valores.

## Checklist de Entendimento

Antes de seguir, você deve conseguir responder:

- Qual é a consulta interna?
- Qual é a consulta externa?
- Que resultado a subquery devolve?
- Por que `IN` aparece em muitos exemplos?
- Quando uma subquery pode piorar a legibilidade?

## Prática Recomendada

- Compare qual versão ficou mais clara.
- Teste mentalmente o que acontece quando a subquery não retorna nada.
