# Conceitos Úteis no MongoDB

## Objetivo

Ao final desta aula, você deve ser capaz de:

- reconhecer os conceitos mais importantes do MongoDB;
- diferenciar embedding e referencing;
- entender o papel de `_id`;
- pensar documentos a partir de padrões de acesso.

## Ideia Central

MongoDB fica mais fácil quando você entende o vocabulário certo: database, collection, document, field, `_id`, embedding, referencing e schema flexível.

O ponto principal é que documentos devem ser modelados para as leituras e escritas mais importantes da aplicação. Às vezes embutir dados é bom. Às vezes referenciar outro documento é melhor.

## Exemplo Prático

```javascript
{
  _id: "usuario-1",
  nome: "Ana",
  preferencias: {
    tema: "escuro",
    idioma: "pt-BR"
  },
  estudosRecentes: [
    { aulaId: "09.01", concluida: true },
    { aulaId: "09.02", concluida: false }
  ]
}
```

Preferências embutidas fazem sentido porque são lidas junto com o usuário. Já aulas completas talvez devam ficar em outra collection.

## Analogias e Intuições

- Embedding é guardar junto o que normalmente viaja junto.
- Referencing é guardar um ponteiro quando o dado tem vida própria.

## Erros Comuns

- Embutir dados que crescem sem limite.
- Referenciar tudo e perder a vantagem do modelo documental.
- Ignorar duplicação intencional.
- Deixar documentos da mesma collection sem padrão mínimo.
- Pensar só na escrita e esquecer a leitura.

## Checklist de Entendimento

Antes de seguir, você deve conseguir responder:

- O que identifica um documento?
- Quando embutir dados?
- Quando referenciar dados?
- Que consulta principal esse documento precisa atender?
- O documento pode crescer demais?

## Prática Recomendada

- Modele um usuário com preferências.
- Modele uma aula com tags.
- Decida se progresso fica embutido no usuário ou separado.
- Explique sua decisão pensando nas consultas.
