# Operadores de Query no MongoDB

Ao final desta aula, você deve ser capaz de:

- entender como consultar documentos no MongoDB;
- reconhecer operadores de comparação;
- combinar filtros simples;
- comparar filtros MongoDB com `WHERE` em SQL.

## Ideia Central

Consultas no MongoDB usam documentos de filtro. Em vez de escrever uma cláusula `WHERE`, você descreve critérios em uma estrutura parecida com objeto.

Operadores como `$eq`, `$ne`, `$gt`, `$gte`, `$lt`, `$lte`, `$in`, `$and` e `$or` ajudam a criar filtros mais expressivos.

## Tópicos da Aula

- Filtro como documento.
- Igualdade simples.
- Comparação.
- Pertencimento com `$in`.
- Combinação com `$and`.
- Alternativa com `$or`.
- Consulta em campos embutidos.

## Exemplo Prático

```javascript
{
  status: "publicada",
  duracaoMinutos: { $gte: 10 },
  tags: { $in: ["banco-de-dados"] }
}
```

Esse filtro procura aulas publicadas, com duração mínima de dez minutos e tag relacionada a banco de dados.

## Analogias e Intuições

- O filtro é um objeto que descreve o formato da busca.
- Operadores são verbos pequenos para comparar valores.

## Erros Comuns

- Confundir igualdade simples com operadores.
- Esquecer que arrays podem participar da consulta.
- Fazer filtros amplos demais.
- Não pensar em índices para consultas frequentes.
- Copiar operador sem entender o resultado esperado.
