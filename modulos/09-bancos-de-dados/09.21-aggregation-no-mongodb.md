# Aggregation no MongoDB

## Ideia Central

Aggregation no MongoDB permite processar documentos em um pipeline. Cada estágio recebe documentos, transforma, filtra, agrupa ou projeta campos, e passa o resultado para o próximo estágio.

Essa ideia é útil para relatórios, resumos, contagens, agrupamentos e transformações que vão além de uma busca simples.

## Tópicos da Aula

- Pipeline.
- Estágio.
- `$match`.
- `$group`.
- `_id` como chave de agrupamento.
- Acumuladores.
- `$project`.
- Comparação com `GROUP BY`.

## Exemplo Prático

```javascript
[
  { $match: { status: "publicada" } },
  {
    $group: {
      _id: "$autorId",
      total: { $sum: 1 }
    }
  }
]
```

O primeiro estágio filtra aulas publicadas. O segundo agrupa por autor e conta quantas aulas cada autor possui.

## Analogias e Intuições

- Pipeline é uma esteira de processamento.
- Cada estágio faz uma transformação pequena e passa adiante.

## Erros Comuns

- Colocar etapas em ordem confusa.
- Agrupar sem entender a chave `_id` do `$group`.
- Fazer aggregation para uma busca simples.
- Ignorar impacto de volume e índices.
- Criar pipelines longos demais para uma primeira versão.

## Checklist de Entendimento

Antes de seguir, você deve conseguir responder:

- O que é um pipeline?
- O que `$match` faz?
- O que `$group` faz?
- Por que `_id` aparece no agrupamento?

## Prática Recomendada

- Crie um pipeline para contar aulas por status.
- Adicione um filtro antes do agrupamento.
- Projete apenas os campos importantes no resultado.
- Compare com uma query SQL usando `GROUP BY`.
