# Atividade Final do Módulo — Visão e Backlog Inicial do PetCare OS

## O Que Você Vai Produzir

Nesta atividade final, você vai consolidar tudo o que estudou no módulo em um pacote de documentos de produto versionado no seu repositório do PetCare OS. O objetivo é deixar claro, antes de qualquer linha de código de frontend ou backend, qual problema o PetCare OS resolve, para quem e quais são as primeiras entregas.

O resultado será a base sobre a qual os próximos módulos vão construir interfaces, APIs, bancos de dados e arquitetura.

## Pré-requisitos

Antes de começar, você deve ter:

- Concluído os exercícios 01, 02, 03 e 04 deste módulo.
- O repositório base do PetCare OS criado no módulo 05 (`petcare-os-base`).
- Familiaridade com Git, Markdown e escrita de documentos simples.

## Tarefa

Dentro do seu repositório do PetCare OS, crie uma pasta chamada `docs/produto/` e produza os seguintes documentos:

### 1. `visao-de-produto.md`

Uma visão curta e clara do PetCare OS. Deve conter:

- **Problema principal:** qual dor do mundo real justifica o produto?
- **Públicos:** tutores, recepcionistas, clínicos e administradores — com uma necessidade curta para cada um.
- **Proposta de valor:** em uma frase, o que o PetCare OS promete entregar.
- **Escopo inicial:** o que está dentro da primeira versão e o que está fora.
- **Restrições e premissas:** o que o time está assumindo sobre regulamentação, dispositivos ou integrações.

### 2. `jornada-principal.md`

Descreva a jornada principal do PetCare OS do ponto de vista do tutor. Deve conter:

- **Início:** tutor deixa o pet na clínica.
- **Desenvolvimento:** tutor recebe informações e precisa tomar decisões.
- **Fim:** tutor busca o pet e o atendimento é finalizado.
- **Pontos de dor e oportunidade:** pelo menos três momentos onde o produto pode ajudar.
- **Atores envolvidos:** tutor, recepcionista, clínico.

### 3. `backlog-inicial.md`

Uma lista priorizada das primeiras funcionalidades. Deve conter:

- Pelo menos **cinco itens** de backlog.
- Cada item deve ter: título, breve descrição, motivo de prioridade e dependências.
- Identifique pelo menos **um item Must have**, **um Should have** e **um Won't have (nesta versão)**.

### 4. `decisoes-e-riscos.md`

Registre decisões que você tomou ao montar os documentos anteriores. Deve conter:

- **Decisões:** pelo menos duas escolhas importantes que você fez sobre escopo ou prioridade.
- **Riscos:** pelo menos dois riscos que podem impedir o sucesso da primeira versão.
- **Perguntas abertas:** pelo menos três perguntas que ainda precisam ser respondidas.
- **Próximos passos:** o que deve acontecer antes do módulo 07 (Frontend Moderno).

### 5. Registro no Git

- Crie uma branch chamada `feature/visao-e-backlog-inicial`.
- Faça commits pequenos para cada documento ou grupo de mudanças.
- Abra um pull request ou merge local com uma descrição clara do que foi entregue.

## Critérios de Aceite

- A visão de produto está escrita na linguagem do problema, não na linguagem técnica.
- A jornada principal tem início, meio e fim claros.
- O backlog inicial é priorizado e justificado.
- As decisões e riscos mostram que você pensou sobre escopo e incerteza.
- Os documentos estão em Markdown, bem formatados e dentro de `docs/produto/`.
- O Git possui branch, commits e descrição de merge organizados.

## Como Este Trabalho Continuará

Este pacote de documentos será retomado nos próximos módulos:

- No **módulo 07 (Frontend Moderno)**, a jornada do tutor virará telas e componentes.
- No **módulo 08 (Next.js)**, o portal do tutor será estruturado com rotas e renderização.
- No **módulo 09 (Bancos de Dados)**, os atores e a jornada orientarão a modelagem inicial.
- No **módulo 10 (Backend com Node.js)**, os itens do backlog ajudarão a definir os primeiros endpoints.

Por isso, não é apenas um exercício de escrita: é a fundação sobre a qual o PetCare OS vai crescer.
