# Feature Flags

## Objetivo

Ao final desta aula, você deve ser capaz de:

- explicar o que é uma feature flag;
- entender por que ela reduz risco de entrega;
- diferenciar deploy de release;
- reconhecer cuidados e custos de manter flags;
- saber o que estudar em seguida.

## Ideia Central

Feature flag é um mecanismo para ligar ou desligar uma funcionalidade sem precisar mudar o código naquele momento. Ela permite separar duas coisas que muita gente mistura: colocar código em produção e liberar a funcionalidade para usuários.

Nesta etapa do curso, a ideia não é usar uma plataforma complexa de flags. A ideia é entender o raciocínio: se uma funcionalidade tem risco, incerteza ou precisa ser liberada aos poucos, uma flag pode dar mais controle.

## Tópicos da Aula

- O que é uma feature flag.
- Deploy vs release.
- Flags booleanas simples.
- Liberação gradual.
- Flags por ambiente, usuário ou grupo.
- Riscos: código morto, combinações confusas e flags esquecidas.
- Quando não usar feature flag.

## Exemplo Prático

Imagine uma funcionalidade de favoritos.

Sem flag:

```text
Ao fazer deploy, todos os usuários passam a ver a funcionalidade.
```

Com flag:

```ts
const isFavoritesEnabled = true;

if (isFavoritesEnabled) {
  renderFavoritesButton();
}
```

Em um projeto real, essa flag poderia vir de configuração, variável de ambiente, banco de dados ou serviço especializado. O importante é que o comportamento da aplicação muda sem precisar reescrever a funcionalidade inteira.

## Analogias e Intuições

- Deploy é colocar o código no lugar. Release é abrir a porta para o usuário.
- Feature flag é um interruptor de produto dentro do sistema.
- O interruptor ajuda, mas interruptores esquecidos deixam a casa confusa.

## Erros Comuns

- Usar flag para toda pequena mudança sem necessidade.
- Esquecer de remover flags antigas.
- Criar nomes vagos como `newFeature`.
- Não testar os dois estados da flag.
- Achar que feature flag substitui critério de aceite ou teste.

## Checklist de Entendimento

Antes de seguir, a pessoa deve conseguir responder:

- Qual risco esta flag reduz?
- Quem deve ver a funcionalidade quando a flag estiver ligada?
- O sistema funciona corretamente com a flag desligada?
- Existe um plano para remover a flag?
- A flag muda apenas comportamento de produto ou também regra crítica?

## Prática Recomendada

- Escolha uma funcionalidade pequena.
- Descreva como ela seria entregue sem feature flag.
- Descreva como ela seria entregue com feature flag.
- Liste o que precisa ser testado com a flag ligada e desligada.
- Defina quando a flag deve ser removida.
