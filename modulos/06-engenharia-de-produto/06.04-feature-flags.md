# Feature Flags

## Objetivo

Ao final desta aula, voce deve ser capaz de:

- explicar o que e uma feature flag;
- entender por que ela reduz risco de entrega;
- diferenciar deploy de release;
- reconhecer cuidados e custos de manter flags;
- saber o que estudar em seguida.

## Onde Esta Aula Entra no Curso

- Modulo: Engenharia de Produto

- Proxima aula: Experimentos e A/B testing

## Ideia Central

Feature flag e um mecanismo para ligar ou desligar uma funcionalidade sem precisar mudar o codigo naquele momento. Ela permite separar duas coisas que muita gente mistura: colocar codigo em producao e liberar a funcionalidade para usuarios.

Nesta etapa do curso, a ideia nao e usar uma plataforma complexa de flags. A ideia e entender o raciocinio: se uma funcionalidade tem risco, incerteza ou precisa ser liberada aos poucos, uma flag pode dar mais controle.

## Exemplo Pratico

Imagine uma funcionalidade de favoritos.

Sem flag:

```text
Ao fazer deploy, todos os usuarios passam a ver a funcionalidade.
```

Com flag:

```ts
const isFavoritesEnabled = true;

if (isFavoritesEnabled) {
  renderFavoritesButton();
}
```

Em um projeto real, essa flag poderia vir de configuracao, variavel de ambiente, banco de dados ou servico especializado. O importante e que o comportamento da aplicacao muda sem precisar reescrever a funcionalidade inteira.

## Analogias e Intuicoes

- Deploy e colocar o codigo no lugar. Release e abrir a porta para o usuario.
- Feature flag e um interruptor de produto dentro do sistema.
- O interruptor ajuda, mas interruptores esquecidos deixam a casa confusa.

## Erros Comuns

- Usar flag para toda pequena mudanca sem necessidade.
- Esquecer de remover flags antigas.
- Criar nomes vagos como `newFeature`.
- Nao testar os dois estados da flag.
- Achar que feature flag substitui criterio de aceite ou teste.

## Checklist de Entendimento

Antes de seguir, a pessoa deve conseguir responder:

- Qual risco esta flag reduz?
- Quem deve ver a funcionalidade quando a flag estiver ligada?
- O sistema funciona corretamente com a flag desligada?
- Existe um plano para remover a flag?
- A flag muda apenas comportamento de produto ou tambem regra critica?

## Pratica Recomendada

- Escolha uma funcionalidade pequena.
- Descreva como ela seria entregue sem feature flag.
- Descreva como ela seria entregue com feature flag.
- Liste o que precisa ser testado com a flag ligada e desligada.
- Defina quando a flag deve ser removida.
